# 第二大脑知识库 — Claude Code 项目指南

## 语言与格式规范

- 所有内容使用**简体中文**
- 日期格式：`YYYY-MM-DD`
- 文件名使用中文
- Markdown 格式遵循 Obsidian 语法

## PARA 结构规则

```
obsidian-second-brain/
├── 0-收集箱/    # 临时捕获，≤7天清理
├── 1-项目/      # 有明确起止日期，命名「[状态] 项目名」
├── 2-领域/      # 六大生活领域
├── 3-资源/      # 按类型分类的参考材料
├── 4-归档/      # 已完成项目和不活跃材料
├── MOC/         # 导航枢纽（Map of Content）
├── 日记/        # 每日日记
├── _模板/       # Templater 模板
└── _scripts/    # 自动化脚本（Claude Code 工具）
```

### 文件夹详细规则

| 文件夹 | 用途 | 命名规范 |
|--------|------|---------|
| `0-收集箱/` | 快速捕获，未经处理的想法 | 自由命名，≤7天内分类 |
| `1-项目/` | 有起止日期的具体项目 | `[进行中]`/`[计划中]`/`[已完成]` + 项目名 |
| `2-领域/` | 持续维护的生活领域 | 子目录：健康、财务、职业、人际关系、个人成长、家庭 |
| `3-资源/` | 参考材料和知识沉淀 | 子目录：书籍笔记、课程笔记、文章摘要、思维模型 |
| `4-归档/` | 已完成/不活跃材料 | 只移入不删除 |
| `MOC/` | 导航枢纽 | 不存放主内容，仅索引和链接 |
| `日记/` | 每日日记 | 模板在 `日记/日记模板.md` |
| `_模板/` | Templater 模板文件 | 以「模板」结尾 |
| `_scripts/` | Shell 自动化脚本 | `.sh` 文件 |

## Frontmatter 约定

### 项目笔记
```yaml
创建日期: YYYY-MM-DD
状态: 进行中 | 计划中 | 已完成
截止日期: YYYY-MM-DD
tags: #项目
```

### 资源笔记
```yaml
创建日期: YYYY-MM-DD
类型: 书籍 | 课程 | 文章 | 视频
作者: 作者名
关键词:
复习间隔: 1, 7, 21, 60
复习日期: YYYY-MM-DD
tags: #资源
```

### 概念笔记
```yaml
创建日期: YYYY-MM-DD
别名:
相关概念:
标签: #概念
```

## 链接规范

- 使用 `[[笔记名称]]` 内部链接
- 每篇笔记至少包含一个出链
- 典型链接关系：
  - 项目 → 相关资源
  - 日记 → 当日项目进展
  - 概念 → 应用示例
  - 资源 → 相关概念

## 渐进式总结层级

| 层级 | 格式 | 说明 |
|------|------|------|
| L1 | 原始文本 | 初始捕获的原始内容 |
| L2 | `**加粗**` | 标记关键点和核心语句 |
| L3 | `> 引用块` | 用自己的话总结理解 |
| L4 | `- [ ] 任务` | 转化为具体行动清单 |

## 行为规则

1. **创建笔记时使用对应模板结构**（见 `_模板/` 目录）
2. **不修改 `.obsidian/` 配置**
3. **不删除笔记**，需要归档时移至 `4-归档/`
4. **建议链接前验证目标笔记存在**（用 Glob 或 Grep 检查）
5. **不主动执行 git commit**（Obsidian Git 插件每 30 秒自动提交）

## 会话上下文加载

每次会话开始时建议：
1. 读取 `MOC/主页.md` 了解导航结构
2. 检查 `0-收集箱/` 待处理项数量
3. 查看 `1-项目/` 中 `[进行中]` 项目了解当前重点

## 已安装插件

Dataview、Templater、Tasks、Obsidian Git、QuickAdd、Excalidraw、Spaced Repetition、Minimal Settings 等。脚本可利用这些插件的数据格式（如 Dataview frontmatter、Tasks 语法 `- [ ] 任务 📅 日期`）。

## 自动化脚本

`_scripts/` 目录下的脚本用于知识库维护：

| 脚本 | 用途 |
|------|------|
| `vault-health.sh` | 知识库健康检查（孤立笔记、断裂链接、空笔记等） |
| `vault-stats.sh` | 统计仪表盘（笔记数、标签分布、链接密度等） |
| `inbox-process.sh` | 收集箱分类建议（`--dry-run` 或 `--move`） |
| `weekly-review.sh` | 周报数据生成（基于 git log） |
| `suggest-links.sh` | 链接建议（发现未链接的提及） |
| `progressive-summary.sh` | 渐进式总结进度扫描 |
| `review-reminder.sh` | 复习提醒（过期复习、停滞项目） |
