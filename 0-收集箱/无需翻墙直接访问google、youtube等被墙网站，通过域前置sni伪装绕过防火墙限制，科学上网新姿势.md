## 一句话总结
通过修改浏览器快捷方式配置，利用 [[sni]] 伪装和映射固定 [[IP]] 实现免代理直连谷歌等被墙网站的教程 [1, 2]。

## 目标受众与前置知识
- **适合谁看**：需要免代理访问被墙网站的普通用户及对翻墙底层技术感兴趣的爱好者 [1, 3]。
- **前置知识**：[[http]], [[HTTPS]], [[tls]], [[sni]], [[DNS]], [[IP]], [[ipv6]], 域前置，防火墙拦截机制 [2-4]。

## 核心观点
1. **修改浏览器启动配置即可实现轻度免翻墙**：通过在浏览器桌面快捷方式追加特定配置，可直连访问被封锁的网站 [1, 5]。
   - 为什么重要：提供了一种无需后台运行任何代理工具即可科学上网的简易方式 [1]。
   - 依据：讲者原话演示，在未运行代理工具的情况下直接搜索谷歌并成功加载 [1]。
2. **[[sni]] 伪装是骗过防火墙域名检测的关键**：将请求中的 [[sni]] 值修改为未被拉黑的域名，可绕过防火墙的黑名单拦截 [2, 4]。
   - 为什么重要：揭示了基于预前制（推断：域前置）绕过防火墙检测的核心网络原理 [4]。
   - 依据：讲者举例访问 tj（推断：Telegram），将其 [[sni]] 伪装为未在黑名单的 `cyfm` 从而被防火墙放行 [2, 4]。
3. **映射未封锁的固定 [[IP]] 可解决 [[DNS]] 污染问题**：仅伪装 [[sni]] 不够，还必须在配置中绑定未被封锁的真实服务器 [[IP]] 或代理 [[IP]] [2, 6]。
   - 为什么重要：解决了常规 [[DNS]] 解析会返回被封锁的黑名单 [[IP]] 或遭受 [[DNS]] 污染返回假 [[IP]] 的痛点 [2]。
   - 依据：讲者说明如果在进行 [[DNS]] 解析时获取的是假 [[IP]] 或黑名单 [[IP]]，即使伪造了 [[sni]] 依然会被丢弃数据包 [2]。
4. **播放 YouTube 视频需借助专属客户端与 [[ipv6]]**：直接修改配置只能访问网页，播放视频需通过客户端进行本地劫持 [3, 5, 7]。
   - 为什么重要：指出了纯网页端配置在处理复杂流媒体连接时的局限性，并给出了进阶解决方案 [3, 7]。
   - 依据：讲者原话展示网页版 YouTube 只能加载框架且视频会“一直黑屏转圈”，需要通过 Windows 客户端工具、安装 [[ca根证书]] 并具备 [[ipv6]] 环境才能播放 [3, 5, 7]。

## 概念关系
- [[HTTPS]] → [[tls]]：说明（使用 HTTPS 方式访问网站时，会对 http 数据进行 tls 封装并产生包含域名的 sni 值） [4]。
- 伪造 [[sni]] ↔ 真实 [[tls]] 证书：对比（由于发给服务器的伪造 sni 与服务器返回的真实域名 tls 证书不一致，导致浏览器告警显示不安全，此为预期效果） [2, 6]。
- 常规 [[DNS]] ↔ 映射固定 [[IP]]：对比（常规 DNS 往往返回黑名单 IP，通过映射精心挑选的未封锁 IP 才能确保连通性） [2]。

## 详细笔记
### 网页版基础配置与浏览器支持
讲者 不良林 演示了在不开启代理工具的情况下，直接访问 谷歌 (Google) 和 YouTube 的方法 [1]。该方法支持基于 [[chrome]] 内核的浏览器，包括 [[chrome]]、[[ag]] (推断：Edge) 以及 [[brave]] [1]。操作步骤为：复制网页默认生成的配置段，右键编辑浏览器的桌面快捷方式“属性”，在“目标”路径最后粘贴该配置并应用 [1]。
避坑事项：如果使用的是 [[ag]] (推断：Edge) 浏览器，必须在“系统和性能”设置中关闭前两项后台常驻进程 [1, 5]。确保浏览器完全退出且无后台进程后，双击修改后的快捷方式即可生效 [1, 5]。

### 批量配置抓取与多平台命令行执行
默认配置只能直连 谷歌 和 YouTube 网页端 [5]。若需访问其他被墙网站，可点击网页上的“抓取数据”（数据来源于 gthub，推断：[[github]]），能获取到如 推特 (Twitter)、Facebook、披照/皮站 (推断：Pixiv)、tj (推断：Telegram) 等域名的配置 [5]。
因批量配置文本过长，快捷方式的目标框无法容纳，需要打开文件夹所在位置，在地址栏输入 [[cmd]]，使用命令行将浏览器文件名与配置文件拼接执行 [5]。对于麦克 (推断：[[Mac]]) 端，操作原理相同，需在终端拼接获取到的浏览器默认路径与配置内容并回车执行 [4]。

### 核心原理解析：域前置与 SNI 伪装
该方案的核心是讲者曾在 mik (推断：MikroTik) 那期提到的预前制（推断：域前置）原理 [4]。当访问 tj (推断：Telegram) 等黑名单网站时，[[tls]] 数据包中的 [[sni]] 值为 tj 域名，防火墙检测到后会直接拦截 [4]。
通过浏览器启动配置做的小动作，是给域名添加映射关系：将发送给防火墙的 [[sni]] 伪造成未在黑名单的域名（例如 `cyfm` 或谷歌的 `g.cn`）[2, 6]。防火墙检测到合法的 [[sni]] 就会放行，而接收方服务器解密后仍能读取内层的真实 host 并返回相应资源 [2]。

### 应对 IP 封锁与方案局限性
防火墙不仅检测 [[sni]]，还会拦截被墙网站的专属 [[IP]] [2]。由于常规 [[DNS]] 解析通常会返回黑名单 [[IP]] 或直接进行 [[DNS]] 污染返回假 [[IP]]，因此配置中还执行了另一个动作：给伪装后的 [[sni]] 映射一个精心挑选的未被封锁的 tj 服 务 器 [[IP]]，或是 [[sni]] 代理服务器的 [[IP]] [2, 6]。这种免翻墙看谷歌的原理与之前分享的免翻墙看 奈非/奈飞 (推断：Netflix) 类似 [3, 6]。
此方案局限性在于只适合轻度使用，晚高峰会变得很慢且 [[IP]] 易失效 [3]。讲者建议普通用户使用其频道的专线机场，并提到该机场节点已从 30 多个地区增加到了 70 多个地区（讲者原话） [3]。

### YouTube 视频流播放的高级方案
纯靠网页版配置访问 YouTube 会导致无法播放视频（乱发 sni 对方服务器不一定接受） [3]。讲者提供了一个基于 [[Windows]] 的客户端工具解决此问题，前置条件是电脑需要有 [[ipv6]] 地址，并安装 run 运行环境（推断：.NET runtime）及 ca跟正书 (推断：[[ca根证书]]) [3]。
该工具的代码是讲者使用 [[chatgpt]] 将 cshop (推断：[[csharp]]) 代码转换成 GS (推断：[[JS]]) 实现的 [3, 7]。需要在网页复制 ngex/ang (推断：[[nginx]]) 配置文件到文档中，其原理是将网络请求劫持到 [[nginx]]，由其伪造 [[sni]] 并直连墙比较低的 [[ipv6]] 达到效果 [7]。该软件存在闪退问题，关闭此功能时必须停止全局伪装，并删除 host 文件中的域名映射以恢复正常上网 [7]。

## 金句亮点
- “你乱发 sni 对方的服务器也不一定接受” — 知识 / 说明纯粹进行 SNI 伪装而不配合后端支持的局限性 [3]。
- “通过这种曲线救国的方式实现免翻墙访问谷歌” — 意译 / 总结利用 SNI 代理服务器获取真实 IP 机制的巧妙技术价值 [6]。

## 常见坑与解决方案
- **问题描述**：配置后打开 `[[Edge]]` 浏览器无法生效（无法正常访问谷歌）。
  - 解决方案：`[[Edge]]` 浏览器默认在后台常驻进程。需要在浏览器的“系统和性能”设置中，关闭相关的后台长驻及启动选项，完全退出进程后重新打开浏览器即可 [1, 6]。
- **问题描述**：网页地址栏左侧显示“不安全”的警告告警信息。
  - 解决方案：未明确提及（此为预期现象，因伪造的 `[[SNI]]` 域名与服务器返回的 `[[TLS]]` 证书域名不匹配导致，属于正常生效状态，无需解决） [3, 7]。
- **问题描述**：能够访问 YouTube 网页，但视频一直黑屏转圈无法播放。
  - 解决方案：需要使用带有 `[[IPv6]]` 的网络，安装 `[[CA根证书]]`，并通过 Windows 客户端工具配合 `[[Nginx]]` 配置，将网络请求劫持并伪造 `[[SNI]]` 直连 [4, 5]。
- **问题描述**：关闭视频解锁客户端工具后，之前伪装的网站全都无法访问。
  - 解决方案：重新打开该客户端软件，停止全局伪装功能，并手动编辑系统 `[[hosts]]` 文件，删掉由该软件添加的域名映射并保存 [4]。

## 最佳实践
- 推荐普通用户（大流量/重度使用者）直接使用专业的专线“机场”代理服务，以获取更高的高速性和稳定性 [5]。
- 不推荐（反模式）将客户端解锁 YouTube 视频播放的方式作为主力方案，因为配置繁琐、需要 `[[IPv6]]` 门槛、晚高峰容易拥堵变慢、IP 易失效且软件容易闪退 [4, 5]。

## 关键概念

### [[SNI]]
- **定义**：TLS 封装中的一个明文值，指示客户端尝试连接的服务器域名 [1]。
- **视频上下文**：防火墙通过检测 SNI 来判断用户是否在访问黑名单网站并进行拦截；作者通过伪造 SNI（如将 tg 域名的 SNI 改为 cyfm）来骗过防火墙 [1, 2]。
- **为什么重要**：它是防火墙实施拦截的关键检测点，也是本视频中绕过防火墙限制的核心修改对象 [1, 2]。
- **关联概念**：对比 → [[Host]]

### [[Host]]
- **定义**：HTTP 数据包封装中的一个值，代表用户实际想要访问的真实域名 [1]。
- **视频上下文**：数据包发送给目标服务器时，只有经过解密才能获取到内层正确的 Host 域名并返回相应资源 [1, 2]。
- **为什么重要**：它确保了即使外层的 SNI 被伪装成了其他名称，目标服务器依然能根据内层的 Host 返回正确的网站数据 [2]。
- **关联概念**：未明确提及

### [[TLS]]
- **定义**：一种用于对 HTTP 数据包进行加密封装的安全协议 [1]。
- **视频上下文**：当使用 HTTPS 方式访问网站（如 tg）时，会进行 TLS 封装，其中包含了未加密的 SNI 值 [1]。
- **为什么重要**：TLS 加密保护了内层的真实 Host 等数据不被防火墙直接读取，使得仅通过修改外露的 SNI 绕过检测成为可能 [1, 2]。
- **关联概念**：属于 → [[域前置]]

### [[DNS污染]]
- **定义**：防火墙在进行域名解析时，直接返回一个不属于目标服务器的假 IP 的拦截手段 [2]。
- **视频上下文**：为了防止被 DNS 污染获取到假 IP，作者给伪装后的 SNI 手动映射了一个精挑细选且未被封锁的固定真实 IP [2]。
- **为什么重要**：它解释了为什么仅仅伪装 SNI 是不够的，必须同时绑定真实 IP 才能确保数据包不被防火墙丢弃 [2]。
- **关联概念**：未明确提及

### [[SNI代理服务器]]
- **定义**：当目标网站的所有 IP 都被防火墙屏蔽时，用于接收伪装请求并代为访问目标服务器的中转节点 [3]。
- **视频上下文**：由于谷歌的所有 IP 通常都被屏蔽，请求会先伪装成 g.cn 发给配置好的 SNI 代理服务器，由代理服务器代为获取谷歌的真实 IP [3]。
- **为什么重要**：在目标域名的所有直接 IP 均被封锁的极端情况下，它是实现“曲线救国”免翻墙访问的核心组件 [3]。
- **关联概念**：依赖 → [[SNI]]

### [[IPv6]]
- **定义**：一种网络协议，在视频语境中指代防火墙屏蔽力度（墙）相对较低的网络地址类型 [4, 5]。
- **视频上下文**：为了解决 YouTube 网页能开但无法播放视频的问题，作者提到电脑需要有 IPv6 地址，并通过工具直连墙比较低的 IPv6 达到免翻墙效果 [4, 5]。
- **为什么重要**：它是高级别解锁流媒体视频播放能力的基础网络前置要求 [4, 5]。
- **关联概念**：未明确提及

### [[CA根证书]]
- **定义**：一种用于验证网络连接和数字证书的凭证 [4, 5]。
- **视频上下文**：在使用 Windows 客户端工具启动“全局伪造”以解决 YouTube 视频播放问题之前，必须先安装此证书 [4, 5]。
- **为什么重要**：它是使本地伪装工具能够成功接管和处理 HTTPS 流量的关键配置步骤 [4, 5]。
