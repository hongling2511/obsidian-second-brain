## 一句话总结
本教程演示了在 [[openWrt]] 路由器上编译部署 [[openGFW]]，以实现内网 [[sni]] 阻断、[[DNS污染]]及代理协议探测。

## 目标受众与前置知识
- **适合谁看**：希望了解网络访问控制、代理协议探测原理，或有网络管理需求（如家庭防沉迷、公司管理）的用户。（推断）
- **前置知识**：[[openWrt]]、[[Linux]]、[[go语言]]、[[ssh]]、[[DNS]]、[[加密代理]]

## 核心观点
1. **[[openGFW]] 可在家庭终端实现类似 [[GFW]] 的功能**
   - 为什么重要：能够让用户近距离观察长城防火墙的运行机制，并将算力下放到家庭终端实现网络管控。
   - 依据：讲者原话称其可实现“[[sni]] 阻断、[[DNS污染]]以及 [[shadowsocks]]、[[trojan]]（视频字幕作[[秋卷]]）等加密代理协议的屏蔽”。
2. **明文代理协议极易被特征探测阻断**
   - 为什么重要：解释了传统明文代理为何已无法用于科学上网。
   - 依据：讲者原话指出“由于 [[socks]] 和 [[http代理]] 协议本身是明文的，通过探测可以知道你在使用代理访问哪个网站”，会被规则直接阻止。
3. **全加密流量也会被识别并屏蔽**
   - 为什么重要：打破了全加密协议绝对安全的误解，证明特征检测的范围广泛。
   - 依据：讲者演示测试时，全加密数据的 [[shadowsocks]] 和 [[vmess]]+[[tcp]] 节点均被 [[openGFW]] 判定并 Block（阻断）。

## 概念关系
- [[openGFW]] → [[openWrt]]：[[openGFW]] 程序需经过交叉编译后，运行在基于 [[Linux]] 系统的 [[openWrt]] 路由器的 [[F沃链]] 上。
- [[openGFW]] ↔ [[Hysteria]]（视频字幕作[[C斯底里]]）：两者为同一作者开发，目前 [[openGFW]] 未实现 [[Hysteria]] 的协议解析器，但可通过屏蔽对应的 [[quic]]（视频字幕作[[quck]]）协议来进行间接阻断。
- [[DNS污染]] → [[cloudflare]]：演示中 [[知乎]] 的 [[DNS]] 被污染至 1.1.1.1（[[cloudflare]] 的 IP），导致浏览器访问时出现证书颁发给 [[cloudflare]] 的不匹配警告并报错 10016。
- [[doh]] ↔ [[DNS污染]]：开启安全 DNS（[[dnsoverhttps]]）对查询进行加密，可使 [[openGFW]] 无法进行污染拦截。

## 详细笔记
### 项目背景与翻墙原理简介
- 讲者（不良林）介绍国内访问国际互联网（如[[谷歌]]）需经过 [[GFW]]（长城防火墙），常规手段是通过[[加密代理]]协议（如 [[shadowsocks]]、[[vmess]]、[[trojan]]、[[Hysteria]]）绕过检测。
- 引出 [[openGFW]] 项目，介绍其能够实现 [[sni]] 阻断和 [[DNS污染]] 等功能。讲者提到一个“冷知识”：该项目作者与 [[Hysteria]] 作者是同一人。引用鲁迅名言（意译改写版）表达探究原理的目的。

### 交叉编译与路由器前置配置
- 因为软路由指令集架构不同（讲者演示使用基于 [[arm64]] 的 [[r2s开发版]]），需在 [[Windows]] 电脑上进入 [[github]] 下载项目源码，并安装 [[go语言]] 环境进行交叉编译。
- 讲者演示在 [[cmd.exe]] 中设置环境变量（目标系统 [[Linux]]，目标架构 [[arm64]]），执行编译指令生成 `openGFW` 可执行文件。
- 在路由器（系统为基于 [[openWrt]] 的 iStoreOS 23.05 版本，推断）的防火墙设置中，必须取消勾选“软件流量分载”。如使用旁路由（视频字幕作“跑路由”）结构，需勾选 LAN 区域的“IP动态伪装”。

### 程序安装部署与运行测试
- 使用 [[ssh]] 工具连接路由器，通过 [[opkg]] 安装依赖（遇报错需先执行 update）。
- 使用 [[scp]] 指令将程序上传至 root 目录，并赋予执行权限。
- 讲者演示使用 [[vim编辑器]] 创建配置文件（config.yaml，需工作在 [[F沃链]] 上）和规则文件（rule.yaml）。运行程序后，经过路由器的网络数据即被监控。

### HTTP/HTTPS 与 SNI 阻断演示
- 讲者配置第一条规则记录包含特定关键字的域名。使用命令行（推断为 [[curl]] 指令）访问 [[HTTP]] 网址未触发记录，因为该规则探测的是 [[HTTPS]] 建立时的 [[sni]] 信息。改为 [[HTTPS]] 访问后成功命中规则并被记录。
- 修改规则加入 Block 动作后，目标网页被成功阻断。
- 演示屏蔽[[百度]]：分别添加了屏蔽 HTTP 请求、屏蔽 HTTPS 连接（[[tls]] 连接）、以及屏蔽 [[quic]]（[[http3]]）访问百度的规则，将访问方式全部堵死。讲者补充由于 [[socks]] 和 [[http代理]] 为明文，试图通过它们访问百度也会被规则阻断。

### DNS 污染规则与排错演示
- 讲者演示使用 [[v2a项目]] 的 [[gfset]] 数据库屏蔽 [[B哩哔哩]] 域名。
- 配置 [[知乎]] 的 [[DNS污染]] 规则（目标 IP 设为 1.1.1.1）。初次使用 [[nslookup]] 查询未被污染，排查发现是路由器开启了“[[DNS]]重定向”（在 [[prerouting链]] 上被劫持，未经过 [[F沃链]]）。
- 取消重定向后，污染生效。浏览器访问被污染的知乎会弹出 10016 报错页面，并提示证书与域名不匹配的警告（证书显示颁发给 [[cloudflare]]）。
- 讲者提到可在[[浏览器]]开启安全 DNS（[[doh]]）解决污染，但不建议此操作以免影响代理工具。讲者补充：访问谷歌出现 DNS 污染说明配置一定存在 DNS 泄露。

### 代理协议探测与阻断演示
- 讲者准备了五种节点类型进行测试。开启 [[openGFW]] 后：
  - [[shadowsocks]] 节点：全加密数据被识别并 Block（讲者展示目标端口为 11，推断）。
  - [[vmess]] + [[tcp]] 节点：同为全加密数据，被识别为 shadowsocks 并 Block。
  - [[vmess]] + [[ws]]（[[websocket]]）节点：因非全加密数据，未命中规则，可正常使用。
  - [[trojan]] 节点：未稳定输出日志，但实际用于科学上网时，其端口（讲者展示为 33333，推断）连接部分被屏蔽，导致网页加载不完整。
  - [[Hysteria]]（视频字幕作[[CSDD]]/[[csdd2]]）节点：[[openGFW]] 未实现其协议解析器，无法直接阻断，但可以通过将 [[quic]] 规则的域名改为该节点域名，通过屏蔽 [[quic]] 实现间接阻断。

### 总结与应用场景
- 讲者指出 [[openGFW]] 规则实际应用会更复杂，该项目刚出现不到两月，还在完善中，未来会推出图形化界面。
- 列举实际应用场景：控制家人访问防沉迷（精神鸦片）、公司屏蔽上班期间摸鱼的新手朋友/员工（推断）、[[IDC]] 商家用于屏蔽用户的代理数据。

## 常见坑与解决方案
- **问题描述**：在路由器上通过指令安装 `[[openfw]]` 所需要的依赖时，提示安装失败 [3]。
  - 解决方案：先执行 `opkgupdate` 更新一下软件源，再重新进行安装即可成功 [3]。

- **问题描述**：配置了 `[[DNS]]` 污染规则，但使用 `nslup` 查询时依然返回正确 IP，未成功污染 [5, 8]。
  - 解决方案：进入路由器的网络下的 DNS 设置，取消勾选“开启 DNS 重定向”，并点击保存并应用 [8]。

- **问题描述**：`[[DNS]]` 被成功污染后，尝试使用 HTTPS 访问该网站时会出现警告页面，提示证书和域名不匹配 [8]。
  - 解决方案：未明确提及（视频中仅说明这是污染导致的报错效果，未提供直接消除警告的方案） [8]。

## 最佳实践
- 推荐做法：通过修改 `[[QUIC]]` 规则中的域名为对应节点的域名，以此来屏蔽基于 `[[QUIC]]` 协议的 `[[Hysteria]]` 节点 [7]。
- 反模式：不建议通过在浏览器设置中开启安全 DNS（`[[DoH]]` / DNS over HTTPS）来解决 DNS 污染，这会使你的代理工具无法进行 DNS 劫持从而导致其他问题 [6, 8]。

## 关键概念

### [[opengfw]]
- **定义**：一个可以在家庭路由器上运行，实现类似长城防火墙（GFW）探测与屏蔽功能的开源项目 [1]。
- **视频上下文**：作者演示了如何在 OpenWrt 路由器上进行交叉编译并运行它，通过配置规则来实现 SNI 阻断、DNS 污染和加密代理协议的拦截 [1-3]。
- **为什么重要**：它是整个视频探讨的核心工具，能让用户在家庭网络中近距离观察和模拟网络封锁的实际运作原理 [1]。
- **关联概念**：对比 → [[GFW]]

### [[GFW]]
- **定义**：全称为长城防火墙，是国内用于检测和主动屏蔽用户访问国际互联网黑名单网站的系统 [1]。
- **视频上下文**：视频开篇指出其会导致无法直接访问外部网站，翻墙手段均是为了绕过其检测，而本视频旨在用 opengfw 在本地模拟其行为 [1]。
- **为什么重要**：理解网络审查机制和科学上网（翻墙）技术存在的根本背景 [1]。
- **关联概念**：未明确提及

### [[SNI 阻断]]
- **定义**：通过探查 HTTPS 请求中的 SNI（服务器名称指示）信息来拦截对特定域名的网络连线 [4]。
- **视频上下文**：作者配置规则探查 SNI，当使用 HTTPS 访问包含特定关键字或目标网站（如百度）时，触发 Block（阻断）动作导致网页无法返回内容 [4, 5]。
- **为什么重要**：是网络防火墙阻断加密网页访问的常用和基础技术手段 [4, 5]。
- **关联概念**：未明确提及

### [[DNS 污染]]
- **定义**：在进行 DNS 查询时，防火墙将查询结果篡改为错误的 IP 地址，导致用户无法连接到真实网站 [5]。
- **视频上下文**：作者演示了将知乎的 DNS 查询结果修改为 1.1.1.1，这导致浏览器随后弹出报错页面或证书不匹配的警告页面 [5, 6]。
- **为什么重要**：展示了不直接切断网络连接也能实现网站封锁的另一种主流审查手段 [5, 6]。
- **关联概念**：对比 → [[DoH]]

### [[shadowsocks]]
- **定义**：一种常见的加密代理协议，其数据特征在网络探测中表现为“全加密数据” [7]。
- **视频上下文**：作者测试发现 opengfw 能够识别出全加密的数据包特征，从而成功拦截并阻断使用此协议及类似特征（如 vmess+tcp）的节点连接 [7]。
- **为什么重要**：展示了防火墙如何通过分析流量特征而非仅依靠目标 IP 来识别并屏蔽特定的代理工具 [7]。
- **关联概念**：未明确提及

### [[hysteria]]
- **定义**：一种基于 QUIC 协议的加密代理工具，与 opengfw 的开发者是同一人 [1, 8]。
- **视频上下文**：被作者用来测试“以子之矛攻子之盾”，虽然 opengfw 没有内置该协议的解析器，但可以直接通过屏蔽其节点对应的 QUIC 协议来实现拦截 [8]。
- **为什么重要**：体现了翻墙与封锁之间的技术博弈，以及通过封杀底层传输协议来实现阻断的策略 [8]。
- **关联概念**：依赖 → [[QUIC]]

### [[QUIC]]
- **定义**：一种网络传输协议，在此视频语境中被用作部分网页（http3）或代理工具（如 hysteria）的底层通信协议 [4, 8]。
- **视频上下文**：作者在规则中演示了将使用 QUIC 访问百度的途径堵死，随后又通过屏蔽对应 SNI 的 QUIC 协议成功阻挡了 hysteria 节点的连接 [4, 8]。
