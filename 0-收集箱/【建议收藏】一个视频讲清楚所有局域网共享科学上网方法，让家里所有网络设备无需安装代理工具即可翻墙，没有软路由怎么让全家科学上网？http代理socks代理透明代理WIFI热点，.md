## 一句话总结
本视频详解了在局域网内通过[[代理共享]]、[[网关共享]]与[[路由共享]]三种模式实现全设备科学上网的原理及实操步骤。

## 目标受众与前置知识
- **适合谁看**：想让家中所有网络设备（无[[软路由]]）无需单独安装代理工具即可实现科学上网的用户（推断）。
- **前置知识**：[[IP地址]]、[[网关]]、[[局域网]]、基本代理工具（如 [[clash]]）概念。

## 核心观点
1. **代理共享模式最简单但有局限**：一键开启，但缺点显著。
   - 为什么重要：实现门槛最低，所有主流客户端均支持。
   - 依据：讲者演示在 [[v2ng]] 或 [[clash]] 中仅需勾选“允许局域网连接”即可（讲者原话[1, 2]）。但其缺陷在于每台设备需单独设置，[[http代理]] 不支持 [[udp]] 数据，且部分应用会绕过系统代理走直连（讲者原话[3, 4]）。
2. **网关共享模式（透明代理）更完美**：弥补了代理共享的全部缺点。
   - 为什么重要：可以接管局域网设备的所有流量，设备本身无感。
   - 依据：通过将设备的默认网关设为开启了 [[网关共享]] 的设备的 [[IP地址]]（如讲者演示设为 [[Cali]] 虚拟机的 192.168.0.120，屏幕展示[5]），流量强制经过代理处理，还能轻松实现 [[链式代理]]（讲者原话[6]）。
3. **路由共享模式适合无局域网环境**：脱离现有局域网独立运作。
   - 为什么重要：适合外出出差等场景，连上热点直接翻墙。
   - 依据：代理设备自身分配IP和网关（讲者原话[5]）。但缺点是多了一层 [[net]]（推断指 [[NAT]]，讲者原话[7]），且大部分 [[安卓]] 手机必须 [[Root]] 配合第三方工具（如 [[VPN热点]]）才能将 [[VPN]] 网络共享出去（讲者原话[7, 8]）。

## 概念关系
- [[代理共享]] ↔ [[网关共享]]：前者可能因应用不走代理导致流量直连，后者直接在网关层级接管所有流量（透明代理）。
- [[http代理]] ↔ [[socks代理]]：前者不支持传输 [[udp]] 数据，后者支持（讲者原话[3]）。
- [[软路由]] ↔ 闲置 [[安卓]]：前者在路由功能上更专业，后者学习成本低且可突破设备数量限制（讲者原话[9]）。

## 详细笔记
### 1. 家庭网络基础与代理共享模式
- **网络基础**：光猫连接路由器，路由器作为局域网的 [[网关]]（假设IP为192.168.0.1），给设备分配内网IP。设备上网请求先交给网关处理[1]。
- **代理共享原理**：设备请求先交给开启共享的电脑/手机 → 代理软件处理 → 网关 → 互联网[1]。
- **实操教程**：
  - **手机共享给电脑**：在 [[v2ng]] 中勾选“允许局域网连接”，电脑系统代理填入手机的 [[IP地址]] 和 [[http代理]] 端口（屏幕展示：10809）。或使用 [[clash]] 设置混合端口（如屏幕展示：7890）[1, 2]。
  - **电脑共享给手机**：电脑端 [[v2n]] 或 [[clash]] 勾选允许局域网连接；手机连接 [[WiFi]] 时修改配置，将代理设为手动，填入电脑IP和对应端口（如屏幕展示：10811或7890）[3]。
  - **补充说明**：若需代理 [[udp]] 流量，需借助第三方 [[VPN]] 工具连接电脑的 [[socks代理]] 端口（如屏幕展示：10810）[3]。[[iPhone]] 和 [[苹果手机]] 用户亦可同理修改 WiFi 代理，或用 [[小火箭]] 开启代理共享[4]。

### 2. 网关共享模式（透明代理）
- **核心原理**：不改变路由器原本的网关身份，而是让路由器将某台代理设备（如电脑）的IP作为网关分配给其他设备，或手动修改单台设备的网关[10]。
- **实现步骤（以 Windows 借助虚拟机为例）**：
  - [[Windows]] 系统原生不支持此模式。讲者演示通过带有图形化界面的 [[Linux]] 发行版（如 [[CY虚拟机]] / [[Cali]]）实现[6]。
  - 虚拟机网络使用桥接模式。开启 IP 转发后，在 [[clash for Window]] 中安装服务模式并开启 [[ton模式]]（推断指 TUN 模式）[6]。
  - 修改被代理设备（如电脑或手机的 [[WiFi]] 设置）的默认 [[网关]]，将其改为虚拟机的 [[IP地址]]（屏幕展示：192.168.0.120）。修改后设备即可无感访问 [[谷歌]] 等网站[5]。

### 3. 路由共享模式
- **核心原理**：开启共享的设备单独创建一个新网段，充当路由器，负责给连接的设备分配 IP 和网关。适合脱离局域网的环境（如出差）[5, 7]。
- **Windows 实操**：需双网口（可通过 USB 转网卡）。在电脑开启 [[clash]] 的 [[ton模式]]（推断指 TUN 模式）后，将其创建的虚拟网卡通过系统设置“共享”给另一个空闲的局域网物理网卡。其他设备用网线连接该网口或连接后接无线AP即可[7, 11]。
- **安卓实操**：
  - 通过开启 [[WiFi]] 热点或 [[USB共享]]。部分手机自带“允许客户端使用VPN”功能，开启后连入热点即可科学上网访问 [[谷歌]] [11]。
  - 若手机不支持该功能，必须先将手机 [[Root]]，随后安装名为 [[VPN热点]] 的第三方工具。打开对应网络接口（WiFi 热点或 USB 共享网络接口），即可将 [[VPN]] 共享给其他设备[8]。

## 步骤拆解
1. **手机代理共享给电脑（代理共享）**
   - 具体操作描述：手机端代理软件（以 [[v2ng]] 或 [[clash]] 为例）开启“允许局域网连接”，查看手机连接 WiFi 后的内网 IP 以及代理软件提供的 http 代理端口。在电脑的系统代理设置中启用代理，填入手机 IP 及对应端口 [2, 3]。
   - 关键代码/命令：未明确提及。
   - 注意事项：需保持手机端 VPN 服务始终开启，关闭手机 VPN 会导致电脑无法访问外网 [3]。

2. **电脑代理共享给手机（代理共享）**
   - 具体操作描述：电脑端代理软件勾选“允许局域网连接”，获取电脑所在的内网 IP 及 http 代理端口。进入手机 WiFi 配置页面，将“配置代理”改为手动，填入电脑 IP 和端口并保存 [4, 5]。苹果手机同理 [5]。
   - 关键代码/命令：未明确提及。
   - 注意事项：系统层面的 http 代理不支持传输 [[UDP]] 数据；并且部分应用会绕过系统代理直连，无法接管全部流量 [4, 5]。

3. **借助虚拟机实现网关共享（透明代理）**
   - 具体操作描述：在 [[Windows]] 电脑上运行带有桥接网络模式的 [[Linux]] 虚拟机（与物理机处于同局域网），切换 root 用户并开启 IP 转发。打开 Clash for Windows，安装服务模式并开启 [[TUN 模式]]。最后修改局域网其他设备或路由器 DHCP 的默认网关为虚拟机的 IP 地址 [7, 10]。
   - 关键代码/命令：未明确提及（视频仅说明“使用这条指令开启 IP 转发”，未展示具体指令本身） [7]。
   - 注意事项：作为网关的设备若关机，会导致网内修改过网关的设备断网，此时需将设备的网关手动改回默认路由器网关并重启 WiFi [10, 11]。

4. **电脑开启路由共享模式**
   - 具体操作描述：为电脑准备一张用于局域网的主网卡和一张空闲的辅助网卡。开启 [[clash]] 的 [[TUN 模式]]，在生成的虚拟网卡属性中勾选共享（允许其他网络用户通过此计算机的 Internet 连接来连接），并将其指向那张空闲的物理网卡。通过网线将空闲网口连接至其他电脑或无线 AP 设备 [8, 11]。
   - 关键代码/命令：未明确提及。
   - 注意事项：此模式会单独开启一个新的网段，产生额外的一层 [[NAT]]，对性能有一定影响；默认情况下新旧两个网段设备无法互相通信 [8]。

5. **安卓手机开启路由共享模式（WiFi 热点 / USB 共享）**
   - 具体操作描述：手机开启 WiFi 热点或 USB 网络共享，在热点设置中勾选“允许客户端使用 VPN”。如果手机系统本身没有该选项，必须 [[ROOT]] 手机，安装并打开第三方工具 [[VPN Hotspot]]，开启对应网络接口的开关 [9, 11]。
   - 关键代码/命令：未明确提及。
   - 注意事项：若未勾选/未开启 VPN 共享权限，其他设备即使连上热点或 USB 也仅能访问普通国内网络，无法科学上网 [11]。

## 常见坑与解决方案
- **问题描述**：手机 WiFi 设置中的系统代理通常是 [[http 代理]]，不支持传输 [[UDP]] 数据 [4]。
  - 解决方案：借助第三方 VPN 工具（如 [[v2n]]）创建 [[socks 代理]]（例如 [[SS 代理]]）建立连接，同时前提是所使用的节点本身必须支持代理 UDP 数据 [4]。
- **问题描述**：在代理共享模式下，有些应用会绕过系统代理走直连，导致各种网络问题，无法接管所有网络流量 [5]。
  - 解决方案：改用网关共享模式（[[透明代理]]），在网关底层强制接管所有设备流量，解决特定应用或设备不支持代理的问题 [6]。
- **问题描述**：安卓手机开启 WiFi 热点或 USB 网络后，其他设备连上却无法实现科学上网 [11]。
  - 解决方案：在热点设置中打开“允许客户端使用 VPN”。若系统自带功能缺失，需对手机进行 [[ROOT]]，通过安装 [[VPN Hotspot]] 来强行共享 VPN 网络 [9, 11]。
- **问题描述**：关闭作为网关的电脑/虚拟机后，原本修改了网关的设备无法再连接任何网络 [11]。
  - 解决方案：进入设备的网络或 WiFi 设置，重新修改回默认网关并重启网络连接 [11]。

## 最佳实践
- 推荐使用 [[clash]] 提供的复写混合端口功能，使得 http 与 socks 使用同一个代理端口，在配置上比分离端口的 [[v2ray]] 更方便 [3]。
- 推荐使用网关共享模式来实现 [[链式代理]]。由于网关模式下设备默认处于代理环境，无需复杂配置，只需再连接落地节点即可轻松完成链式代理部署 [7]。
- 在没有局域网环境（如外地出差）的场景下，推荐使用路由共享模式（开启手机热点），这能方便地直接共享手机流量和网络环境 [8, 10]。

## 关键概念

### [[代理共享]]
- **定义**：通过在一台设备上运行代理软件并开启允许局域网连接，让其他设备通过设置该设备的IP和端口来实现科学上网的模式 [1]。
- **视频上下文**：作为最常见且最简单的共享模式被首个介绍，作者详细演示了在手机和电脑间互相共享代理的具体流程 [1-4]。
- **为什么重要**：虽然实现简单且客户端支持丰富，但存在需逐台设置、部分应用会绕过代理走直连以及不支持UDP传输等致命缺点 [4]。
- **关联概念**：对比 → [[网关共享]]

### [[网关共享]]
- **定义**：将开启了科学上网环境的设备（如手机或开启Tun模式的虚拟机）的内网IP设置为局域网其他设备的默认网关，从而接管所有网络流量的模式 [5, 6]。
- **视频上下文**：作者演示了如何利用安卓手机或Linux虚拟机作为局域网网关，让局域网内所有设备免设置直接翻墙 [5-7]。
- **为什么重要**：能接管系统所有流量（包括不走系统代理的应用），且设备本身不知道自己被代理了，这种方式也被称为透明代理，弥补了代理共享模式的缺陷 [5]。
- **关联概念**：依赖 → [[网关]]

### [[路由共享]]
- **定义**：共享设备单独开启一个网段（如WiFi热点或USB共享），承担路由器功能负责给连接的设备分配IP和网关等信息的共享模式 [7, 8]。
- **视频上下文**：被介绍为适合在没有局域网环境（如出差）时使用的方案，作者通过Windows空闲网卡和安卓手机WiFi热点进行了配置演示 [7-9]。
- **为什么重要**：设备连上就能直接使用且无需单独去设置网关，能脱离局域网环境，但对设备性能有一定影响且会多出一层NAT转换 [8]。
- **关联概念**：对比 → [[网关共享]]

### [[系统代理]]
- **定义**：在设备的系统网络设置中手动配置的代理IP和端口，用于将设备的网络访问请求优先转发给运行代理软件的节点处理 [1-3]。
- **视频上下文**：在演示代理共享时，作者在Windows的网络设置和手机的WiFi配置中多次填入IP和端口来启用此功能 [2, 3]。
- **为什么重要**：并不是所有应用程序都会走系统代理，部分应用会直接绕过它走直连，导致无法接管所有网络流量并引发各类网络问题 [3, 4]。
- **关联概念**：属于 → [[代理共享]]

### [[http代理]]
- **定义**：一种不支持传输UDP数据包的代理协议格式 [3, 4]。
- **视频上下文**：在设置设备的系统代理时被反复指出，大部分设备（如WiFi设置中）的系统代理默认使用的就是这种协议格式 [2-4]。
- **为什么重要**：其不支持UDP的特性构成了代理共享模式的一大缺陷，使得某些依赖UDP传输的应用无法正常工作 [3, 4]。
- **关联概念**：对比 → [[socks代理]]

### [[socks代理]]
- **定义**：一种能够支持传输UDP数据包的网络代理协议格式 [3]。
- **视频上下文**：作者在演示手机端WiFi设置不支持直接配置socks代理时，说明需要借助第三方VPN工具创建该类型代理来专门处理UDP数据 [3]。
- **为什么重要**：能够弥补http代理的缺陷，但其实际能否成功传输UDP最终仍取决于所连接的节点服务器本身是否支持代理UDP数据 [3]。
- **关联概念**：对比 → [[http代理]]

### [[网关]]
