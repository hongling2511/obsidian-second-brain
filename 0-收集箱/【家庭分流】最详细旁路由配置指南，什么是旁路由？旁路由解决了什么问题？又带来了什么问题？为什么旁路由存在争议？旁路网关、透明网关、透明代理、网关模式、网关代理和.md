## 一句话总结
讲者 [[不良林]] 详细演示了 [[旁路由]] 的配置与按设备分流方法，并剖析了导致国内网站访问异常及无法翻墙的网络故障原理。

## 目标受众与前置知识
- **适合谁看**：需要家庭网络部分设备科学上网，且不想改变现有网络拓扑的用户。（推断）
- **前置知识**：[[主路由]]、[[软路由]]、[[IP]]、[[网关]]、[[dhcp]]、[[交换机]]、[[NAT]]

## 核心观点
1. **旁路由能实现灵活的按设备分流且不破坏原有网络拓扑**
   - 为什么重要：避免改动 [[主路由]] 导致全家断网，解决多设备不同代理需求。
   - 依据：视频中演示通过配置 [[openwrt]] 的 [[dhcp]] 服务，利用标签为 [[苹果]]、[[安卓]] 设备分配不同的 [[网关]]（主路由或旁路由）实现按需分流（屏幕展示）。
2. **旁路由的非标准拓扑极易引发非对称路由问题**
   - 为什么重要：会导致设备能正常翻墙，但无法访问国内网站。
   - 依据：访问 [[百度]] 等国内网站的回程数据直接由主路由发给电脑，不经过旁路由，被旁路由防火墙判定为无效数据包而丢弃（讲者原话）。
3. **开启 IP 动态伪装可解决 WiFi 设备访问国内网站异常，但损耗性能**
   - 为什么重要：提供了解决 [[网桥]] 和 [[iptables]] 机制导致的丢包问题的方案，但需权衡性能。
   - 依据：开启 [[IP动态伪装]]（[[NAT]]）后，相当于将 [[非对称路由]] 强制转为对称路由，但数据会经过两次 NAT 处理，对性能有一定影响（讲者原话）。
4. **IPv6 是导致旁路由无法翻墙的常见原因**
   - 为什么重要：排查网络问题的关键点，建议普通用户直接关闭。
   - 依据：[[IPv6]] 不通过 [[dhcp]] 下发网关，获取到 IPv6 的设备会绕过旁路由直接通过主路由发到互联网（讲者原话）。

## 概念关系
- [[旁路由]] ↔ [[主路由]]：主路由负责连接互联网和分发公网数据，旁路由作为辅助设备连接在局域网内，专门负责处理科学上网等特定设备的加解密数据。
- [[dhcp]] → [[网关]]：通过修改 DHCP 服务的下发规则，可以决定特定设备获取到的默认网关是主路由还是旁路由，从而控制其流量走向。
- [[非对称路由]] → [[丢弃无效数据包]]：非对称路由导致设备请求与回程的数据包路径不一致，旁路由未能完整追踪 [[TCP]] 连接状态，导致国内网站数据被防火墙的「丢弃无效数据包」规则拦截。
- [[IP动态伪装]] → [[NAT]]：IP 动态伪装本质上是进行 NAT 转换，修改数据包源 IP，强制主路由识别为全新数据流并重新 NAT，解决流量被丢弃的问题。

## 详细笔记
### 旁路由的定义与解决的痛点
- 传统 [[主路由]] + [[二级路由]] 模式在分流配置上繁琐，且改动主路由易导致全家断网。若 [[软路由]] 运行多个 [[插件]] 易引发冲突 [1]。
- [[旁路由]] 不改变原网络结构，接在主路由的 [[LAN口]]（L口）旁，专门负责处理科学上网数据的加解密。数据通过旁路由加密后交由主路由转发给互联网上的 [[节点]]，解密后再访问 [[谷歌]] 等网站 [2]。

### 旁路由的基础网络配置
- 确认当前局域网网段（演示中为 192.168.2.x 系列，主路由IP假设为 192.168.2.1）[3]。
- 电脑插到刚刷好 [[openwrt]] 的软路由 L 口，通过分配的默认网关进入后台（若IP为 169.254.x.x 说明插错口，讲者原话）[4]。
- 在接口页面编辑 L 口，协议改为静态地址。将 IPv4 地址改为同网段空闲IP（演示改为 192.168.2.2），网关和 [[DNS]] 指向主路由（192.168.2.1）[5]。
- 删除 [[WAN口]]（万口），将空闲的物理网口（eth1）桥接到 [[网桥]]（br-lan），相当于作为 [[交换机]] [5, 6]。
- 关闭 [[小米]]、[[华硕]]、[[tplink]] 等品牌主路由的 [[dhcp]] 服务，由旁路由统一提供 DHCP [6, 7]。

### 按设备分流的进阶配置
- 利用 [[openwrt]] 默认的 [[dnsmasq]] 提供 DHCP 和 DNS 服务，为不同设备分配不同网关 [8, 9]。
- **步骤**：在 DHCP 静态地址分配中，根据设备的 [[MAC地址]] 进行静态 IP 绑定，并设置自定义主机名和标签（如 dect 代表直连，poxy 代表代理）[10-12]。
- **小部分设备直连，大部分设备代理**：在 DHCP 高级选项中输入 `tag:Dect,3,192.168.2.1` 和 `tag:Dect,6,192.168.2.1`，使标签为 Dect 的 [[苹果]] 和 [[安卓]] 手机网关/DNS指向主路由，其余设备默认指向旁路由 [11]。
- **小部分设备代理，大部分设备直连**：修改选项为 `tag:!poxy,3,192.168.2.1`，意为非 poxy 标签的设备全部分配主路由网关，仅标为代理的设备分配旁路由网关 [13]。

### 旁路由常见网络故障与排查（非对称路由）
- **现象**：电脑网关指向旁路由后，能正常访问 [[推特]]，但神乎其技地无法访问 [[百度]]、[[知乎]] [14]。
- **原因一（防火墙机制）**：因偷开防火墙的 [[丢弃无效数据包]] 功能。电脑向目标发起 [[TCP]] [[三次握手]]，[[代理工具]] 判定国内 IP 不走代理，交主路由转发并进行 [[NAT]]。目标服务器同意连接的回程数据经主路由 [[反向NAT]] 后直发电脑，不经过旁路由。形成 [[非对称路由]] 后，旁路由因未收到回程包，将电脑后续发包判定为无效并丢弃 [14-16]。
- **解决一**：取消勾选「丢弃无效数据包」。讲者同时提及 [[SYN泛洪防御]] 功能关不关无所谓，不影响数据转发 [16, 17]。
- **原因二（WiFi连接设备异常）**：通过网线连接正常，但通过 [[WiFi]] 连接的设备无法访问国内网站。WiFi 数据需转为以太网帧经虚拟 [[网桥]]。部分主路由默认将网桥互访数据交由 [[iptables]]（或新版 [[nftables]]）处理，记录其 [[五元组]] 且不进行 NAT。数据经旁路由回传至主路由时，匹配上原五元组仍不进行 NAT，导致携带 [[内网 IP]] 的数据包在互联网被丢弃 [17-19]。
- **解决二**：开启 LAN 区域的 [[IP动态伪装]]。强制对 WiFi 数据进行 [[NAT]] 处理，源 IP 替换为旁路由 IP（192.168.2.2），迫使主路由识别为新数据流并执行第二次 NAT 处理。此法虽解决丢包，但会因两次 NAT 影响性能 [20, 21]。

### 旁路由的争议、IPv6及扩展应用
- [[IPv6]] 不通过 DHCP 下发网关，设备易直接通过主路由发往互联网绕过旁路由导致无法翻墙，讲者建议普通用户直接关闭 IPv6 [21, 22]。
- 旁路由非标准网络拓扑，不适合跑 [[BGP]] 的小型 IDC 用户 [22]。
- 广义旁路由形式：在非路由系统跑 [[v2ray]] 或 [[clash]] 做 [[透明代理]]（又称 [[旁路网关]]、[[透明网关]]、[[网关模式]]、[[网关代理]]），或使用 [[Windows]]、[[Linux]]、[[Apple TV]]、[[Mac mini]] 共享网络（如 电玩科技AK 给 王四充 演示的四级网关模式），只要开启 IP 转发，本质皆为旁路由 [23]。
- 讲者提及 [[DNS泄露]] 也是类似争议话题。最后讲者建议过年多陪家人以及隔壁村的 [[春花]] 和 [[小美]]，少折腾软路由 [24]。

## 常见坑与解决方案
- **问题描述**：配置完成后，电脑或手机依然无法科学上网，查看默认网关还是主路由的 IP [21, 22]。
  - 解决方案：由于旧的 [[DHCP]] 租约未到期，需要将电脑网卡禁用后重新启用，或者手机断开 WiFi 重连，以重新获取旁路由分配的 IP 信息 [21, 22]。
- **问题描述**：设备可以科学上网，却无法访问国内（不需要代理的）网站 [4]。
  - 解决方案：在防火墙设置中，取消勾选“丢弃无效数据包”（该问题由非对称路由引发，导致 [[TCP]] 握手数据包被旁路由拦截）[4, 23]。
- **问题描述**：通过网线连接的设备可正常上网，但通过 WiFi 连接的设备只能科学上网，无法访问国内网站 [24]。
  - 解决方案：在防火墙中开启 LAN 区域的“IP 动态伪装”（即开启 [[NAT]]，解决 [[iptables]] 对未经过以太网桥的 WiFi 数据包的判定问题）[5-7]。
- **问题描述**：由于获取到 [[IPv6]] 地址导致流量绕过旁路由，直接通过主路由发往互联网，出现无法翻墙的情况 [7]。
  - 解决方案：直接关闭 [[IPv6]] [8]。

## 最佳实践
- 条件允许的情况下，建议将电脑等设备通过网线直接插回主路由，这样设备间的互访可以直接通过交换芯片，不需要经过旁路由的虚拟网桥，效率更高 [14]。
- 大部分用户如果没有公网访问需求，强烈建议直接关闭 [[IPv6]]，以省去因配置不当导致的各种网络烦恼 [8]。
- 防火墙设置中的“SYN 泛洪防御”选项对数据转发没有影响，开启或关闭均可 [23, 24]。
- 旁路由并不适合网络结构复杂的场景（如小型 IDC 或运行 [[BGP]] 的环境），用户应根据自身实际家庭网络需求选择是否使用旁路由 [8]。

## 关键概念

### [[旁路由]]
- **定义**：接在主路由旁边，负责处理科学上网等额外工作而不改变原有网络结构的路由设备 [1, 2]。
- **视频上下文**：作者指出它通过让网关指向自己来接管部分或全部设备的流量，从而实现分流和代理，但因其非标准网络拓扑结构而存在争议 [1, 3, 4]。
- **为什么重要**：视频的核心探讨对象，是实现家庭网络分流的高级手段，也是容易导致网络故障的源头 [3, 4]。
- **关联概念**：对比 → [[主路由]]

### [[主路由]]
- **定义**：负责连接互联网、拨号获取公网IP并作为家庭局域网最初核心设备的路由器 [2, 5]。
- **视频上下文**：视频中主路由负责原有的网络基础工作，当设备不需要代理时，数据会直接经过主路由访问互联网 [1, 3]。
- **为什么重要**：理解旁路由工作原理的基础，两者共同构成了完整的网络拓扑结构 [2]。
- **关联概念**：对比 → [[旁路由]]

### [[网关]]
- **定义**：局域网内所有网络设备访问其他网络时必须经过的IP地址 [6]。
- **视频上下文**：作者通过将设备的网关修改为指向主路由或旁路由，来决定该设备的流量是否要经过代理，从而实现按设备分流 [3, 7, 8]。
- **为什么重要**：控制设备流量走向的核心配置，旁路由起作用的关键就是接管设备的默认网关 [3]。
- **关联概念**：依赖 → [[dhcp]]

### [[dhcp]]
- **定义**：负责为局域网内的每一台网络设备自动分配IP地址、网关、DNS等信息的服务 [6]。
- **视频上下文**：为解决手动修改设备网关过于繁琐的问题，作者关闭主路由的dhcp，改由旁路由提供服务，并利用高级选项为不同标签的设备下发不同的网关 [3, 7, 9, 10]。
- **为什么重要**：实现局域网设备自动化网络配置和精细化分流（按设备分配网关）的基础底层服务 [3, 6]。
- **关联概念**：未明确提及

### [[nat]]
- **定义**：网络地址转换处理，将数据包里的源IP替换为路由器的公网IP或内网IP [11]。
- **视频上下文**：作者在讲解主路由如何转发数据给百度以及旁路由解决WiFi设备无法访问国内网站的问题时，详细描述了nat及反向nat的处理过程 [11, 12]。
- **为什么重要**：理解数据包在内外网及不同路由器之间穿梭机制的核心概念 [11]。
- **关联概念**：扩展 → [[IP动态伪装]]

### [[非对称路由]]
- **定义**：往返的数据包经过不同网络路径的现象 [13]。
