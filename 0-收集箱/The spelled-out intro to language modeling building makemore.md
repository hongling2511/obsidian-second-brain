# The spelled-out intro to language modeling: building makemore

## 一句话总结
从零构建基于字符的 [[bigram language model]]，涵盖从纯统计计数到基于 [[神经网络]] 和 [[梯度下降]] 的完整实现过程 [1-3]。

## 内容类型
tech_tutorial

## 目标受众与前置知识
- **适合谁看**：希望从底层逻辑深入理解 [[大型语言模型]] (LLM) 运作机制的 AI 初学者、开发者，以及想要夯实 [[PyTorch]] 基础的人群。
- **前置知识**：基本的 [[Python]] 编程能力，基础的 [[微积分]]（了解导数和梯度），基础的 [[线性代数]]（如 [[矩阵乘法]]），以及对基本计算图和反向传播有初步概念（如了解 [[micrograd]]） [1, 4, 5]。

## 核心观点
1. **[[bigram language model]] 的核心是预测下一个字符的概率**：通过给定前一个字符，模型学习序列中下一个最可能出现的字符分布。
   - 为什么重要：这是理解更复杂的现代模型（如基于 [[Transformer]] 的 [[GPT]]）的基础构建块 [6, 7]。
   - 说服力评估：极强。作者通过训练一个极其简单的模型成功生成了类似真实名字的伪造词汇（如 dontel, irot 等），直观验证了该逻辑的有效性 [6]。

2. **统计计数法与基于 [[梯度下降]] 的 [[神经网络]] 在数学本质上可以等价**：直接统计字符共现频率并归一化，与随机初始化权重并通过最小化 [[negative log likelihood]] 训练单层线性网络，最终会得到相同的概率分布矩阵。
   - 为什么重要：它在传统统计学和现代深度学习之间搭建了一座认知桥梁，解密了 [[神经网络]] 黑盒 [3, 8]。
   - 说服力评估：极强。作者分别用两种方法计算，在代码层面得出了完全一致的 [[损失函数]] 值（约 2.45），用不可辩驳的运行结果证明了这一点 [3, 9, 10]。

3. **[[PyTorch]] 的 [[broadcasting]] 机制极其强大但也极易引发隐蔽的 Bug**：在进行张量维度操作（如 `torch.sum` 时不使用 `keepdim=True`）时，系统会自动补齐维度，可能导致行归一化变成列归一化。
   - 为什么重要：这是新手在构建多维 [[张量]]（[[tensor]]）操作时最常踩的坑，会导致模型默默地学错而没有任何报错 [11, 12]。
   - 说服力评估：强。作者故意演示了这段错误代码，展示了它是如何不报错但产生垃圾结果的，让人印象深刻 [13, 14]。

## 详细笔记
- **数据集准备与基础统计（约 0-20% 进度）**
  加载 `names.txt` 数据集，将其拆分为包含 32000 个名字的列表 [1, 17]。引入首尾特殊标识符（如 `<S>` 和 `<E>`，后优化为单独的 `.`），使用 [[Python]] 字典和 `zip` 函数统计字符两两相邻出现的频率，建立最基础的 [[bigram language model]] [18-21]。
- **使用张量重构与采样（约 20-40% 进度）**
  引入 [[PyTorch]] 的 2D [[张量]] (`torch.tensor`) 替换字典以提高效率，构建一个 27x27 的计数矩阵 [22, 23]。通过按行求和并进行除法操作，将绝对计数转化为概率分布 [24, 25]。深入探讨了 [[broadcasting]] 语义及其潜在陷阱 [11, 26]。随后使用 `torch.multinomial` 结合确定的随机数生成器进行字符采样，生成新的名字序列 [25, 27]。
- **模型评估指标构建（约 40-60% 进度）**
  引入 [[最大似然估计]]（Maximum Likelihood Estimation），解释为什么我们需要将多个概率连乘，并为了计算方便将其转化为对数相加，推导出模型评估的核心指标：[[negative log likelihood]] (负对数似然损失) [28-30]。为解决某些未出现的字符组合导致概率为 0（损失为无穷大）的问题，引入了模型平滑（[[model smoothing]]）技术 [31, 32]。
- **基于神经网络的重构（约 60-80% 进度）**
  转变思路，将字符映射为整数后，使用 [[one-hot encoding]] 将其转换为可以输入神经网络的浮点数向量 [33-35]。构建了一个无偏置项、无非线性激活函数的单层线性神经网络（本质是 [[矩阵乘法]]）[4, 36]。网络的输出被称为 [[logits]]（对数计数），通过求指数运算（`exp`）和归一化，实现了 [[softmax]] 层的效果，输出新的概率分布 [37-39]。
- **梯度下降与正则化（约 80-100% 进度）**
  利用 [[PyTorch]] 的反向传播（`loss.backward()`）自动计算梯度，通过更新权重矩阵 $W$ 来最小化损失 [40, 41]。在完整数据集上运行多轮迭代，发现神经网络最终收敛的损失值（约 2.45）与纯统计计数法完全一致 [9, 10]。最后，解释了在神经网络损失函数中加入权重衰减（即 [[正则化]]）等同于统计学方法中的模型平滑，两者在控制概率分布均匀性上起到了异曲同工的作用 [42, 43]。

## 步骤拆解
1. **数据加载与预处理**
   - 具体操作描述：读取 `names.txt` 中的单词，提取所有的唯一字符，并构建字符到整数（`s2i`）和整数到字符（`i2s`）的查找表。定义特殊的起始/结束标记（单一特殊字符 `.`，索引设为 0）[9-11]。
   - 关键代码/命令：
     `chars = sorted(list(set(''.join(words))))` [10, 12]
     `s2i = {s:i+1 for i,s in enumerate(chars)}; s2i['.'] = 0` [11, 12]
   - 注意事项：需要将所有的字符串操作映射到基于整数索引的张量操作上，以便后续计算[10]。

2. **构建基于统计计数的 [[bigram language model]]**
   - 具体操作描述：遍历数据集统计所有相邻两个字符（bigram）出现的频次，并将计数存入一个 27x27 的二维 PyTorch 张量 `N` 中[1, 3, 11]。
   - 关键代码/命令：`N[ix1, ix2] += 1` [3]

3. **计算概率分布与文本采样**
   - 具体操作描述：将计数矩阵 `N` 的每一行归一化，转换为概率分布矩阵 `P`。随后基于给定的当前字符，根据概率分布抽取下一个字符的索引，循环直到生成结束标记[8, 13, 14]。
   - 关键代码/命令：
     `P = (N + 1).float(); P /= P.sum(1, keepdim=True)` （加1是为了平滑）[13, 15, 16]
     `ix = torch.multinomial(P[ix], num_samples=1, replacement=True, generator=g).item()` [17, 18]
   - 注意事项：`torch.multinomial` 接收概率分布，输出基于该概率采样的索引。使用 `generator` 可以保证采样的可重复性[8]。

4. **模型评估：计算 [[negative log likelihood]] (NLL)**
   - 具体操作描述：为了用单一数值衡量模型好坏，提取训练集真实 bigram 在模型中对应的概率，计算它们的乘积（[[likelihood]]），转换为对数求和（[[log likelihood]]），最后取反并求平均，作为损失函数 (Loss)[19-22]。
   - 关键代码/命令：`nll = -log_likelihood / n` [22]
   - 注意事项：引入 [[model smoothing]]（如在计数矩阵中全部加 1），以避免在遇到模型未见过的组合时输出零概率，从而导致对数概率变为负无穷大（Infinite Loss）[16, 23, 24]。

5. **神经网络化：构建数据集与 [[one-hot encoding]]**
   - 具体操作描述：不直接统计计数，而是将问题转化为分类任务。构建输入集 `xs` 和目标集 `ys`。将输入的整数索引转换为独热编码（One-Hot）向量，使其能够与权重矩阵进行相乘[25-27]。
   - 关键代码/命令：`xenc = F.one_hot(xs, num_classes=27).float()` [27, 28]

6. **实现前向传播与 [[softmax]]**
   - 具体操作描述：创建一个单层线性神经网络（无偏置，无非线性激活函数），通过输入向量和权重矩阵 `W` 相乘得到 [[logits]]（可理解为对数计数）。然后对 logits 取指数（转换为正数计数），最后按行归一化得到概率分布[28-31]。
   - 关键代码/命令：
     `logits = xenc @ W` [6]
     `counts = logits.exp()` [30]
     `probs = counts / counts.sum(1, keepdims=True)` [32]
   - 注意事项：这个取指数再归一化的操作序列在神经网络中被称为 [[softmax]] [31]。

7. **反向传播与 [[gradient descent]] 优化**
   - 具体操作描述：提取网络对真实下一个字符预测的概率，计算平均负对数似然损失。清理之前计算的梯度，进行反向传播计算每个权重的梯度，并沿负梯度方向更新权重[33-38]。
   - 关键代码/命令：
     `W.grad = None` [36]
     `loss.backward()` [36]
     `W.data += -0.1 * W.grad` [38]
   - 注意事项：基于梯度的优化结果最终与纯统计计数的矩阵结果一致，但基于神经网络的架构具有极强的扩展性，是通往复杂模型（如 Transformers）的基础[39, 40]。

8. **引入 [[Regularization]]**
   - 具体操作描述：在损失函数中添加对权重矩阵 `W` 的惩罚项（例如权重的平方均值），促使权重趋向于 0，使输出的概率分布更均匀[41]。
   - 关键代码/命令：`loss += 0.01 * (W**2).mean()` [41]
   - 注意事项：神经网络中的正则化（Regularization）等价于统计计数方法中的标签平滑（Label Smoothing）[42, 43]。

## 常见坑与解决方案
- **问题描述**：张量的 [[broadcasting]] 导致计算逻辑错误且不报错（Silent Bug）。例如在归一化矩阵求和时丢弃了维度 `keepdim=False`，使得行向量在除法时被错位广播，变成了对列进行归一化[44-46]。
  - **解决方案**：在进行张量求和用于逐行相除运算时，严格设置 `keepdim=True`。时刻检查张量的 `.shape`，熟悉从右向左对齐的 broadcasting 语义规则[15, 47]。
- **问题描述**：PyTorch Tensor API 构建差异导致的隐式类型错误。使用大写的 `torch.Tensor` 和小写的 `torch.tensor` 行为不同，前者默认返回 Float，后者推断类型（如传入整数返回整数类型）[48, 49]。
  - **解决方案**：推荐始终使用小写的 `torch.tensor` 初始化，并通过 `.dtype` 显式确认类型，避免输入到神经网络时出现数据类型不支持的错误[26, 49]。
- **问题描述**：模型遇到训练集中未出现的连续字母组合（如 "jq"），预测概率为 0，导致对数损失求值为负无穷大 `-infinity` [16, 23]。
  - **解决方案**：使用 [[model smoothing]]，即为所有的频率统计基础加上一个极小的基数（例如 `+ 1`），使得任何字符组合都有极低的“保底”概率，消除 0 概率节点[16, 24]。

## 最佳实践
- **深入理解并敬畏张量的 [[broadcasting]] 规则**：在进行维度不完全匹配的张量加减乘除时，严格画出维度变化，不要带有侥幸心理（"play fast and loose"），它极易引发难以排查的 bug[44, 47]。
- **使用原地操作 (In-place operations)**：如 `W.data += ...` 替代创建新的张量赋值，这不会增加底层内存分配，能够提升运行效率[47]。
- **梯度清零的效率优化**：在每一轮反向传播前，使用 `W.grad = None` 代替将其设置为零向量，这在 PyTorch 内部能更高效地表示梯度的缺失[36]。
- **使用向量化索引替代循环**：在计算整个 batch 的 Loss 时，避免使用 for 循环去提取概率。利用 `torch.arange` 配合目标索引张量 `ys`，可以高效地一次性摘取出对应位置的预测概率（如 `probs[torch.arange(5), ys]`）[34, 35]。
