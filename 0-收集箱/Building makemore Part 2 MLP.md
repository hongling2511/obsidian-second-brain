# Building makemore Part 2: MLP

## 一句话总结
用 [[PyTorch]] 构建 [[MLP]] 字符级语言模型，解决上下文维度爆炸问题并优化工程训练。[1, 2]

## 内容类型
tech_tutorial

## 目标受众与前置知识
- **适合谁看**：想要深入理解 [[神经网络]] 底层原理、学习深度学习代码调试技巧及构建基础 [[语言模型]] 的开发者、学生或 AI 研究者。
- **前置知识**：[[Python]] 编程基础，矩阵乘法与基础线性代数，[[PyTorch]] 张量基础操作，以及前作所建立的 [[bigram]] (二元语法) 和 [[反向传播]] (back propagation) 概念。

## 核心观点
1. **观点概括**：采用 [[多层感知机]] (MLP) 和 [[词嵌入]] (embeddings) 能有效解决随上下文增加导致的维度爆炸问题。[2-5]
   - 为什么重要：指出了从传统统计模型向神经网络模型演进的关键动机，使模型能够泛化到未见过的短语组合中。
   - 说服力评估：极强。作者通过对比计算证明，在传统 [[bigram]] 中取三个字符作为上下文会导致组合数暴增至 20000 种，数据稀疏问题严重，而映射到低维嵌入空间的策略成功解决了这个问题。[2, 6]

2. **观点概括**：在实际生产中，应首选框架自带的优化函数（如 `F.cross_entropy`）而非手动实现复杂的前向/反向传播逻辑。[7-10]
   - 为什么重要：直接关乎模型训练的计算效率和数值稳定性，是深度学习工程实践中的核心经验。
   - 说服力评估：强。作者展示了当 logits 值极大（如 100）时，手动实现 Softmax 出现的 `NaN` 报错，并解释了内置函数如何通过 offset 底层操作解决数值溢出，并融合内核以提升速度，极具实战说服力。[8-10]

3. **观点概括**：寻找合适的 [[学习率]] (learning rate) 需要科学的指数级步进搜索策略。[11-14]
   - 为什么重要：学习率是深度学习中最难调的参数之一，盲目猜测会导致模型无法收敛或震荡。
   - 说服力评估：强。作者现场演示了从 $10^{-3}$ 到 $10^0$ 进行指数级步进测试，并绘制损失随学习率变化的曲线，直观找出了山谷底部的最佳学习率区间。[13-16]

4. **观点概括**：正确划分数据集（[[training split]], [[dev split]], [[test split]]）是防止模型 [[overfitting]] (过拟合) 并真实评估模型泛化能力的基石。[17-19]
   - 为什么重要：区分了“模型只是死记硬背训练集”和“模型真正学会了规律”，这是机器学习科学方法论的核心。[17, 18]
   - 说服力评估：强。作者通过逻辑推理论证，随着模型参数量变大，它可以轻易记住仅仅 32 个样本的内容让 loss 降为 0，但这完全不具备生成新名字的能力。[17, 18, 20]

## 副观点与隐含立场
- **副观点**：编写深度学习代码时，理解张量在内存中的存储结构（如 `view` 机制）能大幅避免内存拷贝并提升效率。[21-24]
  - 如何为核心观点服务：辅助观众构建不仅能跑通，而且高效的 [[PyTorch]] 生产级代码。
  - 隐含前提：假设观众不仅希望了解模型背后的数学原理，还关心软件工程层面的最佳实践。

- **副观点**：故意在代码中暴露错误是学习和调试的绝佳方式。[25]
  - 如何为核心观点服务：通过展示数据类型错误（期望 float 却得到 long）等，增强观众面对复杂框架报错时的排障能力。[25, 26]
  - 隐含前提：学习代码是一个动态的试错过程，遇到报错是常态而非异常。

## 详细笔记
- **[0-10%] 引入与痛点分析**：回顾前作 [[bigram]] 模型的局限性，一旦增加历史上下文（如 3 个字符），状态矩阵的行数就会呈指数级爆炸（增至 20000 种），导致数据稀疏。[1, 2] 引出基于 Bengio 2003 年论文的 [[MLP]] 网络，其核心是将词/字符映射为低维的 [[embeddings]]。[3, 4]
- **[10-30%] 构建数据集与 Embedding 查找表**：设定 `block_size=3` 构建上下文 `X` 和预测目标 `Y` 的数据集。利用 [[PyTorch]] 的张量列表索引功能（如 `C[X]`）实现等效于 [[one-hot encoding]] 配合全连接层的超快 [[embeddings]] 查找。[25, 27-31]
- **[30-50%] 构建隐藏层与张量视图重塑**：搭建含有 `tanh` 激活的 [[hidden layer]]。详细对比了低效的 `torch.cat` 拼接操作与利用底层 `storage` 机制实现的极速 `view`（重塑张量形状）操作，最终完成矩阵乘法。[22, 32-35]
- **[50-70%] 交叉熵损失重构与数值稳定性**：先手写输出层 logits 的 softmax 过程以阐明原理，随后强烈推荐改用内置的 `F.cross_entropy`。深入剖析了当面临极端数值时（比如 logit 达到 100），手写实现会产生 `NaN`，而底层 API 通过偏移处理保障了数值稳定性（Numerical Stability）且具备更高的前向/反向传播效率。[7-10, 36, 37]
- **[70-85%] 优化策略：Mini-batch 与 学习率探索**：由于一次性处理 22 万条数据极慢，引入了 [[mini-batch]] 随机抽取数据。[38, 39] 接着重点演示了如何使用 `torch.linspace` 在指数空间（从 $10^{-3}$ 到 $10^0$）内搜索理想的初始学习率，并绘制曲线找出山谷低点（约 0.1）。[11-15]
- **[85-100%] 数据集划分与超参数调优**：讲解了为什么必须将数据按比例切割为 [[training split]], [[dev split]], 以及 [[test split]]，严防 [[overfitting]] (过拟合)。尝试调整隐藏层神经元数量和 [[embeddings]] 维度大小以突破性能瓶颈，最终展示了模型能够生成出更像名字的文本，并留给观众超越当前损失分数的课后挑战。[18, 19, 40-44]

- **策略有效性**：强。最有效的策略是**逻辑论证与反面论证结合**。当他手写一套数学上绝对正确的 Softmax 代码，却在输入稍大数值时现场报错崩溃（NaN），极大地增强了后续推崇框架底层优化（`F.cross_entropy`）的说服力。

## 技术栈与环境要求
- **核心技术**：[[PyTorch]], [[Python]], [[Jupyter Notebook]], [[Matplotlib]], [[Google Colab]], [[MLP]] (多层感知机), [[Language Model]] (语言模型) [1-4]。
- **环境要求**：需要安装 Python 及其科学计算依赖库（主要为 PyTorch 和 Matplotlib）[3]。如果不希望在本地配置环境，可以直接在浏览器中使用 [[Google Colab]] 运行相关代码 [4]。
- **前置技能**：
  - 了解基础的 Bigram（二元语法）语言模型和概率归一化概念 [1]。
  - 具备基本的 Python 编程能力和对张量（Tensor）操作的基础认知。
  - 了解基本的神经网络概念（如线性层、权重、偏置项）[1]。

## 步骤拆解
1. **理论基础与论文引入**
   - 具体操作描述：讲解传统 Bigram 模型在扩展上下文时会导致矩阵维度呈指数级爆炸的问题，引入 Bengio 等人 2003 年发表的论文，决定使用 [[MLP]] 来构建字符级别的 [[Language Model]] [2, 5]。
   - 注意事项：原论文构建的是词级别的模型（17,000个词汇），而本教程采用相同的架构来构建字符级别的模型（27个字符）[5, 6]。

2. **数据集构建 (Dataset Construction)**
   - 具体操作描述：通过滑动窗口机制构建输入 `X` 和标签 `Y`。定义 `block_size` 作为上下文窗口大小（如使用3个字符预测第4个）[3, 7, 8]。
   - 关键代码/命令：`block_size = 3` [3]。
   - 注意事项：在序列起始位置，需要使用特殊的占位符（如 `.` 对应的整数0）进行填充 (Padding) [7, 8]。

3. **构建 [[Embedding]] 层**
   - 具体操作描述：创建一个由随机数初始化的查找表 `C`，将每个离散的字符索引映射到一个低维稠密向量空间（如二维或十维）中 [9, 10]。
   - 关键代码/命令：直接使用整数张量作为索引 `emb = C[X]` [11]。
   - 注意事项：直接使用整数索引查找表，在数学上等价于将 One-hot 编码向量与权重矩阵相乘，但直接索引的速度要快得多 [12, 13]。

4. **构建隐藏层与输出层**
   - 具体操作描述：初始化隐藏层的权重 `W1` 和偏置 `b1`，使用 `tanh` 作为非线性激活函数。然后构建输出层 `W2` 和 `b2` 生成 logits。为了进行矩阵乘法，需要将三维的 embedding 张量展平（拼接）为二维张量 [14-16]。
   - 关键代码/命令：使用 `.view()` 方法进行高效展平重塑，例如 `emb.view(-1, 6) @ W1 + b1` [15, 16]。

5. **计算损失函数 (Loss Function)**

## 常见坑与解决方案
- **问题描述：One-hot 向量与浮点型权重矩阵相乘时报错 `RuntimeError: expected scalar type Long but found Float`** [12, 28]。
  - 解决方案：PyTorch 中的 `F.one_hot` 默认生成 Long（64位整数）类型的张量，必须显式将其转换为 Float 类型才能与浮点型张量相乘 [12]。

- **问题描述：使用 `torch.cat` 拼接高维张量导致内存效率极低** [29, 30]。
  - 解决方案：使用 PyTorch 的 `.view()` 方法。`.view()` 仅通过修改底层的 strides 和 shapes 属性来改变张量的解析方式，不涉及任何内存的复制或移动，性能极高 [15, 30, 31]。

- **问题描述：手动计算 softmax 时遇到极端正数值导致数值溢出（出现 `NaN`）** [32, 33]。
  - 解决方案：直接使用 PyTorch 的内置函数 `F.cross_entropy`。该函数在底层实现时，会找出 logits 中的最大值并将其从所有值中减去（保持概率分布不变），从而保证数值稳定性 [18, 33]。

- **问题描述：广播机制（Broadcasting）可能导致矩阵加法出现不易察觉的逻辑错误** [30, 34]。
  - 解决方案：在进行 `矩阵 + 偏置项` 操作前，务必检查维度的对齐情况（例如 `32x100` 加上 `100` 的张量，PyTorch 会在偏置项左侧补齐维度变成 `1x100` 并垂直复制，符合预期逻辑）[30, 34]。

## 最佳实践
- **优先使用内置函数而非手写数学公式**：强烈推荐使用 `F.cross_entropy` 替代手动实现的对数概率和 softmax。内置函数不仅由于内核融合（fused kernels）能显著提升前向和反向传播的运行效率，还能通过数学化简提供更高效的解析解导数，且更具数值稳定性 [18, 33, 35, 36]。
- **通过单一批次验证模型（Overfitting a single batch）**：在跑完全量数据前，先拿一个极小的数据集（如32个样本）让模型训练。如果模型无法达到接近 0 的损失，说明代码架构或反向传播有严重 Bug [19, 37]。
- **科学的学习率调度策略**：不要瞎猜学习率。先通过指数步长搜索定位合理的初始 [[Learning Rate]]。训练后期当损失进入平台期时，使用**学习率衰减（Learning Rate Decay）**，通常将其缩小 10 倍以进行精细收敛 [25, 38, 39]。
- **警惕 [[Overfitting]] 与 [[Underfitting]]**：如果你发现模型在训练集上的损失极低，但在评估模型生成的样本时效果很差，这说明模型发生了 [[Overfitting]]（死记硬背）[26, 40]；相反，如果训练集和验证集的损失差不多且偏高，说明模型 [[Underfitting]]，此时应该增加模型的参数容量（例如增加隐藏层神经元数量或扩大 [[Embedding]] 维度）[10, 41, 42]。
- **杜绝硬编码（Magic Numbers）**：在构建网络维度尤其是经过 view 展平操作时，避免直接写死维度数字（如硬编码输入特征为 `6`）。应使用张量自身的 `.shape` 属性或使用 `-1` 让 PyTorch 自动推导，以适应后续修改 `block_size` 或 `embedding` 维度的需求 [10, 16, 43]。
