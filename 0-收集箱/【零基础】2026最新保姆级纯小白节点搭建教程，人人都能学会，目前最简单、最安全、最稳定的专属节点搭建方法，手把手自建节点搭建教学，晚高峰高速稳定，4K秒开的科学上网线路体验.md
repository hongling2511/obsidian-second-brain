## 步骤拆解

1. **购买与获取 [[VPS]] 信息**
   - 具体操作描述：选择合适的线路（如美国 DC6 的 GIA 线路或日本软银线路）并完成购买付款 [6, 8]。支付成功后，等待接收名为“OS Complet”的邮件，从中获取 [[VPS]] 的 IP 地址、端口号以及 root 密码 [6]。
   - 注意事项：新购 [[VPS]] 有一定的退款期，不满意可申请退款；可使用优惠码获取折扣 [8]。

2. **通过 [[SSH]] 连接服务器**
   - 具体操作描述：打开电脑终端，输入连接命令，首次连接确认 `yes`，随后输入 root 密码进行登录 [6]。
   - 关键代码/命令：`ssh root@<IP> -p 22` [6]。
   - 注意事项：粘贴密码时界面不会有任何显示（看起来像没输入），只需按回车即可。如果提示密码错误，建议手动输入并注意区分大小写 [6, 7]。

3. **运行搭建脚本与 [[SNI]] 域名测速**
   - 具体操作描述：连接成功后，粘贴教程提供的指令开始跑代码（需等待3-5分钟），遇到提示直接回车 [7]。接着运行测速指令，挑选并保存一个延迟较低的域名（如 `www.icloud.com`），最后输入 `exit` 退出连接 [7]。
   - 关键代码/命令：遇到 `y/-n` 提示时按回车；输入 `exit` 退出 [7]。
   - 注意事项：如果中途出现卡死或意外，直接关闭黑窗口，重新跟着教程操作一遍即可 [7]。

4. **配置 [[SUI面板]] 及节点协议**
   - 具体操作描述：根据教程指令生成本地访问链接并在浏览器登录面板。在 TLS 设置中添加 [[Reality]] 模板并填入测速好的低延迟域名生成公钥和私钥；接着添加另一个允许不安全的 TLS 模板 [2]。最后在“入站管理”中分别创建 443 端口的 [[VLESS]]（对应视频中的 WS/valance 节点）和 [[Trojan]]（对应视频中的 T 节点）[2, 3]。
   - 注意事项：节点搭建完成之前，不要关闭运行代码的黑窗口 [2]。

5. **客户端导入与配置**
   - 具体操作描述：各平台下载对应客户端：Windows/Mac 下载 [[v2rayN]]，iOS 下载 [[Shadowrocket]]，Android 下载 [[NekoBox]] [3-5]。通过扫描二维码或粘贴链接导入刚创建的两个节点 [3-5]。
   - 关键操作：**必须**双击或编辑节点配置，将默认地址修改为你 [[VPS]] 的真实 IP 地址 [3-5]。
   - 注意事项：对于 [[Trojan]]（T节点），必须在配置中将“跳过证书验证”或“允许不安全的链接”改为 `true` 或启用状态 [3-5]。

## 常见坑与解决方案

- **问题描述**：[[SSH]] 登录时输入密码一直提示错误 [6]。
  - **解决方案**：粘贴操作由于不显示字符容易出错，建议直接手动输入密码，并严格注意区分大小写 [6, 7]。
- **问题描述**：代码运行中途卡死或出现意外情况 [7]。
  - **解决方案**：直接关闭当前的黑窗口，重新从第一步开始跟做 [7]。
- **问题描述**：将节点导入客户端后无法连接，无法科学上网 [3]。
  - **解决方案**：检查是否忘记将节点配置中的服务器地址替换为自己 [[VPS]] 的真实 IP 地址；若使用的是 [[Trojan]] 备用节点，需检查是否开启了“允许不安全”验证 [3-5]。
- **问题描述**：使用美国节点时，[[Trojan]] 节点的测速延迟看起来偏高 [3]。
  - **解决方案**：这是正常的 TLS 握手延迟，不代表实际的网速表现，不会影响正常使用，无需特殊处理 [3, 9]。

## 最佳实践

- **主备节点搭配**：建议日常作为主力使用第一个 [[Reality]] 节点，将第二个 [[Trojan]]（T节点）作为备用节点使用 [4]。
- **机房平滑迁移**：如果对当前线路延迟不满意，可以在 VPS 控制面板免费进行机房切换（例如从美国迁往日本软银以降低延迟）。迁移完成后原 IP 会被回收，只需在客户端把节点 IP 改为新 IP 即可恢复连接 [9]。
- **避免频繁迁移（反模式）**：虽然机房迁移是免费的且能更换 IP，但迁移过程会消耗 [[VPS]] 的流量，因此“非必要不频繁进行迁移” [9]。
- **定期续费管理**：务必留意商家的续费邮件（一般有七天操作期），如忘记续费会导致 [[VPS]] 被回收，届时需要重新购买和从头搭建 [5]。
