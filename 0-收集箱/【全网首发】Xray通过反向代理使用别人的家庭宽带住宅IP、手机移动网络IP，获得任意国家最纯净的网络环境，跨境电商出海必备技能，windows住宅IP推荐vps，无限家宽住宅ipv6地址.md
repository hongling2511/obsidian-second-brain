## 一句话总结
本教程演示如何通过 [[Xray]] 的 [[反向代理]] 技术，将无 [[公网 IP]] 的家庭 [[宽带]] 或手机网络转化为纯净的专属代理 [[节点]]。

## 目标受众与前置知识
- **适合谁看**：有跨境电商、加密货币交易及养号需求，受限于普通代理 IP 导致账号被风控的用户。
- **前置知识**：[[vps]]、[[节点]]、[[公网 IP]]、[[SSH]]（推断）、[[科学上网]]

## 核心观点
1. **普通机房 IP 容易触发风控**
   - 为什么重要：指出了跨境电商和资金交易中账号异常的根本网络环境原因。
   - 依据：使用 [[机房 IP]] 进行敏感操作会导致 [[tiktok]] 发视频没流量、[[Paypal]] 判定 IP 存在风险导致封号或冻结资金（讲者原话）。
2. **当地家庭宽带是获取最纯净代理的最佳方式**
   - 为什么重要：提出了解决 IP 风控问题的终极网络环境标准。
   - 依据：依据较弱（讲者仅从逻辑上陈述直接办理当地运营商宽带可解决风控，未提供对比数据）。
3. **无公网 IP 时可通过反向代理实现高速互联**
   - 为什么重要：解决了大部分家庭网络无公网 IP、躲在防火墙后无法配置 [[正向代理]] 的技术痛点。
   - 依据：讲者通过演示 [[Linux]]、[[Windows]] 及移动设备连接中转 [[vps]]，成功获取到 [[ATT]] 运营商的 [[住宅 IP]] 并在 [[ipinfo]] 上得到验证（屏幕展示）。

## 概念关系
- [[机房 IP]] ↔ [[住宅 IP]]：对比关系。前者易导致风控和封禁，后者属于真实的家庭宽带网络，环境纯净。
- [[正向代理]] ↔ [[反向代理]]：对比与依赖关系。正向代理要求住宅端有 [[公网 IP]]；反向代理则通过带 [[公网 IP]] 的中转机，让无公网的住宅设备主动连接，实现数据转发。
- [[中转 vps]] → [[节点]]：依赖关系。在反向代理拓扑中，中转服务器作为桥梁接收用户请求，并将其转交给具体的真实住宅网络节点。
- [[3xui]] → [[Xray]]：依赖关系。[[3xui]] 是可视化面板，用于便捷配置底层的 [[Xray]]（视频中音译为插瑞）核心的入站、出站及 [[路由规则]]。

## 详细笔记
### 代理 IP 风控痛点与反代思路
- 核心问题：用户（包括频道主播不良林）进行跨境出海业务时，常用的 [[机房 IP]] 极易触发风控。
- 终极方案：使用真实当地 [[移动 IP]] 或运营商分配的 [[住宅 IP]]。
- 技术难点：朋友家的电脑往往没有 [[公网 IP]] 或被防火墙拦截，无法直接通过 [[正向代理]] 发送数据。若双方都没公网 IP 强行直连，线路会卡顿（讲者原话）。
- 解决方案：引入一台带 [[公网 IP]] 的中转 [[vps]] 进行 [[反向代理]]（区别于以前讲过的 [[CF]] 反代）。双方主动连接中转机，由中转机转交数据。

### 中转机环境准备与基础节点搭建
- 中转机选购：推荐购买 [[cn2gia]] 线路的 [[vps]]，晚高峰高速稳定（推断：为保证国内连通率）。使用优惠码有 6% 的折扣（讲者原话）。
- 系统配置：通过 [[Windows]] 自带的 [[cmd]] 或 Mac 系统的 [[终端]]，使用 [[ssh]] 指令连接中转机。
- 面板安装：粘贴官方指令安装 [[3xui]]，通过 `IP:2053/随机路径`（屏幕展示）登录。
- 搭建基础节点：在入站列表添加 [[vless]] + [[reality]] 协议组合（推荐 [[端口]] 443）。将随机生成的邮箱改为 `proxy`。
- 验证：通过 [[v2rayN]]（视频称 v2n）连接，访问 [[ipinfo]]（视频称 ipino）验证落地 IP 为中转机自身 IP。

### Linux 住宅机反向代理配置
- 住宅端准备：通过 [[向日葵]] 远程控制，在朋友的 [[Linux]] 设备安装 [[3xui]]（该设备无需搭节点）。
- 中转端配置：创建全英文备注的 [[ss]] (shsocks) 协议节点，加密方式选 [[as128]]。
- 住宅端出入站规则：
  - 添加入站，将中转机的 [[ss]] 节点链接解析为出站规则。
  - 在反向标签页添加反向代理，类型选 `bridge`，互联选刚建立的 [[ss]]，出站选 `direct`。保存并重启 [[Xray]]。
- 中转端路由映射：
  - 给原来的 [[reality]] 节点新增一个邮箱客户端 `住宅/一`。
  - 添加反向代理，类型为 `portal`，互联选 [[ss]]，入站指定 [[reality]] 节点的 443 端口。
  - 编辑路由规则，指定 user `住宅/一` 的流量走反代。
- 效果验证：切换到 `住宅/一` 节点后，[[ipinfo]] 显示为美国三大电信运营商之一的 [[ATT]]（地位等同于国内的 [[电信]]、[[移动]]、[[联通]]）的 [[双 isp]] 住宅 IP。
- 避坑提示：有商家会用双 isp 的机房 IP 冒充住宅 IP，需留心眼（讲者原话）。

### Windows 住宅机反向代理配置
- 住宅端准备：在朋友的 [[Windows]] 电脑下载 [[Windows]] 版 [[3xui]] 压缩包。安装 Go 环境，通过 [[cmd]] 执行 `main` 启动，使用默认账密 `admin` 登录本机的 2053 端口。
- 防火墙模拟：此 [[Windows]] 有公网 IP，但防火墙拦截了公网访问，正好模拟无公网 IP 状态。
- 反代配置：操作同 [[Linux]]，但反代标签和域名需改为 `2` 以作区分。若重启报错，在高级设置中修改 API 端口为 6666（讲者原话）。
- 中转端路由映射：新增客户端 `住宅/2`，配置标签为 `2` 的 `portal`，路由规则绑定 `住宅/2`。
- 多节点实现：通过切换不同客户端，实现一台中转机同时套用原本中转 IP、Linux 住宅 IP 和 Windows 住宅 IP。

### 商业静态住宅 VPS 推荐（免折腾方案）
- 适用场景：在国外无朋友或朋友不愿 24 小时开机的用户。
- 服务商优势：商家在美国有办公场所直接办理 [[ATT]] 宽带，提供静态住宅 IP，且自带 [[公网 IP]]，无需配置反向代理（讲者原话）。
- 配置与定价：
  - 垃圾 IP 的 2核4G 都要 28美金/月（讲者原话）。此家最低配 2核4G 装 [[Windows]] 勉强，建议选 8G 内存标准版（推断：为了图形界面流畅）。
  - 带宽最低选配 15兆bps，实际给 17兆bps，看 2k 视频或单台 [[tiktok]] 直播无压力；4k 视频需选 30兆bps（屏幕展示与讲者原话）。
  - 附赠一段 -64 前缀的 [[ATT]] [[ipv6]] 地址，可搭建无限 [[ipv6]] 的站群服务器（讲者原话）。
  - 对小白提供直连的 [[socks代理]]。对大老板提供 60个 IP 和 1G 带宽的独服。
  - 系统支持安装 [[Ubuntu 22.04]] 或 [[Windows 2022]]。
  - 使用优惠码 "不良林" 可享 5% 优惠；季付95折，半年9折，年付85折（讲者原话）。
- 欺诈值提醒：IP 欺诈检测仅供参考，国内运营商分配的公网 IP 欺诈值可能更高。

### 移动设备（安卓/苹果）网络反代配置
- 方案选择：可让手机发射 [[WiFi热点]] 给电脑做反代，或直接在手机上配置。
- [[安卓]] 手机配置：使用 [[v2rayN]]（视频称 vng）。修改讲者提供的代码模板，替换中转机 [[ss]] 信息与密码，编号改为 `3`。以“从本地导入”方式添加自定义配置。在进阶设置将模式从 [[VPN]] 改为“仅代理”后启动。中转机配置对应的 `住宅/3` 客户端及反代规则。实测下载文件消耗了手机的蜂窝网络流量（屏幕展示）。
- [[苹果]] 手机配置：使用 [[v2box]]（视频称 vbx）。模板编号改为 `4`，加密协议对应修改。通过添加“手动配置 -> JS”粘贴代码，滑块连接。中转机配置对应的 `住宅/4` 客户端及反代规则。实测落地 IP 变为德国的苹果手机网络（屏幕展示）。

6. **在中转机上配置反向中转与路由分发**

7. **移动端（安卓/苹果）反向代理配置**
   - 具体操作描述：将反向代理 JSON/JS 模板中的中转机 SS 节点信息及设备序号（如 3 或 4）修改好后，作为自定义配置导入至移动端代理工具（如安卓的 `[[v2rayN]]` 或苹果的 `[[vbx]]`），并在中转机上重复步骤 6 创建对应的客户端和 portal 配置 [6, 7, 10]。
   - 关键代码/命令：未明确提及 [6, 7, 10]。

- **问题描述**：`[[Windows]]` 默认开启防火墙拦截，无法从公网访问 `[[3x-ui]]` 面板 [5]
  - 解决方案：未明确提及 [5]。
- **问题描述**：配置好反代节点后测试没有延迟 [9]

## 最佳实践
- 日常科学上网建议使用中转机原生 IP 的普通节点以保障高速稳定，只有涉及资金交易或容易触发风控的敏感业务时，再切换使用朋友家住宅 IP 的节点，以实现最纯净的伪装 [4, 11]。
- 推荐使用反向代理的方式利用他人宽带，因为该方式不需要改动住宅的网络拓扑，无需公网 IP，也不需要放行防火墙，只需在设备上安装面板即可 [4]。
- 若在当地没有朋友提供宽带，推荐直接购买当地运营商直连、自带公网静态住宅 IP 的 `[[vps]]`（如 ATT 宽带的 VPS），无需折腾反代即可 24 小时保持运行 [12]。

## 关键概念

### [[反向代理]]
- **定义**：在双方均无公网IP或受防火墙限制时，借助一台具备公网IP的中转服务器来互相转发数据，从而实现使用目标端网络环境的代理技术 [1, 2]。
