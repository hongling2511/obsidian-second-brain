## 一句话总结
通过常见代理协议与节点分流配置，打破代理工具的内网直连限制，实现从公网远程且灵活地访问家庭内网服务。

## 目标受众与前置知识
- **适合谁看**：需要从外网环境远程访问家中内网设备（如 [[NAS]]、路由器）且对上行带宽有分配需求的用户。
- **前置知识**：[[代理工具]]基础使用、[[软路由]]搭建基础、[[公网]]与[[内网]]网络概念、[[ipv4]] / [[ipv6]] 端口与防火墙规则、[[ddns]]。

## 核心观点
1. **代理协议可直接替代传统 VPN 访问内网**
   - 为什么重要：比传统的 [[微p]]（[[VPN]]）更灵活方便，可以避免全局流量绕行家庭网络。
   - 依据：讲者原话：“现在的代理工具都有完善的节点分流策略，可以实现访问内网才走 [[home]] 节点，翻墙就使用其他节点，不用忍受家里那小的可怜的上行带宽”。
2. **代理工具防泄漏机制是导致无法“回家”的根本原因**
   - 为什么重要：点明了即便搭建了节点依然无法访问内网的痛点。
   - 依据：讲者原话：“大部分代理工具为了防止潜在的网络问题，即使你设置为全局代理，访问内网网段的数据包还是会绕过代理走直连”。
3. **两端网段冲突将导致内网访问失败**
   - 为什么重要：这是远程访问过程中极易被忽视的网络基础错误。
   - 依据：讲者原话：“本地局域网和家里的局域网不能在同一网段...如果两端都是一样的网段，你将只能访问到公司的局域网”。

## 概念关系
- [[代理工具]] → [[节点分流策略]]：代理工具通过分流规则识别请求目标，决定是发往翻墙节点还是家庭内网节点。
- [[公网 IP]] / [[ddns]] → [[路由器防火墙]]：外网客户端通过域名和端口寻址到家庭路由器，需通过防火墙（端口转发或通信规则）放行才能建立连接。
- [[局域网直连]] ↔ [[内网访问需求]]：代理客户端默认的跳过局域网直连机制与用户需要通过代理访问内网的需求形成对立（推断），必须通过修改配置来解除限制。

## 详细笔记
### 1. 代理内网的原理与痛点
- **原理**：讲者 [[不良林]] 介绍无需 [[微p]]，直接使用 [[V2]]、[[clash]]、[[小火箭]] 等工具，通过常见的 [[socks]]、[[ss]]、[[w]]（推断为 vmess）等代理协议即可代理内网 [[IP]] [1]。公网请求交由客户端，分流配置将其识别并发往家中的 [[home]] 节点，路由器解密后代为访问内网中“192.168.2.110”（讲者原话示例）的 [[NAS]] 服务并返回数据 [1]。
- **痛点**：默认情况下，代理工具会将内网网段数据包强制走直连导致访问失败 [1]。

### 2. 路由器端节点搭建（服务端配置）
- 需在家中部署节点。以下为三种常见路由插件的配置方法：
  - **[[home proxy]]**：添加节点，选择 [[socks]] 或 [[ss]] 类型。设置空闲端口如“7788”（讲者原话），设置密码和加密方式，勾选启用及自动配置防火墙 [2]。
  - **[[pw]]**：配置同上，但必须额外勾选“接受局域网访问” [2]。
  - **[[open clash]]**：仅演示 [[socks]] 节点。在流量控制中取消勾选“仅允许内网”，在复写设置中将 S5 的默认端口由“7891”改为“7788”（讲者原话），并在设置鉴权中勾选启用并设置账号密码 [2]。

### 3. 防火墙设置与连通性测试
- **[[ipv4]]**：如有公网 IPv4，需在防火墙设置端口转发，将外部和内部端口均设为“7788”（讲者原话），指向路由器的内网 IP [2, 3]。
- **[[ipv6]]**：无需端口转发，直接在通信规则添加操作为“接受”的规则，目标端口设为 7788，协议栈改为“仅 ipv6” [3]。
- 测试：配置完成后，通过视频下方的测试网页输入 IP 和端口，显绿色即为拼通 [3]。为应对 IP 变化，建议提前配置好 [[ddns]] [4]。

### 4. macOS 与 iOS 客户端配置
- **[[小火箭]]**（支持 [[macOS]]/[[iOS]]）：通过手机 [[5G]] 网络测试。添加基于 [[ddns]] 域名和端口的 [[home]] 节点。进入通用配置中的“跳过代理”，删掉家里的内网网段；进入规则添加 `ip/cidr`，填入内网网段如“192.168.2.0-24”（讲者原话），策略绑定 home 节点 [4, 5]。
- **[[圈插]]**：只支持图形化添加 [[ss]] 节点（socks5需手动改文件）。添加 `ip/cidr` 分流配置，在配置文件“排除路由”里删除内网网段，本地规则中将策略设为 home 节点 [6]。

### 5. Windows 等桌面端客户端配置
- **[[clash for Windows]]**（支持 [[windows]]）：将 home 节点插入当前配置，添加内网网段的 `ip/cidr` 规则。需在 parsers/rules 模块后追加防 DNS 泄露规则。系统代理模式下，需在代理设置页面中将内网网段从绕过列表中删除 [5, 6]。完成后查看日志，访问 [[谷歌]] 走香港节点，访问内网走 home 节点 [5, 6]。
- **[[sbox]]**（配置文件模式）：在 profiles 里的 outbound 插入模板配置，并在 rule 里插入将内网网段指向 home 的路由配置 [6, 7]。
- **[[v2n]]**：因难以方便添加分流策略，需在路由设置编辑“绕过大陆”模式，输入并置顶内网网段规则，在系统代理设置中删除局域网绕过列表，需访问家时手动切换至 home 节点 [7]。

### 6. Android 客户端配置与排错建议
- **[[android]]** 客户端在 [[WiFi]] / 5G 切换下配置：
  - **[[v2ng]]** / **[[nobx]]** / **[[sb]]**（推断 sb 指类似 NekoBox 核心端）：用法同 v2n，需手动切换节点。nobx 需在设置中取消“绕过局域网地址”，若是 IPv6 节点需启用 ipv6 路由 [7, 8]。
  - **[[nakebo]]**（推断为某分支客户端，讲者原文提及 [8]）：（未明确提及详细单独步骤，与 v2ng 逻辑并列）。
  - **[[冲浪版]]**：复制配置编辑，在跳过代理中删掉内网网段，在 proxy 中插入 home 节点。注意其不支持 [[ss]] 节点的 IPv6，需改为 [[socks]] 等协议。最后勾选局域网流量过滤 [8, 9]。
- **总结建议**：万变不离其宗的三步：添加 home 节点，添加分流规则，删除局域网直连路由。本地与家里局域网绝不能在同网段 [9]。

## 行动建议
- **做什么**：在家庭路由器的代理服务端插件中，创建一条指定端口（如7788）的 ss 或 socks 节点，并配置对应公网协议的防火墙放行规则。
- **关联观点**：1
- **验证标准**：通过公网连通性测试网站，输入 DDNS 域名和指定端口后能成功 PING 通（显示绿色）。

- **做什么**：在客户端设备的代理软件中，找到“绕过局域网（跳过代理）”的设置项，将其中的家庭局域网网段（如 192.168.x.x）删除，并手动添加一条将该网段指向 home 节点的分流规则。
- **关联观点**：2
- **验证标准**：在外部网络环境下开启代理，能通过浏览器直接打开家中路由器的内网 IP 管理界面，且代理软件日志显示流量经由 home 节点。

- **做什么**：检查当前所处外网环境（如公司）的内网 IP 网段，核对是否与家中内网 IP 网段完全一致。
- **关联观点**：3
- **验证标准**：确认两端网段不同（如公司为 192.168.3.x，家里为 192.168.2.x），以保证路由请求能正确跳出本地局域网发往远端。

## 技术栈与环境要求
- **核心技术**：`[[SS]]`, `[[socks5]]`, `[[vmess]]`, `[[clash]]`, `[[Shadowrocket]]`, `[[Quantumult X]]`, `[[clash for Windows]]`, `[[v2rayN]]`, `[[v2rayNG]]`, `[[NekoBox]]`, `[[Surfboard]]`, `[[sing-box]]`, `[[home proxy]]`, `[[passwall]]`, `[[open clash]]`, `[[ddns]]`, `[[ipv4]]`, `[[ipv6]]` [1-8]。
- **环境要求**：操作系统涉及 `[[macOS]]`, `[[iOS]]`, `[[windows]]`, `[[android]]` [1]；软件版本：未明确提及；依赖项：未明确提及。
- **前置技能**：代理分流规则基础、防火墙与端口转发配置基础、`[[ddns]]` 配置基础 [3, 4]。

## 步骤拆解
1. **路由器节点搭建**
   - 具体操作描述：在软路由上使用 `[[home proxy]]`、`[[passwall]]` 或 `[[open clash]]` 搭建 `[[SS]]` 或 `[[socks5]]` 节点，配置密码与加密方式，并开放局域网访问权限 [2]。
   - 关键代码/命令：示例端口为 `7788` [2]；使用 `[[open clash]]` 时需在流量控制中取消勾选“仅允许内网”，并将 `[[socks5]]` 默认端口 `7891` 更改为 `7788` [2]。
   - 注意事项：未明确提及。

2. **配置防火墙与外部访问**
   - 具体操作描述：将搭建好的节点暴露在公网，允许外部连接 [2]。
   - 关键代码/命令：若为 `[[ipv4]]` 环境，添加端口转发，外部和内部端口均填 `7788`，内部 IP 填路由器内网 IP [2, 3]；若为 `[[ipv6]]` 环境，在通信规则中添加规则，原区域设为 `wan`，目标区域设为 `设备`，目标端口 `7788`，操作为 `接受`，高级设置的协议栈设为 `仅ipv6` [3]。
   - 注意事项：配置完成后，需通过在线工具输入路由器的 IP 和端口测试连通性，必须显示绿色（ping通）方可进行后续配置 [3]。

3. **客户端配置：移除局域网直连限制**
   - 具体操作描述：在代理客户端的配置中，找到“跳过代理”或“排除路由”等设置，将家中的内网网段从绕过列表中删除 [4, 6-8]。
   - 关键代码/命令：内网网段示例，如 `192.168.2.0-24` 或 `192.168.x.x` [4, 5]。
   - 注意事项：在 Windows 环境中，如果使用系统代理模式，还必须进入操作系统的“代理服务器设置”页面，将内网网段从绕过列表中手动删除 [5-7]。

4. **客户端配置：添加节点与分流策略**
   - 具体操作描述：将家中的 `[[ddns]]` 域名和配置好的端口作为节点加入客户端，并添加分流规则使访问家中的内网网段时走该节点代理 [4-6]。
   - 关键代码/命令：规则类型选择 `ip/cidr`，将对应内网网段指向家中的节点 [4-6]；在 `[[clash for Windows]]` 的 rules 模块末尾添加 `no-resolve` 参数 [5]；在 `[[sing-box]]` 中修改配置文件在 outbounds 和 route 中插入对应的 `[[SS]]` 或 `[[socks5]]` 模板 [6]。
   - 注意事项：`[[Quantumult X]]` 的图形化界面默认只能添加 `[[SS]]` 节点，若使用 `[[socks5]]` 需手动修改配置文件 [6]；`[[v2rayN]]` 和 `[[v2rayNG]]` 由于分流策略添加不便，建议需要连接内网时通过手动方式切换至家中节点 [7]。

## 常见坑与解决方案
- **问题描述**：配置防火墙后，在线测试 IP 和端口显示红色（无法从公网访问） [3]。
  - 解决方案：仔细检查路由器的端口转发或 `[[ipv6]]` 通信规则配置是否有误 [3]。
- **问题描述**：Windows 下配置好代理工具分流后，系统代理模式下依然无法访问内网 [5-7]。
  - 解决方案：进入 Windows 系统的代理设置页面，将家里的内网网段从绕过列表中删掉 [6, 7]。
- **问题描述**：`[[Quantumult X]]` 用户无法通过图形界面添加 `[[socks5]]` 节点 [6]。
  - 解决方案：手动在配置文件中添加相关节点信息 [6]。
- **问题描述**：`[[NekoBox]]` 开启代理后无法连接家中内网 [8]。
  - 解决方案：在设置中取消勾选“绕过局域网地址”，若节点为 `[[ipv6]]` 地址则需启用“IPv6 路由” [8]。
- **问题描述**：`[[Surfboard]]` 中的 `[[SS]]` 节点无法使用 `[[ipv6]]` 地址 [8]。
  - 解决方案：在软路由中将节点换成 `[[socks5]]` 或 `[[vmess]]` 协议，并在客户端对应修改模板类型及参数 [8]。
- **问题描述**：远程所在的本地局域网（如公司）与家里的局域网处于同一网段，导致无法访问家里的内网 [9]。
  - 解决方案：确保两端网络不在同一网段，必须修改其中一端的网段 [9]。

## 最佳实践
- 建议配置 `[[ddns]]` 以应对公网 IP 地址发生变化 [4]。
- 在添加分流规则时，启用“不解析域名”或添加 `no-resolve` 参数，以防止 DNS 泄漏 [4, 5]。
- 如果路由器没有公网 `[[ipv4]]` 地址，推荐直接使用公网 `[[ipv6]]` 地址并在防火墙配置通信规则，不需要设置端口转发 [3]。
- 所有客户端的通用配置核心逻辑可归纳为三步：添加回家节点、添加分流规则、删除局域网直连路由 [9]。

## 关键概念

### [[代理协议]]
- **定义**：无需搭建VPN，可用于代理内网IP并对数据进行加密和解密的网络通信协议（如socks、SS、vmess等） [1]。
- **视频上下文**：作者指出很多人以为访问内网只能用VPN，但其实通过在路由器或内网设备上搭建SS或socks等节点，将请求交由代理工具即可实现远程访问 [1, 2]。
- **为什么重要**：它是本视频实现内网穿透的核心技术基础，被认为比真正的VPN更灵活方便 [1]。
- **关联概念**：对比 → [[VPN]]

### [[节点分流]]
- **定义**：代理工具中用于根据访问目标（如内网或外网）决定使用特定代理节点的流量分发策略 [1]。
- **视频上下文**：作者在多个客户端（如小火箭、Clash等）中演示添加规则，让访问内网网段的流量交由home节点代理，而科学上网的流量交由其他节点代理 [1, 3, 4]。
- **为什么重要**：它使得翻墙时不需要消耗家庭网络较小的上行带宽，实现了多节点的灵活共用 [1, 4]。
- **关联概念**：未明确提及

### [[绕过代理走直连]]
- **定义**：代理工具为了防止潜在网络问题而默认采用的机制，会导致访问内网网段的数据包不经过代理节点而直接发送 [1]。
- **视频上下文**：作者强调即使开启了全局代理，内网数据包依然会触发此机制导致无法连接回家，并在各类客户端配置中反复演示必须删掉“跳过代理”或“绕过局域网地址”中的内网网段 [1, 3, 5, 6]。
- **为什么重要**：这是导致常规代理配置无法直接远程访问家庭内网的核心障碍，必须手动解除该限制 [1, 7]。
- **关联概念**：未明确提及

### [[端口转发]]
- **定义**：在路由器防火墙中配置的规则，用于将外部公网的端口请求转发到内部局域网特定IP的对应端口上 [2, 8]。
- **视频上下文**：作者指出如果路由器有公网IPv4地址，就必须在防火墙设置端口转发（例如将外部7788端口转发到内部7788端口），以暴露搭建好的节点 [2, 8]。
- **为什么重要**：它是让处于公网环境的用户能够成功连接并访问到家庭内网代理节点的必要步骤 [2, 8]。
- **关联概念**：对比 → [[公网IPv6]]

### [[公网IPv6]]
- **定义**：路由器获取的可直接在公网上路由和通信的IPv6地址 [8]。
- **视频上下文**：作者提到如果路由器没有公网IPv4地址，就可以使用公网IPv6地址，并演示了只需在防火墙通信规则中设置为“接受”即可，无需其他复杂转发 [8]。
- **为什么重要**：它是IPv4的替代方案，使用该方案不需要设置端口转发即可让外部设备直连代理节点 [8]。
- **关联概念**：对比 → [[端口转发]]

### [[DDNS]]
- **定义**：用于应对公网IP地址发生变化的动态域名解析服务 [3]。
