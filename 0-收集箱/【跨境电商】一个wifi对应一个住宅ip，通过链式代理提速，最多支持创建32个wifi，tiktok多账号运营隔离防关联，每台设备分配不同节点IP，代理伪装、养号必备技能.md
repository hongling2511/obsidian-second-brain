## 一句话总结
通过 [[软路由]] 配置多 WiFi 或绑定设备局域网 IP，结合 [[链式代理]]，实现 [[TikTok]] 多账号单设备单节点防关联运营。

## 目标受众与前置知识
- **适合谁看**：跨境电商从业者、[[TikTok]] 多账号运营人员。
- **前置知识**：[[软路由]]、[[openwrt]]、[[节点 IP]]、[[内网网段]]、[[DHCP]]。

## 核心观点
1. **将代理节点转移至路由器集中管理可提高伪装度并简化操作**
   - 为什么重要：避免在每台手机上单独运行代理工具，防止被业务方检测出挂代理导致账号关联。
   - 依据：讲者原话，“由于手机上没有运行带的工具，伪装也会更好”。
2. **通过软路由可以实现单个 WiFi 对应一个专属节点 IP**
   - 为什么重要：能在物理连接层面对不同运营账号的网络环境进行严格隔离。
   - 依据：讲者原话，“每个平段最多能创建16个WiFi，也就是我这台设备总共能创建32个WiFi”。
3. **不支持发射 WiFi 的软路由也能通过静态 IP 绑定实现设备分流**
   - 为什么重要：不需要软路由本身自带无线网卡，兼容性更广，适合外接 AP 路由器的场景。
   - 依据：讲者明确演示将 [[安卓]] 手机局域网 IP（192.168.2.235）和电脑 IP（192.168.2.222）分别设置不同的分流规则，均能成功分配不同节点。
4. **使用住宅 IP 必须配合链式代理（前置代理）以保证连通性**
   - 为什么重要：国内网络环境无法直接稳定连接海外纯住宅 IP 节点。
   - 依据：讲者原话，“不能直接用国内直连 [[SS]] 和 [[socks]] 简直就是送死，所以还需要配置练识代理用其他稳定的节点来拉住者 IP”。

## 概念关系
- [[软路由]] → [[多WiFi创建]]：依托刷入 [[openwrt]] 系统的软路由功能，可以在 2.4G 和 5G 频段虚拟出多个独立网络接口。
- [[内网网段]] → [[节点分流]]：为不同 WiFi 或设备分配不同的局域网 IP 段，是代理插件识别流量并路由到不同节点的基础。
- [[链式代理]] → [[住宅 IP]]：通过稳定的机场节点或自建节点作为前置通道，拉动/连接真实的 [[住宅 IP]]，解决直连封锁问题。
- [[私有无线局域网地址]] ↔ [[静态 IP]]：开启设备上的私有 MAC（轮替）可能导致 DHCP 重新分配 IP；将其关闭或固定为静态 IP，可确保分流规则持久生效。

## 详细笔记

### 需求背景与方案选型
- **背景痛点**：跨境电商需同时运营多个账号，直接在设备上运行代理工具配置繁琐，且易被业务平台检测出代理，导致账号关联。
- **解决方案**：讲者 [[不良林]] 提出将 [[节点 IP]] 转移至 [[软路由]] 运行，路由器为每台设备分配不同节点 IP。
- **硬性要求**：需一台刷了 [[openwrt]] 系统的路由设备。无设备者可在 [[咸鱼]] 购买，需向卖家确认是否支持 WiFi 及内置 [[pword]]（推断为 PassWall）插件。
- **方案分支**：
  1. 一 WiFi 对应一节点（要求软路由支持发射 WiFi）。
  2. 同 WiFi 下不同设备对应不同节点（兼容所有软路由，讲者推断更推荐）。

### 方案一：创建多个 WiFi 并分配独立网段
- **演示设备**：[[红米 a6000]]，搭载固件 [[emot wrt23.05]]。
- **创建过程**：
  - 在“网络-无线”页面，2.4G或5G频段添加新 WiFi（例如输入 ssid: tk01）。
  - 设置至少八位数的密码（讲者原话）。
  - 高级设置中配置不重复的接口名称（如 ap/tk01）。
  - 在“接口”页面将协议改为“静态地址”，分配避开现有网段的 IP（如现有 192.168.2.x，新设为 192.168.11.1），子网掩码选 24（屏幕展示）。
  - 开启 [[DHCP]] 服务，防火墙选择 lan 区域。
  - 重复操作创建 tk02，分配网段 192.168.12.1。
- **补充设置**：为防邻居查探，可开启“隐藏 ssid”，但需自行在 [[谷歌]] 搜索连接方法。
- **容量上限**：讲者测试指出每个频段最多 16 个 WiFi，设备总计可创 32 个 WiFi（讲者原话）。

### 导入代理节点与配置链式代理
- **代理插件**：以 [[pword]] 为例演示。建议配置前清空还原默认设置并备份。
- **导入住宅 IP**：
  - 演示使用美国 [[ATT]] 宽带运营商的 [[住宅 IP]]（类型为双 ISP）。
  - 支持通过购买的 [[socks]] 代理手动填入 IP、端口、账号密码添加；或通过 [[vps]] 上 [[xui]] 搭建的节点链接直接粘贴导入。
- **配置链式代理**：
  - 由于国内直连 [[SS]] 或 [[socks]] 会被封锁，需导入机场订阅节点或自建节点。
  - 将机场节点类型改为 [[插瑞]]（推断为 [[Xray]]），通过订阅链接导入全地区节点作为前置代理。

### 分流规则设置与运行验证
- **设置规则**：
  - 在规则管理中添加规则，原地址输入对应网段（如 tk01 对应 192.168.11.0-24）。
  - 域名填入 `Re:.*`（讲者原话“Re冒号点心”），IP 填入 `0.0.0.0-0`。tk02 同理填写网段 192.168.12.0-24。
- **应用分流**：
  - 添加 [[插瑞]] 类型的分流协议节点，为对应规则绑定 [[住宅 IP]]。
  - 开启“前置代理”，选择与住宅 IP 同地区的机场节点拉动流量。
  - 在基本设置中，TCP 和 UDP 节点均选择配置好的住宅分流节点。
- **运行与验证**：
  - 启动后需等待 [[dnsmq]] 重启完成，日志显示 tcp/udp/DNS 运行中。
  - [[安卓]] 手机连 tk02 访问 [[谷歌]] 查 IP，结尾为 178（讲者原话）；连 tk01，IP 变为结尾 164（讲者原话），证明分流成功。

### 方案二：同 WiFi 下不同设备基于 IP 分流
- **适用场景**：软路由无无线网卡，需通过网线外接无线 AP 路由器。
- **配置步骤**：
  - 在同一网段内（如 192.168.2.x），识别不同设备的局域网 IP。
  - 修改此前创建的分流规则：将源地址的网段删掉，直接替换为设备具体 IP（如手机 192.168.2.235，电脑 192.168.2.222）。
- **IP 稳定性保障**：
  - 讲者指出路由器 DHCP 有机制保障同一 WiFi 下虚拟 MAC 也不易改变 IP。
  - 终极防护：可在路由器概览页将设备 IP 设为“静态”，或在设备端手动配置静态 IP。
  - 系统特例：[[ios18]] 以上用户需将“[[私有无线局域网地址]]”设置为关闭或固定（推断为防 MAC 轮替改变局域网 IP）。

## 常见坑与解决方案
- **问题描述**：在创建第 10 个 WiFi 的时候会出现问题 [2]。
  - 解决方案：在高级设置里配置当前接口的名称，随便设置一个不重复的名称 [2]。
- **问题描述**：手机 WiFi 列表中并没有出现新创建的 WiFi（SSID） [4]。
  - 解决方案：编辑刚才添加的 WiFi，取消勾选隐藏 SSID [4]。
- **问题描述**：直接用国内网络直连 [[ss]] 和 [[socks]] 会“送死”被检测 [5]。
  - 解决方案：配置链式代理，用其他稳定的机场节点来拉住宅 IP [5]。
- **问题描述**：插件启动出现问题 [7]。
  - 解决方案：点击清空防火规则，并尝试重新启动 [7]。
- **问题描述**：通过 [[dhcp]] 自动分配的 IP 改变导致分流规则失效 [6]。
  - 解决方案：在概览页面将设备 IP 设置为静态绑定，或在手机端手动将设备配置为静态 IP 地址 [8]。

## 最佳实践
- 更推荐“同 WiFi 下不同设备对应不同节点”的配置方式（单 WiFi 绑 IP 模式），不需要软路由本身能发射 WiFi，设备兼容性更好 [2, 6]。
- 建议在 [[pword]] 中先做好备份，再点击恢复默认配置，清空以前的配置和节点 [3]。
- 链式代理的前置节点选择原则：住宅 IP 是哪个地区，就选择对应地区的机场节点做前置代理 [7]。
- iOS 18 以上的用户，不要将私有无线局域网地址（Mac 地址）设置为轮替，建议设置为关闭或者固定，防止设备 IP 变化 [8]。

## 关键概念

### [[软路由]]
- **定义**：搭载 OpenWrt 等系统的路由设备，用于集中运行代理工具并为连接的设备分配不同的节点IP [1]。
- **视频上下文**：作者指出传统在每台手机上运行代理工具比较繁琐且容易被业务检测，建议将节点转移到刷了 OpenWrt 系统的软路由上进行集中管理和伪装 [1]。
- **为什么重要**：它是实现“一机一IP”、创建多WiFi隔离以及在路由器底层进行节点分流的硬件与系统基础 [1]。
- **关联概念**：扩展 → [[节点分流]]

### [[住宅 IP]]
- **定义**：由真正宽带运营商（如美国ATT）提供的纯正家庭宽带网络IP地址（包含双ISP类型） [2, 3]。
- **视频上下文**：作者推荐跨境电商（如TikTok）用户使用真正的住宅IP（VPS或Socks代理）作为运营节点的最终出口，以实现不同账号的隔离 [1-3]。
- **为什么重要**：伪装度极高，能有效防止跨境电商多账号同时运营时被平台判定为账号关联 [1, 2]。
- **关联概念**：依赖 → [[链式代理]]

### [[链式代理]]
- **定义**：通过一个稳定的前置代理节点去拉动并连接另一个目标节点（如住宅IP）的技术 [3, 4]。
- **视频上下文**：作者强调不能在国内直接连接（直连）住宅IP的SS或Socks节点，否则会被封锁（“送死”），必须配置链式代理，用自建或机场的稳定节点作为前置代理来拉动住宅IP [3, 4]。
- **为什么重要**：它是保障国内网络安全、稳定地连通纯正海外住宅IP节点的必备配置技能 [4]。
- **关联概念**：未明确提及

### [[节点分流]]
- **定义**：在路由器的代理插件中，根据设备的局域网IP或WiFi网段，将流量导向不同代理节点的规则机制 [2, 3]。
- **视频上下文**：作者在代理插件（如 PassWall）中配置规则，让连接 `tk01` WiFi（192.168.11网段）和 `tk02` WiFi（192.168.12网段）的设备分别走不同的住宅IP节点；或者针对同WiFi下不同设备的特定IP设置不同节点 [2-5]。
- **为什么重要**：这是实现多账号运营时“一个WiFi对应一个节点”或“一台设备对应一个节点”的核心配置逻辑 [1, 2, 5]。
- **关联概念**：依赖 → [[软路由]]

### [[DHCP]]
- **定义**：局域网中自动为连接的设备（如手机、电脑）分配内网网段IP地址的服务 [5, 6]。
- **视频上下文**：作者在创建新的WiFi接口时开启了DHCP服务为其分配独立网段；并在后续解答了用户关于DHCP自动分配IP是否会导致节点分流规则失效的担忧 [5, 6]。
- **为什么重要**：DHCP分配的IP稳定性直接影响基于IP设置的节点分流规则是否能持续生效 [5]。
- **关联概念**：对比 → [[静态IP]]

### [[静态IP]]
- **定义**：为设备固定绑定或手动配置的、不随设备重新连接而改变的局域网IP地址 [6, 7]。
- **视频上下文**：为防止由于虚拟MAC或DHCP重新分配导致设备IP改变，作者建议在路由器后台将手机IP设置为静态，或在手机端手动配置静态IP [7]。
- **为什么重要**：确保设备始终获取相同的内网IP，从而保证节点分流规则不会因为IP变动而失效 [5, 7]。
- **关联概念**：对比 → [[DHCP]]
