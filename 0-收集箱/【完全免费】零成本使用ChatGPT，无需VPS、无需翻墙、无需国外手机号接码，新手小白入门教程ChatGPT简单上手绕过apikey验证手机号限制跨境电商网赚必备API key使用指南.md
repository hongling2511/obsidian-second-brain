## 一句话总结
讲者不良林教新手通过提取 [[sensitive ID]] 结合 [[反向代理]] 或自建 [[webui]]，免 [[翻墙]]、免海外手机号零成本使用 [[ChatGPT]]。

## 目标受众与前置知识
- **适合谁看**：想突破网络和手机号限制、希望零成本稳定使用 [[ChatGPT]] 的新手小白及普通用户。
- **前置知识**：了解 [[VPN]]/[[翻墙]] 的基础概念，知晓什么是 [[API key]]，以及基础的 [[域名]] 解析操作（推断）。

## 核心观点
1. **利用网络抓包可绕过手机号验证获取 API 调用凭证**
   - 为什么重要：解决了国内用户没有海外手机号而无法生成官方 API 凭证的卡脖子问题。
   - 依据：讲者演示在未验证手机号的情况下，通过浏览器开发者面板拦截带有黄色图标的 login 请求，从 response 中提取出的 [[sensitive ID]] 可以完美充当 [[API key]] 使用（讲者原话）。
2. **自建反向代理是兼顾免翻墙与密钥安全的必要手段**
   - 为什么重要：第三方提供的代理节点有窃取敏感密钥的风险，而官方原始 API 接口已被国内屏蔽。
   - 依据：讲者指出第三方 [[webui]] 会利用反向代理拿到用户的 key；通过注册 [[cloudflare]] 的 pages 部署 `_work.js` 代码并绑定自定义 [[域名]] 解决 DNS 污染，就能避免把 key 暴露给陌生人且实现免翻墙（讲者原话）。
3. **通过多密钥轮询可突破免费额度的速率限制**
   - 为什么重要：极大地提升了重度用户的交互体验，避免聊天中断报错。
   - 依据：讲者演示单 key 在一分钟内发送第四条信息时会报错超限；但在自建的 [[webui]] 环境变量中用英文逗号分割输入两个 key 后，实现了在一分钟内发送六条信息而不报错（讲者原话）。

## 概念关系
- [[sensitive ID]] → [[API key]]：在没有完成手机验证的账号中，前者被提取出来后可完全作为后者的平替凭证用于调用 API。
- [[API key]] → [[token]]：通过接口进行对话交互时，会根据上下文长度扣除账号内对应的 token 计费单位。
- [[cloudflare]] pages → [[反向代理]]：利用前者的 Serverless 平台上传代码，将其作为中间节点转发对官方 API 接口的请求，从而绕过屏蔽。
- 第三方 [[webui]] ↔ 自建专属 [[webui]]：前者无需配置即开即用但存在 key 泄露风险和单点故障；后者通过 [[github]] 部署，支持多 key 轮询、密码保护和环境高度自定义。

## 详细笔记
### 1. 账号注册与网页版限制
- **主讲人**：不良林
- **注册步骤**：进入 [[openai]] 旗下的 [[ChatGPT]] 官网，输入邮箱和密码并点击验证邮件。目前注册已无强制海外手机号验证限制（若弹窗提示可尝试换节点或 +86 手机号）。
- **网络要求**：必须使用非香港节点的 [[VPN]]/[[翻墙]] 工具。
- **网页版痛点**：对节点要求严苛，极易触发人机验证、封锁节点导致聊天中断。

### 2. 绕过验证提取 [[API key]]
- **概念**：[[API key]] 是身份凭证（类似密码），可在第三方应用中调用模型。因未验证手机号，官网的创建按钮不可点击。
- **提取小技巧（讲者在 [[playground]] 偶然发现）**：
  1. 登录平台后按 F12 打浏览器开发者面板，进入“网络”标签并刷新。
  2. 找到首个带黄色图标的 `login` 链接（区分于白色图标）。
  3. 查看 `response`（响应），提取 `sensitive ID` 字段的值。此值等效于 [[API key]]。
- **重置机制**：若泄露，需在 `platform` 网站重新输入账号密码登录才会使旧 key 失效生成新 key；仅退出登录无效。

### 3. 第三方 [[webui]] 与计费规则
- **连接方式**：打开开源 [[webui]] 网页，在设置中启用“自定义接口”，粘贴提取到的 key 即可免翻墙对话。
- **额度与计价（讲者原话）**：
  - 新用户自带 **5 美金** 免费额度。
  - 计费单位为 [[token]]：1个英文单词或1个中文字符约消耗 1 个 token（如 "Hello"=1 token，"你好"=2 token）。
  - 1000个 token 消耗 **0.002 美金**，5 美金约合 **250 万个 token**。
- **速率限制（讲者原话）**：免费版 API 限制为每分钟 **3 次请求** 或 **4万 token**，每日最高 **200 次请求**。

### 4. 解决隐私风险：利用 [[cloudflare]] 搭建 [[反向代理]]
- **风险来源**：免翻墙的第三方网页实质是做了一层反向代理，用户的 key 会暴露给网站搭建者。若改回官方接口 `api.openai.com` 则无法免翻墙。
- **部署反代**：
  1. 注册 [[cloudflare]] (CF)，进入 `workers and pages` 创建应用程序。
  2. 上传讲者提供的包含反代代码的 `.zip` 压缩包（内含 `_work.js`）。
  3. 部署后当网页显示“小猫咪”图案即代表生效。
- **解决 DNS 污染**：默认分配的 `.pages.dev` 域名在国内部分地区被墙（推断：DNS污染），需在 CF 添加自定义 [[域名]] 并激活 CNAME 解析记录，完成后替换原接口地址。

### 5. 突破速率限制：通过 [[github]] 部署专属 [[webui]]
- **为什么自建**：防止第三方网站倒闭、被攻击或速率超限。
- **部署流程**：使用 [[github]] 账号授权，点击 `install` 和 `deploy` 一键部署（推断：使用了支持 GitHub 联动的云部署平台）。
- **核心配置**：
  - **多 Key 轮询**：填入多个 API key 并用英文逗号分隔，可突破 1分钟3条的限制（讲者演示配置 2个 key 后一分钟发送了 6条信息）。
  - **访问密码**：设置 `Code` 防止公网陌生人爆破盗刷。
- **收尾**：自带域名同样被墙，需绑定自定义 [[域名]] 并添加 CNAME 解析，关闭代理状态。

### 6. 移动端使用与高级环境变量
- **多端适配**：自建的 [[webui]] 自动适配手机端浏览器，在电信 5G（未开VPN）下可直连访问。
- **参数修改**：可在环境变设置中增删 key、修改密码。
- **特殊环境变量**：
  - 设置某参数为 `1` 可禁止他人手动在网页输入 key。
  - 设置某参数为 `1` 可开启界面的 API 余额查询功能。
  - 修改后必须进入部署页面点击 `redeploy`（重新部署）并勾选“使用缓存”方可生效。
- **补充说明**：除 API 之外，还有基于 [[access token]]（视频发音类似 xess token）的调用方式，优缺点不同，视频未展开。

## 常见坑与解决方案
- **问题描述**：使用网页版聊天遇到节点被屏蔽、频繁出现人机验证或聊天中断 [15]。
  - 解决方案：改用 `[[WebUI]]` 配合 `[[API key]]` 的方式进行调用 [15]。
- **问题描述**：使用第三方反代网站会导致 `[[API key]]` 有泄露给第三方的风险 [8]。
  - 解决方案：将接口地址改回官方 API 接口（需翻墙），或使用 `[[Cloudflare]]` 自建反向代理来杜绝泄露风险 [10]。
- **问题描述**：Cloudflare Pages 分配的域名或自建 WebUI 的默认域名无法访问（DNS 污染或被墙）[11, 13]。
  - 解决方案：添加自定义域名，并设置 CNAME 的解析记录 [4, 13]。
- **问题描述**：免费用户调用 API 存在一分钟 3 次请求的速率限制，连续发信会报错 [9]。
  - 解决方案：自建 WebUI 时填入多个 API key（用逗号分隔），利用轮询机制突破单账号的请求次数限制 [9, 12]。

## 最佳实践
- **保护接口与防滥用**：在公网部署专属 `[[WebUI]]` 时，必须设置复杂且高强度的访问密码（Code），防止被他人爆破和滥用导致 API 额度耗尽 [12]。
- **API 凭证安全管理**：将 `[[API key]]` 视作密码妥善保管。如不慎泄露，务必通过“重新输入账号密码登录平台”的方式强制刷新令牌，使其失效 [1, 6]。
- **突破限制方案**：推荐通过配置多个 `[[API key]]` 的轮询模式，来规避单节点/单账号由于频繁请求触发的速率封控拦截 [9, 12, 13]。

## 关键概念

### [[API key]]
- **定义**：ChatGPT 的一种身份凭证，允许开发者在应用程序中调用 API 与 ChatGPT 进行交互 [1, 2]。
- **视频上下文**：作者强调它类似于密码，用于在 WebUI 等第三方工具中调用 ChatGPT 并实现免翻墙使用，必须妥善保管以防泄露 [2-4]。
- **为什么重要**：它是脱离官方网页版限制、使用第三方工具或自建代理调用大模型的核心前置条件 [1, 3]。
- **关联概念**：依赖 → [[sensitive ID]] 

### [[WebUI]]
- **定义**：一个提供前端图形界面的开源网页项目，用于配合 API key 来进行聊天交互 [3]。
- **视频上下文**：作者演示了使用他人搭建或自建的 WebUI 来替代官方网页版，解决节点被屏蔽、人机验证断连等问题，并实现多设备免翻墙访问 [1, 3, 5]。
- **为什么重要**：它提供了一个可视化的交互窗口，让普通用户无需编程即可通过 API 方式便捷地使用大模型 [3, 6]。
- **关联概念**：依赖 → [[API key]]

### [[Token]]
- **定义**：ChatGPT 独有的计费和计量单位 [6]。
- **视频上下文**：作者解释一个英文单词大约消耗一个 Token，中文字符视情况消耗一到两个，且连续聊天携带大量上下文会显著增加 Token 消耗 [4, 6]。
- **为什么重要**：直接决定了通过 API 方式使用 ChatGPT 的成本以及免费额度（如 5 美金）的消耗速度 [4, 6]。
- **关联概念**：未明确提及

### [[sensitive ID]]
- **定义**：在免手机号验证的情况下，通过浏览器开发者工具的网络面板拦截到的可充当 API key 的敏感标识 [2, 7]。
- **视频上下文**：作者利用抓包 playground 的 login 响应获取此 ID，以此作为绕过 OpenAI 手机号验证限制来生成并提取 API 凭证的技巧 [2, 7]。
- **为什么重要**：提供了一种零成本获取 API 调用权限的非正规途径，极大降低了新手的入门门槛 [7]。
- **关联概念**：属于 → [[API key]]

### [[反向代理]]
- **定义**：代为向 ChatGPT 官方接口发送请求的中间服务器机制 [4, 8]。
- **视频上下文**：作者指出第三方 WebUI 帮用户做了一层反向代理所以能免翻墙，但这会将 API key 暴露给网站提供者；随后教导用 Cloudflare 自建反向代理以确保安全 [4, 8, 9]。
- **为什么重要**：是国内用户实现“免翻墙”访问接口的核心技术原理，同时也是决定 API 凭证是否安全的关键环节 [4, 8]。
- **关联概念**：依赖 → [[Cloudflare Pages]]

### [[轮询]]
- **定义**：一种配置多个 API key 轮流进行接口请求的调用机制 [10-12]。
- **视频上下文**：作者演示在自建 WebUI 部署时填入多个用逗号分隔的 API key，使得系统在发送消息时自动切换 key [11, 12]。
- **为什么重要**：解决了普通 API 账号存在“一分钟只能聊三句”的速率限制（Rate limit）痛点，大幅提升了连续对话的体验 [4, 10, 12]。
- **关联概念**：依赖 → [[API key]]

### [[Cloudflare Pages]]
