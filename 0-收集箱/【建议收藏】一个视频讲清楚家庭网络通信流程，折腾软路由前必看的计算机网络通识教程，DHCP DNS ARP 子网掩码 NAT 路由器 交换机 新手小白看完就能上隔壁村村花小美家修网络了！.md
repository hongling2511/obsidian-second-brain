## 核心观点
1. **理解家庭网络基础通信流程是配置高级网络设备的前提**
   - 为什么重要：为后续的[[软路由]]教程提供必备的基础。
   - 依据：“由于折腾软路由需要一定的网络基础，所以作为开篇的第一期我们不讲软路由，来讲讲一般家庭的网络基本通讯流程”（讲者原话）。
2. **[[交换机]]与[[路由器]]在网络模型中负责不同层级的任务**
   - 为什么重要：解释了局域网数据如何被准确送达。
   - 依据：视频演示普通的家用[[交换机]]只能解析[[网络接口层]]数据（即[[MAC地址]]），而[[路由器]]能够处理[[网络层]]和[[传输层]]的复杂服务如[[DHCP]]、[[NAT]]等（讲者原话）。
3. **[[NAT]]是局域网设备访问互联网的关键技术**
   - 为什么重要：解决了内网私有IP无法在公网路由的问题。
   - 依据：讲者指出，[[路由器]]在将数据发往公网前，必须通过[[NAT]]将源私有[[IP]]转换为路由器WAN口的公网IP并映射端口，才能在公网上进行路由（讲者原话）。

## 概念关系
- [[DHCP]] → [[IP]]：设备连接网络时，依赖[[DHCP]]服务自动获取动态[[IP]]、[[子网掩码]]、[[网关]]和[[DNS]]信息。
- [[DNS]] → [[IP]]：浏览器访问域名（如[[百度]].com）时，依赖[[DNS]]服务将域名解析为实际的[[IP]]地址。
- [[ARP]] → [[MAC地址]]：设备只知道目标[[IP]]时，需要通过发送[[ARP]]广播请求来获取目标设备的[[MAC地址]]。
- [[IP]] ↔ [[MAC地址]]：[[IP]]负责[[网络层]]的逻辑寻址（确定发给哪个网络设备），而[[MAC地址]]负责[[网络接口层]]的物理寻址（网卡的唯一标识）。

- 典型家庭网络拓扑：运营商分配[[光猫]]，用户购买[[路由器]]。
- [[路由器]]的 WAN 口通过网线连接[[光猫]]，通过 [[pppoe]] 拨号获取公网IP，或直接从光猫获取内网[[IP]]。
- [[路由器]]有多个颜色一样的 LAN 口，可将其抽象为一台[[交换机]]连接到路由器的虚拟网口。该网口拥有内网[[IP]]（如192.168.1.1），作为局域网的[[网关]]。设备通过有线或 [[WiFi]] 连接到该[[交换机]]。

### 局域网IP获取流程（DHCP工作原理）
- 电脑接入网络，通过 [[DHCP]]（动态主机配置协议）自动获取IP。
- **数据封装过程**（基于 [[tcpip4层网络模型]]）：
  - [[传输层]]：使用 [[UDP]] 协议，客户端源端口68，服务端目标端口67。
  - [[网络层]]：因不知服务端IP，目标[[IP]]设为广播地址 255.255.255.255，源[[IP]]为 0.0.0.0。
  - [[网络接口层]]：源[[MAC地址]]为电脑网卡MAC（如CC），目标MAC为全F的广播地址。
- **交换机转发**：[[交换机]]接收数据，将其对应关系记录在MAC地址映射表，因目标是全F广播，向所有接口转发。
- **路由器响应**：一般家用[[路由器]]提供多种服务（包含 [[DHCP]], [[DNS]], [[DDNS]], [[QoS]], [[UPnP]], [[DMZ]], 端口转发，防火墙等）。[[路由器]]的[[DHCP]]服务监听到[[UDP]] 67端口数据，从IP地址池选择未占用的IP（如192.168.1.3）、结合[[子网掩码]]、[[网关]]、[[DNS]]及租约信息进行分配。
- [[路由器]]封装回包（目标[[IP]]为全255广播，目标MAC为电脑的MAC），[[交换机]]根据MAC表定向发给电脑，电脑完成网络配置。
- **租约与续期机制**：电脑在租约过半（如12小时的1/2，即6小时）时请求续约；失败则在剩余时间1/8时再试。彻底到期未响应则自动分配 169.254.x.x 的网段以确保局域网通信。公共场所网络租期短（十来分钟），家庭网络一般默认720分钟（讲者原话）。若局域网存在多台DHCP服务器会导致网络问题（DHCP攻击）。

### 域名解析流程（DNS与ARP工作原理）
- 浏览器输入 [[百度]].com 后，需通过 [[DNS]] 寻找对应IP。
- **DNS请求封装**：源端口随机（如9527），目标端口53，使用 [[UDP]] 传输。目标[[IP]]为[[网关]]（路由器IP）。
- **获取路由器MAC**：因电脑不知路由器MAC，发起 [[ARP]] 广播请求（我是192.168.1.3，谁是192.168.1.1，目标MAC全F）。[[路由器]]收到后回应自己的MAC（如BB），双方均将对方的MAC保存在自己的 ARP缓存 列表中。
- **DNS转发与响应**：电脑拿到MAC后发给[[路由器]]。[[路由器]]不知道百度的IP，转发给上游[[DNS]]服务器（通过拨号获取）。拿到[[百度]]IP后，路由器将其保存在 DNS缓存，并将结果返回给电脑，电脑端也会根据 TTL 时间进行缓存。

### 公网通信与网络地址转换（NAT工作原理）
- 浏览器向 [[百度]] IP发起 [[http]] 请求。目标端口80，使用 [[TCP]] 传输。
- **跨网段判断**：电脑通过 [[子网掩码]]（如255.255.255.0，即 CIDR表示法 中的 /24）计算出目标IP与自己不在同一网段，需将数据发给默认[[网关]]。若普通设备有权限开启IP转发，也可帮忙转发当网关（俗称 旁路由）。
- **网络地址转换（NAT）**：[[路由器]]发现源[[IP]]是局域网私有IP（192.168.1.3），无法在公网路由。必须执行 [[NAT]]（网络地址转换），将源IP改为路由器WAN口的公网IP，源端口改为随机空闲端口（如4134），并在NAT映射表中记录。
- **数据返回与渲染**：公网路由十几个设备后到达百度的服务器。[[百度]]返回网页数据给路由器的4134端口。[[路由器]]查NAT映射表，转换回内网IP 192.168.1.3 和端口 9527，通过[[交换机]]发给电脑。浏览器收到数据并渲染呈现。

## 步骤拆解
1. **建立物理网络与外部接入**
   - 具体操作描述：使用网线将路由器的 WAN 口与光猫连接，路由器通过拨号或通过光猫分配以获取外部公网或内网 IP 地址 [1]。
   - 关键代码/命令：未明确提及
   - 注意事项：通常情况下颜色不一样的接口就是路由器的 WAN 口 [1]。

2. **终端获取局域网 IP（[[DHCP]] 流程）**
   - 具体操作描述：设备接入网络后，[[DHCP]] 客户端自动发起获取 IP 地址的请求 [2]。由于不知晓服务端位置，请求会在网络层和接口层封装为全网广播，由交换机转发给所有连接接口 [3, 4]。路由器的 [[DHCP]] 服务收到后，从地址池中分配未被占用的 IP、子网掩码、网关、[[DNS]] 和租约等信息，并单播返回给目标设备 [4, 5]。
   - 关键代码/命令：客户端默认使用 [[UDP]] 端⼝ `68`，服务端默认使用 [[UDP]] 端⼝ `67` [2]；目标 IP 为广播地址 `255.255.255.255`，原 IP 设为 `0.0.0.0` [3]；目标 MAC 为全 F 的广播地址 `FF:FF:FF:FF:FF:FF` [3]。
   - 注意事项：客户端会在租约时间剩余二分之一时发起续约请求 [5]。

3. **终端手动配置 IP（可选流程）**
   - 具体操作描述：在设备设置中手动配置 IP 地址、子网掩码、网关和 [[DNS]]，其中网关和 [[DNS]] 一般设置为路由器的 IP 地址 [6]。
   - 关键代码/命令：未明确提及
   - 注意事项：手动设置的 IP 必须与路由器在同一网段内，且需要避免与其他设备的 IP 产生冲突 [6]。

4. **域名解析与获取物理地址（[[DNS]] 与 [[ARP]] 流程）**
   - 具体操作描述：浏览器发起访问时，先通过本地随机空闲端口向路由器发送 [[DNS]] 请求以获取域名对应的 IP [6, 7]。若设备未缓存路由器网关的 MAC 地址，会先发起 [[ARP]] 广播请求询问网关 MAC 地址，收到响应并缓存后再将 [[DNS]] 数据包发给路由器 [7, 8]。路由器会将请求发给上游 [[DNS]] 服务器并将获取的结果缓存及返回给终端 [8, 9]。
   - 关键代码/命令：[[DNS]] 服务默认使用 [[UDP]] 传输及 `53` 号端⼝ [6, 7]。
   - 注意事项：电脑和路由器均会在本地缓存 [[ARP]] 列表和 [[DNS]] 结果一段时间，缓存有效期内无需重新发起请求 [7, 8]。

5. **公网通信与地址转换（[[HTTP]] 与 [[NAT]] 流程）**
   - 具体操作描述：浏览器向目标 IP 发起 [[HTTP]] 请求 [9]。电脑通过子网掩码计算出目标 IP 不在同网段，将数据发给默认网关（路由器）帮忙转发 [9, 10]。路由器执行 [[NAT]] 转换，将局域网源 IP 替换为路由器的公网 IP，并随机替换源端口，将映射关系保存在表中后发往互联网 [11]。收到公网响应后，路由器根据 [[NAT]] 映射表转回原始局域网 IP 和端口，交还给终端 [11]。
   - 关键代码/命令：[[HTTP]] 协议默认端口为 `80`，默认传输方式为 [[TCP/IP]] [9]。
   - 注意事项：普通网络设备收到非发给自己的包会直接丢弃，但开启了 IP 转发功能的设备（旁路由）可帮忙转发 [10]。

## 最佳实践
- 公共场所（如机场、商场）的 WiFi [[DHCP]] 租期建议设置得较短（一般十来分钟），确保人员离开后 IP 能尽快释放，防止地址池被占满 [12]。

## 关键概念

### [[DHCP]]
- **定义**：全称动态主机配置协议，用于给网络设备动态分配和获取 IP 地址、子网掩码、网关和 DNS 等信息的应用层协议 [1, 2]。
- **视频上下文**：当电脑连接网络且设置为自动获取 IP 时，会通过发送广播包寻找 DHCP 服务器（通常由路由器提供）；服务器收到后会从地址池中分配一个未被占用的 IP 并赋予一定的租约时间（如12小时） [1-4]。
- **为什么重要**：它是局域网中设备实现即插即用、正常联网的核心基础服务；若局域网内缺乏该服务，设备只能获取到 169.254 开头的无效 IP，导致无法上网 [5]。
- **关联概念**：属于 → [[路由器]]

### [[MAC 地址]]
- **定义**：也叫物理地址，是每个网卡独有的唯一识别码 [3]。
- **视频上下文**：数据包在网络接口层会封装源 MAC 地址和目标 MAC 地址；电脑初始不知道目标设备位置时，会使用全 F 的广播 MAC 地址发包 [2, 3]。
- **为什么重要**：在局域网内部，底层数据传输完全依赖 MAC 地址进行物理定位和投递，它是交换机识别并转发数据的唯一依据 [3, 4]。
- **关联概念**：扩展 → [[ARP]]

### [[交换机]]
- **定义**：一个只能解析网络接口层数据（即 MAC 地址），并根据 MAC 地址对数据包进行转发的设备 [3]。
- **视频上下文**：视频将路由器的四个 LAN 口视作一台交换机；它内部维护一个 MAC 地址映射表，收到数据时，会根据目标 MAC 地址查找对应接口进行精确转发或广播 [1-4]。
- **为什么重要**：它是连接局域网内所有终端设备的核心物理枢纽，负责在硬件层面把数据线缆连通并正确投递数据包 [2, 3]。
- **关联概念**：依赖 → [[MAC 地址]]

### [[路由器]]
- **定义**：连接光猫（外网）和局域网（内网），并负责提供网关、路由转发及多项网络附加服务的核心网络设备 [1, 2, 6]。
- **视频上下文**：路由器通过 WAN 口获取运营商的公网或内网 IP，通过 LAN 口连接家庭设备，同时在其内部提供 DHCP、DNS 缓存、网关路由以及 NAT 转换等服务 [2, 6-9]。
- **为什么重要**：它是家庭局域网与外部互联网进行跨网段通信的“海关”，没有它，内网设备无法实现对外部公网的访问 [8, 9]。
- **关联概念**：扩展 → [[NAT]]

### [[DNS]]
- **定义**：负责将人类易于记忆的域名（如百度的网址）转换为计算机通信实际所需的 IP 地址的服务 [10]。
- **视频上下文**：浏览器访问网站前，会通过 UDP 发起 DNS 请求寻找目标域名的 IP；路由器接收请求后会查找上游服务器，并将获取到的 IP 保存在本地 DNS 缓存中一段时间 [7, 10-12]。
- **为什么重要**：它免去了人类记忆复杂数字 IP 的麻烦，并且其本地缓存机制能有效提升网页重复访问时的加载速度 [7, 10]。
- **关联概念**：未明确提及

### [[ARP]]
- **定义**：一种在局域网内通过已知的目标 IP 地址来查询对应设备目标 MAC 地址的广播请求 [11]。
- **视频上下文**：当电脑需要把数据发给路由器（网关）但只知道其 IP 而不知道其 MAC 地址时，会发送 ARP 广播询问；得到回应后，电脑会将该映射关系保存在本地的 ARP 缓存列表中 [7, 11]。
- **为什么重要**：它是连接网络层（IP 寻址）和网络接口层（MAC 寻址）的关键桥梁，缺失它将导致局域网内设备无法建立物理层的通信 [11]。
- **关联概念**：依赖 → [[MAC 地址]]

### [[子网掩码]]
- **定义**：用于计算机判断目标 IP 地址与本机 IP 地址是否处于同一个网段的二进制掩码标识（如 255.255.255.0 或 /24） [8, 12]。
- **视频上下文**：当电脑需要访问外部服务器时，会将本机 IP、目标 IP 与子网掩码转换为二进制进行对比；如果前 24 个比特位（网络部分）相同，说明在同网段可直接访问；如果不相同，则说明在不同网段，必须把数据交给网关处理 [8, 12]。
- **为什么重要**：它决定了网络设备在发包时的基础逻辑：是直接在局域网内寻找目标，还是必须将数据委派给网关进行跨网路由 [8]。
- **关联概念**：依赖 → [[网关]]

### [[网关]]
- **定义**：当设备需要与不同网段（如公网）通信时，负责帮忙转发数据包的默认出口设备，通常是路由器的内网 IP 地址 [1, 8]。
- **视频上下文**：电脑发现要访问的目标 IP（如百度服务器）与自己不在同一网段时，会将包含目标数据的包发送给配置好的网关 IP（192.168.1.1），请求其代为转发到互联网 [8]。
- **为什么重要**：它是所有局域网内部设备通向外部世界（广域网）的唯一必经大门 [8]。
- **关联概念**：属于 → [[路由器]]

### [[NAT]]
- **定义**：网络地址转换，将只能在局域网内部使用的内网 IP 修改为可以在公网上进行路由的公网 IP 的技术过程 [9]。
- **视频上下文**：路由器在将内网电脑的数据发往外网前，会将其源 IP 和源端口替换为路由器自身的公网 IP 和随机端口，并将此映射关系记录在 NAT 映射表中；等外网数据返回时，再依据此表转换回内网设备的 IP 和端口 [9]。
- **为什么重要**：它解决了局域网私有 IP 无法在互联网上被识别和路由的根本问题，允许多台内网设备共享同一个公网 IP 上网 [9]。
- **关联概念**：依赖 → [[路由器]]
