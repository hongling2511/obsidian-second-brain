## 一句话总结
由讲者[[不良林]]主讲，从零开始讲解[[软路由]]概念、设备选购、固件刷入及系统配置，实现全家庭网络设备[[科学上网]]。

## 目标受众与前置知识
- **适合谁看**：从未接触过[[软路由]]、需要解决全家网络设备（含无法安装代理的设备）科学上网痛点的新手（如讲者比喻的隔壁村[[春花]]、[[小美]]）。
- **前置知识**：[[家庭网络]]拓扑、[[IP地址]]网段划分、[[网关]]概念、[[DHCP]]服务原理（推断，讲者明确提到需回看上期家庭网络诉通视频相关概念）。

## 核心观点
1. **[[软路由]]是解决家庭全设备[[科学上网]]的通用且高效方法**
   - 为什么重要：因为部分网络设备无法直接安装[[代理工具]]（如 [[v2a]]、[[clash]]、[[小火箭]]）。
   - 依据：视频列举了[[电视盒子]]看 [[YouTube]]、[[VR头显]]（vr头衔）首次激活需访问 [[Facebook]] 等场景。将代理工具运行在[[路由器]]上，所有局域网设备通过它连接[[节点服务器]]解密数据，无需单独安装客户端即可访问被墙网络（讲者原话演示逻辑）。
2. **新手入门建议购买小主机折腾，而非直接刷硬路由**
   - 为什么重要：刷机风险低，可拓展性强。
   - 依据：[[硬路由]]系统安装在不可更换的主板芯片 [[Flash闪存]] 中，操作不当会导致设备变砖难以恢复；而小主机的系统安装在外置的 [[SD卡]] 或 [[U盘]] 里，随时可以更换（讲者原话）。
3. **推荐直接使用官方纯净固件或其分支项目，不推荐使用别人编译好的固件**
   - 为什么重要：避免系统臃肿和潜在的安全隐患。
   - 依据：别人编译的固件可能带有永远不用的插件，最重要的是可能“夹带了撕货”，运行收集信息的脚本甚至留下主机上线后门（讲者原话）。

## 概念关系
- [[硬路由]] ↔ [[软路由]]：硬路由指功能被厂商固化的常规路由器，而软路由是可以安装软件的路由器系统（形态可为[[迷你PC]]、[[工控小主机]]、[[笔记本电脑]]、[[开发板]]等）。
- [[光猫]]桥接模式 → [[路由器]][[PPPoE]]拨号：避免内网数据进行两次 [[NAT]] 处理，以此减少网络性能损耗（讲者明确表示双重nat有性能损耗）。
- [[ext4]] ↔ [[squashfs]]：两种文件系统格式的固件，[[squashfs]] 修改的配置文件写入 [[overlay分区]]，支持抹除分区恢复出厂设置；[[ext4]] 无此机制无法恢复出厂设置（讲者原话）。

## 详细笔记
### 痛点与软路由概念科普
- **科学上网痛点**：普通家庭网络中，[[电脑]]、[[手机]]可以通过运行代理工具访问[[谷歌]]、[[百度]]，但[[电视盒子]]、[[游戏机]]、[[VR头显]]等无法安装代理工具。
- **软路由定义**：除电脑外，受芯片控制的如[[冰箱]]、[[空调]]、[[洗衣机]]、[[电视]]、[[音响]]等算作[[嵌入式设备]]。软路由不是具体产品，而是能安装路系统的设备（如安装 [[Windows]] 就是电脑，装路由系统就是软路由）。
- **常见系统**：[[openwrt]]（简称[[op]]，本教程重点，免费开源）、[[pfsense]]、[[Rose]]、[[爱快]]、[[梅林]]等。

### 设备选购与硬件科普
- **支持设备查询**：访问官网搜索如 [[小米]]、[[TP LINK]]、[[华硕]]、[[红米]]（如型号 [[ax6000]]）判断是否可刷。
- **推荐选购标准**：必须有至少两个网口。
- **设备演示与对比**：
  - [[nanopr2s]]（或称[[r2s]]、由[[有善]]/推断为友善生产）：属于 [[ARM]] 架构开发板，价格 250 元（讲者原话），待机功耗不到两瓦（讲者原话），无内置存储，依靠外置 SD 卡启动。适合不折腾仅科学上网。
  - [[红米ax6000]]：[[硬路由]]，价格 350 元（讲者原话），自带无线 [[WiFi]]，待机功耗 7 瓦左右（讲者原话），[[Flash闪存]] 容量 128 兆（讲者原话）。需注意硬路由刷机依赖第三方 [[ubut]] (推断为 U-Boot) / 原厂 ubut 的分区布局差异。
  - [[x86]]小主机：杂牌，价格 850 元（讲者原话），CPU型号为[[英特尔]]的 [[n100]]（常见型号还包括面向低功耗的 [[j1900]]、[[n5105]]、[[奔腾]]、[[赛扬]]，以及桌面级的 [[i5]]、[[i7]]、[[i9]]），含 166g内存加512g硬盘（讲者原话，推断实际为16G内存），支持虚拟化，性能过剩。

### 固件获取与编译方式
- **获取固件三种途径**：他人编译版（不推荐）、自行编译（对新手繁琐不推荐）、官方固件及分支项目（推荐）。
- **推荐版本**：讲者推荐使用 openwrt 的分支项目 [[Moto wrt]]（发音和屏幕显示为 [[amoto]] / [[amoto wrt]]，推断为 immortalwrt），因其带有常用插件源且有 CDN 加速。
- **固件细节与差异**：以最新版 23.05.01 为例，防火墙工具由 [[IP Table]] 变更为 [[nf tables]]，可能导致旧插件如 [[ssrp]] 不兼容。下载时需辨认文件格式：[[combine]] 表示内核与根文件系统在固件中，[[efi]] / [[legacy]] 对应不同主板引导类型，[[img]] 为格式，[[jz]] (推断为 gz) 为压缩包。
- **在线编译**：可使用官网 [[Image builder]] 在线剔除或增加插件（如不需要[[IPv6]]可删减），但消耗官方服务器资源，不推荐新手高频使用。

### 固件刷入过程（写盘）
- **Windows系统写盘**：讲者首推 [[refess]] 工具（推断为 Rufus），体积为 1.4兆大小（讲者原话），将固件写入 SD 卡或 U 盘。写入时会被清空数据。
- **Mac/Linux系统写盘**：可使用另一款免安装便携版工具（未明确提及工具名，拼写为 mik 和 Linux 用户可用）进行烧录操作。
- **引导启动**：将装好系统的U盘插入 x86 小主机。若主机内原本安装了 [[Win11]] 等系统，开机需按 delete 等键进入 [[BIOS]]，将 boot option 的启动优先级从 Hard Disk 切换至 USB 设备引导。进入后系统通过命令行如 `ip a` 查询网卡，默认管理地址为 192.168.1.1。

### OpenWrt初始化与网络配置
- **LAN/WAN口定义**：区别于硬路由的颜色区分，软路由通过软件定义网口，如默认 [[eth0]] 为 LAN 口（[[lco]]），[[eth1]] 为 WAN 口。
- **登录管理后台**：电脑网卡设为自动获取IP，插对LAN口后，浏览器输入默认IP访问 [[lcy]] (推断为 LuCI) 管理界面，初始用户名为 ROOT，密码为空。
- **基础网络配置**：
  - 关闭所有 [[IPv6]]：在接口和 DHCP 服务器中禁用所有相关服务和解析，避免透明代理出现问题。
  - 配置网桥（[[br-lan]]）：将除 eth1 外多余的网口桥接到 LAN 口，相当于给[[交换机]]添加网口。
  - 防止网段冲突：WAN 口连接光猫后，若获取的IP（如 192.168.1.13）与 LAN 口在同一网段，必须修改 LAN 口网段（如改为 192.168.2.1）。系统有 90秒（讲者原话）防误触回滚机制。

### 拨号上网与旧路由变身无线AP
- **网络模式优化**：若光猫处于路由模式，建议改为桥接模式。在软路由 WAN 口将协议切换为 [[PPPoE]] 拨号，输入宽带账号密码获取公网 [[IPv4]]。
- **无线覆盖恢复**：软路由大多无无线功能，需将退役的硬路由（以小米路由器为例）设为有线中继模式（或手动关闭 DHCP、LAN口相连并修改管理IP同网段），作为无线的接入点（AP）。

### 插件安装与节点测试
- **终端与界面美化**：进入系统软件包，更新列表。安装主题 [[lucy-theme-argon]]（讲者搜索 ug / ly them ag）和网页命令行终端 [[ttyd]] 及其汉化包。
- **代理插件安装**：演示安装 [[open clash]]、基于 [[sbox]] 内核的 [[home proxy]]、[[password]]（推断指 Passwall 插件）及其汉化包。
- **节点导入验证**：在 password 插件中演示导入自建节点链接或通过本频道推荐的专线机场订阅链接（含解锁 [[ChatGPT]] (chgpt)、[[奈飞]] 流媒体）。开启主开关，通过访问谷歌测试是否生效。若遇依赖未安装提示需重启路由器。
- **后续预告**：讲者提及本次演示为“主路由模式”，下期将更新“[[旁路由]]”模式。

## 金句亮点
- 简单理解软路由就是可以安装软件的路由器，从形态上来讲它可能都不是一个路由器。 — 概念定义 / 科普价值
- 刷别人（编译）的固件存在安全风险，可能夹带了私货，跑他的脚本收集你的信息，甚至给你安装有主机上线后门。 — 经验避坑 / 安全警告
- 如果你买的这个硬路由能通过一些手段刷成其他路由系统，那么它的称呼就从硬路由变成了软路由了。 — 概念定义 / 原理揭示

## 行动建议
- **排查光猫工作模式**：登录家庭光猫后台，查看是桥接模式还是路由模式，若为路由模式，联系客服或自行将其改为桥接模式。
  - **关联观点**：对应视频提到的“减少双重NAT性能损耗”环节。
  - **验证标准**：软路由WAN口从DHCP获取IP变为手动配置PPPoE拨号，且能成功获取IP并上网。
- **下载与烧录官方纯净固件**：使用闲置U盘或SD卡，下载OpenWrt官方纯净版或amoto分支版固件，使用refess（推断Rufus）工具将其写入。
  - **关联观点**：3
  - **验证标准**：U盘插入小主机后，能成功引导进入OpenWrt系统的命令行界面并显示IP地址。
- **改造旧路由器为无线AP**：将软路由配置完毕后，把换下来的旧硬路由切换至“有线中继”模式，或关闭DHCP并修改管理IP与软路由同网段后将两者的LAN口直连。
  - **关联观点**：1
  - **验证标准**：手机等无线设备能连上旧路由器的WiFi信号，并成功通过软路由上网。

## 步骤拆解
1. **固件获取**
   - 具体操作描述：访问 [[Moto wrt]] 网站，选择对应的 op 版本（23.05.01）。根据设备型号搜索并下载纯净版固件。根据需求选择 ext4 格式（不支持恢复出厂设置）或 squashfs 格式（支持恢复出厂设置）[1, 5]。
   - 关键代码/命令：未明确提及。
   - 注意事项：不建议使用 Image builder 在线编译，因为耗时且会消耗服务器资源，建议直接下载纯净固件 [6, 7]。

2. **固件刷写（写盘）**
   - 具体操作描述：使用 [[Rufus]]（Windows 环境）或其他写盘工具，将 img 固件解压（或直接拖入压缩包）并完整写入到 U盘或 SD 卡中 [8-10]。
   - 关键代码/命令：未明确提及。
   - 注意事项：写盘操作会抹掉磁盘内所有分区信息和数据，需提前做好备份 [9]。

3. **软路由启动与基础网络配置**
   - 具体操作描述：对于 x86 设备，通电开机并进入 BIOS，将 Boot Option 1 修改为 USB 设备引导。使用网线将电脑连接至软路由 LAN 口（默认为 eth0），通过 DHCP 自动获取 IP。访问管理后台（默认 IP 为 192.168.1.1，用户名 root，密码为空）。在接口设置中，删除 WAN6 接口并禁用 LAN 口的所有 IPv6 服务，过滤 IPv6 解析。将多余网口桥接至 br-lan [2, 10-15]。
   - 关键代码/命令：`ip a` （用于在系统命令行中查看网卡信息）[11]。
   - 注意事项：未明确提及。

4. **WAN 口连接与网段防冲突**
   - 具体操作描述：将光猫的网线插入软路由 WAN 口（eth1）。修改 LAN 口的网段 IP（如改为 192.168.2.1）以防止与 WAN 口所在的光猫网段冲突。如光猫为桥接模式，则在 WAN 口将协议修改为 PPPoE 并输入宽带账号密码进行拨号 [3, 4, 16, 17]。
   - 关键代码/命令：未明确提及。
   - 注意事项：修改 LAN 口 IP 后，系统有 90 秒等待时间，若未在 90 秒内使用新 IP 访问管理页面，配置将自动回滚以防失联 [3, 16]。

5. **硬路由改造为无线 AP**
   - 具体操作描述：进入家中原有的硬路由管理页面，将其工作模式切换为“有线中继”。或者手动关闭硬路由的 DHCP 服务，将其管理 IP 修改为与软路由 LAN 口同网段的空闲 IP，然后将软路由 LAN 口与硬路由的 LAN 口相连 [18, 19]。
   - 关键代码/命令：未明确提及。
   - 注意事项：中继模式下，硬路由的 WAN 口也可以作为 LAN 口使用；而手动配置模式下则只能空置 WAN 口 [19]。

6. **安装插件与配置科学上网**
   - 具体操作描述：在软件源更新列表，搜索并安装终端插件 [[ttyd]] 及其汉化包。分别搜索并安装 [[OpenClash]]、[[PassWall]]、[[home proxy]] 及其汉化包。在对应插件界面导入节点/订阅链接并启动。对于 [[OpenClash]]，需进入版本更新页面下载对应架构内核，开启订阅转换并更新配置后启动 [20-26]。
   - 关键代码/命令：未明确提及。
   - 注意事项：同一时间只能运行一款科学上网插件，否则会造成冲突 [24]。

## 常见坑与解决方案
- **问题描述**：固件版本升级导致防火墙工具从 iptables 变为 [[nftables]]，部分旧插件可能存在不兼容 [1]。
  - 解决方案：未明确提及（视频仅表示“踩到坑了再说吧”）。
- **问题描述**：电脑连接网口后，网卡获取到 169.254.x.x 的未识别网络 IP，说明插错了网口 [13]。
  - 解决方案：更换另一个网口插入，直到成功获取 DHCP 分配的 IP [13]。
- **问题描述**：光猫使用路由模式负责拨号，导致软路由与光猫之间形成双重 NAT，存在性能损耗 [4, 17]。
  - 解决方案：将光猫连接模式改为桥接模式，由软路由直接进行 PPPoE 拨号 [17]。
- **问题描述**：安装依赖较多的插件（如 [[OpenClash]]）时，前端浏览器出现报错 [21]。
  - 解决方案：无需干预，后台仍在自动执行安装，稍等片刻后刷新网页即可 [21, 22]。
- **问题描述**：配置完 [[PassWall]] 节点并开启后，显示未运行，日志提示“依赖未安装” [24]。
  - 解决方案：重启路由器让设置生效 [24]。
- **问题描述**：未翻墙状态下，[[OpenClash]] 无法连接 GitHub 下载内核 [25, 26]。
  - 解决方案：先使用 [[PassWall]] 或 [[home proxy]] 开启代理后再下载，或者进入“复写设置”更换 CDN 的下载地址挨个尝试 [26]。

## 最佳实践
- **推荐做法**：新手建议购买使用外置 SD 卡或 U盘 的小主机（如 x86 或 ARM 开发板）来折腾，因为硬路由 Flash 容量受限且操作不当容易彻底变砖难恢复 [27]。
- **推荐做法**：路由器应以稳定为主，不应盲目追新 [1]。
- **推荐做法**：避免刷别人预编译好的固件（可能夹带私货脚本、收集信息或过于臃肿），推荐使用纯净官方固件（或 [[Moto wrt]]）按需自行在软件源下载插件 [1, 28, 29]。

### [[软路由]]
- **定义**：可以安装软件（如路由系统）的路由器，形态多为迷你 PC、工控小主机、闲置笔记本或开发板 [1]。

- **为什么重要**：突破了传统家用路由器功能的限制，可通过安装代理插件实现局域网内所有连网设备的自动科学上网 [3, 4]。
- **关联概念**：对比 → [[硬路由]]

### [[硬路由]]
- **定义**：我们在网上随便买的普通家用路由器，功能基本被厂商固化，只能满足最基本的路由需求 [1, 3]。
- **视频上下文**：作者指出其因系统安装在不可更换的 Flash 闪存中，无法自行添加新功能（如安装代理工具），如果通过手段刷入其他系统，其称呼就变成了软路由 [1, 3, 5]。
- **为什么重要**：是普通家庭最常见的网络设备，但在高级网络配置中存在局限性，通常会在软路由网络拓扑中被降级用作无线 AP（交换机）使用 [3, 6]。
- **关联概念**：对比 → [[软路由]]

### [[OpenWrt]]
- **定义**：一个开源免费的路由系统，提供完整的路由功能并且能安装各式各样的第三方插件 [1, 4]。

### [[固件]]
- **定义**：在嵌入式开发领域中，指安装并运行在设备里的操作系统文件（如 img 格式文件） [7, 12]。
- **视频上下文**：作者介绍了获取固件的三种途径（使用别人编译好的、自己编译、官方纯净固件），并详细演示了利用写盘工具将固件镜像完整拷贝到 U 盘或 SD 卡的刷机过程 [7, 8, 11-13]。
- **为什么重要**：是软路由设备启动和运行的基础，不同格式（如 ext4 和 squashfs）和来源的固件决定了设备的安全性、可恢复性以及功能拓展性 [7, 14]。
- **关联概念**：属于 → [[OpenWrt]]

### [[网关]]
- **定义**：局域网中所有设备通往互联网络的数据出口，通常为路由器的内网 IP 地址 [15, 16]。
- **视频上下文**：作者在讲解家庭网络拓扑时指出，电脑、手机等访问百度的请求，必须先经过作为网关的路由器转发到互联网 [15-17]。
- **为什么重要**：理解网关是配置软路由接管局域网流量、排查网络连通性以及后续配置主路由或旁路由模式的基础前提 [15, 16, 18]。
- **关联概念**：未明确提及

### [[DHCP]]
- **定义**：由路由器或光猫提供的一种服务，用于给连接到该网络的设备自动分配局域网 IP 地址 [19-21]。
- **视频上下文**：作者在测试软路由 LAN 口连通性时，通过观察电脑网卡是否能自动获取到 IP 地址，来验证软路由的 DHCP 服务是否正常运行并确认是否插对了网口 [20, 22]。
- **为什么重要**：自动化了局域网内设备的网络配置过程，如果不同设备的 DHCP 网段冲突，会导致网络丢失或无法上网 [20, 23]。
- **关联概念**：未明确提及

### [[NAT]]
- **定义**：网络地址转换处理，指将内网设备的源 IP 地址替换为路由器 WAN 口 IP 或光猫公网 IP 的数据转发过程 [21]。
- **视频上下文**：作者解释当光猫处于路由模式时，数据包从电脑发出后，会先在软路由进行一次 NAT 处理，接着在光猫再进行一次 NAT 处理，形成双重 NAT [21, 24]。
- **为什么重要**：双重 NAT 会带来一定的网络性能损耗，是判断是否需要更改光猫工作模式的核心技术原因 [24]。
- **关联概念**：依赖 → [[路由模式]]

### [[桥接模式]]
- **定义**：光猫的一种网络连接模式，在此模式下光猫仅负责将光信号转换为电信号，由路由器来负责 PPPoE 拨号获取 IP [21, 24, 25]。
- **视频上下文**：作者建议如果想要减少一重 NAT 的性能损耗，应将光猫改为桥接模式，并在软路由的 WAN 口配置宽带账号密码进行拨号上网 [24]。
- **为什么重要**：能够减少网络转发层级（避免双重 NAT），是优化家庭网络架构的高级操作 [24]。
- **关联概念**：对比 → [[路由模式]]

### [[路由模式]]
- **定义**：光猫的一种网络连接模式，光猫不仅进行信号转换，还自行完成 PPPoE 拨号获取公网 IP，并在 LAN 口提供 DHCP 服务 [21]。
- **视频上下文**：作者发现自己的软路由无需配置拨号就能直接访问百度，从而判断出家中的光猫正处于路由模式 [21, 25]。
- **为什么重要**：在此模式下软路由直接作为下级客户端运行，会导致软路由下的设备经历两层 NAT 处理 [21, 24]。
- **关联概念**：对比 → [[桥接模式]]

### [[无线 AP]]
- **定义**：带有 WiFi 无线发射功能的交换机，负责提供无线网络接入节点 [6]。
- **视频上下文**：由于绝大多数软路由没有无线功能，作者教导用户将家中淘汰下来的硬路由改为“有线中继”模式或手动关闭 DHCP，将其作为无线 AP 连接到软路由下使用 [6, 26, 27]。
- **为什么重要**：是弥补软路由无 WiFi 发射能力这一硬件短板、恢复全家无线设备网络连接的必经步骤 [6]。
- **关联概念**：扩展 → [[硬路由]]
