## 一句话总结
解析翻墙节点速度慢的根本原因，科普跨运营商互联原理、国际线路分类及中转节点的作用。

## 目标受众与前置知识
- **适合谁看**：使用翻墙节点但受困于网络限速或高延迟的普通用户，或想要购买/搭建 [[vps]] 的用户。
- **前置知识**：[[vps]]、[[路由器]]、基础网络通讯与代理原理。

## 核心观点
1. **VPS速度主要取决于线路而非硬件**
   - 为什么重要：纠正了用户单纯追求高配置服务器的误区，指明购买VPS时评估网络链路的必要性。
   - 依据：讲者原话指出“vps速度的快慢和你vps的硬件配置没有关系...主要的话就是看你这条链路走的是什么线路”，例如晚高峰走163骨干网就是垃圾，走cn2就是传家宝。
2. **国际出海网络拥堵及QoS导致直连速度慢**
   - 为什么重要：解释了为何优质配置的VPS在晚高峰依然表现极差。
   - 依据：讲者举例海底光缆带宽有限（如500兆，讲者原话），由于国内用户宽带提速，出海数据量过大排队，且针对未知协议会被降级甚至丢包（讲者原话）。
3. **中转节点能有效解决跨网/跨国绕路问题**
   - 为什么重要：提供了解决直连速度慢、丢包严重的实用方案，解释了机场中转节点的作用原理。
   - 依据：讲者说明移动直连 [[cmi]] 很快，而电信去程绕路 [[NTT]] 会丢包严重，连一个 [[tcp]] 确认报文都发不过去（讲者原话）。在移动网络搭建 [[中转机]] 转发即可避免绕路。
4. **回程路由对翻墙体验至关重要**
   - 为什么重要：提醒用户评估线路时必须双向考量，不能仅看去程。
   - 依据：讲者原话指出，访问国外往往是拉取数据，去程可能只有几十KB，回程却可能拉取几百兆的文件，回程优化远比去程重要。

## 概念关系
- [[bgp]] → [[自制系统]]：不同的 [[自制系统]]（AS）通过 [[bgp]] 协议连接，实现全互联以共享路由IP信息。
- [[直连]] ↔ [[中转机]]：[[直连]] 容易遭遇国际出口拥堵或跨运营商绕路；而通过 [[中转机]] 可以在境内的不同网络间抄近道，优化出境路由。
- [[163骨干网]] ↔ [[cn2]]：两者皆为中国电信网络，前者是普通民用宽带接入，晚高峰极度拥堵；后者是服务政企大客户的精品网络，成本高但通畅。

## 详细笔记
### 宽带接入与国内网络互联原理
- 用户（如 [[不良林]] 的家用电脑）每年花1000多块钱（讲者原话）拉100兆宽带接入 [[中国电信]] 的 [[163骨干网]]，访问接在对应网络内的 [[淘宝]] 或 [[哔哩哔哩]]。
- 跨运营商（如联通访问电信的淘宝）需通过边界网关协议（[[bgp]]，分内部 [[ibgp]] 与外部 [[egp]]）互联各自的 [[自制系统]]。
- [[中国电信]]（AS4134）和 [[中国联通]]（AS4837）体量相当，是对等互联（免流量费）。[[中国移动]] 早期由于体量小，付费接入联通（推断：后来体量变大改为对等）。
- 大型企业为优化访问，会部署跨运营商线路建立机房，或申请独立IP和AS成立数据中心（如 [[阿里云]]）。

### 国际互联与长城防火墙机制
- 国内网络通过横跨太平洋的海底光缆或互联网交换中心（[[ipx]]）与国外网络连接（如访问挂在某个AS下的 [[谷歌]]）。
- 由于国内宽带从1兆、2兆提速至百兆千兆（讲者原话），而海缆带宽（如假设的500兆、300兆，讲者原话）提升有限，导致晚高峰国际出口极度拥堵。
- [[长城防火墙]] 部署在国际出口，一旦检测到访问黑名单，即丢弃数据或返回假地址。
- 代理原理：流量发送至海外接通优质线路（如 [[NTT]]）的 [[vps]]，由其代为访问目标网站，然后通过加密协议绕过防火墙返回本机解密。

### 直连网络速度慢的原因与各运营商线路分类
- **QoS限速与丢包**：拥堵时，设备通过 [[qos]] 设定优先级。正常的 [[http]] 流量优先，而无规则字节流（如代理协议、[[ss]] 等）被降级排队甚至丢弃。
- **电信网络**：
  - 普通网：[[163骨干网]]（4134），晚高峰极度拥堵。
  - 精品网：[[cn2]]（4809），服务政企，昂贵但不拥堵。分 [[cn2gt]]（先走163，出口处接入cn2）和 [[cn2gia]]（就近无缝接入cn2）。
- **联通网络**：
  - 普通网：169骨干网（4837），因目前使用人数相对少，表现尚可。
  - 精品网：[[a网]]（9929），对标cn2。
- **移动网络**：
  - 国内骨干网（9808）无国际访问能力，必须统一通过自家的国际网络 [[cmi]]（58453）接入国外运营商。

### 中转节点的作用与实现成本
- **问题**：电信或联通用直连访问标榜“三网回程 [[cmi]]”的代服器时，去程由于不让走移动网络，可能会绕远路（如走电信 → [[NTT]] → [[cmi]]），导致晚高峰严重丢包。
- **解决方案**：引入 [[中转机]]。在移动网络建立中转节点（如“广移中转”），电信/联通先发送给中转节点，由它原封不动将数据传入 [[cmi]] 进而送达代服器。
- **端口转发与隧道转发**：千兆中转机每月成本高达几千块钱（讲者原话）。用户可通过月付十几块、二十块（讲者原话）购买商家提供的 [[端口转发]]（如配置1080端口原样转发）或 [[隧道转发]]（对数据进行再封装加解密）服务，极大降低使用门槛。

### 路由追踪实战演示与下期预告
- 使用第三方网站追踪去程路由：移动宽带访问经过武汉到广州（国际出口），然后接入香港的 [[cmi]] 线路，37毫秒的超低延迟（讲者原话）证明机房物理位置在香港（讲者调侃若标称为美国则按不住 [[牛顿]] 的棺材板）。
- 电信网络测试同一IP：路由显示经过上海出口后绕路去东京接入 [[NTT]]，再跳回香港，证明直连存在绕路现象。
- AS信息查看：可点开对应编号，查看其包含的 [[ipv4]]、[[ipv6]] 地址总数量，以及上下游连接信息。
- 讲者预告下节将讲解无需花钱的提速手段：[[bbr]] 拥塞控制算法，以及利用 [[cloudflare]] 的免费 [[cdn]] 服务筛选优选IP。

   - 关键代码/命令：未明确提及
   - 注意事项：结合网络延迟判断物理距离，不要被 IP 归属地误导（例如测得延迟仅为37毫秒，考虑到光的传播速度，机房绝不可能在美国，实际应在香港） [10]。

2. **回程路由测试**
   - 具体操作描述：登录到拥有的服务器上运行测试脚本，或者使用商家自行搭建的 [[Looking Glass]] 网站来测试回程路由的情况以及文件的下载速度 [9]。
   - 关键代码/命令：未明确提及
   - 注意事项：由于用户访问国外服务器通常是拉取大量数据（去程仅几十KB，回程可能高达几百兆），因此回程路由的质量比去程更重要 [6]。

3. **配置端口转发搭建中转节点**
   - 具体操作描述：购买中转机的端点转发服务，获取商家提供的高位随机端口（如1080） [8]。在本机发起请求时填写中转机的 IP 地址和该端口，并配置中转机的端口规则，使其收到数据后原封不动转发至目标代理服务器 [8]。
   - 关键代码/命令：未明确提及
   - 注意事项：[[隧道中转]] 的业务逻辑与端点转发相似，但涉及对数据的再封装和再加密。需要确认代理服务器端是否配置好了解密隧道，或确认商家是否已经做好了落地解密 [8]。

## 常见坑与解决方案
- **问题描述**：晚高峰期间，使用无规则字节流协议（如 [[VMess]]、[[SS]]）在拥堵的163骨干网中被 [[QoS]] 降低优先级甚至直接丢包，导致节点速度极慢无法使用 [3]。
  - 解决方案：购买接入了优质网络（如电信 [[CN2]] 线路）的 [[VPS]]，或通过搭建中转线路来改善路由 [5, 11]。
- **问题描述**：购买了优质线路的 [[VPS]]，去程走了 [[CN2]] 精品网，但回程走了拥堵的普通163骨干网，导致实际访问体验依然糟糕 [6]。
  - 解决方案：购买 [[VPS]] 时必须看清楚产品描述，明确确认商家是否保证“回程也走 [[CN2]] 网络” [7]。
- **问题描述**：电信或联通用户直连移动 [[CMI]] 线路的代理服务器时，由于跨网互联问题发生严重绕路（例如先绕道日本 NTT 再连入 CMI），导致丢包严重根本无法建立连接 [11]。
  - 解决方案：引入中转机（例如在移动网络端架设一台广移中转服务器），先将请求发送至中转机，再由中转机直接进入 [[CMI]] 网络发给代理服务器，从而避免绕路 [11, 12]。

## 最佳实践
- 购买 [[VPS]] 线路时，一定要仔细辨别产品描述中关于去程和回程的说明（例如是否为三网回程 [[CMI]] 或是回程 [[CN2]]），防范不良商家玩文字游戏 [7, 10, 13]。
- 针对无法承担昂贵独享中转服务器带宽费用的普通用户，推荐通过购买月付十几块钱的 [[端口转发]] 或 [[隧道中转]] 服务，利用商家提供的端口来实现低成本的高速中转 [8, 12]。
- 在确认购买代理节点之前，建议利用路由追踪工具交叉对比电信、联通、移动不同网络下的去程路由，验证是否真实不绕路 [9, 10]。

## 关键概念

### [[AS (自治系统)]]
- **定义**：一个拥有独立编号的大型网络系统，代表一个运营商或机构的内部网络（如中国电信的163骨干网AS4134）[1]。
- **视频上下文**：作者在解释不同运营商（电信、联通、移动）和大型互联网公司（如淘宝）如何进行跨网络互联互通时，引入了自治系统及其编号的概念[1, 2]。
- **为什么重要**：理解自治系统是理解互联网底层物理架构和跨运营商路由互联的基础[1]。
- **关联概念**：依赖 → [[BGP (边界网关协议)]]

### [[BGP (边界网关协议)]]
- **定义**：不同自治系统（AS）之间用来相互连接、共享IP信息并实现网络全互联的外部路由协议[1]。
- **视频上下文**：作者用它来解释电信、联通、移动以及像阿里云或谷歌这样的数据中心如何通过对等或不对等的方式跨网络互相交换数据[1, 2]。
- **为什么重要**：它决定了不同独立网络之间如何寻找彼此的通信路径，是影响跨运营商及跨国网络访问质量的核心机制[2, 3]。
- **关联概念**：依赖 → [[AS (自治系统)]]

### [[163骨干网]]
- **定义**：中国电信的基础骨干网络（AS编号4134），承载了绝大多数普通家庭宽带业务[1, 4, 5]。
- **视频上下文**：作者以此为例解释普通VPS节点速度拉垮的原因，指出由于普通宽带用户基数庞大且国际出口带宽有限，晚高峰时该网络极易发生严重拥堵[4, 6, 7]。
- **为什么重要**：直接解释了绝大多数普通宽带用户访问国际互联网时速度慢、丢包率高的根本物理原因[6, 7]。
- **关联概念**：对比 ↔ [[CN2网络]]

### [[CN2网络]]
- **定义**：中国电信建设的精品网络（AS编号4809），主要服务于政企大客户，拥有极其通畅的国际出海线路[5, 8]。
- **视频上下文**：作者将CN2（分为CN2 GT和CN2 GIA）作为解决晚高峰拥堵的高端替代方案介绍，指出接入此网络的VPS能实现高速国际互联，但价格昂贵[5, 9]。
- **为什么重要**：这是选购高质量、低延迟VPS节点的核心线路指标之一[5, 10]。
- **关联概念**：对比 ↔ [[163骨干网]]

### [[长城防火墙]]
- **定义**：主要部署在国家网络国际出口位置的审查系统，会检测异常流量或丢弃发往黑名单网址（如谷歌）的数据包[7]。
- **视频上下文**：在讲解国内流量如何访问海外服务时，作者说明了防火墙的拦截机制，从而引出必须使用代理服务器（VPS）进行加密和转发的需求[7]。
- **为什么重要**：它是催生翻墙代理技术、节点搭建需求及各类流量伪装协议的根本原因[7]。
- **关联概念**：未明确提及

### [[QoS]]
- **定义**：一种服务质量控制机制，在网络拥堵时会设置数据优先级，降低未知或无规则协议的优先级甚至进行撤流性丢包[11]。
- **视频上下文**：作者用其解释晚高峰时期，原本就拥堵的链路会对代理协议（如无规则的字节流）进行限速或丢包，导致节点无法使用[11]。
- **为什么重要**：揭示了在网络拥堵环境下，特定代理协议受到的负面影响会远大于普通网页浏览（HTTP流量）的原因[11]。
- **关联概念**：未明确提及

### [[回程路由]]
- **定义**：数据从目标代理服务器返回到用户本地电脑时所经过的网络传输路径[12]。
- **视频上下文**：作者强调数据的去程和回程可能走完全不同的路线，且由于平时访问国外主要为拉取下载大文件，回程走优质线路（如三网回程CMI）往往比去程更为关键[9, 12]。
- **为什么重要**：打破了“网络连接双向同路”的直觉，是评估VPS实际下载体验和线路质量的最关键标准[9, 12]。
- **关联概念**：扩展 → [[绕路]]

### [[绕路]]
- **定义**：数据传输没有选择最优物理距离路径，而是绕道其他国家或运营商网络（如从日本回中国却先跑到美国）的低效路由现象[12-14]。

- **为什么重要**：直接导致高延迟和严重丢包，是跨国网络通信中需要被诊断并规避的核心问题[12, 13]。

### [[中转]]
- **定义**：在数据到达最终的海外代理服务器之前，先利用一台国内或特定运营商（如移动）的服务器进行中间转发的技术[8, 13, 15]。
- **视频上下文**：作者介绍中转是为了解决跨运营商（如电信连移动）直连目标服务器时发生的绕路或丢包问题，通过中转机直连优质出海口（如CMI）以大幅提升速度[13, 15]。
- **为什么重要**：它是目前多节点机场为了兼顾不同运营商用户的高速访问需求而采用的核心解决方案[15]。

- **为什么重要**：为普通用户提供了一种高性价比搭建个人高质量中转网络节点的实操方案[16]。
