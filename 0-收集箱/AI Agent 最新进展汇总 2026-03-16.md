## 一、技术架构与协议

### 1. Google Chrome 146 推出 WebMCP
- **作者**: @xiaohu（小互）
- **要点**: AI agent 操作网页一直有个根本性问题：它得像人一样"看"页面，截图识别按钮，猜输入框该填什么格式。Google 在 Chrome 146 里推了 **WebMCP**，思路完全反过来——让网站主动把自己的能力以结构化工具的形式暴露出来，AI agent 直接调用，拿到结构化结果。
- **意义**: 从"AI 模拟人操作网页"转向"网站主动为 AI 提供接口"，大幅降低 token 消耗和延迟

### 2. Skill Graphs > SKILL.md
- **作者**: @arscontexta（Heinrich）
- **要点**: 结构化知识远比人们想象的更有力量。当前人们写的 skill 只捕获某一方面的能力（如总结、编码），而 **Skill Graph** 能将知识以图结构组织，使 AI agent 具备更复杂的推理和应用能力
- **热度**: 1K 转发, 8K 点赞, 3.4M 浏览

## 五、思考与探讨

### 9. Agent 时代的"数字人"核心要素
- **作者**: @yenlin2015（catfish）
- **要点**: 探讨 Agent 时代"什么才是一个人"的问题。从人的不完美出发，提出五种打造数字人的核心要素。涉及 Discord、Telegram、云端、本地等多平台 AI agent 的实践经验

### 10. 动画知识编码为 Agent Skills
- **作者**: @0xNotthatsundar
- **要点**: 将动画知识编码为 agent skill 是正确的抽象方式。大多数 AI 生成的动画默认使用线性缓动和 300ms，因为训练数据中这些最常见。一个编码了审美品味的 skill.md 可以显著提升 AI 生成动画的质量

## 关键趋势总结

1. **协议层变革**: WebMCP 代表了 AI agent 与 Web 交互范式的根本转变，从"模拟人"到"原生接口"
2. **工具链成熟**: AgentPearl 等工具表明 AI agent 开发的工程化程度在提高
3. **硬件支撑**: NVIDIA GTC 2026 显示硬件厂商正在为 AI agent 提供专属优化
4. **商业落地加速**: 从交易 agent 到创业变现，AI agent 正在产生实际经济价值
5. **本地化趋势**: 开源模型 + 本地部署的 agent 方案越来越可行
