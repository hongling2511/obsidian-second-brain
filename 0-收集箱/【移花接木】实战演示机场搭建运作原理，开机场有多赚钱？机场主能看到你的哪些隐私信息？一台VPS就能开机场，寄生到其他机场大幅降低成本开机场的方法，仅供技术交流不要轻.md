## 一句话总结
讲者不良林实战演示使用 [[xboard]] 面板与 [[XrayR]] 后端搭建机场全过程，并揭秘利用 [[二级代理]] 降低运营成本的方法。

## 目标受众与前置知识
- **适合谁看**：对机场运作原理感兴趣，或试图通过运营机场盈利但缺乏对其技术及成本风险认知的技术爱好者。
- **前置知识**：[[x-ui]]、[[VPS]]、[[Docker]]、[[DNS]]、[[TLS]]

## 核心观点
1. **搭建机场的技术门槛极低**
   - 为什么重要：打破了普通人对商业代理服务（机场）存在高深技术壁垒的固有认知。
   - 依据：“搭建机场的门槛非常低，如果你会使用 xui 搭建节点，那么搭建机场对你来说也就不是什么难事了”（讲者原话）[1]。
2. **开设普通机场利润微薄且面临诸多风险**
   - 为什么重要：直接劝退想要盲目进入该领域赚钱的人，揭示了行业的隐性成本。
   - 依据：讲者列举了 VPS 每月 50 元成本（讲者原话假设）、20% 推广佣金（讲者原话）、10% 第三方支付抽成（讲者原话），加上购买中转和 [[DNS 解锁]]的费用，最终“大概还剩个 30% 的利润”（讲者原话），且面临攻击勒索与法律风险 [2][3]。
3. **机场主能轻易获取用户的上网隐私数据**
   - 为什么重要：提醒普通用户在使用第三方节点时需注意个人隐私安全。
   - 依据：讲者说明日志中记录了“什么时间哪个用户使用哪个 IP 访问了哪个网站”，这些信息机场主都能看到（讲者原话）[4]。
4. **通过 [[二级代理]] 能够以极低成本伪造多国节点和解锁流媒体**
   - 为什么重要：揭露了部分小机场降低成本、夸大节点规模的技术手段，即“寄生”在其他机场上。
   - 依据：讲者演示了在一台 VPS 上配置 [[XrayR]] 路由，将流量按端口转发给其他机场的现成节点，实现一机多地并继承被寄生机场的流媒体解锁能力 [3][5]。

## 概念关系
- [[v2board]] / [[xboard]] / [[SS Panel]] ↔ [[XrayR]]：说明：前者仅为实现注册、订阅和支付的前端 Web 面板，不具备翻墙功能；后者是进行过魔改以对接面板 API 的后端翻墙内核 [1][6]。
- [[API key]] → 用户隐私：说明：后端内核通过该通讯密钥调用开放 API 接口获取用户的密码、套餐限制等核心信息，极度敏感 [7]。
- [[二级代理]] → [[成本降低]]：说明：利用内核的端口转发功能将数据发往其他机场的节点，无需自行购买维护多地 [[VPS]] 即可提供全球节点服务 [3]。

## 详细笔记
### 运作原理概述与前端面版部署
- **原理概念**：机场相当于加了多用户、注册订阅和支付系统的 [[x-ui]]。主要由前端 Web 界面和后端魔改版内核组成 [1]。
- **面板部署工具与步骤**：讲者在系统为 [[Ubuntu]] 22.04（推断）的 [[VPS]] 上，放弃已停更的 [[v2board]]，选择在 [[GitHub]] 上通过 [[Docker]] 部署还在更新的 [[xboard]] 面板。若不习惯 Docker，也支持 [[宝塔]] 面板部署 [8]。
- **组件安装**：克隆仓库后，启用内置的 [[SQLite]]（讲者称 circite）代替 [[MySQL]]，并启用 [[Redis]]，设置管理员邮箱账号即可完成面板部署 [8]。
- **域名解析与加密**：在 [[DNS]] 设置中添加 A 记录和 WWW 记录指向 VPS IP，开启代理状态，将 [[SSL]] 加密模式设为灵活，并使用 Orange Rose 规则将流量重写到 7001 端口，实现域名访问和自动 [[TLS]] 证书套用 [9]。

### 机场后台配置与支付/节点添加
- **后台基础设置**：为防止数据库被刷，需开启邮箱验证并配置 [[SMTP]] 服​​务器。此外需设置极为复杂的通讯密钥（用于对接后端）[9][10]。
- **支付对接方式**：支持 [[支付宝]] 当面付、加密货币，或使用抽取约 10% 手续费的第三方支付平台提供的 [[微信]]、支付宝收款渠道 [10]。
- **套餐设置**：建立不同的权限组，设定套餐价格、月度流量、限速与设备数量，开启销售状态供用户购买 [10]。
- **前端节点添加**：以添加香港 [[SS 节点]]（Shadowsocks）为例，设置倍率（1倍表示消耗1G计费1G）、权限组、VPS IP 及用户订阅连接端口。若非直连，服务端口需针对性设置。此步骤涉及的加密算法、混淆参数等依赖讲者往期教程知识 [6][11]。

### 后端内核对接与隐私测试
- **XrayR 部署与对接**：使用一键脚本安装开源内核 [[XrayR]]。建议使用带 [[sftp]] 功能的 [[Termius]] 下载配置文件到本地编辑器修改。配置内容包括：面板类型选 xbard、API host 填入机场域名、填入后台设置的 [[API key]]、节点 ID 与节点类型（shsocks）[11][7]。
- **验证与隐私风险**：保存重启后，面板节点由红转黄再转蓝，代表对接成功并产生连接。此时后端的日志会记录用户的连接时间、来源 IP 和目标网站，揭示了机场主能窥探用户隐私的事实 [7][4]。

### 多节点演示、中转机制与运营成本
- **单机多节点**：讲者复制节点并修改端口，说明一台 VPS 实际上可以搭建无数个节点，不应通过节点数量判断机场好坏（推断）[4]。
- **中转节点添加**：中转本质是端口转发。演示添加名为 [[IPLC]] 的广港专线（假 IP 入口），分配高级权限组以区别于基础套餐 [2]。
- **盈利难点分析**：运营机场需支付 VPS、中转服务、[[DNS 解锁]] 等硬性成本，以及高昂的推广抽成和支付平台手续费。同时还要应对小白用户答疑、被攻击勒索、跑路和法律查水表等风险，利润空间被极大压缩 [2][3]。

### 降本奇技淫巧：利用二级代理借鸡生蛋
- **实现原理**：仅使用一台 VPS 搭建面板和后端，让 [[XrayR]] 套用其他大型机场的现成节点。数据到达自己 VPS 后直接转发给目标机场，从而省去购买多地 VPS 的成本，并白嫖其流媒体解锁能力 [3]。
- **配置步骤**：修改 XrayR 的 `outbound` 配置，替换为目标机场各地区的节点参数；修改 `routing` 路由配置，设置不同端口接收到的数据交由对应国家（如 HK、TW、JP 等）的节点发出。回到面版建立不同端口的区域节点并匹配 ID，即可实现一机全亮 [3][5]。
- **进阶应用**：为防止被被“寄生”的机场限制连接数，可购买多个套餐对接，实现[[负载均衡]]和[[故障转移]] [5]。

- **环境要求**：操作系统为 [[Ubuntu]] 22.04（视频原话为“无头 22.04”），依赖项为 [[Docker]] [2]。软件版本：未明确提及。
- **前置技能**：[[x-ui]] 节点搭建经验、DNS 解析配置、基本的服务器端口放行及防火墙操作、对代理协议及加密算法（如 TLS、传输协议）工作原理的基础了解 [1, 3, 5]。

## 步骤拆解

1. **面板前端部署**
   - 具体操作描述：在 VPS 上安装 [[Docker]]，克隆 [[Xboard]]（视频中称为“插播”）仓库并执行安装。数据库选择启用 [[SQLite]]，缓存选择启用 [[Docker]] 内置的 [[Redis]]，最后设置管理员邮箱与密码并启动前端面板 [2]。
   - 关键代码/命令：未明确提及（视频中仅指代“使用这条指令”，未提供具体命令内容） [2]。
   - 注意事项：确保服务器防火墙已提前放行 7001 默认端口 [2]。

2. **域名解析与 SSL 证书配置**
   - 具体操作描述：在 DNS 设置中添加 A 记录（如 @ 和 www）指向 VPS 的 IP 并开启代理状态。将 SSL 加密模式设置为“灵活”。利用页面规则（Page Rules）创建重写规则，将域名访问流量重写转发至 7001 端口，从而实现域名访问及自动配置 TLS 证书 [5]。
   - 关键代码/命令：未明确提及。
   - 注意事项：未明确提及。

3. **面板后台基础与节点配置**
   - 具体操作描述：登录面板后台，配置非常关键的“通讯密钥”（用于与后端通信），添加不同等级的权限组、设置订阅套餐（包含价格、流量、设备数限制等）并开启销售状态。随后在“节点管理”中添加具体节点（以 [[Shadowsocks]] 为例），配置节点 IP、连接端口、服务端口、倍率与权限组 [3, 6, 7]。
   - 关键代码/命令：未明确提及。
   - 注意事项：通讯密钥至关重要，绝对不能被泄露，且设置得越复杂越好 [4, 6]。若为直连节点，服务端口必须与连接端口一致 [3]。

4. **后端内核心安装与对接**
   - 具体操作描述：在提供节点服务的 VPS 上一键安装 [[XrayR]] 后端。通过 [[SFTP]] 工具（如 [[Termius]]）将位于 `/etc/XrayR/` 目录的配置文件下载到本地修改。清空特定行后的内容，设置面板类型为 `New V2board`，填入面板 API 域名与通讯密钥，将节点 ID 修改为面板中对应的 ID，并将节点类型改为 `Shadowsocks`。保存后上传覆盖原文件并重启后端 [3, 4]。
   - 关键代码/命令：`XrayR`（查看选项菜单）、`XrayR restart`（重启后端）、`XrayR log`（查看日志信息） [3, 4]。
   - 注意事项：未在后端配置加密方式和密码是因为这些用户信息均通过通讯密钥调用面板 API 自动获取 [4]。

5. **配置二级代理（寄生中转）**
   - 具体操作描述：为了大幅降低成本，购买其他机场套餐获取现成节点。在本地修改 [[XrayR]] 的 `outbound` 配置文件，填入目标机场的节点信息（如香港、台湾节点）；修改 `route` 配置文件，设置不同端口（如 30000、30001）接收到的数据分别由指定的机场节点转发出去。取消相关配置的注释，并在面板与后端按对应端口批量创建多个地区的节点 [8, 9]。
   - 关键代码/命令：未明确提及。
   - 注意事项：可以直接通过购买多个不同套餐对接来突破目标机场的连接数限制，甚至能做负载均衡与故障转移 [9]。

## 常见坑与解决方案
- **问题描述**：安装 [[Xboard]] 时提示数据库创建失败。
  - 解决方案：不用管，继续在提示“使用 [[Docker]] 内置的 [[Redis]]”时同样选择启用即可 [2]。
- **问题描述**：修改后端配置文件后无法启动。
  - 解决方案：注意复制粘贴配置时的空格要严格对齐，否则会导致解析错误无法启动 [10]。
- **问题描述**：在同台服务器上添加多个节点时端口冲突。
  - 解决方案：将其修改为其他未被占用的空闲端口（如 30000 被占用则改为 30001） [10]。

## 最佳实践
- 推荐使用支持 [[SFTP]] 的工具（如 [[Termius]]）将繁杂的配置文件下载到本地使用编辑器修改，修改完成后再上传覆盖，比直接在命令行中修改更加方便 [3]。
- 推荐利用“二级代理”（寄生到其他具备流媒体解锁能力的机场节点上），仅用一台 VPS 就能实现多地区节点部署与流媒体解锁，大幅降低服务器和专线成本 [8]。
- 反模式：切忌通过机场客户端列表中的节点数量多少来判断机场的质量好坏，因为一台 VPS 实际上可以克隆出无数个节点 [10]。
- 反模式：不要轻易尝试为了赚钱而实际运营机场，存在明确的法律风险与各方勒索、跑路压力，仅建议用于技术交流 [1, 9]。

## 关键概念

### [[机场]]
- **定义**：一个包含注册系统、订阅系统、支付系统以及代理节点的多用户管理平台 [1]。
- **视频上下文**：作者指出搭建机场门槛低，其核心是前端Web界面加上负责翻墙的后端内核组成的系统，供其他人注册和订阅使用 [1]。
- **为什么重要**：它是整个视频探讨的核心商业与技术模式，理解其构成是理解后续代理技术细节的基础。
- **关联概念**：依赖 → [[机场面板]]

### [[机场面板]]
- **定义**：仅提供前端Web界面和管理功能（如注册、计费），本身不具备翻墙能力的开源程序（如v2board或Xboard） [1, 2]。
- **视频上下文**：作者演示了使用Docker部署Xboard面板，并在后台配置套餐和节点信息，强调它只负责多用户管理 [2, 3]。
- **为什么重要**：澄清了普通用户对代理服务的误区，明确了前端管理与后端代理在物理和逻辑上的分离。
- **关联概念**：对比 → [[XrayR]]

### [[XrayR]]
- **定义**：基于Xray修改的开源后端内核，专门用于对接机场面板的API接口，实现增删用户和流量统计等功能 [1]。
- **视频上下文**：作者演示了在独立VPS上安装XrayR，并通过修改本地配置文件将其对接到前端面板，指出它是真正执行翻墙工作的核心 [2, 4, 5]。
- **为什么重要**：揭示了多用户代理节点如何实现统一管理、计费以及验证的底层技术支撑。
- **关联概念**：依赖 → [[通讯密钥]]

### [[通讯密钥]]
- **定义**：在机场面板与后端XrayR节点之间建立通信连接时使用的核心认证密码 [6]。
- **视频上下文**：作者在面板后台设置该密钥，并在配置XrayR（作为API key）时填入，强调其极其重要，后端依靠它获取用户密码、套餐、限速等隐私信息 [5, 6]。
- **为什么重要**：它是连接管理前端与代理后端的唯一凭证，直接关系到整个系统的数据交互与用户信息安全。
- **关联概念**：未明确提及

### [[节点]]
- **定义**：实际部署了后端内核并运行在特定端口上的代理服务器，面向多用户、多地区的使用场景 [2, 4]。
- **视频上下文**：作者演示了在后台添加香港和台湾节点，配置IP和端口，并强调一台VPS实际上可以搭建无数个节点，打破了靠节点数量判断机场好坏的迷思 [2, 7]。
- **为什么重要**：它是代理服务的物理或逻辑实体，决定了用户的落地IP和网络连接质量。
- **关联概念**：扩展 → [[中转]]

### [[中转]]
- **定义**：本质上是干端口转发的活，通过一个专线入口将数据流量转发到实际的落地节点的技术 [8]。
- **视频上下文**：作者在面板中演示了如何设置中转节点，将父节点设置为香港，并提到现代机场基本都会花钱购买中转服务以提升网络质量 [8, 9]。
