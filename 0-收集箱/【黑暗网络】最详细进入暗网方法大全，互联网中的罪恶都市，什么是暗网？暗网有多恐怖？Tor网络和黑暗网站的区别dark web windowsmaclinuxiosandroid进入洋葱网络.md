## 一句话总结
科普[[暗网]]与[[Tor网络]]原理，并详细演示在各操作系统下配置[[洋葱浏览器]]安全连接暗网的实操步骤。

## 目标受众与前置知识
- **适合谁看**：对网络层级结构感兴趣、具有网络安全意识且希望探索或连接[[暗网]]的用户（推断）。
- **前置知识**：[[代理]]工具配置（如[[clash]]、[[v2ray]]）、[[IP]]地址基础、[[科学上网]]原理。

## 核心观点
1. **暗网只是深网的一小部分，且充斥极高比例的非法内容**：
   - 为什么重要：纠正大众对网络层级的误解，揭示真实的暗网生态。
   - 依据：深网在整个互联网占比达90%以上（讲者原话）；使用Tor访问暗网的占比约5%，但其中访问虐待相关内容的占比达80%以上（讲者原话）。
2. **技术本身无罪，暗网初衷是保护隐私与对抗审查**：
   - 为什么重要：客观评价了匿名网络的合法价值。
   - 依据：很多正规网站如[[facebook]]、[[twitter]]、[[bbc]]甚至[[cia]]（美国中情局）也有官方暗网址（讲者原话）。
3. **Tor网络通过多层随机节点加密确保高度匿名性**：
   - 为什么重要：解释了为何暗网难以被追踪以及访问速度极慢的根本原因。
   - 依据：网络会随机选择3个[[中继节点]]进行三层加密，且每10分钟自动更换一次链路（讲者原话）。
4. **受审查地区访问Tor需针对性调整代理或使用网桥伪装**：
   - 为什么重要：解决了大部分用户无法直接连接[[Tor网络]]的现实痛点。
   - 依据：通过配置本地代理端口（如`127.0.0.1`和`7890`）、关闭[[流量嗅探]]（sniffing）或开启[[obfs]]等网桥混淆流量，成功绕过防火墙（屏幕展示与演示）。

## 概念关系
- [[深网]] ↔ [[表网]]：深网是搜索引擎无法找到或需特定权限的内容（如邮箱、[[telegram]]、[[q空间]]），表网是普通网民可通过搜索引擎直接访问的内容。
- [[黑暗网络]] → [[黑暗网站]]：黑暗网站（隐藏服务）必须运行在黑暗网络（如[[Tor网络]]、[[i2p]]）之上，必须先连接网络才能访问网站。
- [[守卫节点]] ↔ [[出口节点]]：守卫节点唯一知道用户的真实[[IP]]但无法解密数据，出口节点解密知道用户的访问意图但不知道真实[[IP]]。

## 详细笔记
### 互联网结构与暗网科普
- **网络分层**：网络分为[[表网]]、身网/生网（即[[深网]]）和[[暗网]]。此外还有[[银网]]和[[马里亚纳网]]，但据传是缺乏真实证据的都市传说（推断为都市传说）。
- **表网与深网**：[[youtube]]视频、[[google]]搜索、[[白度]]等属表网；个人邮箱、[[q空间]]、[[朋友圈]]、需权限的论坛、以及发生N号房事件的[[telegram]]属于深网。深网占整个互联网站比达90%以上（讲者原话）。
- **暗网现状**：暗网（如不良林演示的 不良0.com 博客版）提供匿名性，充斥犯罪活动。官方统计暗网站存活量大概75万个（讲者原话）。讲者警告暗网上骗子比变态多得多，付款99.9%概率只会买到“[[葫芦兄弟]]”视频（讲者原话）。提醒勿追求猎奇（如暗网红房子等都市传说）。

### 黑暗网络原理与Tor网络机制
- **网络与网站的区别**：黑暗网络包含[[Tor网络]]（拓/拖/透网络，英文 rot/The Onion Router）和[[i2p]]（大蒜网络）。在网络中搭建的隐藏服务即黑暗网站，例如以“安”（推断为`.onion`）或`.i2p`结尾的域名。
- **匿名实现机制**：以[[洋葱浏览器]]访问[[google]]为例，系统从约7500个志愿者提供的[[中继节点]]（讲者原话，可由闲置[[vps]]加入）中随机选3个。
- **节点工作流**：
  1. 数据先被节点三、二、一的密钥层层加密（像剥洋葱）。
  2. 交给[[守卫节点]]（节点一）：唯一知道真实[[IP]]，解密第一层，发给节点二。
  3. 普通中继（节点二）：解密第二层，不知真实IP也不知内容。
  4. [[出口节点]]（节点三）：解密最后一层，得知访问意图并代为访问目标网站。易受滥用警告牵连。
- **安全性与追踪难度**：追踪必须同时控制三个节点并在10分钟（更换节点周期，讲者原话）内完成，这在去中心化网络中几乎不可能。被抓多因自身操作泄露信息。

### 电脑端（Windows/Mac/Linux）配置与连接教程
- **软件下载**：必须在官网下载[[洋葱浏览器]]。提及某中文YouTuber曾在视频下方提供带毒浏览器，被[[卡巴斯基]]揪出后频道被[[youtube]]删除。
- **情况一：直接连接**：若所在地区未屏蔽，或软路由/t模式（推断为tun模式）接管流量成功，可直接连接。此时浏览普通网页与[[chrome]]/[[firefox]]无异；若使用暗网版dg（推断为[[duckduckgo]]）访问暗网址，因增加交汇节点需经过6个中继节点（推断），速度雪上加霜。
- **情况二：无代理的受屏蔽地区**：需使用[[网桥]]（未公开且混淆流量的中继节点，类似[[ss]]/ssw）。可选[[obfs]]（伪装成http）、[[snowflake]]（伪装成视频通话流量，易被墙）或 meek/m（速度非常慢）。
- **情况三：有代理的受屏蔽地区**（推荐）：
  - [[clash]]（class）配置：高级设置勾选代理连接，地址填`127.0.0.1`，端口填`7890`（讲者原话），类型选[[socks5]]（sox5）。
  - [[v2ray]]（微2）配置：端⼝填`10808`（讲者原话），**必须关闭[[流量嗅探]]**（sniffing），否则无法连接。
  - 节点问题：自建节点需关服务端 sniffing，可能引发[[dns污染]]；机场节点若无法关嗅探，必须开启内置的[[obfs]]网桥。

### 移动端（安卓与iOS）配置教程
- **安卓端**：官网下载第一项。需先运行微篇/vn（推断为[[vpn]]）代理工具。若因流量嗅探连不上，进入设置开启[[obfs]]网桥功能，退出后台重进。
- **iOS（苹果端）**：官方无[[ios]]版。需在外区[[apple store]]下载斯诺登（斯诺灯）推荐的第三方开源版 Onion Browser。防范捆绑收费的假冒软件。
- **iOS版本适配与冲突**：
  - [[ios15]]以下：需下载旧版。
  - ios15至[[ios17]]之间：软件强制安装免VPN且断开WiFi连Tor，导致软路由用户无法使用。
  - [[ios17]]及以上：恢复之前的连接方式。
- **iOS代理工具配合**：若使用[[shadow rocket]]（小火箭），因其默认进行嗅探覆写且无法手动关闭，**必须开启网桥**才能连接（讲者向作者确认下版本解决）。
- **安全设置**：全平台均可选择安全等级，最高等级禁用[[js脚本]]，可防止类似此前 clash 的[[csrf]]未授权配置重载导致的[[rce]]安全漏洞。

   - 关键代码/命令：未明确提及。

## 常见坑与解决方案
- **问题描述**：下载到带有病毒的伪造 [[洋葱浏览器]]，导致安全风险 [3]。
  - 解决方案：必须严格在官方网站下载安装包 [3]。
- **问题描述**：使用本地代理或软路由后，[[洋葱浏览器]] 依然提示无法连接到网络 [5]。
  - 解决方案：需确保代理客户端和服务端的“流量嗅探”（sniffing）功能均已关闭 [5, 7]。
- **问题描述**：自己搭建的节点关闭 sniffing 嗅探复写功能后，出现 DNS 污染 [7]。
  - 解决方案：未明确提及（作者提示需观看其往期相关视频解决） [7]。
- **问题描述**：使用机场节点且机场端强制开启了嗅探，用户无法手动关闭 [7]。
  - 解决方案：在网络设置中选择内置网桥并开启 [[obfs]] 将流量混淆，或者更换未开启嗅探的专线机场 [7]。
- **问题描述**：iOS 端使用 [[Shadowrocket]]（小火箭）作为代理时无法连接 [6]。
  - 解决方案：因最新版 [[Shadowrocket]] 默认进行嗅探复写且无法手动关闭，必须在 [[洋葱浏览器]] 内开启网桥才能连接 [6]。
- **问题描述**：iOS 15 到 iOS 17 系统版本使用时，内置连接工具会自动断开 WiFi 网络 [8]。
  - 解决方案：建议将系统升级至 iOS 17 以恢复旧版的连接方式，或直接放弃在该版本使用 [8]。

## 最佳实践
- 强烈推荐直接从官网下载软件，防范第三方带毒版本 [3]。
- 除非万不得已（如无法关闭嗅探的机场），尽量不使用网桥（如 [[obfs]]、[[meek]]），因为网桥会使原本就慢的访问速度更加缓慢 [4, 7]。
- 推荐根据需求提高浏览器的安全等级（如最高等级完全禁用 JS 脚本），这可以防止类似 [[Clash]] 的 CSRF 未授权配置重载导致的 RCE 漏洞问题，尽管会影响部分网页功能 [6]。

## 关键概念

### [[表网]]
- **定义**：普通网民不需要任何访问权限，通过搜索引擎就能直接访问的公共互联网内容 [1]。
- **视频上下文**：作者以平时刷youtube视频、使用谷歌搜索、浏览网页等日常行为来举例说明表网的内容 [1, 2]。
- **为什么重要**：它是理解网络分层结构的基础，作为对比深网和暗网的参照物 [1]。
- **关联概念**：对比 → [[深网]]

### [[深网]]
- **定义**：搜索引擎无法找到或者需要特定权限才能访问的互联网内容 [2]。
- **视频上下文**：作者指出个人邮箱、设置了访问权限的QQ空间和朋友圈、需要登录的论坛帖子等都算作深网，并指出其在整个互联网的占比达到90%以上 [2]。
- **为什么重要**：澄清了大众对暗网的误解，说明暗网只是深网的一部分，且许多网络活动（包括非法活动）都在深网中进行 [2]。
- **关联概念**：扩展 → [[暗网]]

### [[暗网]]
- **定义**：必须通过某些特殊手段连接特殊网络才能访问的网站或网络，提供极高的匿名性和隐私保护 [1]。
- **视频上下文**：作者指出暗网包含“黑暗网络”和“黑暗网站”两层意思，常被不法分子利用进行违法犯罪活动，但其初衷是为了保护用户隐私、对抗网络监控 [1, 3]。
- **为什么重要**：视频的核心探讨对象，理解其双刃剑特性（隐匿身份与滋生犯罪）是网络安全认知的重要部分 [1, 3]。
- **关联概念**：包含 → [[黑暗网络]]

### [[黑暗网络]]
- **定义**：运行在互联网上的覆盖网络，不基于用户的真实IP地址直接路由，能隐藏用户访问互联网的IP [3]。
- **视频上下文**：作者将Tor洋葱网络和i2p大蒜网络作为黑暗网络的实例，说明一旦连接该网络，访问表网或隐藏服务时双方的真实身份都会被隐藏 [3, 4]。
- **为什么重要**：明确了暗网在技术架构上的底层网络概念，是搭建暗网站点和实现匿名通信的基础 [3, 4]。
- **关联概念**：依赖 → [[Tor网络]]

### [[隐藏服务]]
- **定义**：只能在特定黑暗网络中路由和访问的网站，即该黑暗网络里的黑暗网站 [4]。
- **视频上下文**：作者举例i2p网络中以 .i2p 结尾的域名，以及Tor网络中以 .onion 结尾的域名（如暗网版博客），说明必须先连接对应的黑暗网络才能访问它们 [4]。
- **为什么重要**：解释了为什么常规浏览器无法打开某些特定后缀的暗网网址，揭示了暗网内容特定的存在形式 [4]。
- **关联概念**：依赖 → [[黑暗网络]]

### [[Tor网络]]
- **定义**：目前主流的黑暗网络，也叫洋葱路由或洋葱网络，通过多层加密和随机节点保证用户的匿名性 [2, 4]。
- **视频上下文**：作者详细讲解了其工作原理，包括数据如何像剥洋葱一样经过三个中继节点进行加密和解密传输，并强调其匿名性难以被监管和追踪 [4-6]。
- **为什么重要**：是目前最被大众熟知的暗网实现方式，也是视频后续演示进入暗网操作所依赖的核心网络环境 [2, 4]。
