## 一句话总结
通过免费配置无限量 [[IPv6]] 地址，实现单台 [[VPS]] 多落地 IP 与防封锁节点。

## 目标受众与前置知识
- **适合谁看**：跨境电商从业者、网络爬虫开发者、批量注册/养号运营者。
- **前置知识**：[[VPS]] 基础操作、[[代理节点]]配置原理（推断）、[[IPv4]] 与 [[IPv6]] 基础网络概念。

## 核心观点
1. **利用 [[IPv6]] 可免费实现单台 [[VPS]] 无线落地 IP**
   - 为什么重要：解决跨境电商多账号防风控需要独立 IP 且购买多个 [[IPv4]] 成本高昂的问题。
   - 依据：购买一个 [[IPv4]] 的 C 段（提供200多个IP）每月需几千块（讲者原话），而利用商家分配的 [[IPv6]] 前缀（如 /64）批量生成地址完全免费，且因数量庞大出现重复概率几乎为零（讲者原话）。
2. **通过不同端口映射不同 [[IPv6]] 落地地址可绕过目标网站封控**
   - 为什么重要：能够突破网站对单一 IP 的访问频率限制或反爬虫机制。
   - 依据：视频演示中，同一 IP 在 10 秒内频繁访问测试网页触发防火墙封锁（讲者原话/屏幕展示），切换至另一端口对应的 [[IPv6]] 节点后限制解除并显示新 IP（屏幕展示）。
3. **利用 [[IPv6隧道]] 可为无公网 [[IPv6]] 的 [[VPS]] 增加地址并解锁流媒体**
   - 为什么重要：扩展了纯 [[IPv4]] 服务器的功能，不仅免费提供公网 [[IPv6]]，还带来额外的流媒体解锁价值。
   - 依据：通过第三方网站创建隧道，不仅成功获取 /64 的公网 [[IPv6]]，且测试新加坡、日本、纽约节点均可解锁 [[Netflix]]（推断：视频提及“耐飞”）非自制剧，其中日本节点可解锁印度地区（讲者原话）。
4. **通过多 [[IPv6]] 入口实现单落地出口可做到“节点永不被墙”**
   - 为什么重要：规避网络审查对单一服务器节点的封锁。
   - 依据：当节点被墙（表现为高丢包率、无法使用）时，只要本地电脑支持 [[IPv6]] 且防火墙不直接封锁整段 /64 前缀，即可在客户端免费更换任意生成的同前缀 [[IPv6]] 地址作为新入口，恢复连接（讲者原话）。

## 概念关系
- [[不同端口]] → [[不同落地 IP]]：在节点配置中切换不同端口，对应 VPS 路由出不同的 IPv6 地址，用于防网站封控。
- [[多 IP 入口]] → [[节点防墙]]：在客户端更换不同 IPv6 地址作为连接入口，连接至同一落地 IP，用于规避防火墙对入口 IP 的封锁。
- [[IPv4]] ↔ [[IPv6]]：前者 C 段独立 IP 资源昂贵（每月几千块），后者可免费获取无限量的独立 IP 用于替代业务落地。
- [[IPv6隧道]] → [[网络线路降级]]：若原 VPS 采用如 [[c2gia]]（推断：指 CN2 GIA）的优质线路，使用隧道作为 IPv6 入口会导致其变成普通线路，得不偿失。

## 详细笔记
### 场景引入与痛点分析
讲者 [[不良林]] 提出，跨境电商、网络爬虫、批量注册养号等业务需要将账号与 IP 绑定以防风控。若购买包含 200 多个 [[IPv4]] 的站群服务器（C段），每月需花费几千块（讲者原话）。本期教程旨在利用无数量限制的 [[IPv6]] 免费为 [[VPS]] 添加无限落地 IP 节点。

### 具备默认 IPv6 的 VPS 配置单入多出（多落地）
1. **初始安装与验证**：在 [[VPS]] 执行指令安装 [[Xray]]（推断：视频提及“x 瑞”），起始端口默认设为 20000（讲者原话），生成包含账号密码的 [[socks]] 代理。使用 [[v2rayN]]（推断：视频提及“VN/v2n”）配置节点。
2. **测试端口与落地 IP 的关系**：通过测试网页验证，频繁刷新触发 10 秒防火墙拦截（讲者原话）。将节点端口改为 20001 并切换后，显示新的 [[IPv6]] 地址且解除限制，证实不同端口对应不同落地 IP。
3. **批量生成无限 IP**：确认 [[VPS]] 拥有前缀小于 128（如 64、48、56 等）的公网 [[IPv6]]。使用提供的工具网页，输入 IP、前缀和网卡名称，设置数量（如 100，讲者原话），生成指令并在 [[VPS]] 执行，即可增加 100 个同前缀的 [[IPv6]] 地址。
4. **批量生成节点端口**：在工具中贴入节点链接，利用批量修改端口功能生成 102 个（起始 20000，结束 20101，讲者原话）不同端口的节点导入客户端。
5. **协议与访问限制**：
   - 网站兼容性：[[IPv6]] 落地仅支持兼容该协议的网站（如 [[Google]]（推断：视频提及“谷歌”）、[[YouTube]]（推断：视频提及“yube”）、[[Facebook]]、[[Instagram]]（推断：视频提及“instagramy”）、[[Amazon]]（推断：视频提及“亚马逊”））；不支持纯 [[IPv4]] 网站（如 [[Twitter]]（推特）、[[TikTok]]）。
   - 协议选择：[[socks]] 代理不适合直连，容易受干扰被 [[Cloudflare]]（推断：视频提及“CF”）防火墙封锁单 IP。讲者建议直连使用 [[VMess]]（推断：视频提及“vmens”/“VM”）结合 [[ws]]（推断：指 WebSocket）协议，工具内同样支持该节点类型生成。
   - 临时性：此脚本配置的 IP 均为临时，重启（`reboot`）后自动清除。

### 无公网 IPv6 的 VPS 隧道配置（持久化）
1. **创建免费隧道**：对于仅有 `f180/fe80` 开头链路本地地址的机器，在第三方网站注册并创建 [[IPv6隧道]]。输入 [[VPS]] 的 [[IPv4]]，选择临近区域（如新加坡）服务器，不仅能获取 /64 的免费公网 [[IPv6]]，还能解锁对应地区的流媒体（如 [[Netflix]]）。
2. **持久化网络配置**：以 [[Ubuntu]] 22.04（推断：视频提及“5半图 22.04/物半图 17”）系统为例，将隧道配置代码通过编辑 `yaml` 文件，写入并使用 [[netplan]]（推断：视频提及“netpl”）的 `apply` 指令应用，使分配的 [[IPv6]] 地址永久生效，重启不丢失。

### 多 IP 入口实现节点防墙（多入单出）
1. **本地环境要求**：需确认本地电脑具有 2 开头的公网 [[IPv6]] 地址（需要在网络属性中勾选 [[IPv6]] 并依赖运营商分配）。
2. **防墙操作原理**：当节点受严重干扰丢包时，讲者演示将节点配置中的服务器地址（入站 IP）替换为脚本生成的其他同前缀 [[IPv6]] 地址。由于访问入口改变，只要防火墙未按前缀封锁，即可继续正常使用，实现“永不被墙”。
3. **复合应用**：此时入口为 [[IPv6]]，若节点内设置访问特定端口（如 20000）走 [[IPv4]] 落地，则依然能正常访问仅支持 [[IPv4]] 的 [[TikTok]] 等网站。但也需注意 [[IPv6]] 路由质量目前普遍不及 [[IPv4]]，强行将优质的 [[c2gia]] 线路改用 [[IPv6]] 入口会导致路由劣化。

## 常见坑与解决方案
- **问题描述**：手动执行指令配置的批量 `[[IPv6]]` 地址在 `[[VPS]]` 重启后会丢失 [9]。
  - 解决方案：如果不需要保留则直接重启清除即可；若需关机不清除、永久生效，需自行在网上搜索相关配置方法（视频未明确提及具体操作） [9]。
- **问题描述**：目标网站可能直接封锁整个 /64 的 `[[IPv6]]` 前缀，导致批量生成的 IP 一同失效 [10]。
  - 解决方案：自行测试业务网站是否按前缀封锁（例如 `[[Cloudflare]]` 的防火墙通常仅封锁单个 IP，不会误伤前缀） [3]。
- **问题描述**：`[[Socks]]` 代理协议直接连接容易出现问题或受干扰 [3]。
  - 解决方案：套用前置代理使用，或者更换为 `[[VMess]]` 加 `[[WS]]` 节点类型 [3]。
- **问题描述**：以 `[[IPv6]]` 地址作为落地时，无法访问推特、TikTok 等仅支持 `[[IPv4]]` 的网站 [9]。
  - 解决方案：切换回使用 `[[IPv4]]` 作落地的节点（如端口 20000），或者采用 `[[IPv6]]` 入站配合 `[[IPv4]]` 落地的配置模式 [9, 12]。
- **问题描述**：拥有 CN2 GIA 等优质路由的 `[[VPS]]`，在使用 `[[IPv6]]` 隧道作为入口后，网络丢包严重或延迟变高 [11, 12]。
  - 解决方案：优质线路不建议使用 `[[IPv6]]` 隧道作为入口，会导致流量走普通线路，视频明确指出“不值得” [12]。
- **问题描述**：本地电脑勾选了 `[[IPv6]]` 选项，但依然没有获取到 `[[IPv6]]` 地址 [11]。
  - 解决方案：未明确提及（需自行在网上搜索运营商是否分配及路由器的相关配置问题） [11]。

## 最佳实践
- 推荐使用商家分配前缀小于 128 的 `[[VPS]]` 进行批量 IP 生成（前缀长度越小越好，如 48、56、64、96、112，其中 64 较为常见） [8]。
- 为不带 `[[IPv6]]` 的 `[[VPS]]` 创建隧道时，推荐选择与 `[[VPS]]` 所在地一致的隧道区域（如新加坡 `[[VPS]]` 选新加坡区域）；如果有解锁奈飞非自制剧的需求，推荐使用新加坡、日本或纽约节点，其中日本节点经测试可解锁印度地区 [4]。
- 反模式：切勿将 `[[IPv6]]` 隧道入口用于本身已具备优质线路（如 CN2 GIA）的 `[[IPv4]]` `[[VPS]]`，这是一种导致线路降级的反面做法 [12]。

## 关键概念

### [[VPS]]
- **定义**：虚拟专用服务器（Virtual Private Server），用于搭建网络节点和执行批量操作的基础设施 [1]。
- **视频上下文**：作者提到许多朋友希望自己的 VPS 能有多个 IP 地址，以避免在跨境电商或网络爬虫操作中因单一 IP 被封控而需要购买多台 VPS 的高昂成本 [1]。
- **为什么重要**：它是配置无限量 IPv6 地址、搭建代理环境的核心载体 [1]。
- **关联概念**：未明确提及

### [[IPv6]]
- **定义**：一种互联网协议版本，用于为服务器提供海量且免费的 IP 地址资源 [1, 2]。
- **视频上下文**：作者演示了如何利用 IPv6 为默认有或没有 IPv6 地址的 VPS 批量生成随机 IP，以此应对网站的 IP 频率限制或封锁 [2-4]。
- **为什么重要**：由于其庞大的地址池，生成的 IPv6 地址出现重复概率极低，可被视为“无限 IP”，是实现免费多 IP 方案的技术核心 [2]。
- **关联概念**：扩展 → [[IPv6 隧道]]

### [[落地 IP]]
- **定义**：配置在代理节点端，最终用于访问目标网站（如电商平台、流媒体）的实际出口 IP 地址 [3, 5]。
