# The spelled-out intro to neural networks and backpropagation: building micrograd

## 一句话总结
从零构建微型自动求导引擎，深入浅出讲解 [[反向传播]] 与 [[神经网络]] 底层原理 [1, 2]。

## 目标受众与前置知识
- **适合谁看**：计算机科学学生、AI 开发者、想透彻理解 [[神经网络]] 底层数学原理的程序员 [1]。
- **前置知识**：基本的 [[Python]] 编程（如面向对象、特殊方法定义）[3, 4]，基础微积分概念（如 [[导数]]、[[链式法则]]）[5, 6]。

## 核心观点
1. **观点概括**：[[神经网络]] 本质上只是一类特定的数学表达式 [7, 8]。
   - 为什么重要：这打破了深度学习的“黑盒”神秘感，将其还原为标量加乘运算的组合，极大降低了理解门槛 [8]。
   - 说服力评估：作者通过从最基础的加法和乘法开始构建 `Value` 对象，一步步将其组装成 [[多层感知机]]，逻辑严密，极具说服力 [3, 9]。

2. **观点概括**：[[反向传播]] 仅仅是 [[链式法则]] 在计算图上的递归应用 [10, 11]。
   - 为什么重要：揭示了训练神经网络的核心算法（反向传播）的数学本质，消除了对其实现复杂度的恐惧 [11]。
   - 说服力评估：作者亲手演算了加法、乘法节点以及复合函数 [[tanh]] 的局部梯度，并将其与全局梯度相乘，每一步都有严密的数学与代码对照，说服力极强 [12-14]。

3. **观点概括**：理解了 `micrograd` 就理解了神经网络的核心，现代框架（如 [[PyTorch]]）的复杂性主要为了计算效率（处理多维 [[张量]]） [15, 16]。
   - 为什么重要：指出了基础原理与工业级实现之间的关系，鼓励学习者不要被庞大的工程代码吓倒 [15]。
   - 说服力评估：作者亲自编写了一段调用 [[PyTorch]] 的代码进行对照，得出了与手写 `micrograd` 完全相同的梯度数值，证据确凿，极具说服力 [17, 18]。

## 副观点与隐含立场
- **副观点**：节点的实现粒度是随意的，你可以实现整个复合函数（如 [[tanh]]），也可以将其拆分为基础运算（如求指数、除法） [17, 19]。
  - 如何为核心观点服务：强化了“只要能写出局部导数就能参与 [[反向传播]]”的核心理念，证明了计算图机制的灵活性 [17]。
  - 隐含前提：作者假设观众能够理解不管函数多复杂，只要其内部输入输出的导数是确定的，对外部梯度的流向就没有影响 [17]。

## 详细笔记
- **理解导数与微积分直觉（0-10%）**：介绍 `micrograd` 的功能，回顾 [[导数]] (`derivative`) 的微积分定义，通过输入微小偏移量 $h$ 观察函数输出的响应来估算斜率，为后续梯度计算打下直觉基础 [1, 5, 20]。
- **构建表达式图与前向传播（10-30%）**：从零构建 `Value` 对象，重载 [[Python]] 的数学运算符（加法、乘法），记录节点的子节点和操作符，从而在后台建立完整的数学表达式计算图 (`expression graph`)，并编写代码可视化前向传播结果 [3, 21-23]。
- **手动执行反向传播（30-50%）**：利用 [[链式法则]] (`chain rule`) 手动计算每个节点的梯度。分析得出加法节点的作用是将梯度均匀分发（路由），而乘法节点则是将梯度与输入值交叉相乘 [6, 13, 24, 25]。
- **自动化反向传播与拓扑排序（50-70%）**：在节点中封装 `_backward()` 函数实现局部链式法则。引入 [[拓扑排序]] (`topological sort`) 确保节点按依赖关系从后往前计算梯度。解决由于变量被多次使用导致的梯度覆盖 bug，改为使用梯度累加 (`+=`) [26-29]。
- **引入更多运算与对比 PyTorch（70-85%）**：增加了指数函数、减法、除法和常数运算支持 [30-32]。证明了将复合操作 [[tanh]] 拆解为基础算子后的反向传播结果是一致的 [19, 33]。随后，引入 [[PyTorch]] 的 [[张量]] (`tensor`) API，证明工业级库在单标量情况下的行为与 `micrograd` 完全一致 [18, 34]。
- **构建神经网络并执行训练（85-100%）**：构建 `Neuron`、`Layer` 和 `MLP` (多层感知机) 类 [9, 35, 36]。定义一个微型数据集和均方误差 [[损失函数]] (`loss function`)，利用求得的梯度执行参数更新（[[梯度下降]]）[37-39]。展示并修复了初学者常犯的 `zero_grad`（未清零梯度）bug，最终总结了整个训练循环 [40-42]。

## 步骤拆解
1. **理解导数与数值梯度**
   - 具体操作描述：定义一个简单的标量函数，通过引入一个微小的步长 `h`，利用公式 $(f(x+h) - f(x))/h$ 来计算数值导数，从而建立对函数斜率和灵敏度的直观理解 [14]。
   - 注意事项：`h` 不能设置得过小（比如带太多 0），否则受限于计算机浮点数的有限精度表示，会导致计算出错误的答案 [15]。

2. **构建基础 [[Value]] 对象与前向传播**
   - 具体操作描述：定义一个 `Value` 类，用于包装标量数据。通过实现 Python 的特殊方法使其支持加法和乘法。在运算时，不仅记录计算结果，还保存其子节点（`_prev`）和对应的操作符（`_op`），从而构建出完整的数学表达式图 [11, 16-18]。
   - 关键代码：`def __add__(self, other): ...` [11]

3. **手动实现 [[backpropagation]] (反向传播)**
   - 具体操作描述：从最终的输出节点开始，逆向遍历图节点。利用 [[chain rule]] (链式法则)，将全局导数与当前操作的局部导数相乘。发现加法器（`+`）的作用是直接“路由”或分发梯度，乘法器（`*`）的作用则是将梯度乘以另一个输入值 [10, 19-21]。

4. **封装单个人工 [[neuron]] (神经元)**
   - 具体操作描述：实现 $w \cdot x + b$ 的计算逻辑。将多个输入的权重与对应数据相乘后求和，加上偏置，最后通过 [[tanh]] 激活函数引入非线性 [12, 13, 22, 23]。

5. **自动化 [[backpropagation]] (反向传播)**
   - 具体操作描述：为每种操作定义一个 `_backward` 闭包函数来处理局部链式法则。使用 [[topological sort]] (拓扑排序) 算法构建包含所有节点的线性排序，确保在调用任意节点的反向传播前，它依赖的所有后续节点都已经计算完毕 [24-27]。
   - 关键代码：`def backward(self): ...` 包含拓扑排序的递归实现及遍历调用 `_backward()` [27]。

6. **扩展数学操作与完善操作符重载**
   - 具体操作描述：进一步实现指数函数（`exp`）、幂函数（`pow`）、除法（转化为负一次幂）和减法（转化为加上负数）[28-30]。

7. **构建 [[神经网络]] API**
   - 具体操作描述：参考 [[PyTorch]] 的接口设计风格，依次封装 `Neuron`（单神经元）、`Layer`（神经元层）、`MLP`（多层感知机）类，使网络可以顺序前向传播 [31-33]。

8. **训练循环与 [[gradient descent]] (梯度下降)**
   - 具体操作描述：定义一个微型数据集和均方误差（MSE）[[loss function]] (损失函数)。然后进入循环：前向传播计算 Loss -> 清零梯度 -> 调用 Loss 的反向传播 -> 沿着梯度的反方向（乘以学习率步长）更新所有的权重和偏置参数 [34-36]。
   - 关键代码：`p.data += -learning_rate * p.grad` [37]。

## 常见坑与解决方案
- **问题描述**：变量在计算图中被多次使用时（例如 `b = a + a` 或 `d = a * b; e = a + b`），自动计算的梯度不正确，后续的反向传播会覆盖掉之前的梯度值 [38, 39]。
  - **解决方案**：根据多元微积分的链式法则，梯度必须被**累加**。在 `_backward` 函数中，需要将 `=` 赋值改为 `+=` 累加操作（`self.grad += ...`）[40]。

- **问题描述**：使用类似 `a + 1` 或 `2 * a` 时程序报错，因为普通数字（int/float）并没有实现 `data` 属性，也不是 `Value` 实例对象 [41, 42]。
  - **解决方案**：在方法中做类型检查，如果 `other` 不是 `Value` 对象，则先将其包装为 `Value(other)` [43]。同时实现 `__rmul__` 和 `__radd__` 作为后备机制（Fallback），当 Python 发现常数无法乘以/加上对象时，会反向调用对象的这些方法 [28, 42]。

- **问题描述**：训练网络时不仅没有收敛，损失（Loss）反而表现出虚假繁荣或极端不稳定，因为梯度在上一次循环后留存了下来 [44, 45]。
  - **解决方案**：在每次调用 `backward()` 之前，必须遍历所有参数并将它们现有的梯度显式重置为零（即实现类似于 PyTorch 中的 `zero_grad` 功能），否则所有的梯度会随着迭代不断累积叠加 [45, 46]。

- **问题描述**：更新参数时跨步太大（学习率设置过高），导致越过了损失函数的极小值点，甚至使损失爆炸（blow up） [47, 48]。
  - **解决方案**：仔细调优学习率步长（Step size）。这门手艺通常被称为“微妙的艺术”。在实际操作中，还可以引入 [[learning rate decay]] (学习率衰减)，在训练初期使用较大的学习率，后期逐渐调小以微调参数 [48, 49]。

## 最佳实践
- **用标量解构黑盒，忽略底层张量（Tensors）**：为了真正理解背后的数学原理，抛弃多维张量，全部用标量（Scalars）操作来重构框架。因为从数学角度而言，张量计算只是对标量计算的并行打包（仅为了提升计算效率），核心的微积分逻辑完全不变 [2, 50]。
- **数值梯度校验（Gradient Check）**：在实现复杂的解析梯度/自动求导之前，始终先使用极小的扰动 $h$ 计算数值梯度，来对你推导出的数学反向传播公式进行内联检查 [51]。
- **灵活的局部抽象层级**：自动求导引擎不需要仅仅构建在最原子的运算（如加、乘）上。像 [[tanh]] 这种复合函数，只要你能够写出它的局部导数公式（Local derivative），就可以把它封装成一个独立的宏观节点来参与链式求导，抽象粒度完全由开发者决定 [4, 52-54]。
- **梯度向量的反方向应用**：梯度代表的是使得目标函数（Loss）**增加**的斜率方向。因为我们的目标是最小化损失，所以在更新参数时，务必要添加负号（`-learning_rate * grad`）以向梯度的反方向移动 [37, 55]。

### [[Backpropagation (反向传播)]]
- **定义**：一种用于高效评估损失函数相对于神经网络权重梯度的核心算法。
- **视频上下文**：贯穿整个视频的核心内容。它通过在表达式图中从输出节点向输入节点反向工作，并递归计算梯度，来告诉我们如果微调参数，结果将如何变化 [4-6]。

### [[Chain Rule (链式法则)]]
- **定义**：微积分中的一种法则，通过将局部导数与全局导数相乘来计算复合函数的导数。
- **视频上下文**：被描述为反向传播背后的数学核心。视频通过“汽车比自行车快，自行车比人快”的直观例子解释了为什么要将各个局部斜率相乘 [5, 7, 8]。
- **关联概念**：属于 → [[Backpropagation (反向传播)]]

### [[Derivative (导数) / Gradient (梯度)]]
- **定义**：衡量函数输出对其输入微小变化的响应敏感度（即函数在某一点的斜率）。
- **视频上下文**：视频最初通过给变量加上极小的数值 `h` 来数值化地逼近导数。在后续的神经网络训练中，梯度指示了误差增加的方向 [9-12]。
- **关联概念**：依赖 → [[Gradient Descent (梯度下降)]]

### [[Forward Pass (前向传播)]]
- **定义**：输入数据通过计算图（或神经网络）并逐步计算以产生最终输出预测的过程。
- **视频上下文**：在求梯度之前，系统必须先用加法、乘法等节点构建出数学表达式图并执行前向计算得出结果（例如计算出预测值或损失值） [5, 13, 14]。
- **关联概念**：对比 → [[Backpropagation (反向传播)]]

### [[Loss Function (损失函数)]]
- **定义**：一个衡量神经网络当前预测表现与真实目标之间误差的单一数值。
- **视频上下文**：视频示例中使用了均方误差（Mean Squared Error, MSE）。损失值越低，说明网络预测得越准确。训练的最终目的就是最小化这个数值 [4, 15, 16]。
- **关联概念**：依赖 → [[Gradient Descent (梯度下降)]]

### [[Neuron (神经元)]]
- **定义**：神经网络的基本计算结构单元，计算输入与权重的点积并加上偏置，最后通过非线性激活函数。
- **视频上下文**：在视频中，神经元被建模为包含权重（w）、偏置（b）输入并应用 `tanh` 挤压函数（squashing function）的数学表达式模块 [17-19]。
- **关联概念**：依赖 → [[Activation Function (激活函数)]]

### [[Activation Function (激活函数)]]
- **定义**：一种将非线性特性引入神经元输出的函数，例如将任意范围的输入挤压到特定区间内。
- **视频上下文**：视频中详细实现了 `tanh` 作为非线性激活函数，它会将大的正数平滑地限制在 1，大的负数限制在 -1 [18, 20, 21]。
- **关联概念**：属于 → [[Neuron (神经元)]]

### [[Topological Sort (拓扑排序)]]
- **定义**：一种对有向无环图进行排序的算法，确保所有节点的依赖项都在它之前被处理（边从左指向右）。
- **视频上下文**：为了实现自动反向传播，视频使用拓扑排序确保在调用任何节点的 `.backward()` 方法之前，其所有后续依赖节点的梯度都已计算完毕 [22-24]。
- **关联概念**：属于 → [[Backpropagation (反向传播)]]

### [[Gradient Descent (梯度下降)]]
- **定义**：一种优化策略，通过沿着梯度的相反方向（负方向）不断微调参数来最小化损失函数。
- **视频上下文**：视频展示了经典的训练循环：前向传播、反向传播，然后使用负梯度乘以步长更新神经网络参数（权重和偏置），使预测逐渐逼近目标 [12, 25, 26]。
- **关联概念**：依赖 → [[Learning Rate (学习率)]]

### [[Learning Rate (学习率) / Step Size (步长)]]
- **定义**：在梯度下降更新参数时，控制每次朝着梯度方向移动幅度大小的超参数。
- **视频上下文**：视频提到调整学习率是一门微妙的艺术。如果设置得太低，网络收敛会非常慢；如果设置得太高，训练会变得不稳定，甚至导致损失值爆炸 [27, 28]。
- **关联概念**：属于 → [[Gradient Descent (梯度下降)]]

### [[Zero Grad (梯度清零)]]
- **定义**：在每个反向传播周期开始前，将所有参数累积的梯度重置为零的操作。
- **视频上下文**：讲师将其称为深度学习中最常见的 Bug 之一。因为微积分链式法则在节点被多次使用时需要累加梯度（`+=`），若不手动清零，历史梯度就会错误地累积到下一轮更新中 [29-32]。
- **关联概念**：依赖 → [[Gradient Descent (梯度下降)]]
