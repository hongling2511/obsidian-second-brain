## 一句话总结
讲者 Drake 介绍了从掌握提示词框架到最终构建非确定性 [[AI Agents]] 的四个进阶阶段。

## 目标受众与前置知识
- **适合谁看**：想要学习构建 [[AI Agents]] 以实现工作流自动化的人群（推断）。
- **前置知识**：[[LLM]]（大型语言模型）的基本概念与 [[Chat GBT]] 的日常交互基础（推断）。

## 核心观点
1. **掌握提示词框架（Prompt Frameworks）是构建智能体的基础**
   - 为什么重要：它能重塑创建指令的思维方式，在 [[LLM]] 层获取更好的结果。
   - 依据：讲者展示了使用 [[GCAO]]（Goal, Context, Action, Output format）框架，在 [[Chat GBT]] 中输入目标（增肌 10 磅减脂到 10%）、背景（当前体重 155磅/13%体脂，可用器械）、动作和表格输出格式，生成了高度精确的 6 个月锻炼和饮食计划。
2. **自定义指令（Custom Instructions/System Prompts）能稳定控制模型后台响应**
   - 为什么重要：允许用户在后台转换响应，使前端输入更简捷，并保证输出语气与格式的绝对一致。
   - 依据：讲者在 [[Chat GBT]] 后台植入预设的模板提示，使其变成一个自动化收集"主体"和"背景"信息的引导程序，每次都能稳定生成带有流行色彩的漫画风格图片（如水下古城的孟加拉虎）。
3. **自动化（Automations）提供所见即所得的固定工作流**
   - 为什么重要：将 [[LLM]] 与外部应用系统连接，完成可预测的、固定的序列任务，不需要人工干预。
   - 依据：讲者使用 [[N8N]]（字幕记为 NAND/NADN）搭建工作流，将 [[Telegram]] 中输入的词汇发送给 [[OpenAI]] 生成图片，随后自动上传至 [[Google Drive]] 并发回给 [[Telegram]]。
4. **[[AI Agents]] 的核心在于赋予模型自主选择工具的非确定性能力**
   - 为什么重要：有别于固定的自动化流程，Agent 能够根据触发器的动态信息，自主决定调用哪些工具来完成复杂任务。
   - 依据：讲者对比了固定在每天早晨6点发送包含特定温度数据的天气预报自动化流程；而在 Agent 模式下，挂载获取天气和发送邮件工具后，[[LLM]] 根据用户的随机聊天提问，自动查询天气并动态生成了一封包含天气双关语的幽默邮件。

## 概念关系
- [[Prompt Frameworks]] → [[Custom Instructions]]：前者的结构化思维和测试结果，是编写后者后台系统提示词的依据。
- [[Automations]] ↔ [[AI Agents]]：前者是确定的（Deterministic）、单一路径的触发执行流；后者是非确定的（Non-deterministic），由大型语言模型动态决定工具的调用。
- [[GCAO]] → [[LLM]]：利用目标、背景、动作和输出格式四个维度的信息，显著提高大型语言模型输出的有效性和一致性。

## 详细笔记
### 阶段一：使用和创建提示词框架 (Prompting with Frameworks)
- **概念介绍**：讲者 Drake 指出这是最重要的一步，提示词框架是基于首字母缩写的提示模板。讲者展示了其社区推荐的 [[GCAO]] 框架（Goal目标、Context背景、Action动作、Output format输出格式）。
- **工具与演示**：
  - 讲者在 [[Chat GBT]] 使用 [[GPT-4o mini]]（字幕记为 04 Mini）演示了 [[GCAO]] 框架的威力，输入个人的体型数据和健身目标，获取了表格形式的营养与锻炼计划。
  - 讲者进一步展示了如何反向让模型生成框架：先在对话中找出一个喜欢的漫画风格大象图片，随后要求模型提取其中的风格参数，生成一个带括号占位符（如[Subject], [Setting]）的模板提示词，甚至可以将其保存在 [[Notion]] 页面中备用。

### 阶段二：自定义指令 (Custom Instructions)
- **概念介绍**：等同于系统提示词（System Prompts），用于在后台限定 [[LLM]] 的回复行为。讲者提到不仅可以在 [[Chat GBT]] 使用，也可以在 [[Claude]]（字幕记为 clude）中使用。
- **工具与演示**：
  - 讲者将第一阶段生成的漫画风格图片模板嵌入到 [[Chat GBT]] 的"Custom Instructions"中，并指示模型通过提问的方式引导用户输入。
  - 演示中，讲者仅输入"Hello"，模型便自动开始分步询问用户想要的动物（孟加拉虎）和场景（水下古城），收集完毕后自动输出漫画风格图片。讲者强调，如果不掌握这种一致性输出，在后续构建自动化时会感到非常沮丧。

### 阶段三：构建预定义自动化 (Building Automations)
- **概念介绍**：预定义自动化是每次执行结果一致（讲者原话："What you see is what you get"）的工作流，始于触发器，然后执行后续应用程序的操作。
- **工具与演示**：
  - 讲者展示了在自动化平台 [[N8N]]（视频出现 NAND/NADN/Nad/NAN 多种拼写）中的连线操作。该平台可连接 [[Google]] 生态、[[Microsoft]] 生态、[[Notion]]、[[PayPal]]、[[Stripe]] 等全套工具。
  - 讲者连接了 [[Telegram]]、调用 [[OpenAI]] 的节点（植入了之前的漫画图片风格指令），以及 [[Google Drive]]。
  - 实际效果：在 [[Telegram]] 输入"zebra in a cafe"，工作流在 3 到 5 分钟内（讲者原话）生成图片，以上下文命名文件并存入 [[Google Drive]]，最后将图片回传到 [[Telegram]]。讲者还提到可以延伸节点，比如自动配文并发布到 X（原Twitter）或 [[Facebook]]。

### 阶段四：构建 AI Agents
- **概念介绍**：Agent 与 Automations 类似，但核心区别是为 [[LLM]] 提供了一组工具（Tools），模型根据传入的触发器内容自主决定使用哪个工具（非确定性）。
- **理论案例说明**：
  - 构建一个邮件分类 Agent，提供 [[Google Calendar]] 的相关工具（搜索、创建、更新日历事件）。
  - 当收到一封邀请晚餐或参加 [[Zoom]] 电话会议的邮件时，Agent 会自主搜索日程表。若有空则创建日历事件并草拟同意回复；若忙碌则分类不同并回复拒绝。讲者提及还可挂载 [[Air Table]]、[[WordPress]]、[[Trello]]、[[Notion]] 等数据库让 Agent 执行查询和操作记录。
- **演示对比**：
  - 讲者在 [[N8N]] 中展示了原先单一固定的天气自动化（每天早晨6点发送位于芝加哥的特定温度数据邮件）。
  - 随后将其替换为使用 [[AI Agent Node]]，将"获取天气"和"发送邮件"作为可调用的工具，且不提供任何硬性步骤指令。当讲者在提问输入"今天天气如何，用这些信息给我发一封搞笑邮件草稿"时，Agent 自主查询了天气，并发送了一封包含天气双关语的动态邮件。

### 社区推广与资源

- "Automations are genuinely where things start to get fun" — 价值判断 / 强调了当不同的应用程序和模型被连接流转时，能为用户带来极大的创造乐趣与效率提升。

- **关联观点**：1
- **验证标准**：获得一份包含具体操作步骤，且严格按照指定格式（如表格）输出的定制化方案。

- **做什么**：将你调试好的优秀提示词模板复制到 ChatGPT 的「自定义指令（Customize ChatGPT）」后台，并要求它在每次新对话时主动分步向你收集必要变量。

- **核心技术**：[[ChatGPT]] (包含 04 Mini 模型), [[Claude]], [[n8n]] (视频字幕中显示为 NAND/NADN/Nad), [[Telegram]], [[OpenAI]], [[Google Drive]], [[Google Calendar]] [1-6]。
- **环境要求**：未明确提及。
- **前置技能**：需要掌握基础的 Prompt 框架编写（如 GCAO：Goal, Context, Action, Output format）[7, 8]。需要掌握大语言模型后端自定义指令（Custom Instructions / System Prompts）的配置机制 [2]。

## 步骤拆解
1. **使用 GCAO 框架构建基础 Prompt**
   - 具体操作描述：明确提示词的四个核心部分：目标 (Goal)、提供背景上下文 (Context)、指定动作 (Action) 以及设定输出格式 (Output format，如表格) [1, 7-10]。将构建好的框架输入至大语言模型进行前端测试，以获取一致性的结果 [1]。
   - 关键代码/命令：未明确提及。
   - 注意事项：在 Action 部分，建议只给大语言模型分配一到两个动作指令，避免分配四五个动作导致效果下降 [1, 10]。

2. **配置自定义指令 (Custom Instructions) 提取模板**
   - 具体操作描述：在得到满意的结果后，要求模型反向生成一个带括号占位符的模板化 Prompt [11, 12]。进入 [[ChatGPT]] 的 "Customize ChatGPT" 设置，在 "What traits should ChatGPT have" 区域填入该模板，并指示模型在每次对话时主动引导用户提供相关变量（如主题、场景）来替换占位符 [13-15]。
   - 关键代码/命令：未明确提及。
   - 注意事项：利用自定义指令不仅可以隐藏前端冗长的 Prompt，还能固定每次对话的交互流程与输出格式 [2, 16]。

3. **在自动化平台构建预定义工作流**
   - 具体操作描述：在 [[n8n]] 等平台创建基于节点的自动化 [3, 4]。设置触发器（Trigger，如 [[Telegram]] 接收消息），然后连接 [[OpenAI]] 节点生成内容（如配置 API key 调用模型生成图片）[4, 5]。最后连接目标应用节点（如将文件上传至 [[Google Drive]] 并将结果回传至 [[Telegram]]）[5, 17, 18]。
   - 关键代码/命令：未明确提及。
   - 注意事项：此类自动化的特点是完全固定、预定义，除了用户的输入变量外，所有数据流向和文件命名方式都在配置节点时被写死 [19-21]。

4. **构建非确定性 (Non-deterministic) 的 AI Agent**
   - 具体操作描述：在工作流中使用 AI agent node 连接大语言模型，不再提供固定的操作动作指令，而是为其配备一组外部工具（Tools，如获取天气、发送邮件、操作 [[Google Calendar]] 搜索/创建日程）[6, 21-23]。由模型根据接收到的输入动态决定调用哪些工具以及如何处理原始数据 [22, 24, 25]。
   - 关键代码/命令：未明确提及。
   - 注意事项：由于模型行为是动态的，需要在提示或设定中为其操作范围建立安全护栏（Guard rails）和边界限制 [26]。

## 常见坑与解决方案
- **问题描述**：在没有掌握如何引导模型获取一致性结果之前就去构建自动化或 Agent，会导致实际运行时无法输出精确结果，从而引发困惑与压力 [16]。
  - 解决方案：先在基础的、低风险的前端界面中熟练掌握 Prompt 框架和自定义指令的使用，确保能够稳定复现期望的输出结果，然后再进入自动化工具的搭建阶段 [2, 16, 27]。
- **问题描述**：给大语言模型一次性分配过多的操作动作（如四五个动作），会导致模型执行效果变差 [1, 10]。
  - 解决方案：在构建 Prompt 框架的 Action 阶段，严格限制指令数量为一到两个 [10]。

## 最佳实践
- 推荐使用首字母缩写来固化你的提示词结构（如 GCAO），以便在不同的任务场景下都能套用此逻辑并获得良好结果 [7, 8]。
- 先通过普通对话寻找并确定理想的输出风格，然后直接要求模型为你生成包含占位符的模板化 Prompt 框架，而不是一开始就尝试手动编写模板 [11, 12, 28]。
- 在为 Agent 开放工作邮箱、日程表等高权限工具时，务必设置护栏（Guard rails）和边界来规范模型的处理动作 [26]。

## 关键概念

### [[Prompt Frameworks]]
- **定义**：基于首字母缩略词的提示词结构，旨在让用户更快地从大语言模型获得更好的结果 [1]。
- **视频上下文**：作为构建 AI Agent 的第一阶段，作者强调必须首先掌握如何使用和创建它，并在聊天界面的前端测试以找到满意的风格和输出 [1-3]。
- **为什么重要**：它是理解底层模型工作原理的基础，能够重塑编写指令的思维方式，并为后续自动化提供一致且可靠的结果 [1, 4]。
- **关联概念**：扩展 → [[GCAO Prompt Framework]] [1]

### [[GCAO Prompt Framework]]
- **定义**：由目标 (Goal)、上下文 (Context)、行动 (Action) 和输出格式 (Output format) 四个缩写要素组成的具体提示词框架 [1, 5]。
- **视频上下文**：作者以此为示例，演示了如何通过输入体重、可用设备等具体上下文，让大语言模型生成定制的健身和饮食计划表格 [6-9]。
- **为什么重要**：它提供了一个标准化的模板，无论用户遇到什么问题，都能确保模型生成高质量的结果 [5]。
- **关联概念**：属于 → [[Prompt Frameworks]] [1]

### [[Custom Instructions]]
- **定义**：在后台运行的指令（也称 System Prompts），用于精确控制大语言模型以特定的语气、方式或输出格式进行回复 [10]。
- **视频上下文**：作者将测试好的图片生成 Prompt 模板输入到 ChatGPT 的自定义设置中，让模型自动引导用户输入相关细节，从而稳定生成漫画风格的图片 [11-13]。
- **为什么重要**：它让用户在前端无需反复输入冗长复杂的提示词，并且是未来创建每一个 AI Agent 时必须具备的核心后端组件 [10]。
- **关联概念**：依赖 → [[Prompt Frameworks]] [10, 14]

### [[Automations]]
- **定义**：一种预先定义的、固定的工作流，在触发器启动后依次执行特定的应用程序操作，结果始终如一 [15, 16]。
- **视频上下文**：作者展示了如何建立一个自动化流程：当在 Telegram 输入文本时，程序会固定调用 ChatGPT 生成图片，然后自动上传到 Google Drive 并发送回 Telegram [16-18]。
- **为什么重要**：能够实现多步骤任务的无人值守运行，过程所见即所得且完全可预测，是连接不同应用生态的桥梁 [16, 19]。
- **关联概念**：对比 → [[AI Agents]] [20]

### [[AI Agents]]
- **定义**：一种高级工作流，大语言模型在其中被赋予了一组工具，并根据传入的触发器（如查询或事件）自行决定使用哪个工具 [20]。
- **视频上下文**：作者对比了固定的天气邮件自动化，展示了 Agent 如何根据用户的随意提问，动态决定调用“获取天气工具”，并起草包含双关语的个性化邮件 [21-23]。
- **为什么重要**：它具有非确定性和高度动态性，无需为每个场景编写死板的规则即可智能处理复杂多变的任务（如查阅日历、规划时间、分类邮件） [22, 24, 25]。
- **关联概念**：对比 → [[Automations]] [20]

### [[Tools]]
- **定义**：用户提供给大语言模型的外部应用程序接口或执行权限（如搜索/更新 Google 日历、获取天气等） [20, 26]。
- **视频上下文**：在构建 AI Agent 节点时，作者没有给模型写死指令，而是为其接入了获取天气预报和发送邮件的工具，让模型根据聊天内容自行决定如何调用它们 [21, 22]。
- **为什么重要**：赋予大语言模型与外部系统交互和执行实际操作的能力，是系统从“被动响应”转变为“主动代理 (Agentic)”的关键变量 [21, 23, 26]。
- **关联概念**：属于 → [[AI Agents]] [20]
