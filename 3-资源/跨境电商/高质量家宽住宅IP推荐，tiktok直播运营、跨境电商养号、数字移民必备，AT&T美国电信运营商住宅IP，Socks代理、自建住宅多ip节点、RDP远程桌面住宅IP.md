## 核心观点

## 概念关系
- [[机房IP]] ↔ [[住宅IP]]：前者（包含[[广播IP]]）用于业务风控风险较高，后者由当地网络运营商（如 [[AT&T]]）分配给真实家庭用户，质量最高且最稀缺。
- [[VPS]] → [[多IP节点]]：通过购买支持分配多个 IP 的虚拟专用服务器，并在面板内配置不同的出站路由规则，实现单一服务器提供多个不同的 IP 出口。
- [[Reality]] → [[住宅IP]]：前者作为抗封锁的高速加密前置代理协议，被用来代理（复活）原本容易被墙阻断的后者（[[Socks5]]/[[SS]]）。

## 详细笔记
### 代理 IP 风控原理与住宅 IP 价值
- 互联网高价值业务（如 [[Tiktok]]、[[Paypal]]）对网络环境要求极高。
- 使用普通的机场节点（[[机房IP]] / [[广播IP]]）会导致账号被限流或封禁。
- [[住宅IP]] 是当地真实家庭宽带分配的 [[原生IP]]，风控阈值最宽松，是开展业务的理想状态。
- IP检测网址只能作为参考，因运营商分配的动态住宅IP在部分数据库（如 [[Ping]]）中仍可能被误判为机房（IDC）IP。

### 选购 AT&T 住宅 IP 产品及参数分析
- 推荐商家提供真实美国 [[AT&T]] 住宅IP（相当于中国电信地位）。
- 产品分四款：
  1. [[Socks5]] 代理：直接导入工具，适合纯小白。
  2. 普通 [[VPS]]：需自装系统建节点，一机可绑多个IP。
  3. 旗舰版 VPS：适合直接使用远程桌面（[[RDP]]）的高配置用户。
  4. 独立服务器：256G内存、61个静态IP、1G带宽，无限流量，适合大客户（数据：讲者原话/屏幕展示）。
- 带宽建议：15兆 bps 满足基本与 Tiktok 单设备直播；45兆 可满足解锁 [[Netflix]]（奈飞）及看 4K 视频（数据：讲者原话）。

### 方案一：Socks5 节点配置与导入
- 购买后在“产品管理”获取分配的 IP、端口、用户名和密码。
- 工具导入：可直接填入 [[指纹浏览器]]，或通过链接/二维码扫码导入苹果端小火箭（推断为 [[Shadowrocket]]）、安卓端 [[N Box]]（推断为 NekoBox）、[[Windows]] 端 [[V2N]]（推断为 v2rayN）。
- 注意：直接导入极易被墙，需按后续章节进行“复活”配置。

### 方案二：VPS 自建多 IP 节点操作步骤
- 购买包含3个IP的入门版 VPS，安装 [[Ubuntu]] 20.04（原音：物班图2.04）系统。
- 登录服务器：[[Mac]] 用终端，[[Windows]] 用 cmd（命令提示符），通过 `ssh root@IP` 输入密码登入。
- 搭建面板：执行指令安装 [[X-UI]]（原音：xui/插瑞面板），登录面板后台（推断该面板控制 [[Xray]]）。
- 创建主节点：在入站列表添加 [[SS]]（原音：shad）协议，加密选择 [[aes-128-gcm]]（原音：as128gm）。
- 配置多IP路由：
  1. 面板添加不同客户“电子邮箱”标识（如住宅-2）。
  2. 在 xui 设置下添加“出站”，绑定额外分配的IP尾数（如98、100）。
  3. 添加路由规则，将对应客户标记指派到对应出站IP，最后重启 [[Xray]] 服务生效。

### 方案三：旗舰版 VPS 配置远程桌面 (RDP)
- 购买 8G 内存旗舰版 VPS（因 4G 跑不动 Windows，数据：讲者原话）。
- 安装 [[Windows]] 图形化操作系统。
- 连接方式：[[Mac]] 用户从 App Store 下载 [[Windows App]]；Windows 用户使用系统自带的远程桌面连接工具。
- 操作：输入 IP 和 3389 端口（端口：屏幕展示/推断），填入账号密码连接，获得 100% 伪装的美国本地电脑环境（含 [[IPv6]] 地址）。

### 住宅代理“复活”与提速教程
- **前提**：因直连超时阻断，需购买并借助高速稳定的 [[GIA]] / [[软银]] 线路 VPS。
- **前置配置**：通过 SSH 登入高速 VPS，安装面板，搭建 443 端口的 [[Reality]] 节点，目标网站设为 1.1.1.1:443。此节点本身可用作高速 [[科学上网]]。
- **方式一：通过 X-UI 面板出站路由代理（二级代理）**
  - 复活 Socks5：在高速 VPS 面板添加客户端（标识 sox5-1），配置 xui 出站协议为 socks，填入被墙的住宅 Socks 节点信息，配置对应路由规则并重启。
  - 复活 SS 节点：同理添加客户端，在出站设置选 JSON 格式，解析粘贴被墙的 SS 节点链接，配置路由并重启。
- **方式二：复活远程桌面连接（端口转发）**
  - 在面板入站添加 [[dokodemo-door]]（原音：扣do）协议，本地端口设为 3389，目标地址填入住宅远程桌面IP，目标端口 3389。通过高速 VPS IP 即可连接远程桌面。
- **方式三：客户端链式代理 / 电视代理**
  - 小火箭（[[Shadowrocket]]）/ [[V2N]]：在节点设置的“代理通过”中直接选中 Reality 节点。
  - 安卓 [[N Box]]：使用底层“链式代理”（原音：练视代理）功能，服务配置依次添加 Reality 节点和住宅节点。
  - Windows [[V2N]] 全局别名：创建分组，将所有住宅节点放入，配置全制代理别名为 Reality 节点名称。代理远程桌面可直接开启客户端的 [[TUN]] 模式（管理员身份运行）。

- **问题描述**：通过 [[SSH]] 登录 VPS 时出现 warning 警告（多数出现在重装系统之后）[1]。

- **问题描述**：在面板中设置了多 IP 路由规则，但实际访问网页时 IP 未发生切换 [4]。
  - 解决方案：保存配置后，必须重启面板（插瑞）才能生效 [4]。

  - 解决方案：使用高速 [[VPS]] 搭建二级代理进行中转，或在代理客户端中使用链式代理配置复活节点 [6, 7, 9]。
- **问题描述**：在 [[v2rayN]] 中准备使用链式代理代理远程桌面流量时，找不到 TUN 选项 [10]。
  - 解决方案：退出 [[v2rayN]]，以管理员权限重新运行即可显示并启用 [10]。

- 进行跨境电商或数字移民等对风控要求高的业务时，推荐使用住宅 IP 代理，而非机房 IP [11]。

## 关键概念

### [[住宅IP]]
- **定义**：当地网络运营商分配给当地真实家庭用户使用的IP地址 [1]。
- **视频上下文**：作者推荐将其用于跨境电商、TikTok直播和数字移民养号等业务，以解决IP不干净导致的问题 [1, 2]。
- **为什么重要**：它的质量最高且最稀缺，风控阈值与机房IP不同，使用它做业务出问题的概率较小 [1, 2]。
- **关联概念**：对比 → [[机房IP]]

### [[机房IP]]
- **定义**：类型为IDC机房的IP地址，通常风险较高 [1]。
- **视频上下文**：作者指出使用机场节点或普通自建节点大概率是此类IP，其检测结果可能显示为隶属广播IP [1]。
- **为什么重要**：使用这种IP做业务效果会大打折扣甚至完全无法使用，容易触发网络平台的风控 [1]。
- **关联概念**：对比 → [[住宅IP]]

### [[风控]]
- **定义**：互联网平台（如TikTok、PayPal等）判定IP存在风险并进行限制的机制 [1]。
- **视频上下文**：作者提到如果代理IP不干净，很容易触发风控导致账号被限流、直播不进人或被封禁，在互联网上离钱越近的行为风控等级越高 [1]。
- **为什么重要**：它是做跨境业务、数字移民养号时必须重点应对和规避的核心障碍 [1, 2]。
- **关联概念**：未明确提及
