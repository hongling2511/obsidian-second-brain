## 一句话总结
[[clash]] 内核存在路径穿越与跨域攻击漏洞，黑客可通过暴露的API远控设备，需及时升级内核或修改配置设防 [1-3]。

## 目标受众与前置知识
- **适合谁看**：使用 [[clash]] 及相关衍生代理客户端（如 [[clash for Window]]、[[clash verge]]、[[clash.meta]]）的用户、在 [[vps]] 或 [[Linux]] 部署相关服务的运维人员 [1, 4-6]（推断）。
- **前置知识**：[[clash]]、[[RESTful API]]、[[CORS]]、路径穿越（推断）。

## 核心观点
1. **[[clash]] 内核漏洞影响面广且危害极高**
   - 为什么重要：不仅限于单一客户端，覆盖所有使用该内核的主流操作系统。
   - 依据：讲者原话指出漏洞影响 [[Windows]]、[[Linux]]、[[maacos]]，只要使用该内核全中招 [1]；实战演示受害者点击链接后被植入木马，电脑被远控弹计算器、监视屏幕及窃取文件 [2, 7]。
2. **暴露的 [[RESTful API]] 结合 [[CORS]] 易导致无感知的跨域攻击**
   - 为什么重要：用户容易放松警惕，即使有安全意识也可能在点击普通网页时中招。
   - 依据：讲者演示中，受害者打开伪造网站跳转至 [[百度]]，后台却暗中调用 API 修改代理配置中的 [[prox provider]]，进而通过路径穿越将恶意指令写入 [[PowerShell]] 配置或自启目录 [2, 3, 8]。
3. **公网上存在大量未修复的裸露节点已沦为“肉鸡”**
   - 为什么重要：暴露在公网的设备无需任何诱导点击，黑客可直接提交恶意指令入侵。
   - 依据：讲者原话/屏幕展示指出，搜索结果显示有 8000多条 在公网开放的 [[clash]] API 肉鸡，且断言 99.9% 没升级存在漏洞；另有 700多条 [[clash.meta]] 结果同样任人宰割 [5, 6]。

## 概念关系
- [[RESTful API]] + [[CORS]] → 跨域攻击：[[clash]] 为方便 [[webui]]（如官方 [[dashboard]]、[[yacd]]）控制而支持 [[CORS]]，导致任意网站可调用 API 篡改内部配置 [7, 8]。
- 跨域攻击 + 路径穿越 → 远控木马植入：攻击网站通过 API 下发包含恶意指令的 [[prox provider]]，利用路径穿越漏洞将指令写入任意位置（如 [[PowerShell]] 的默认配置），系统重启或运行终端时即被远控 [3, 8]。
- 修改外部控制端口 + 设置授权密码 → 阻断跨域攻击：通过更改默认的 9090 端口并加设密码，使攻击者无法轻易访问 API 接口，从根源切断攻击路径 [3]。

## 详细笔记
### 漏洞背景与影响范围
讲者 不良林 指出，1月份 [[clash for Window]] 爆出的 RCE 漏洞中，[[clash]] 内核因存在路径穿越漏洞负有一半责任 [1]。该漏洞允许在任意位置写入文件，导致 [[Windows]]、[[Linux]]、[[maacos]] 等所有使用该内核的系统全部中招，危害比此前的漏洞更严重 [1]。尽管官方于 4月16号（讲者原话）发布了修复版本，但其分支项目 [[clash.meta]]（[[clashm]]）尚未修复，且多数用户未意识到需升级 [1, 4]。

### 实战演示：跨域攻击与木马控制
讲者通过两台虚拟机模拟攻击：受害者使用最新版 [[clash verge]]（v1.3.2，讲者原话），内核切换为存在漏洞的 [[clashm]] [2, 4]。攻击者发送一个跳转至 [[百度]] 网页的链接诱导点击 [2]。受害者点击后看似无异样，但其电脑的 [[clash]] 因暴露出通过 [[RESTful API]] 控制的接口，并支持跨域资源共享（[[CORS]]），导致恶意网页成功调用 API 替换了配置文件 [2, 7, 8]。新的配置文件引导客户端向指定的 [[prox provider]] 下载节点，其中夹带的恶意指令因路径穿越漏洞被写入系统敏感路径 [8]。受害者一旦打开 [[PowerShell]] 或重启电脑，指令即触发运行，攻击者借此完全控制受害者电脑（包括弹计算器、监控屏幕、浏览和上传下载文件等） [2, 7]。

### 安全防范与配置修复教程
针对漏洞，讲者提出不可仅依赖杀毒软件，应从根源解决 [7]：
- **方案一（首选）**：直接升级 [[clash]] 内核至最新修复版本 [3]。针对 [[clash for Window]] 用户，可直接在软件内点击将内核升级至最新 [3]。
- **方案二（针对未修复版本）**：针对暂无修复版的 [[clash.meta]] 用户，需进入设置将外部控制 API 的默认端（9090）改为其他端口，并设置 API 的授权密码，使攻击者无法找到或访问接口 [3]。针对 [[clash for Window]] 不想升级的用户，可开启“随机 API 接口”并生成授权密码 [3, 9]。
- **方案三（手动配置）**：对于使用 [[Linux]] 做透明网关的用户，需手动修改配置文件中的外部控制端口和授权密码 [5]。
讲者还补充提及，若使用他开发的节点测速工具 [[N catch]] 且为代理设置了密码，会报错 401 未授权错误，由于该软件源码已丢失不再更新，建议使用测速工具时不要设授权密码 [9]。

### 澄清质疑与公网安全现状
针对评论区称其“制造焦虑”的指责，讲者坦言做视频有流量考量，但强调安全意识极为重要 [5]。为证明严重性，讲者展示了搜索结果，显示公网直接暴露了 8000多条（讲者原话）[[clash]] API 肉鸡记录，以及 700多条（讲者原话）[[clash.meta]] 记录 [5, 6]。讲者推断这些设备 99.9% 存在路径穿越漏洞，黑客甚至无需诱导点击，直接通过公网 API 即可提交恶意指令 [5, 6]。最后讲者提醒有 [[vps]] 或家中 [[Linux]] 设备的用户排查是否暴露了 API，并致谢了为此期视频提供参考的某位大佬的博客文章 [6]。

## 常见坑与解决方案
- **问题描述**：使用节点测速工具 N catch 测速时，出现 401 未授权错误 [6]。
  - 解决方案：该工具共用了 [[clash]] 的配置但并未适配密码填写，且软件源码丢失已停止更新。若必须使用此测速工具，则不要为 [[clash]] 设置 API 授权密码 [6]。
- **问题描述**：电脑安装了杀毒软件，依然面临中招风险 [8]。
  - 解决方案：不能完全依靠杀毒软件来拦截，一旦被绕过即宣告沦陷。必须从根源上（升级内核或修改 API 端口/密码）解决路径穿越漏洞带来的问题 [8]。

## 最佳实践
- **推荐做法**：最简单的安全防范方式是直接将 [[clash]] 内核升级到最新版本以修复漏洞 [4]；如果在 VPS 或家里的 Linux 设备上使用 [[clash]]，务必检查是否在公网暴露了 API 接口 [9]。
- **反模式**：保持 [[clash]] API 为默认的 9090 端口且不设置密码，或将 API 直接暴露在公网环境下。这种做法极其危险，黑客可以直接通过 API 提交恶意指令，无需诱导点击链接即可控制设备 [4][7][9]。

## 关键概念

### [[clash 内核]]
- **定义**：一种代理软件的核心组件，支持跨平台运行，自身存在任意目录写入文件的安全缺陷[1]。
- **视频上下文**：作者指出 Windows、Linux 和 macOS 平台只要使用了该组件，就全都受到了漏洞影响[1]。
- **为什么重要**：作为多个代理客户端（如 clash for Window 等）的底层基础，它的安全缺陷导致了大面积的系统安全事件[1]。
- **关联概念**：扩展 → [[clash.meta]]

### [[clash.meta]]
- **定义**：clash 的一个分支项目，同样存在路径穿越漏洞且未修复[2]。
- **视频上下文**：作者在虚拟机实战演示中，使用它作为未修复漏洞的受害者环境，展示了被黑客植入木马并控制电脑的完整过程[2, 3]。
- **为什么重要**：作为大量用户正在使用的工具分支，其未修复状态和不当配置极易成为攻击者的首选目标[2]。
- **关联概念**：属于 → [[clash 内核]]

### [[路径穿越漏洞]]
- **定义**：一种允许在系统任意位置（如开机自启目录或默认配置位置）写入恶意文件的安全漏洞[1, 4, 5]。
- **视频上下文**：作者解释，由于此漏洞的存在，跨域下载的恶意指令可以被保存在电脑的任意危险位置，并在重启或打开 PowerShell 时自动执行[4, 5]。
- **为什么重要**：它是引发严重后果的核心漏洞，使得原本单纯的配置文件覆盖行为演变成了可以直接控制受害者电脑的木马病毒攻击[1, 5]。
- **关联概念**：依赖 → [[clash 内核]]

### [[RESTful API]]
- **定义**：clash 暴露在外部（默认端口9090）用于控制内部设置并支持跨域资源共享的接口[5, 6]。
- **视频上下文**：视频提到它是为了方便 WebUI（如官方 Dashboard 等）配置而设计的，但如果不修改默认端口或设置授权密码，就会被黑客恶意调用[4, 5, 7]。
- **为什么重要**：它是外部攻击者向受害者电脑静默下发包含恶意指令的配置文件的直接通道[4, 8]。
- **关联概念**：依赖 → [[CORS]]

### [[CORS]]
- **定义**：跨域资源共享（Cross-Origin Resource Sharing），即允许不同域名共享相同资源的机制[6]。
- **视频上下文**：作者解释 clash 支持该机制是为了方便在外部网站上直接配置代理，但这也为跨域攻击提供了便利条件[4, 6]。
- **为什么重要**：它是导致任何外部恶意网站都能够调用并篡改本地 clash API 接口的前置条件[4, 6]。
- **关联概念**：扩展 → [[跨域攻击]]

### [[跨域攻击]]
- **定义**：通过网页跳转或链接诱导，在后台静默调用未受保护的本地 API 接口以执行恶意指令的攻击手段[4, 5, 8]。
- **视频上下文**：作者演示了受害者仅打开一个看似普通的网页跳转链接，电脑即在后台通过这种攻击方式被植入木马并被完全控制[3, 4]。
- **为什么重要**：这是黑客利用 clash 漏洞实施入侵的主要网络手段，用户防范门槛极高，往往在不知情中即可中招[3, 4]。
- **关联概念**：依赖 → [[RESTful API]]

### [[肉机]]
- **定义**：因为接口暴露在公网且存在漏洞，被黑客直接远程提交恶意指令并完全控制的设备[7, 8]。
- **视频上下文**：作者通过公网搜索展示了数千台暴露了 clash API 的服务器，指出这些设备绝大多数存在漏洞并可以直接被当作黑客的工具[7, 8]。
- **为什么重要**：直观揭示了网络安全意识薄弱和错误配置在真实公网环境下造成的灾难性后果[7, 8]。
- **关联概念**：未明确提及
