## 一句话总结
大量第三方在线 [[订阅转换]] 工具存在配置泄漏与 [[RCE]] 漏洞，攻击者可借此入侵服务器并盗取节点订阅信息。

## 目标受众与前置知识
- **适合谁看**：使用第三方在线服务进行节点转换的普通用户、提供公共订阅转换服务的站长或机场主。
- **前置知识**：[[订阅转换]]、[[vps]]、[[RCE]]、[[API Mode]]、[[token]]

## 核心观点
1. **使用第三方订阅转换服务存在极大的隐私泄露风险。**
   - 为什么重要：即使你对服务提供者有绝对的信任，提供者自身的服务器也可能因为漏洞被黑客拿下，导致所有用户的节点信息被窃取。
   - 依据：“即使你對他提供的服務有00%的信任（推断：原意为100%），堅信他絕對不會到取你的節點，也可能導致你的節點信息洩露”（讲者原话）。

2. **[[Subconverter]] 工具存在严重的 [[RCE]]（远程命令执行）漏洞。**
   - 为什么重要：只要服务器开启了缓存功能，由于路径穿越过滤不严谨，攻击者就可以将恶意指令写入缓存目录并触发执行，直接获取服务器的 [[ROOT]] 权限。
   - 依据：讲者实战演示将带有反弹 [[shell]] （视频字幕记作 share）的恶意指令转为 [[md5]] 和 [[bas64]] 写入目标服务器，最终在自己的 [[vps]] 上监听“1880端口”（讲者原话）成功拿到服务器权限。

3. **尽管官方进行过代码修复，但大部分公共服务仍存在漏洞。**
   - 为什么重要：目前 GitHub 编译的最新发布版仍是未修复版本，且新的接口依然会泄露关键的 [[token]]，使得现有的鉴权机制失效。
   - 依据：GitHub 项目里的最新 release 版本是“2022年4月3号”（讲者原话）编译的未修复漏洞版本；即使是删除了 [[xsript]] 的新版本，其 [[cvert]] 接口同样可暴露配置内容。

## 概念关系
- [[第三方订阅转换]] → [[节点泄露风险]]：用户通过第三方网页将 [[v2a]] 转换为 [[clash]] 时，第三方后台及入侵第三方的黑客都能截获节点信息。
- [[a CASH]] ↔ [[RCE]]：当配置中的 [[a CASH]]（推断：缓存）开启时，才具备将恶意网址计算为 [[md5]] 写入缓存目录的条件，进而导致 [[RCE]]。
- [[API Mode]] ↔ [[token]]：如果 [[API model]]（讲者原话）为 true，必须配合正确的 [[token]] / [[ten]]（讲者原话）才能执行脚本；如果为 false，则直接无需鉴权即可利用漏洞。
- [[cvert]] 接口 → [[token]] 泄露：通过访问特定接口，可越权查看配置文件，导致服务器密码凭证直接暴露。

## 详细笔记
### 隐私风险引入与背景
不良林指出，[[订阅转换]] 功能常被用来将 [[v2a]] 分享链接转为 [[clash]]，或将 [[cllash]]（讲者原话）转为 [[v2v]]（讲者原话）。很多人图方便使用“肥羊”、“品云”、“边缘”等第三方服务。这存在根本性隐私问题：不仅站长能在后台看到节点，且因开源工具漏洞，黑客能轻易拿下这些提供商的服务器（如“am”和“奶西”这两个转换网站），从而盗走所有人的配置。

### 漏洞探测与配置文件泄露原理
攻击者首先会在后端地址加上 `gversion` 或 `gvergin`（讲者原话）探测版本。如果返回 404 说明加了 [[wb中间键]]。若访问到特定版本（如讲者展示的“0.7.2”版本，屏幕展示），可通过另一接口获取默认配置文件内容。由于路径穿越过滤不严谨，配置中的 [[API model]]（即 [[API Mode]]）、关键的 [[ten]] 值（推断：token），以及决定是否存在利用条件的 [[a CASH]]（值为2说明开启缓存，推断：cache）都会直接泄露给攻击者。

### 实战演示一：常规配置下获取 ROOT 权限
在不需要 [[token]] 的情况下：
1. **写入缓存**：构造一条包含恶意指令的网址拼接到转换地址中。服务器转换失败，但会以网址的 [[md5]] 为文件名保存在缓存目录。
2. **植入脚本**：将恶意网址内容替换为 [[bas64]] 编码的明文，指令作用为下载反弹脚本连接攻击者 [[vps]] 的“1880端口”（讲者原话）。再次访问触发写入。
3. **触发执行**：替换链接中的 [[md5]]，执行缓存目录下的文件。攻击者在其 [[vps]] 端直接拿到对方的 [[ROOT]] 权限，可读取日志并偷走所有正在转换的节点链接。

### 版本迭代与漏洞现状
讲者梳理了代码修复时间线：
- “2022年4月4号”（讲者原话）：作者通过添加鉴权要求正确 [[token]] 才能执行脚本。但 [[穿xsript]] / [[trrip]]（推断：xscript）接口仍会暴露配置文件中的 [[token]]。
- “2022年8月22号”（讲者原话）：删除了该接口，无法再直接暴露。
- **坏消息**：当前 GitHub 最新 Release 版是“2022年4月3号”（讲者原话）编译的，包含“边缘订阅转换”在内的大量公共站长仍在使用这个老版本，处于完全未设防状态。

### 实战演示二：盲打与接口泄露利用
若接口不可见，攻击者可直接“盲打”尝试将指令写入缓存并用 [[vps]] 监听。如果无反应，说明开启了 [[API Mode]] 需要凭据。攻击者可猜测并附加默认的 `rlpword`（推断：默认配置的密码字段）进行测试。
此外，即使是“某一线机场”使用的 [[Math的分支版本0.8.2]]（讲者原话），虽然删了旧接口，但其 [[cvert]] / [[cvat]]（讲者原话）接口同样可暴露 [[token]]。只要确认对方开启了缓存并拿到 [[token]]，即可轻松入侵。

### 防御与缓解措施
目前公共转换服务超一半存在此情况。讲者建议：
- 站长端：使用 [[N反待]]（推断：Nginx反代）隐藏 [[cvert]] 接口，或直接关掉缓存服务（如“品云”的做法，但这会增大服务器负荷）。建议自行编译最新代码。
- 用户端：强烈推荐使用本地 [[订阅转换]]，讲者“两年前”（讲者原话）曾出过教程，信别人不如信自己。

## 步骤拆解
1. **获取并检测后端版本**
   - 复制订阅转换的后端地址，在网址后面拼接特定路径进行访问，以查看后端的版本号 [2]。
   - 关键代码/命令：在网址后拼接 `/gversion` [2]。
   - 注意事项：如果网页返回 404，说明该后端可能使用了 Web 中间件进行了反向代理 [2]。
2. **获取配置文件以确认配置状态**
   - 尝试通过特定接口获取默认配置文件内容，检查 API 模式（API Mode）是否为 true、获取关键的 token 值，并查看最下方是否开启了缓存（cache） [9]。
   - 关键代码/命令：访问 `xsript` 接口或 `cvert` 接口 [6, 8]。
   - 注意事项：由于路径穿越过滤不严谨导致配置文件泄露；只要开启了缓存，就大概率存在 RCE 风险 [9]。
3. **将恶意指令写入缓存目录**
   - 创建一条包含恶意指令（反弹 Shell 脚本，让目标连接攻击机 `[[VPS]]` 的特定端口）的网址，将其加密成 `[[md5]]`，再编码为 `[[base64]]` 替换原明文内容，最后拼接到订阅转换的网址后进行访问 [3]。
   - 关键代码/命令：使用 `[[md5]]` 和 `[[base64]]` 处理载荷；攻击机预先监听 `1880` 端口 [3]。
   - 注意事项：服务器会以恶意网址计算得出的 `[[md5]]` 当做文件名保存在缓存目录中；网页可能返回“无法正常进行订阅转换”，但只要能正常显示恶意的代码串，即说明指令成功写入对方缓存目录 [3, 9]。
4. **触发恶意指令执行**
   - 将上一步生成的 `[[md5]]` 值替换到执行链接中，意思是执行缓存目录下的对应脚本，访问该触发链接 [5]。
   - 关键代码/命令：在触发网址后附带 token 参数（如 `token=password`） [8]。
   - 注意事项：访问触发链接时，攻击者的 `[[VPS]]` 会收到对方 `[[VPS]]` 的 Shell 并拿到 ROOT 用户权限 [5]。

## 常见坑与解决方案
- **问题描述**：通过配置接口无法暴露 token（新版代码删除了特定接口），或被反向代理隐藏了接口，导致无法得知对方是否开启 API Mode 与缓存 [6, 7]。
  - 解决方案：进行盲打测试，假设对方开启了缓存，并在触发链接中猜测并拼接默认弱口令（如 `password`）尝试连接 [7, 8, 10]。
- **问题描述**：目标服务器的 API Mode 设置为 true，导致无法直接执行脚本 [6]。
  - 解决方案：必须获取或猜解出正确的 token 加在请求中才能执行 [6, 10]。

## 最佳实践
- **推荐做法**：对外提供公共订阅转换服务时，必须将 API Mode 设置为 true [7]。
- **推荐做法**：使用 `[[Nginx]]` 反向代理隐藏 `cvert` 等接口，防止配置文件泄露 [4]。
- **推荐做法**：直接关掉缓存服务（如品云订阅转换的做法），虽然对服务器负担较大，但能规避此 RCE 风险 [4]。
- **推荐做法**：自行编译最新代码，或者作为使用者直接使用本地订阅转换，彻底杜绝隐私与安全风险 [4, 6]。
- **反模式**：不要将 token 设置为默认的弱口令（如 password），否则依然会被盲打攻击导致 RCE [8]。
- **反模式**：本地使用订阅转换时，没必要将 API Mode 设置为 true，这会导致无法转换本地文件 [7]。
- **反模式**：继续使用未修复漏洞的旧版本（如 2022年4月3日前的版本）提供公共服务 [6]。

## 关键概念

### [[订阅转换]]
- **定义**：将一种代理节点的分享链接格式转换为另一种客户端（如Clash或V2ray）支持的订阅链接格式的服务 [1]。
- **视频上下文**：作者指出日常使用频率很高，用户常使用第三方服务，但这存在隐私泄漏风险和服务器被入侵的危险 [1, 2]。
- **为什么重要**：是整个漏洞产生的业务背景，广泛使用导致漏洞影响面极大 [1, 2]。
- **关联概念**：依赖 → [[Subconverter]]

### [[Subconverter]]
- **定义**：用于搭建订阅转换服务的开源项目工具 [2]。
- **视频上下文**：视频提到目前订阅转换服务都是用这个开源项目搭建的，且被曝存在 RCE 安全风险 [2]。
- **为什么重要**：漏洞的核心载体，无论使用哪个版本都可能导致服务器被黑客入侵 [2, 3]。
- **关联概念**：未明确提及

### [[RCE]]
- **定义**：远程代码执行漏洞，允许攻击者在目标服务器上执行任意恶意指令 [3, 4]。
- **视频上下文**：作者演示了如何通过该漏洞向服务器缓存目录写入恶意脚本，并触发执行，最终拿到对方 VPS 的 Shell 和 Root 权限 [5, 6]。
- **为什么重要**：这是视频预警的核心高危漏洞，直接导致所有人的节点和机场订阅链接被盗取 [2, 6]。
- **关联概念**：依赖 → [[缓存]]

### [[路径穿越]]
- **定义**：一种由于过滤不严谨导致的安全漏洞，允许攻击者获取本不应被暴露的配置文件内容 [3]。
- **视频上下文**：视频提到由于“路径穿越过滤不严谨”，导致订阅转换的默认配置文件内容泄露，暴露出 API Mode 和 Token 等关键信息 [3]。
- **为什么重要**：是发动攻击、获取关键配置信息的初始突破口 [3]。
- **关联概念**：扩展 → [[Token]]

### [[Token]]
- **定义**：配置文件中的一种鉴权密钥，用于验证执行脚本的权限 [3, 4]。
- **视频上下文**：在开启 API Mode 时，需要正确的 Token 才能执行脚本 [4]。但由于接口泄露或管理员使用弱口令，攻击者依然可以获取或猜解中 Token 从而执行恶意指令 [7, 8]。
- **为什么重要**：本应是保护服务器免受恶意脚本执行的安全机制，但常因配置不当或接口暴露而失效 [4, 8]。
- **关联概念**：属于 → [[API Mode]]

### [[API Mode]]
- **定义**：Subconverter 配置文件中的一项模式设置，用于控制是否需要鉴权才能执行脚本 [3, 4]。
- **视频上下文**：如果设置为 false，则不需要 Token 即可执行脚本；如果对外提供公共服务，必须设置为 true，否则存在 RCE 的风险 [4, 9]。
- **为什么重要**：其状态决定了 RCE 漏洞利用的难易程度和所需条件 [9]。
- **关联概念**：依赖 → [[Token]]

### [[缓存]]
- **定义**：Subconverter 服务中用于临时保存生成文件（或被恶意利用保存脚本）的机制和目录 [3]。
- **视频上下文**：攻击者通过特定链接将恶意指令文件写入到缓存目录中（文件名以网址的 md5 命名），随后触发执行 [3, 5, 6]。
- **为什么重要**：是 RCE 漏洞被成功利用的必要条件，关闭缓存服务可以直接杜绝相关风险 [7, 10]。
- **关联概念**：未明确提及

### [[盲打]]
- **定义**：在无法获取对方配置文件、不知道是否开启特定防御机制的情况下，直接尝试发送攻击载荷 [7, 9]。
- **视频上下文**：作者演示即使相关接口被反向代理隐藏或被删，仍可以通过盲打尝试执行恶意指令，配合弱口令字典猜解 Token [7-9]。
- **为什么重要**：展示了即使修补了部分信息泄露接口，默认或薄弱的配置依然能在实战中被攻破 [9, 10]。
- **关联概念**：未明确提及
