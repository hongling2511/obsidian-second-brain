## 核心观点
1. **[[clash]] 配置不当会导致流量被盗跑**：未设置 [[API]] 鉴权密码会将个人设备变成免费公共代理。
   - 为什么重要：导致机场流量被偷跑，甚至遭受恶意指令注入等安全风险。
   - 依据：讲者原话指出三个月内开放 [[API]] 的人数从 8000 多上升到 1 万多，[[clash meta]]（字幕记为 clash matter）从 700 上升到 1000 多 [1]。
2. **暴露 [[API]] 是用户配置问题而非 [[clash]] 漏洞**：官方文档已强调暴露在公网必须设置授权密码。
   - 为什么重要：纠正部分用户对工具安全性的误解，促使用户自查配置。
   - 依据：讲者原话“clash 的官方文档也强调 API 暴露在公网上必须设置授权密码，只是很多人并没有在意” [2]。
3. **修改默认密码和端口能有效防范扫描**：[[openclash]] 默认密码 123456 形同虚设，默认 7890 端口易被扫。
   - 为什么重要：大量扫描针对默认端口和弱口令，修改可大幅提升安全性。
   - 依据：讲者原话测试多台 [[openclash]] 使用 123456 默认密码即可直登 [3, 4]；指出建议不要使用默认的 7890 端口以防有人全网扫 [5]。
