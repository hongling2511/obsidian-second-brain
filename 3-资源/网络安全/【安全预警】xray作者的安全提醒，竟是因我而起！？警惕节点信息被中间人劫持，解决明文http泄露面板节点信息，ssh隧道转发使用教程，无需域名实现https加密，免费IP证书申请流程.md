## 一句话总结
不良林复盘 [[xray]] 面板明文 [[http]] 泄露争议，并演示如何通过 [[ssh]] 隧道与免费 IP 证书实现 [[https]] 加密访问。

## 目标受众与前置知识
- **适合谁看**：零基础节点搭建小白用户，以及使用 [[xui]]、[[3xui]] 等代理面板的用户（讲者原话：“教程面向的是零基础存小白用户”）。
- **前置知识**：[[vps]]、[[节点]]、[[代理工具]]、[[xray]]、[[http]]、[[https]]。

## 核心观点
1. **使用明文访问面板存在严重安全风险**
   - 为什么重要：由于请求是明文的，中间人能看到用户在面板上进行的所有操作，包括节点配置、密码和私钥。
   - 依据：[[xray]] 作者 rpx 在官方 [[tg]] 频道发布的安全建议（讲者原话）；2023年7月30日（讲者原话）不良林发布的 [[YouTube]] 教程中使用了明文 [[http]] 访问被指出风险。
2. **在已翻墙状态下访问面板拥有相对安全性**
   - 为什么重要：如果请求已经过节点加密，[[gfw]] 实际上看不到内容；但若代理工具关闭或走直连分流，则依然存在暴露风险。
   - 依据：讲者分析自身教程受众已处于翻墙网络环境，但也承认“面板确实还是 [[http]] 的访问方式”（讲者原话）。
3. **面板开发者必须强制实施加密通道访问**
   - 为什么重要：仅靠提醒用户不足以消除安全隐患，会导致公网上持续暴露大量未加密面板。
   - 依据：用户反映各个 Web 面板默认配置会被扫描且大部分使用 [[http]]；[[xray]] 作者提出 PR 强制要求并移除了未遵守规则的面板（讲者原话）。

## 概念关系
- [[http]] ↔ [[https]] / [[ssh]]：对比关系。前者是明文传输，会被 [[gfw]] 看见操作内容；后者是加密隧道或加密协议，中间人无法查看数据。
- [[xray]] 作者要求 → [[xui]] / [[3xui]]：因果关系。[[xray]] 作者强制要求面板使用加密通道，导致未遵循规定的 [[xui]] 和 [[3xui]] 被移出官方推荐列表。
- [[tls]] 证书 + 路径配置 → 防扫描与防窃听：依赖关系。单纯套用 [[tls]] 只能保证数据加密（防窃听），必须配合复杂的路径设置才能有效防止面板被公网扫描。

## 详细笔记

### 争议事件复盘与起因
- 2023年7月30日（讲者原话），YouTuber 不良林发布零基础节点搭建教程，演示中通过 [[http]] 加 IP 和端口访问 [[xui]] 面板后台。
- 次日，[[xray]] 作者 rpx 在 [[tg]] 频道发出安全警告，指出此方式会让中间人（[[gfw]]）看到节点的密码和私钥等敏感信息。
- 讲者初步回应认为，观看教程的用户默认已处于翻墙环境，数据经过节点加密，[[gfw]] 看不到内容，因此问题被暂时搁置。

### 矛盾激化与处理结果
- 今年的10月4号（讲者原话，推断年份为2024年），有用户反映大量面板默认配置导致被公网扫描，且大多仍是 [[http]] 访问。
- [[xray]] 作者发布 PR，强制规定基于 [[xray]] 的 Web 面板必须且只能通过加密通道访问，否则移出官方推荐列表。
- 经过长达近一个月的争论（反方主张提醒即可，正方主张强制执行），未遵循该规定的面板（如 [[xui]]、[[3xui]]）被移出推荐列表。
- [[3xui]] 作者发起投票，有将近一半的用户（讲者原话）希望保留 [[http]] 访问。

### 解决方案一：[[ssh]] 隧道转发配置
- 在 [[vps]] 上安装面板并设置访问端口，例如 20000（讲者原话）。
- 不使用节点的情况下，利用本机的空闲端口（建议同为 20000，讲者原话）建立 [[ssh]] 隧道加密访问。
- 在 [[Windows]] 电脑命令行中输入包含 [[vps]] 用户名、IP 地址、默认 [[ssh]] 端口 22（讲者原话）的指令连接。
- 成功连接后，在浏览器通过 `localhost` 访问面板，数据由 [[ssh]] 加密转发，关闭连接后隧道断开。
- （进阶）可通过加入 `/i` 参数配合密钥文件免密登录，加入 `GN` 参数（讲者原话）可仅作端口转发而不进入 S 终端（推断指 Shell）。

### 解决方案二：免费 IP 证书申请与 [[tls]] 部署
- 考虑到不是所有人有域名，讲者演示申请 90 天（讲者原话）有效期的免费 IP 证书。
- 在指定网站注册账号（不需验证邮箱），输入 [[vps]] 的 IP 地址并下载验证文件。
- 在 [[vps]] 运行指令创建文件夹并启动临时 [[http]] 服务器（需占用 80 端口，讲者原话）进行验证。
- 验证成功后，通过 [[sfp]] 工具将 ca机构（讲者原话）颁发的证书文件及私钥（private.key）上传至 [[vps]]。
- 通过 `realpath` 指令获取完整路径，填入面板设置中并重启，即可强制通过 [[https]] 访问面板。

### 安全进阶：配置防扫描路径与局限性说明
- 仅配置 [[tls]] 无法防止公网扫描，必须在面板配置专属“路径”（输入一串字符即可）。
- 保存重启后直接访问会报 404 错误（讲者原话），必须在 URL 后方加入设置的路径才能进入面板。
- 局限性：免费 IP 证书无法通过 [[acm]]（推断指 ACME 协议）自动续签；单账号限额 3 个（讲者原话）免费证书，到期后需手动重做教程步骤。

## 步骤拆解
1. **安装并初始配置面板**
   - 具体操作描述：在 vps 上安装面板，并设置面板的访问端口。教程中以原版 `[[x-ui]]` 为例演示，将端口设置为 20000 [2]。
   - 关键代码/命令：未明确提及 [2]。
   - 注意事项：不可再直接通过 `[[http]]` 明文方式从浏览器访问该面板端口 [2]。

2. **建立 SSH 加密隧道**
   - 具体操作描述：在本地 `[[Windows]]` 电脑的命令行中执行端口转发指令，通过 vps 的 `[[ssh]]` 功能实现加密隧道访问。填入 vps 用户名、IP、`[[ssh]]` 端口（默认 22）、面板真实端口（如 20000）以及本地空闲端口，连接成功后通过访问 `localhost` 及本地端口进入面板 [1, 2]。
   - 关键代码/命令：如果使用密钥登录，在指令后加入 `/i` 和密钥文件路径；如果仅做端口转发不登录服务端，在后面加入 `GN`（字幕标注） [1]。
   - 注意事项：退出 vps 终端后，该 `[[ssh]]` 隧道即会断开连接 [1]。

3. **申请免费 IP 证书**
   - 具体操作描述：在提供免费 IP 证书的网站（网址未明确提及）注册账号，输入 vps 的 IP 地址，选择 90 天有效期方案并下载验证文件 [1]。
   - 关键代码/命令：未明确提及 [1]。
   - 注意事项：注册邮箱不需要验证，但建议填写自己的真实邮箱以便接收到期提醒 [1]。

4. **验证并部署证书**
   - 具体操作描述：在 vps 上创建对应文件夹，写入下载的验证文件内容。创建一个临时的 `[[http]]` 服务器进行验证，验证成功后下载证书并解压到桌面，随后通过 `[[sftp]]` 工具将证书上传到 vps [1, 3]。
   - 关键代码/命令：`realpath`（用于查看并复制证书所在的完整路径） [3]。
   - 注意事项：临时 `[[http]]` 服务器使用的 80 端口不能被其他应用程序占用 [3]。

5. **配置 HTTPS 与面板安全路径**
   - 具体操作描述：再次通过 `[[ssh]]` 隧道登录面板，在面板设置中填入刚刚上传的证书和私钥（同目录下后缀为 `.key` 的文件）的完整路径，保存并重启面板，此时便可通过公网 IP 直连访问 `[[https]]` 面板。为防扫描，进一步在设置中随意输入一串字符作为“路径”，保存重启 [3]。
   - 关键代码/命令：未明确提及 [3]。
   - 注意事项：配置路径并重启面板后，原网页会弹出 404 错误，此时必须将设置好的路径添加到 URL 尾部重新访问才能进入面板 [3]。

## 常见坑与解决方案
- **问题描述**：直接使用明文 `[[http]]` 访问面板进行配置，会导致中间人看到节点信息、密码和私钥等敏感数据 [4]。
  - 解决方案：通过建立 `[[ssh]]` 加密隧道进行访问，或为面板配置 `[[tls]]` 证书实现 `[[https]]` 加密访问 [1, 2, 4]。
- **问题描述**：在验证 IP 证书时，临时 `[[http]]` 服务器可能验证失败 [3]。
  - 解决方案：检查并确保 vps 的 80 端口没有被其他应用占用 [3]。
- **问题描述**：套了 `[[tls]]` 证书后只能保证数据加密，依然无法防止面板被别人扫描探测 [3]。
  - 解决方案：在面板的安全设置中额外配置一段访问路径（Path） [3]。
- **问题描述**：IP 证书无法通过 `[[acme]]`（字幕标注为 acm）自动申请更新 [5]。
  - 解决方案：90 天证书到期之后，必须按照教程步骤重新手动操作申请一遍 [5]。

## 最佳实践
- 推荐在建立 `[[ssh]]` 端口转发时，将本地的空闲端口设置成与 vps 端面板真实的端口一致（例如均为 20000），便于记忆和管理 [1, 2]。
- 推荐在拥有 `[[https]]` 加密的基础上，务必在面板中配置复杂的自定义访问路径，以彻底防范主动扫描探测 [3]。
- 反模式：绝对不要在没有开启分流代理（如节点直连/关闭代理工具）的情况下，使用 `[[http]]` 明文结合公网 IP 端口直连访问节点控制面板 [2]。

## 关键概念

### [[Xray]]
- **定义**：一个代理工具的核心组件或项目，其作者发布了针对面板安全的强制加密要求[1, 2]。
- **视频上下文**：Xray作者发布PR，强制要求Web面板必须且只能通过加密通道访问，否则将把未遵守规定的面板移出官方推荐列表[1, 2]。
- **为什么重要**：它是引发整个安全合规事件、引发长达一个月讨论以及后续技术调整的核心源头[1, 2]。
- **关联概念**：扩展 → [[Web面板]]

### [[Web面板]]
- **定义**：用于进行节点搭建和配置操作的后台可视化管理界面，例如x-ui和3x-ui[1, 2]。
- **视频上下文**：此前许多用户和教程默认通过明文HTTP访问Web面板，导致安全风险，未遵循加密规定的面板最终被移除了Xray官方推荐[1, 2]。
- **为什么重要**：它是用户进行代理节点配置的直接入口，其访问方式的安全性直接关系到节点信息是否会被泄露[1]。
- **关联概念**：依赖 → [[Xray]]

### [[HTTP]]
- **定义**：一种明文传输协议，通过该方式访问面板会导致所有操作数据对网络中间人可见[1]。
- **视频上下文**：视频指出，如果不使用加密节点，通过HTTP访问面板进行配置，会导致搭建的节点、密码和私钥等信息被中间人以明文形式获取[1, 3]。
- **为什么重要**：理解HTTP的明文特性是认识到面板被扫描和节点信息泄露等严重安全风险的关键[1, 2]。
- **关联概念**：对比 → [[HTTPS]]

### [[GFW]]
- **定义**：网络中间人，能够监控和看到通过明文传输的网络请求内容[1]。
- **视频上下文**：Xray作者警告，通过HTTP访问面板的操作会被中间人GFW看到，从而暴露节点信息[1]。
- **为什么重要**：它是节点信息泄露的主要防范对象，也是促使面板访问必须加密的核心原因[1]。
- **关联概念**：未明确提及

### [[SSH隧道转发]]
- **定义**：一种利用VPS的SSH端口建立的加密数据通道，用于安全地访问服务器上的本地服务[3]。
- **视频上下文**：在用户当前没有可信代理节点的情况下，作者演示了如何利用VPS的SSH端口转发功能，将本机的空闲端口映射到VPS的面版端口，实现对Web面板的加密隧道访问[3, 4]。
- **为什么重要**：它提供了一种在无需配置TLS证书的情况下，临时且安全地访问后台面板的有效技术手段，防止中间人窃听[3, 4]。
- **关联概念**：未明确提及

### [[HTTPS]]
- **定义**：一种加密网络访问方式，通过配置TLS证书来防止面板数据暴露在公网[4, 5]。
- **视频上下文**：为了避免每次登录都创建SSH隧道的麻烦，并解决面板默认按HTTP暴露的问题，作者教大家为面板配置证书以实现HTTPS访问，并建议配合自定义路径使用[4, 5]。
- **为什么重要**：它是解决Web面板明文传输安全漏洞、确保数据在公网传输安全的根本方法[4, 5]。
- **关联概念**：依赖 → [[IP证书]]

### [[IP证书]]
- **定义**：一种直接绑定VPS IP地址的TLS证书，无需购买域名即可申请并用于加密访问[4, 5]。
- **视频上下文**：考虑到并非所有人都有域名，作者演示了如何通过特定网站免费申请90天有效期的IP证书，并将其配置到面板中以开启HTTPS[4, 5]。
- **为什么重要**：极大降低了普通小白用户实现HTTPS加密访问的门槛和成本[4, 5]。
- **关联概念**：属于 → [[HTTPS]]
