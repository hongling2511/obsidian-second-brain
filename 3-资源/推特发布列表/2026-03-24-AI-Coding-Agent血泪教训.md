错了。我的经历恰恰相反。

Claude Code 一次重构，删了我的数据库 😱

这是我学到的5个血泪教训👇

---

1/ Git 不是可选项，是救命稻草

那天我没 commit 就让 agent 重构，20分钟后它把 migration 和生产数据一起删了。

坦白讲，我差点哭了。

就像开车必须系安全带，用 agent 必须先 commit。

铁律：动代码前 git commit，跑完后 git diff，不确定就开新分支。

本质上 agent 没有"后果意识"。

---

2/ Context Engineering 比 Prompt Engineering 重要100倍

大多数人卡在「不会写 prompt」，真正的问题是上下文太模糊。

我发现一个关键规律：

❌ 「帮我加个登录功能」→ 失败率80%
✅ 「这是 auth 模块代码，加 OAuth，保持接口不变」→ 成功率90%

说白了，agent 就像能力很强但不了解你项目的新同事。背景越清晰，输出越靠谱。

今天 awesome-claude-code 在 GitHub 涨了 429⭐

---

3/ 任务越小，agent 越可靠

❌ 「帮我重构整个项目」→ 我的惨痛经历
✅ 「把这个200行函数拆成5个，保持接口不变」→ 一次成功

核心策略：把大任务拆成10个小任务，每个都能在5分钟内验证。

我的经验：
✅ 单文件修改（成功率 >90%）
⚠️ 跨文件重构（~60%）
❌ 自主理解系统架构（不可能）

底层逻辑：agent 不是替代你思考，而是加速你的执行。

---

4/ 不 review 就 merge，是在给未来的自己挖坑

Simon Willison 引用的定义让我印象深刻：
「slop = 消费它花的时间 > 生产它花的时间」

我的判断：AI 本质上是放大器。
好的工程师 + AI = 超级工程师
坏的习惯 + AI = 超级灾难
