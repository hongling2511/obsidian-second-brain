## 一句话总结
由[[不良林]]详细解析比特币[[solo独立挖矿]]、[[pool矿池挖矿]]及[[solo矿池挖矿]]原理，并演示普通人如何零成本参与[[乐透彩票挖矿]] [1]。

## 目标受众与前置知识
- **适合谁看**：对加密货币挖矿深层原理感兴趣，或想尝试低成本/零成本参与比特币挖矿的普通用户（推断）。
- **前置知识**：[[比特币网络]]运行流程、基础[[挖矿原理]] [1]。

## 核心观点
1. **普通家用电脑进行主网[[solo独立挖矿]]已无实际意义**
   - 为什么重要：指明了当前挖矿的硬核现实，避免普通人盲目投入主网单机挖矿。
   - 依据：普通家用电脑算力太低，几乎不可能挖到，现在都是使用专业的[[ASIC]]（讲者原话） [2]。
2. **[[pool矿池挖矿]]门槛和风险较高，普通人不建议参与**
   - 为什么重要：解释了主流挖矿方式的现状及其不适合散户的原因。
   - 依据：目前以T的算力收益极低，500T算力的矿机回本周期达到500多天（屏幕展示/讲者原话），且中国大陆地区明令禁止挖矿（讲者原话） [3-5]。
3. **[[solo矿池挖矿]]结合了两者优点，适合博取小概率高收益**
   - 为什么重要：为普通用户提供了一种极低门槛参与比特币网络并可能获得高额回报的“彩票”方式。
   - 依据：无需运行[[全节点钱包]]下载600多G的区块数据，挖到区块后除2%手续费外独享98%奖励，曾有区块奖励高达26万美元（屏幕展示/讲者原话） [5, 6]。

## 概念关系
- [[全节点钱包]] → [[挖矿软件]]：前者向后者提供上一个区块的[[哈希]]、[[挖矿难度]]、[[内存池]]交易等数据，以便后者打包成[[区块头]]并计算[[随机数]] [1, 7]。
- [[solo独立挖矿]] ↔ [[pool矿池挖矿]]：前者独享收益但概率极低且需下载全量区块数据维护节点；后者按算力分配稳定收益，无需下全量数据，但收益低且需分成 [5]。
- [[pool矿池挖矿]] ↔ [[solo矿池挖矿]]：前者无论是否挖到区块都按算力分配奖励（提供[[份额]]即有收益）；后者不按算力分配奖励，只有自己真实挖到区块才能获得独享大额奖励 [5, 6]。

## 详细笔记
### 比特币挖矿流程回顾与测试网环境准备
- 复习挖矿流程：[[全节点钱包]]获取区块与交易信息，[[挖矿软件]]请求数据打包[[区块头]]，计算包含手续费的哈希，并修改[[随机数]]寻找符合条件的区快哈希，成功后广播至网络 [1, 7]。
- 讲者演示下载并安装[[比特币核心客户端]]，为避免主网80多万个区块同步过慢，在快捷方式属性中添加 `-testnet4` 使用[[测试网]]4环境进行演示（测试网4目前只有7万多个区块，10分钟左右可同步完成，讲者原话） [7]。
- 在客户端中创建钱包并新建收款地址用于接收挖矿奖励 [2]。

### CPU挖矿实操与Python截持演示
- 讲者下载早期CPU[[挖矿软件]]（提示因杀毒软件误报可在虚拟机操作），在目录下创建 `bitcoin.conf` 配置[[RPC接口]]的用户名、密码与端口，以便挖矿软件与钱包通信 [2, 8]。
- 运行终端指令，使用[[SHA256D]]算法计算哈希，开始挖矿后在[[任务管理器]]中可见CPU占用率达到100%（讲者电脑20个线程算力约160兆哈希每秒，讲者原话） [8]。
- 为加速演示，讲者编写[[Python]]脚本截持555端口通信，将钱包获取到的真实[[挖矿难度]]修改为8个0返回给挖矿软件，相当于计算了3亿多次即找到哈希值（讲者原话） [9]。
- 根据网络规则，脚本卡在20分钟后提交难度为1的区块，成功在测试网获得50多枚比特币（包含系统设定的手续费） [9, 10]。

### 区块链分叉现象演示
- 讲者展示由于测试网20分钟未出块即接受难度为1的规则，导致多个矿工（或脚本）卡在同一时间提交区块，产生频繁的区块链[[分叉]] [11, 12]。
- 遵循最长链原则，当某节点以更高算力（无需等待20分钟）挖出新区块时，其所在分支成为主链，讲者之前挖出的由于分叉形成的短链区块全部作废（丢弃） [11, 12]。

### Pool矿池挖矿原理与限制
- 矿池获取钱包数据后创建注币交易（收款地址为矿池），给接入矿工下发较低难度的任务（如实际需要15个0，只要求8个0） [13]。
- 矿工不断提交符合低难度的[[份额]]，矿池据此统计矿工的[[算力]] [3]。
- 当某矿工幸运挖出符合真实高难度（15个0）的区块时，矿池广播区块获取奖励，扣除手续费后按所有人算力分配 [3]。
- 讲者以某鱼池为例（推断为“鱼池/F2Pool”）演示后台界面，将通信地址修改为矿池地址，使用[[Sent]]协议（讲者明确提及）连接，但因CPU算力太低无法提交份额，无任何收益 [3, 4]。

### Solo矿池挖矿与低成本“乐透”方案
- [[solo矿池挖矿]]无需下载600G区块，直接连接矿池，挖到后全额保留收益（仅扣除矿池手续费，如[[CK矿池]]收2%） [5]。
- 讲者演示使用开源的[[SPV客户端]]替代全节点钱包，配置挖矿软件连接CK矿池（其下发难度设置为10000仅作统计用）进行“乐透”挖矿 [6, 14]。
- 推荐小算力用户连接设定难度仅为0.1的[[public]]矿池，以便在网页端直观查看算力统计工作是否正常 [14]。
- 介绍多种低成本设备方案：开源硬件[[Btex Gam]]（20瓦功耗1T算力，价格1000左右，讲者原话）、针对[[ESP32]]开发板的[[net miner]]系统（类似给路由器刷[[OpenWrt]]，几十K算力，价格100左右，讲者原话） [14, 15]。
- 零成本方案：利用24小时开机的电脑后台运行[[CPU minor]]、闲置手机或自建有节点的[[VPS]]（可限制10% CPU占用率），以极大概率不中但无限期参与的形态进行“彩票开奖” [15, 16]。

## 金句亮点
- “八兆算力和8K算力从挖到的概率来讲没啥区别，都是无限趋近于零，能不能中主要是看你命里有没有。” — 观点 / 真实道破了小算力乐透挖矿的本质概率极低 [16]。
- “说白了就是在算力有限的情况下博一把大的，要么下海干活，要么会所嫩模。” — 观点 / 形象概括了Solo矿池挖矿以极小投入博取极端收益的投机属性 [6]。

- **关联观点**：对应核心观点 3
- **验证标准**：通过服务器系统监视器查看到挖矿进程的CPU占用稳定在10%左右，并在矿池网页端输入自己的钱包地址后能查看到算力统计数据。

## 步骤拆解

1. **本地节点同步与配置（以测试网为例）**
   - 具体操作描述：下载安装 [[Bitcoin Core]] 客户端，修改快捷方式以接入测试网 4，启动并等待区块同步。创建钱包及收款地址，然后在当前目录创建 `bitcoin.conf` 配置文件，填入 RPC 通信所需的用户名、密码及端口，重启客户端以开启通信接口 [1, 4, 5]。
   - 关键代码/命令：在快捷方式目标属性后加上 `testnet4` [1]。
   - 注意事项：由于主网已有 80 多万个区块同步极慢，教学演示推荐使用测试网 4（目前仅 7 万多区块，约十分钟同步完成） [1]。

2. **CPU 挖矿软件配置与运行**
   - 具体操作描述：下载早期的 CPU 挖矿软件，通过终端执行带有 RPC 接口信息及钱包地址的挖矿指令。挖矿软件会从 [[Bitcoin Core]] 提取数据并开始哈希计算 [4, 5]。
   - 关键代码/命令：添加 `-t [数字]` 参数可指定运行线程数 [5]。
   - 注意事项：挖矿软件常被杀毒软件误报或捆绑木马，建议在虚拟机中操作；挖矿极度消耗资源，会导致 CPU 满载并大幅增加耗电 [4, 5]。

3. **测试网难度劫持挖矿（可选进阶）**
   - 具体操作描述：利用测试网“20 分钟无人挖出区块则难度重置为 1”的规则，运行 [[Python]] 脚本劫持挖矿软件与钱包之间的通信，将获取的难度修改为 8 个 0 并返回给挖矿软件，待上一个区块产出 20 分钟后再提交给网络 [3, 6]。
   - 关键代码/命令：`python [脚本名称]`（截持通信端口为 5555） [3]。
   - 注意事项：即使成功提交，也可能遇到区块链分叉问题，如果邻居节点以正常难度挖出区块，难度为 1 的测试网区块及奖励将会作废 [7, 8]。

4. **接入矿池 / Solo 矿池挖矿**
   - 具体操作描述：无需下载 600G 的全节点数据。下载并安装轻量级的 [[SPV]] 客户端，创建钱包并保存秘语种子。将挖矿指令中的通信地址修改为矿池协议（如 [[CKPool]]）地址，填入钱包地址即可直接开始挖矿 [9-11]。
   - 关键代码/命令：未明确提及完整命令字符串。
   - 注意事项：Solo 矿池如果挖到区块，奖励全归个人，但矿池会抽取 2% 左右手续费（普通矿池无论是否挖出区块都会按算力分配微薄收益，Solo 矿池没挖到则为零收益） [9]。

5. **VPS 闲置算力挖矿（零成本乐透）**
   - 具体操作描述：利用国外的 Linux [[VPS]] 全天候挂机运行挖矿程序。下载挖矿软件并解压，在后台启动程序并严格限制 CPU 线程与占用率，防止被服务商封禁 [2, 12]。
   - 关键代码/命令：
     - 查看 CPU 信息：`lscpu` [2]
     - 下载挖矿软件：使用 `wget` 指令（视频发音类似 G 指令） [2]
     - 解压软件：`tar` [2]
     - 后台单线程运行：在运行指令后加 `-t 1` 和 `-B` [2]
     - 查看后台进程：`htop`（视频发音类似 edop / HP） [2, 13]
     - 安装 CPU 限制工具：`apt install cpulimit` [2]
     - 限制占用率：使用 [[cpulimit]] 相关指令限制进程 CPU 占用率为 10% [13]。
   - 注意事项：大部分 [[VPS]] 是 X86-64 架构，需选择对应版本的挖矿软件；务必限制 CPU 使用率，否则会导致 [[VPS]] 被封 [2]。

## 常见坑与解决方案
- **问题描述**：下载的挖矿软件常被浏览器或杀毒软件拦截，提示有木马风险 [4]。
  - 解决方案：下载时选择“保留”，如果不放心，建议在虚拟机中进行操作 [4]。
- **问题描述**：个人电脑算力太低，独立挖矿或连接常规矿池难度极高，难以产生有效份额和收益 [6, 14]。
  - 解决方案：连接专门对小算力友好的开源 Solo 矿池（如 public pool，难度设定仅为 0.1），不仅能测试算力工作是否正常，还能以零成本“买彩票”碰运气 [11]。
- **问题描述**：在家中使用闲置设备挖矿时，可能遇到网络代理问题或部分网络节点屏蔽了挖矿流量 [12]。
  - 解决方案：使用不屏蔽挖矿流量的机场代理，或直接将程序运行在国外的独立 [[VPS]] 上以保证网络和供电的稳定 [2, 12]。
- **问题描述**：在 [[VPS]] 上运行挖矿程序导致 CPU 长期满载，面临账号或机器被服务商封禁的风险 [2]。
  - 解决方案：启动时使用 `-t 1` 参数只开启单线程，并使用 [[cpulimit]] 工具将 CPU 占用率强制限制在 10% [2, 13]。

## 最佳实践
- **推荐做法**：如需独立挖矿但不想下载 600G 全节点数据，推荐使用 Solo 矿池（如 [[CKPool]]）结合轻量化 [[SPV]] 客户端的模式 [9]。
- **推荐做法**：测试本地机器算力配置是否正常联通时，推荐使用开源且难度设置极低（难度 0.1）的 public pool 矿池，能在网页端快速查看到算力统计 [11]。
- **反模式**：极不建议个人购买专业 [[ASIC]] 矿机参与普通矿池挖比特币。当前算力难度下回本周期极长（视频中计算需 500 多天），不仅可能赚不回电费，且在中国大陆地区属于明令禁止行为，承担政策与资金双重风险 [9, 14, 15]。

## 关键概念

### [[Solo 独立挖矿]]
- **定义**：矿工自己单独运行全节点钱包软件下载所有区块数据并进行挖矿，挖到区块后独享所有奖励的挖矿方式 [1, 2]。
- **视频上下文**：作者指出早期比特币主要靠这种方式挖矿，但现在全网算力极高，普通人以此方式挖矿难度太大，且需要维护网络和下载超过600G的区块数据 [1, 2]。
- **为什么重要**：这是理解比特币挖矿的最基础模式，也是理解其他衍生挖矿模式的起点。
- **关联概念**：对比 → [[矿池挖矿]]

### [[矿池挖矿]]
- **定义**：多个矿工将算力接入同一个矿池，矿池统一下发挖矿任务，挖到区块后按矿工提交的算力比例分配奖励的方式 [1, 3]。
- **视频上下文**：作者解释这是目前主流的挖矿方式，矿工不需要运行全节点，只需连接矿池并持续提交份额，矿池会收取一定手续费并分配收益 [2, 3]。
- **为什么重要**：它解决了个人算力难以独立挖出区块的问题，是目前绝大多数比特币的产出方式。
- **关联概念**：依赖 → [[份额]]

### [[Solo 矿池挖矿]]
- **定义**：矿工无需运行全节点，直接连接特定矿池进行挖矿，挖到区块后扣除少量矿池手续费，独享剩余全部奖励的模式 [2]。
- **视频上下文**：作者以 CK 矿池为例，解释这种方式结合了矿池的便捷性和独立挖矿的高收益机制，适合算力有限但想零成本参与“乐透彩票挖矿”博取高收益的用户 [2, 4]。
- **为什么重要**：它极大降低了独立挖矿的设备门槛和存储成本。
- **关联概念**：扩展 → [[Solo 独立挖矿]]

### [[挖矿难度]]
- **定义**：比特币网络规定的一个目标值（Target），矿工计算出的区块哈希值必须小于这个目标值才算挖矿成功 [5, 6]。

### [[随机数]]
- **定义**：区块头中的一个数值，矿工在挖矿过程中需要不断修改它以计算出不同的区块哈希值 [5, 8]。

- **为什么重要**：挖矿本质上就是不断试错寻找正确随机数的工作量证明（PoW）过程。

### [[份额]]
- **定义**：矿工在矿池挖矿中，计算出的符合较低“矿池难度”的区块数据，用于向矿池证明自己的工作量 [3]。
