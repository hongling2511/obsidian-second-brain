## 一句话总结
利用 [[Windows]] 系统配置 [[旁路网关]] 与 [[路由转发]]，实现局域网内全家设备无客户端 [[科学上网]]。

## 目标受众与前置知识
- **适合谁看**：拥有单网口 [[Windows]] 电脑，且需要为 [[m的vr设备]]（如 [[qu2]]）、海外版手机、电视盒子、游戏机等无法运行代理工具的设备提供联网激活或代理环境的用户。
- **前置知识**：[[局域网]]、[[网关共享]]、[[代理共享]]、[[路由共享]]、[[旁路由]]、[[IP]]、[[子网掩码]]、[[网关]]、[[DNS]]、[[DHCP]]。

## 核心观点
1. **[[Windows]] 可直接作为 [[旁路网关]] 实现全家设备科学上网**
   - 为什么重要：解决了传统上 [[Windows]] 无法实现网关共享，必须通过安装 [[Linux]] 虚拟机（虽然比较鸡肋）才能实现的痛点。
   - 依据：讲者原话指出“今天我们介绍的方法不需要安装 [[Linux]] 虚拟机，直接使用你的 [[Windows]] 系统实现网关级的共享”。
2. **作为旁路网关，[[v2n]] 比 [[clash]] 更值得推荐**
   - 为什么重要：使用 [[clash]] 的规则模式容易因 IP 判定问题导致 [[路由环路]] 甚至软件卡死。
   - 依据：讲者原话表示“在 [[v2n]] 中 [[ton模式]] 的默认配置不会导致路由环路”，因为它通过调用 [[sbox]] 内核实现了该模式。
3. **[[SS Tab]] (ssab) 是配置最便捷的备选方案，但性能折损明显**
   - 为什么重要：无需配置复杂的额外路由表，适合怕麻烦的小白用户，但需权衡网络质量。
   - 依据：讲者原话表明 [[SS Tab]] “速度衰减的也比较厉害”，且由于不进行 DNS 劫持，设置国外 DNS “会造成 [[CDN]] 舍近求远的情况”。

## 概念关系
- [[代理共享]] ↔ [[网关共享]]：[[clash]] 或 [[v2r]] 自带的局域网共享属于前者，而本教程实现的类似 [[旁路由]] 的全家网络接管属于后者。
- [[Windows]] 电脑开启 [[路由转发]] → 成为 [[旁路网关]]：这是使 Windows 接管局域网其他设备流量的必要前置步骤。
- 规则模式下域名 IP 非国内 → [[路由环路]]：在基于 IP 分流的 [[clash]] 中，此情况会导致流量死循环并让软件卡死。
- [[SS Tab]] (ssab) → [[clash]] / [[v2r]] ([[v2a]])：因前者只支持 SS 节点，必须依赖后者开启的 socks 代理端口进行连接。

## 详细笔记
### 背景介绍与核心痛点
- 痛点：海淘电子产品（如 [[m的vr设备]]、[[qu2]]、海外版手机）需要 [[科学上网]] 环境激活，但用户只有一台能科学上网的 [[Windows]] 电脑。
- 历史方案对比：此前视频讲解过代理共享、网关共享、路由共享。[[Windows]] 无法直接实现网关共享，传统只能装 [[Linux]] 虚拟机。本期通过直接配置 [[Windows]] 弥补此遗憾。

### Windows 前置网络设置与工具准备
- 工具：[[不良林]] 制作的带有精美 UI 的小工具，用于降低配置难度。
- 设置电脑静态 IP：检测物理网卡，将电脑 IP 固定为同网段（讲者原话演示为 192.168.0.140，默认网关 192.168.0.1）。
- 开启 [[路由转发]]：在小工具中执行开启，必要时重启。可通过命令行输入 `IP configureip 路由以启用`（讲者原话），显示“是”即为成功。

### 代理工具配置一：[[clash]]
- DNS 路由设置：将默认的三个 DNS 服务器及 [[fbackdns]] 填入小工具的额外路由条目（或删掉填入局域网默认网关）。
- 节点 IP 填入：将节点服务器的 IP 或域名填入额外路由（中转机场重复 IP 填一个即可，直连机场需全填）。
- 启动：动作选“添加路由”并执行，最后开启 [[ton模式]]。
- 缺陷：通过 IP 分流不完善，规则模式下易造成 [[路由环路]]，建议使用全局模式。关闭时需先关 [[ton模式]]，改动作“删除路由”并执行，最后关闭 [[路由转发]]。

### 其他设备网关修改方式
- 方式一：单台手动修改（适用于手机、[[Windows]]、[[Linux]] 的 [[cali]] 系统）。
  - 将 [[DHCP]] 改为手动/静态。
  - 网关填写为 Windows 电脑 IP（讲者原话：192.168.0.140）。
  - DNS 设为公网 IP（讲者原话：1.1.1.1），可正常访问 [[谷歌]] 且 IP 跟随节点变化。
- 方式二：修改路由器 DHCP 设置（全局生效）。
  - 登录路由器，将 [[DHCP]] 下方的网关设为电脑 IP，DNS 设为公网 IP，全家设备免设置自动联网。
  - 特例：部分路由器（如部分 [[小 米]] 路由器）不支持修改 DHCP 网关，需关闭路由器 DHCP 服务并转移至其他设备（推断）。

### 代理工具配置二：[[v2n]]（推荐方案）
- 原理：调用 [[sbox]] 内核实现 [[ton模式]]，默认配置不会导致 [[路由环路]]。
- 步骤：将不在列表里的节点加入额外路由 → 执行添加路由 → 开启 [[路由转发]] → [[v2n]] 路由改全局 → 启动 [[ton模式]]。此时即使软件设全局，系统已通过 IP 路由实现分流。

### 代理工具配置三：[[SS Tab]] (ssab)（最简但有损方案）
- 步骤：仅需开启 [[路由转发]]，不需小工具配路由。
- 桥接：添加 socks 代理服务器，IP 填 127.0.0.1（讲者原话），端口填 [[clash]] 的 7890 或 [[v2r]] ([[v2a]]) 的 10808（讲者原话）。代理软件不需开 [[ton模式]]。
- 缺陷：不劫持 DNS，易受 DNS 污染；设置公网 DNS 会导致 [[CDN]] 舍近求远；速度衰减厉害。

### 总结与开源互动
- 讲者放弃了像 [[Linux]] 那样仅开路由转发和 [[ton模式]] 就能完美免路由表共享的设想，因此放出小工具源码，呼吁观众提供更完美的实现方式。

## 金句亮点
- “只要你的电脑能科学上网，你家里的手机、电视盒子、游戏机等网络设备无需运行任何代理工具，即可自带科学上网。” — 价值
- “用 Windows 当旁路网关，我更推荐使用 v2n。v2n 是通过调用 sbox 内核实现 ton 模式，在 v2n 中 ton 模式的默认配置不会导致路由环路。” — 技术结论

  - 关联观点：1
  - 验证标准：在命令行输入指令检查，能够看到“路由以启用”显示为“是”。
- **修改局域网设备的默认网关**：进入手机或 [[cali]] 系统的网络设置，将 DHCP 设为静态，把网关修改为刚刚固定的 Windows 电脑 IP。
  - 关联观点：1
  - 验证标准：设备重连网络后，能在未装代理软件的情况下正常访问外部网络，且查询 IP 归属地与电脑端节点一致。
- **配置 [[v2n]] 的旁路路由规则**：使用 6.0 以上版本 [[v2n]]，将节点信息加入额外路由表执行添加，随后将 [[v2n]] 自身路由改为全局，最后启动 [[ton模式]]。
  - 关联观点：2
  - 验证标准：电脑及局域网内其他设备均可正常联网分流，且不会出现类似 clash 规则模式下的路由环路与网络卡死现象。

- **环境要求**：

## 步骤拆解
1. **固定 Windows IP 并开启路由转发**
   - 具体操作描述：打开路由转发表工具，在网卡接口选择当前物理网卡。进入系统的网卡 IP 设置页面，将电脑 IP 固定为同网段的静态 IP（例如 192.168.0.140），子网掩码按默认，默认网关和 [[DNS]] 均填写原局域网网关（如 192.168.0.1）[6]。保存后在工具中点击“开启路由转发”，若提示重启则重启电脑 [6]。
   - 关键代码/命令：`IP configureip 路由以启用` [6]。
   - 注意事项：如果电脑有多张网卡，务必在工具中选择正确的网卡；局域网网关 IP 一般充当了 [[DNS]] 服务器，无需填入额外路由中 [2, 6]。

2. **配置代理工具（以 [[clash]] 为例）**
   - 具体操作描述：以管理员权限运行 [[clash]] 并进入 Tun 设置。将 [[clash]] 默认的 [[DNS]] 服务器 IP，以及所使用节点服务器的地址（域名或 IP）复制填入辅助工具的“额外路由条目”中（中转节点重复的 IP 填一个即可，直连节点需全填）[2, 6]。确保工具中的路由动作是“添加路由”并点击执行，最后打开 [[clash]] 的 Tun 模式 [2]。
   - 关键代码/命令：未明确提及。
   - 注意事项：如果自定义设置了 fallback DNS，也需要一并填入额外路由条目中 [2]。

3. **客户端设备网关修改**
   - 具体操作描述：
     - 单台设备手动修改：进入 [[Windows]]、[[Linux]] 或手机的网卡/WiFi 设置，将 [[DHCP]] 改为静态/手动，IP 设为同网段，网关修改为开启了路由转发的 [[Windows]] 电脑 IP（如 192.168.0.140），[[DNS]] 设为公网 IP（如 1.1.1.1）[2, 3]。
     - 路由器全局修改：在浏览器登录路由器进入 [[DHCP]] 设置，将 [[DHCP]] 下方的网关设为 [[Windows]] 电脑的 IP，并设置一个公网 [[DNS]] 服务器 [3]。
   - 关键代码/命令：未明确提及。
   - 注意事项：手机设备切换节点后，需要先断开 WiFi 连接再重新连接以刷新 IP 查询结果 [3]。

4. **停用路由转发与恢复网络**
   - 具体操作描述：先关闭代理工具（如 [[clash]] 或 [[v2n]]）的 Tun 模式 [4, 5]。将辅助工具的路由动作改为“删除路由”并点击执行，然后关闭“路由转发”功能 [4, 5]。
   - 关键代码/命令：未明确提及。
   - 注意事项：关闭路由转发和删除路由没有先后顺序；路由转发不用时一定要关闭，并将客户端设备的网关重新改回原路由器的 IP，否则客户端会无法上网 [4, 5]。

5. **使用 [[v2n]] 替代实现（推荐方案）**
   - 具体操作描述：以管理员权限运行 [[v2n]]，将使用的节点 IP 填入额外路由条目并执行“添加路由”[4, 5]。打开路由转发，将 [[v2n]] 的路由模式改为“全局”，最后启动 Tun 模式 [5]。
   - 关键代码/命令：未明确提及。
   - 注意事项：[[v2n]] 是通过调用 [[sbox]] 内核实现 Tun 模式的 [4]。

6. **使用 [[SSTap]] 替代实现**
   - 具体操作描述：开启辅助工具的“路由转发”（无需配置额外路由条目）[5]。在 [[SSTap]] 中添加 SOCKS 代理服务器，IP 填写 127.0.0.1，端口填写 [[clash]] 或 [[v2ray]] 开启的监听端口。在 [[SSTap]] 界面选择需要的节点或 IP 分流规则后点击连接 [5, 7]。
   - 关键代码/命令：未明确提及（仅提及本地 IP 为 `127.0.0.1`，[[clash]] 默认端口为 `7890`，[[v2ray]] 默认端口为 `10808`）[5, 7]。
   - 注意事项：此方法下 [[clash]] 和 [[v2ray]] 都不需要开启 Tun 模式 [7]。

## 常见坑与解决方案
- **问题描述**：在 [[clash]] 的规则模式下，如果域名的分流规则设置为直连，且该域名的 IP 不是国内 IP，会导致路由环路（死循环）从而使 [[clash]] 卡死 [4]。
  - 解决方案：将域名手动添加到额外路由条目中；或将 [[clash]] 切换为全局模式；或直接改用 [[v2n]] 代替 [4]。
- **问题描述**：部分路由器（如小米的部分型号）不支持直接修改 [[DHCP]] 服务的网关 [3]。
  - 解决方案：只能通过手动修改单台客户端设备的网关，或者关闭路由器的 [[DHCP]] 服务，将 [[DHCP]] 转移到局域网内的其他设备上 [3, 4]。
- **问题描述**：使用 [[SSTap]] 方案时，由于软件不会进行 [[DNS]] 劫持，如果填国外 [[DNS]]（如 1.1.1.1）会造成国内 CDN 舍近求远，且该工具的速度衰减较为严重 [7]。
  - 解决方案：未明确提及（视频仅提示按自身使用需求在几种方案中权衡选择）[7]。
- **问题描述**：忘记关闭路由转发或未恢复设备网关配置，导致局域网其他设备断网 [4]。
  - 解决方案：停用时必须关闭路由转发，并把目标设备的网关改回原本的路器 IP [4]。

## 最佳实践
- 强烈推荐使用 [[v2n]] 充当旁路网关，因为其 Tun 模式的默认配置天然不会导致路由环路 [4]。
- 若使用 [[clash]]，为了防止产生路由环路，建议直接使用“全局模式”而非规则模式 [4]。
- 修改客户端（如手机、电脑）单台设备的静态网关时，建议将 [[DNS]] 配置为一个公网 IP（如 1.1.1.1），让 [[clash]] 等工具默认进行 [[DNS]] 劫持以正常解析 [3]。
- 辅助配置小工具在“路由转发”等设置执行完成后可以随时关闭，不需要常驻后台运行 [5]。

## 关键概念

### [[网关共享]]
- **定义**：一种使局域网内其他设备无需运行任何代理工具即可实现科学上网的网络共享模式 [1]。
- **视频上下文**：作者指出Windows系统原本无法直接实现此模式，通常需借助Linux虚拟机，但本期视频旨在教大家直接利用Windows系统实现网关级的共享 [1]。
- **为什么重要**：它是解决海淘设备（如电视盒子、游戏机等）因无科学上网环境而无法激活连网问题的核心机制 [1]。
- **关联概念**：对比 → [[代理共享]]

### [[代理共享]]
- **定义**：通过代理工具（如Clash或v2ray）自带的“共享局网”功能来实现网络代理的一种模式 [1]。
- **视频上下文**：作者在解答常见疑问时提到，很多人认为Clash自带局域网共享功能，但这属于“代理共享”，区别于本期重点讲解的网关级共享 [1]。
- **为什么重要**：区分代理共享与网关共享，有助于明确不同网络架构的实现方式和局限性 [1]。
- **关联概念**：对比 → [[网关共享]]

### [[旁路网关]]
- **定义**：在局域网中承担特定流量处理（如代理转发）功能，被其他设备作为网关指向的设备 [1, 2]。
- **视频上下文**：视频的核心操作就是通过配置路由转发和特定代理模式，将一台普通的单网口Windows电脑转变为一台旁路网关（即旁路由） [1, 2]。
- **为什么重要**：它允许用户在不改变主路由器物理结构的情况下，利用现有Windows电脑代替专门的软路由实现全家科学上网 [1, 2]。
- **关联概念**：依赖 → [[路由转发]]

### [[路由转发]]
- **定义**：操作系统中允许设备接收并转发发往其他IP地址数据包的一项网络功能 [3, 4]。
- **视频上下文**：作者使用自制工具协助配置的第一步就是开启Windows的路由转发，并在停用旁路网关时强调必须将其关闭，否则其他设备会断网 [3, 4]。
- **为什么重要**：是普通的Windows电脑能够充当网关并处理局域网内其他设备网络流量的必要前提条件 [3, 4]。
- **关联概念**：属于 → [[旁路网关]]

### [[默认网关]]
- **定义**：局域网设备向外发送数据包时默认指向的出口IP地址 [3, 5]。
- **视频上下文**：要让局域网内的手机或Kali Linux系统通过Windows电脑科学上网，需要将这些设备的网络设置中的“默认网关”修改为该Windows电脑的IP地址 [5]。
