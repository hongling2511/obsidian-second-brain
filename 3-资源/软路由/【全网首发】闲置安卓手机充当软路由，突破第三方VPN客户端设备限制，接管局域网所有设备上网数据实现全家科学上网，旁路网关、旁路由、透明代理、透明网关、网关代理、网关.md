## 一句话总结
利用闲置且已获取 [[root]] 权限的 [[Android]] 手机充当 [[旁路网关]]，实现局域网内所有设备免客户端科学上网的完整教程。

## 目标受众与前置知识
- **适合谁看**：有闲置安卓手机、希望实现全屋设备科学上网以降低 [[软路由]] 学习与硬件成本的用户。
- **前置知识**：[[软路由]]、[[旁路网关]]、[[IP地址]]、[[静态IP]]、[[DHCP]]、[[root]]、[[ADB]]、[[VPN]]、[[DNS]]、[[局域网]]。

## 核心观点
1. **闲置安卓手机可完美替代软路由充当旁路网关**
   - 为什么重要：节省了购买专用软路由的资金，且降低了普通用户的学习门槛。
   - 依据：[[Android]] 与 [[OpenWRT]] 一样基于 [[Linux]] 具有作为软路由的可行性（讲者原话），且手机端拥有如 [[Clash]]、自带节点的第三方 [[VPN]] 加速器等丰富的客户端（讲者原话）。
2. **旁路网关模式能极大减轻手机的性能负担**
   - 为什么重要：手机并非专门的路由器，存在性能限制或省电机制，直接作为主路由可能存在瓶颈。
   - 依据：内网设备之间的数据传输不需经过手机，只有上外网的数据才需发给手机进行路由转发，实测能跑满本地“百兆带宽”（讲者原话）。
3. **有线网络接入能显著提升旁路网关的速度与稳定性**
   - 为什么重要：[[WiFi]] 连接易受干扰且速度存在瓶颈。
   - 依据：使用 [[WiFi]] 连接测速“只跑到了20兆”（讲者原话），而改用网线连接后，测速基本跑满“100兆的带宽”（讲者原话）。

## 概念关系
- [[Android]] → [[软路由]]：安卓底层基于 [[Linux]]，这使得在安卓系统上运行路由转发脚本、充当软路由成为可能。
- [[旁路网关]] ↔ [[传统路由器]]：传统主路由负责内外网所有数据交换，而旁路网关仅接管局域网内其他设备上外网的数据转发请求，对主路由进行补充。
- [[DHCP]] → [[透明代理]]：通过修改主路由的 DHCP 设置将默认网关指向手机 IP，局域网内的设备即可在不知情（透明）的情况下，通过手机的 [[VPN]] 实现科学上网。

## 详细笔记
### 方案可行性与网络环境介绍
- **原理解析**：介绍了 [[旁路网关]]（等同于 [[透明网关]]、[[透明代理]]、[[网关代理]]、[[旁路由]]）的概念。局域网设备访问外网需通过网关（网络出口）。
- **设备展示**：讲者 [[不良林]] 展示其网络拓扑，主路由为2009年产的 [[DIR600M]] 和2010年产的 [[MW3002]]，使用“19.9元包邮长度30米的垃圾网线”（讲者原话）。演示用闲置手机为刷了 [[魔趣系统]] 的 [[坚果 Pro]]（隶属 [[锤子科技]]（推断）），并表示该配置“跑个500兆应该是没有问题的”（讲者原话）。

### 手机端准备工作与环境配置
- **基础前提**：手机必须获取 [[root]] 权限并建议刷原生系统。手机需能正常科学上网（演示访问 [[Google]]/[[谷歌]]），并能在系统设置中将自身的 [[IP地址]] 改为 [[静态IP]]（演示设为“192.168.0.113”，讲者原话）。
- **命令行工具**：
  - 电脑端：使用 [[Windows]] 版的 [[ADB]] 工具。手机需连按版本号“七次”（讲者原话）开启开发者选项，并打开 root 授权、USB调试或网络ADB调试。通过 [[CMD]] 运行 `adb devices` 和 `adb shell` 连接。
  - 手机端：直接使用 [[Termux]]（视频字幕称 thmx/ermix）终端，输入 `su` 切换至 root 用户。

### 旁路网关脚本配置与运行
- **脚本部署**：在命令行输入指令创建 `proxy` 文件夹，使用 [[Vim]]（视频字幕称 VI器）或文件管理器创建 `proxy.sh` 脚本并粘贴讲者提供的代码。
- **权限与启动**：赋予脚本可执行权限，并将其设置为后台运行。此时手机正式成为旁路网关。

### 局域网设备网关接管（全局与单设备）
- **全局代理（通过主路由）**：登录默认网关为“192.168.0.1”（讲者原话）的主路由控制台，修改 [[DHCP]] 设置，将下发网关改为手机的 IP。局域网设备禁用并重新启用 [[网卡]] 后，即可实现免客户端翻墙。
- **注意事项**：在手机上切换不同的 VPN 客户端时（如先用带节点的 [[快连VPN]]，后换为 [[v2rayNG]]/VRNG 或 [[Clash]]），需先断开原有连接再开启新服务。否则虚拟网卡接口名称会变，导致脚本路由规则失效。若访问部分网站异常，大概率是 [[DNS污染]]，需修改 [[DNS]]。

### 进阶优化：网线直连接入与单设备代理
- **网线直连**：由于 [[WiFi]] 存在速度瓶颈，讲者建议使用带充电口的 [[Type-C]]（视频字幕为 TPEC）转网卡转换器，通过网线将手机连入主路由，实现“7乘以24小时长期稳定使用”（讲者原话）。
- **配置修改**：通过 `ip a` 命令查看到网线接口为 `eth0`，分配的新 IP 为“192.168.0.11”（讲者原话）。需关闭原有脚本，修改 `proxy.sh` 脚本中的接口名称为 `eth0` 后重启。建议在路由器端将接口的 [[Mac地址]]（视频字幕为 MD地址）与 IP 绑定。
- **单设备代理**：若不希望全家设备都翻墙，可不修改路由器的 DHCP，而是直接在需要翻墙的电脑上，手动将其网络适配器的默认网关设为手机的 IP。

### 维护事项与备选方案
- **日常维护**：手机重启后网络设置重置，需重新运行脚本。必须关闭手机一切省电选项并清理非必要后台，以保证网络稳定。
- **备选极客方案**：直接使用 [[Linux]] 内核加载 [[Clash]] 内核实现旁路网关，可通过 Web UI 在局域网其他设备上切节点，但无法使用第三方 VPN 客户端。

## 步骤拆解
1. **获取 Root 权限与调试配置**
   - 将手机 root，在“开发者选项”中开启 root 授权与安卓调试。若不使用 USB 直接连接，需开启网络 [[ADB]] 调试 [3]。
   - 关键代码/命令：未明确提及 [3]。
   - 注意事项：各手机厂商开启开发者选项及 root 的方式不同，需自行寻找教程 [3]。

2. **进入命令行环境**
   - 电脑端使用 [[ADB]] 连接手机，或在手机端安装并打开终端工具 [[Termux]]，切换至 root 用户 [3, 4]。
   - 关键代码/命令：
     - `adb devices`（查看已连接的设备）[3]
     - `adb shell`（进入手机命令行）[4]
     - `adb connect <IP地址>`（通过网络连接 ADB）[8]
     - `su`（在 [[Termux]] 中切换 root 用户）[4]
   - 注意事项：命令行提示符需变为 `#` 号，代表已成功获取 root 权限 [4]。

3. **配置固定 IP 地址**
   - 在手机的 WiFi 高级设置中，将 [[DHCP]] 改为静态，并手动设置与自动分配在同网段的固定 IP。或直接在主路由器上将手机的 MAC 地址与 IP 绑定 [5, 7]。
   - 关键代码/命令：未明确提及 [5]。
   - 注意事项：必须确保手机 IP 固定不变，否则作为网关使用时会导致网络中断 [5]。

4. **开启科学上网工具**
   - 运行 [[Clash]] 或其他 [[VPN]] 工具，开启服务并测试手机本地是否能正常访问受限网站 [4]。
   - 关键代码/命令：未明确提及 [4]。
   - 注意事项：未明确提及 [4]。

5. **配置并运行旁路网关脚本**
   - 在终端创建指定目录与脚本文件 `proxy.sh`，粘贴转发规则代码，赋予可执行权限后使其在后台运行 [5]。若使用网线接入，需将脚本中的网卡接口改为对应名称（如 `eth0`）[7]。
   - 关键代码/命令：查看各网络接口设备信息的命令（视频原音读作 "IP 格 A"，推测为 `ip a`）[8]。其他创建目录、后台运行的具体指令“未明确提及” [5]。

6. **修改局域网设备的网关**
   - 方式一：修改主路由器的 [[DHCP]] 设置，将默认网关指向手机的固定 IP，局域网所有设备重连后即可翻墙。方式二：仅在需要翻墙的单台设备（如电脑）上手动配置静态 IP，并将网关设为手机 IP [7, 9]。

## 常见坑与解决方案
- **问题描述**：切换不同的 [[VPN]] 客户端时，虚拟网卡的接口名称会发生变化，导致提供的脚本无法正常添加路由规则，造成无法科学上网 [6]。
  - 解决方案：在启动新的 [[VPN]] 客户端前，先手动断开/关闭原来的 [[VPN]] 服务，虚拟网卡的接口名称即可恢复固定 [6]。
- **问题描述**：出现某些网站无法访问（大概率为 [[DNS]] 污染问题）[7]。
  - 解决方案：将 [[DNS]] 改成可以缓解污染的地址（但可能会带来国内网站解析到国外 IP 的问题，视频中具体操作未明确提及）[7]。
- **问题描述**：手机每次重启后，网络设置均会被重置失效 [10]。
  - 解决方案：需要再次重新启动一键网关脚本 [10]。
- **问题描述**：遇到莫名其妙无法解决的网络配置问题 [10]。
  - 解决方案：直接重启手机，网络设置会被恢复初始状态 [10]。

## 最佳实践
- 强烈推荐购买一根 Type-C 转网口（最好带充电口）的转换器，通过网线将手机接入主路由（对应的网卡接口通常为 `eth0`），比 WiFi 连接的速度更稳定、更快 [8]。
- 建议将手机的一切省电选项全部关闭，性能调整为最优，并关闭没必要的后台进程，以防止手机息屏后降频导致网络不稳定 [10]。
- 对于单台设备的网络调试，修改网关后，在系统的网络设置页面“先将网卡禁用，然后再启用”是快速重获新 IP 和网关的有效做法 [9]。

## 关键概念

### [[旁路网关]]
- **定义**：在主路由器旁边增加的一台充当网关的设备，局域网内其他上网数据会交由它处理，而内网设备间的数据传输不需要经过它 [1, 2]。
- **视频上下文**：作者介绍将闲置安卓手机设置为旁路网关，接管局域网设备的上网数据以实现全家科学上网，它在视频中也常被称为透明代理或透明网关 [1, 3]。
- **为什么重要**：很大程度上减少了手机的性能负担，让网络转发性能不强的安卓手机充当网关成为可能，且突破了第三方VPN客户端的设备限制 [2, 3]。
- **关联概念**：依赖 → [[网关]]

### [[网关]]
- **定义**：局域网设备的网络出口，设备访问任何外网都需要寻找的节点 [1]。
- **视频上下文**：作者解释网关可以简单理解为路由器，本教程的核心就是将局域网其他设备的网关改成安卓手机的IP地址 [1, 4]。
- **为什么重要**：它是理解局域网流量走向的基础，修改网关是实现所有设备透明代理的关键机制 [1, 4]。
- **关联概念**：扩展 → [[旁路网关]]

### [[软路由]]
- **定义**：通常指刷了OpenWRT等基于Linux系统的专业路由设备 [1, 3]。
- **视频上下文**：作者指出安卓系统由于同样基于Linux，本质上也是软路由，且使用手机充当软路由可以省去购买专门设备的钱和学习成本 [3]。
- **为什么重要**：提供了一种技术基准，说明用安卓手机替代传统专用软路由设备的可行性及其优势 [3]。
- **关联概念**：对比 → [[旁路网关]]

### [[root]]
- **定义**：安卓系统的最高权限，获取后才能进入命令行环境执行底层网络配置脚本 [5, 6]。
- **视频上下文**：作者强调这是整个教程最难的第一步，完成root授权后才能使用ADB或手机终端切换为root用户（命令行提示符显示为警号） [5, 6]。
- **为什么重要**：是手机能够执行高级网络转发和代理脚本的强制性前置条件 [5, 7]。
- **关联概念**：未明确提及

### [[ADB]]
- **定义**：一种可以在电脑端通过命令行连接和调试安卓手机的工具 [5]。
- **视频上下文**：作者在电脑端通过CMD运行ADB，在手机开启开发者选项和安卓调试模式后，连接手机进入其命令行环境 [5]。
- **为什么重要**：提供了通过电脑端远程配置手机网络和执行网关脚本的途径 [5, 7]。
- **关联概念**：对比 → [[Termux]]

### [[Termux]]
- **定义**：一个在安卓手机端运行的终端模拟器应用，可直接输入并执行命令行 [5, 6]。
- **视频上下文**：作者提到如果觉得用电脑ADB连接繁琐，可以直接在手机上下载Termux进入命令行环境并切换为root用户 [5, 6]。
- **为什么重要**：提供了一种无需电脑、直接在手机本地进行底层网络配置的便捷备用方案 [5, 6]。
- **关联概念**：对比 → [[ADB]]

### [[静态IP]]
- **定义**：手动设置在手机或电脑上的固定局域网IP地址，避免系统自动分配导致变动 [7]。
- **视频上下文**：作者强烈建议将作为网关的手机IP地址改为静态IP，可以直接通过手机的高级WiFi设置修改 [6, 7]。
- **为什么重要**：如果网关IP随意变动，局域网内指向该网关的其他设备将会断网，无法长期稳定使用 [7]。
- **关联概念**：对比 → [[DHCP]]

### [[DHCP]]
- **定义**：主路由器中用于给局域网设备自动下发IP地址和网关信息的设置功能 [4]。
- **视频上下文**：作者演示了在主路由控制台中修改DHCP的网关设置，使其自动向所有接入设备下发手机的IP作为默认网关 [4]。
- **为什么重要**：是实现全家所有接入网络设备无需单独配置即可自动翻墙的核心机制 [4]。
- **关联概念**：未明确提及
