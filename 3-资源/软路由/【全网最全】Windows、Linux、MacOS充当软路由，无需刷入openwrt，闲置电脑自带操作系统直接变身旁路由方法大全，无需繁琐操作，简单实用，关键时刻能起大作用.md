## 一句话总结
利用闲置设备的自带系统（[[Windows]]/[[Linux]]/[[MacOS]]），开启 [[IP转发]] 并配合代理软件即可充当 [[旁路由]]，实现全家科学上网。

## 目标受众与前置知识
- **适合谁看**：拥有闲置电脑或小主机，希望实现局域网内多设备科学上网，但不想或无法刷入 [[openwrt]] 系统的用户（推断）。
- **前置知识**：[[旁路由]]、[[软路由]]、[[IP转发]]、[[网关]]、[[静态IP]]、[[DHCP]]、[[DNS]]、[[透明代理]]、[[TUN模式]]、[[负载均衡]]。

## 核心观点
1. **主流操作系统自带潜能**：所有主流操作系统都能直接充当 [[旁路由]]。
   - 为什么重要：无需繁琐地刷入专用路由系统（如 [[openwrt]]），降低了局域网科学上网的门槛。
   - 依据：视频演示了 [[Windows]]、[[Linux]]、[[MacOS]]（含苹果 [[M系统]]）只要有权限开启系统的 [[IP转发]] 功能，就能转发他人数据包（讲者原话）。
2. **全局共享的核心在于网关配置**：通过修改主路由器的 [[DHCP]]，可实现无感知的全网段共享。
   - 为什么重要：无需在每一台手机或电脑上手动修改网络设置，方便全家设备接入。
   - 依据：在路由器的 [[DHCP]] 设置中将网关改为电脑的 IP，所有自动获取 IP 的设备都能科学上网，且这并非“[[网关劫持]]”（讲者原话）。

## 概念关系
- [[IP转发]] → [[旁路由]]：开启闲置设备操作系统的 [[IP转发]] 功能，是该设备能接管并转发局域网数据包（成为旁路由）的物理前提。
- [[静态IP]] → [[网关]]：充当旁路由的设备必须设置为 [[静态IP]]，以防重启后 IP 变动导致局域网内指向该网关的设备断网。
- [[TUN模式]] → [[透明代理]]：代理软件（如 [[v2n]]、[[clash for Windows]]）需开启 [[TUN模式]] 才能接管系统所有流量，实现透明代理。

## 详细笔记
### 1. 原理说明：为何不刷机也能当旁路由
- **讲者**：[[不良林]]（讲者原话）。
- **概念解析**：无需刷入 [[openwrt]]，直接使用当前的 [[Windows]]、[[Linux]]、[[MacOS]]（含 [[M系统]]），只要开启 [[IP转发]] 功能即可。讲者此前也演示过用 [[安卓]] 手机充当 [[旁路由]]。将局域网设备网关指向旁路由并非“[[网关劫持]]”，因为主路由器的网关并未改变。

### 2. [[Windows]] 系统旁路由配置
- **工具与版本**：[[ROOT forward]]（不支持 [[IPv6]]）、[[v2n]]（推荐版本 6.33，讲者原话）。
- **配置步骤**：
  1. 将 [[Windows]] 电脑的 [[IPv4]] 设置为 [[静态IP]]（如 192.168.1.2，讲者原话）。
  2. 打开 [[ROOT forward]]，识别物理网卡和默认网关。勾选“路由转发”并按提示重启。
  3. 以管理员权限运行 [[v2n]]（视频中也发音为 wiron），导入节点，复制节点 IP 或域名。
  4. 将节点地址粘贴至 [[ROOT forward]] 的“额外路由”中，动作选择“添加路由”并执行。
  5. 在 [[v2n]] 中将路由规则设为“全局”，开启 [[TUN模式]]（T模式）。
- **客户端接入**：手机连接 [[WiFi]]，将获取 IP 方式由 [[DHCP]] 改为静态，网关指向电脑 IP，网络前缀长度填 24（讲者原话），[[DNS]] 设为公网 IP，即可访问 [[谷歌]]。也可直接修改主路由器的 [[DHCP]] 下发网关。

### 3. 带图形界面的 [[Linux]] 系统配置
- **注意事项**：不要使用 Root 用户（讲者原话），视频中提到“以 C 系统为例”（推断为含图形化桌面的 Linux 发行版）。
- **工具**：[[clash for Windows]]（简称 cfw）。
- **配置步骤**：
  1. 命令行输入指令开启 [[IP转发]]，显示 "1" 表示成功（讲者原话）。
  2. 运行 [[clash for Windows]]（虽已删库但仍可安全使用，讲者原话）。
  3. 安装服务模式（Service Mode），重启软件（reload APP）。
  4. 恢复 [[TUN模式]] 默认配置并开启。
  5. 将系统设为 [[静态IP]]，手机网关指向该 [[Linux]] 设备即可。

### 4. [[MacOS]] 系统旁路由配置
- **适用设备与工具**：[[Mac mini]] 等苹果设备。高级用户（4级用户）可直接使用 [[SG]]（推断为 [[Surge]]）的网关模式。免费方案使用 [[clash for Windows]]。
- **配置步骤**：
  1. 设为 [[静态IP]]（如 192.168.1.106，屏幕展示）。
  2. 命令行执行指令开启 [[IP转发]]（显示 1 即成功，讲者原话）。
  3. [[clash for Windows]] 安装服务模式，重置并开启 [[TUN模式]]。修改其他设备网关指向 Mac。

### 5. 无图形界面的 [[Linux]] 配置（电视盒子等）
- **适用系统**：几十块钱的电视盒子刷入的 [[阿扁系统]]（推断为 Armbian，讲者原话）、[[Ubuntu]] 22.04（发音为物班头/巫班图，讲者原话）。
- **工具**：[[v2a]]（即 v2rayA，默认端口 2017，讲者原话）。
- **配置步骤**：
  1. 设为 [[静态IP]]。
  2. 运行官方指令安装 [[v2a]] 并设置开机自启。
  3. 访问 Web 控制面板（IP加端口2017），创建账号并导入节点。
  4. 设置中启用 [[透明代理]] 并开启 [[IP转发]]。
  5. 启动节点。[[v2a]] 支持多节点 [[负载均衡]]（自动切换最低延迟节点，讲者原话）。

### 6. 虚拟机安装 [[openwrt]] 旁路由
- **工具**：[[VM虚拟机]]、[[wim]]（推断为 vim 编辑器）。
- **配置步骤**：
  1. 下载 x86-64 的 vmdk 格式固件（若是 img 格式需转换）。
  2. 新建虚拟机，网络连接选“桥接”，磁盘选现有虚拟磁盘。
  3. 启动约 10 秒后（讲者原话）按回车进入命令行，默认 IP 为 192.168.1.1（讲者原话）。
  4. 使用 [[wim]] 编辑网络配置文件修改 IP 防冲突，重启网络后访问 Web 界面完成旁路由配置。

## 常见坑与解决方案
- **问题描述**：[[路由转发工具]] 打开时报错 [2]。
  - 解决方案：该工具不支持 IPv6，需在网卡属性中取消勾选 IPv6，或在工具内手动设置默认网关 [2]。
- **问题描述**：电脑通过 DHCP 获取 IP，导致下次开机 IP 变动，旁路由失效 [9]。
  - 解决方案：将充当旁路由的电脑设置为同网段的静态 IP [9]。
- **问题描述**：局域网设备的 DNS 无法劫持，导致无法科学上网 [10]。
  - 解决方案：手机等设备的 DNS 不要设置为 `192.168.1.1` 或旁路由电脑的 IP，应随便设置一个公网 IP（即使不存在也可以，如 `1.2.3.4`） [10]。
- **问题描述**：[[Clash for Windows]] 因以前的改动导致 TUN 模式无法正常使用 [4, 5]。
  - 解决方案：开启 TUN 模式前，将 tunnel 设置 reset 恢复成默认配置 [4, 5]。
- **问题描述**：虚拟机安装 [[openwrt]] 后，默认 IP（`192.168.1.1`）与主路由器冲突 [11]。
  - 解决方案：使用 `vim` 编辑配置文件，将 IP 改为其他同网段地址（如 `192.168.10.10`） [11, 13]。

## 最佳实践
- 强烈建议和教程使用相同版本的软件（如 [[v2n]] 6.33 版），防​​止出现不可预料的问题 [2]。
- 充当旁路由的设备，无论何种操作系统，均建议提前配置为静态 IP [4-6, 9]。

- 在 [[v2a]] 中，建议同时选择多个节点进行负载均衡，系统会自动切换延迟最低的节点 [7]。
- 在 [[v2a]] 设置中，不懂的选项不要改动，按默认配置即可（反模式：随意更改不熟悉的配置） [7]。

## 关键概念

### [[旁路由]]
- **定义**：在不改变现有主网络拓扑结构的情况下，通过修改客户端网关，使网络数据包经过的提供代理转发功能的辅助网络设备 [1]。
- **视频上下文**：作者指出只要开启系统相关功能，不论是闲置的 Windows、Linux 还是 MacOS 电脑，都能充当这个角色，让全家设备实现“科学上网” [1, 2]。
- **为什么重要**：它是本视频的核心网络拓扑方案，配置方便灵活，能在没有专门软路由设备的情况下，利旧现有闲置设备实现网关级的网络环境共享 [1, 2]。
- **关联概念**：依赖 → [[IP 转发]]

### [[IP 转发]]
- **定义**：操作系统的网络功能，允许设备接收并帮忙转发其他网络设备发送的数据包 [1]。
- **视频上下文**：作者强调，不管是哪种操作系统，充当旁路由的先决条件就是有权限开启系统的 IP 转发功能 [1, 2]。Windows 需要专门的路由转发工具辅助，而 Linux 和 Mac 可通过命令行开启 [2-4]。
- **为什么重要**：这是普通电脑操作系统能够变身网络路由节点的最核心底层机制，只有开启它，别人的数据包才能经过本机代理软件 [1, 2]。
- **关联概念**：扩展 → [[旁路由]]

### [[网关]]
- **定义**：局域网内设备访问外部网络时所必须经过的“出口”设备的 IP 地址 [5-7]。
- **视频上下文**：视频多次演示，为了让手机等其他设备科学上网，需要将它们的网络设置中的“默认网关”手动修改为充当旁路由那台电脑的 IP 地址 [4, 6-8]。
- **为什么重要**：如果不将网关指向旁路由设备，客户端的流量就不会经过代理软件，从而无法实现网络环境共享 [6, 7]。
- **关联概念**：对比 → [[DHCP]]

### [[静态 IP]]
- **定义**：手动设置且固定不变的网络 IP 地址，不会随着设备重启或重新连网而发生改变 [7, 9]。
- **视频上下文**：作者强烈建议将充当旁路由的闲置电脑的网络设置为静态 IP，避免使用自动获取 [4, 8-10]。
- **为什么重要**：因为旁路由的 IP 地址将被作为其他局域网设备的网关，如果 IP 发生变动，会导致其他将其设为网关的设备无法上网 [9]。
- **关联概念**：对比 → [[DHCP]]

### [[DHCP]]
- **定义**：由路由器提供的自动为局域网设备分配 IP 地址、网关和 DNS 信息的服务 [7, 9]。
- **视频上下文**：作者提到有些设备不支持手动修改网关，只能通过路由器的 DHCP 自动获取，这时就需要修改主路由器的 DHCP 设置，把下发的网关改为旁路由的 IP [7, 11]。
- **为什么重要**：通过修改主路由的 DHCP 配置，可以实现全屋设备自动接入旁路由网络，而不需要逐个设备进行手动设置 [11]。
- **关联概念**：对比 → [[静态 IP]]

### [[TUN 模式]]
- **定义**：代理软件中的一种运行模式，能够接管操作系统的所有网络流量并生成一个虚拟网卡接口 [6, 8]。
- **视频上下文**：在 Windows 中使用 v2rayN 以及在 Linux/Mac 中使用 Clash 时，作者强调必须以管理员权限运行并开启 TUN 模式（或 Tunel 模式） [4, 6, 8]。
- **为什么重要**：只有开启该模式，代理软件才能成功捕获并处理由 IP 转发机制送进来的局域网其他设备的数据包 [6]。
- **关联概念**：未明确提及

### [[透明代理]]
- **定义**：一种在客户端设备毫无感知且无需额外配置代理规则的情况下，就能让其网络流量被代理的机制 [10, 12]。
- **视频上下文**：在使用 v2a 这个工具搭建无图形界面的 Linux 旁路由时，作者要求在设置中明确勾选并启用“透明代理”功能 [10, 12]。
- **为什么重要**：它是实现无感科学上网的核心目标，让接入的设备只要改个网关就能翻墙，而无需在手机等终端上安装任何代理软件 [10, 12]。
- **关联概念**：依赖 → [[IP 转发]]
