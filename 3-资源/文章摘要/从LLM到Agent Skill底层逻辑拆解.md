### 💡 一句话总结

大模型本质是文字接龙游戏，从最底层的 Token 到最上层的 Agent Skill，每一层都在解决上一层的局限性

---

### 🔥 颠覆认知的 5 个真相

1⃣ **大模型不认识字！**

它本质上就是个巨大的数学函数，跑的全是矩阵运算。人类的文字要靠 [[Tokenizer]]（翻译官）转成数字才能喂给它

2⃣ **它没有记忆！**

你以为它记住了你之前说的话？其实是程序每次都偷偷把历史对话打包重新发送。所谓的「记忆」只是 [[Context]]（临时记忆体）

3⃣ **Token ≠ 词**

「工作坊」三个字可能被切成好几个 Token。1 个 Token ≈ 0.75 个英文单词 ≈ 1.5-2 个汉字。这就是为什么中文消耗 Token 更多

4⃣ **工具不是模型自己调的！**

大模型只会输出「我想调用天气工具」这段文字，真正动手的是平台。它是出主意的军师，不是干活的工兵

5⃣ **[[MCP]] = AI 界的 Type-C 接口**

以前每个平台接工具都要重新写代码，有了 MCP 统一标准，开发者只写一次就能到处用

### 📌 7 个核心概念一图串联

```
文字 → [Tokenizer 编码] → Token → [LLM 预测] → Token → [Tokenizer 解码] → 文字
                                      ↑
                              Context（临时记忆）
                              ├── System Prompt（人设规则）
                              ├── User Prompt（用户指令）
                              └── 对话历史
                                      ↓
                              Tool（外部工具）←── MCP（统一标准）
                                      ↓
                              Agent（自主规划 + 多步调用）
                                      ↓
                              Agent Skill（个性化说明文档）
```

### ✨ 金句收藏

> 大模型本质上是一个庞大的数学函数，里面跑的全是矩阵运算，它接收的是数字，输出的也是数字，压根就不认识人类写的文字

> MCP 就像是所有手机都用 Type-C 接口一样，有了统一标准，大家都会方便很多

> Tokenizer 是人类和大模型之间的翻译官，Token 才是大模型处理文本的最基本单元

---

### 🚫 常踩的 5 个坑

- ❌ 以为大模型一次性输出完整回答 → ✅ 它是一个 Token 一个 Token 蹦出来的
- ❌ 以为 Token 就是「词」 → ✅ 切分规则完全不同，一个字可能被拆成多个 Token
- ❌ 以为大模型有记忆 → ✅ 每次都是「失忆」重来，靠 Context 临时记忆
- ❌ 以为 Prompt 很高深 → ✅ 就是你给模型的问题或指令，说人话就行
- ❌ 以为模型自己能调工具 → ✅ 模型只能说「我想调」，平台才是执行者
