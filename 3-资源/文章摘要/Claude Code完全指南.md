### 💡 一句话理解 Claude Code

它不是聊天机器人，是你的 AI 程序员团队！能自己读代码、改代码、跑测试、部署上线，你只需要当甲方提需求就行

---

### 🔥 颠覆认知的3个核心观点

1⃣ **别追求一次完美，要追求试错速度**

AI 的价值不是一次性 100% 做对，而是光速输出 80%，然后通过测试验证循环逼近完美。搭好反馈闭环比写完美提示词重要 100 倍！

2⃣ **上下文管理 = 省钱 + 提智商**

Token 不是无限的！塞太多信息进去，AI 反而变笨。精简提示词 + 按需加载才是王道

3⃣ **并行开发，效率翻倍**

一个 AI 不够用？那就开一个团队！用子代理 + Git 分支，同时让多个 Claude 干不同的活

### 📌 10个必学技巧（由浅入深）

**入门级 🌱**

① `claude.md` 是项目大脑
- 放在根目录，每次对话自动加载
- 控制在 200-500 行，最重要的规则放最前面
- 全局规则放 `~/.claude/claude.md`，项目规则放本地

② 用 `/init` 一键生成项目配置
- 自动扫描代码库，生成专属系统提示词
- 新项目第一件事就是跑这个！

③ Plan Mode = 省时神器
- 写代码前先进计划模式，让 AI 只看不动
- "花1分钟规划，省10分钟返工"
- 确认方案后再让它动手

**进阶级 ⚡**

④ `/btw` 侧边栏对话
- AI 在干活时，你可以随时插话提问
- 不打断主任务，不浪费 Token 开新会话
- 神器中的神器！

⑤ `/compact` 压缩上下文
- 对话太长时一键压缩历史记录
- 用 `/cost` 随时监控消耗
- 防止"上下文腐烂"导致 AI 变傻

⑥ Skills 技能系统
- 把你的工作 SOP 写成 Markdown 放进 `.claude/skills/`
- 用 Front matter 定义名称和工具权限
- 按需加载，只占几十个 Token！
- 记住：**有 SOP 就有 Skill**

**高阶级 🔥**

⑦ Sub-agents 子代理
- 让主 AI 派小弟去干脏活累活
- 推荐三大子代理：研究员、代码审查员、QA 测试员
- 子代理没有先入为主的偏见，审查更客观
- 可以用便宜模型（如 Sonnet）省钱

⑧ Git Worktrees 并行开发
- 不同功能放不同分支文件夹
- 同时开多个 Claude 实例并行开发
- 最后 AI 辅助合并，效率提升 3-5 倍

⑨ MCP 能力扩展
- 让 AI 直接操作浏览器、调用外部软件
- Chrome DevTools MCP = AI 能看网页截图自我修正
- ⚠️ 注意：有些 MCP 很吃 Token，成熟操作建议转成 Skills

⑩ Auto-research 自动研究
- 给 AI 一个优化指标（转化率、回复率等）
- 让它自己跑 A/B 测试、更新最佳方案
- AI 自我进化，越用越好！

### 🚨 避坑指南（血泪总结）

❌ 别把完整 API 文档塞进 claude.md → 用 Skills 按需加载
❌ 别让 AI 重复犯错 → 让它把正确规则写进 claude.md
❌ 别只靠 AI 生成不验证 → 搭建截图循环 + TDD 测试
❌ 子代理返回太多数据 → 要求只返回摘要不返回原文
❌ AI 生成的支付/用户数据代码 → 绝对不能直接上线！必须人工审计

---

### 💰 赚钱思路（最值钱的部分）

大多数人用 AI 写代码写助手，但真正赚钱的是把 AI 用在业务前端：

✅ 线索抓取 → LinkedIn 数据采集
✅ 冷邮件撰写 → 自动生成个性化邮件
✅ CRM 自动跟进 → 客户管理自动化
✅ 销售提案生成 → AI 写方案你改改就发
✅ 竞品分析 → 自动抓取分析竞争对手网站
