## 一句话总结
[[不良林]] 演示了一款基于 [[v2n]] 改版的全自动 [[clash]] 内核节点测速软件及高阶节点池检索技巧。

## 目标受众与前置知识
- **适合谁看**：需要大批量测试并筛选出可用网络代理节点的用户。
- **前置知识**：[[节点池]]、[[订阅链接]]、[[bas64]]（推断指 [[Base64]]）、[[yaml]]。

## 核心观点
1. **测速内核选择取决于市面主流分享协议**
   - 为什么重要：直接决定了测速软件兼容的节点广度与实用性。
   - 依据：讲者提到网上分享的 [[ssr]] 节点数量较多，而自带的 [[wn]]（包含 [[x 内核]] 与 [[w 内核]]）不支持 [[ssr]]，因此必须更换为支持该协议的 [[clash]] 内核（讲者原话）。
2. **多阶段测速机制大幅提升筛选效率**
   - 为什么重要：有效防止在海量失效节点上浪费长时间的下载测速等待。
   - 依据：面对「2300 多个节点」（讲者原话），软件会先测延迟，剔除无效者，直到将剩余待测数量筛至「100 以内」（讲者原话），才会执行更耗时的下载测速流程。
3. **利用搜索引擎特定语法可低成本获取免费节点**
   - 为什么重要：解决用户有测速工具但缺乏测试源的痛点。
   - 依据：讲者演示通过在 [[Google]] 中使用 `inurl:` 语法加上节点池特有 URL 字段，可以直接搜出大量的免费节点池网页（屏幕展示/讲者原话）。

## 概念关系
- [[wn]] ↔ [[CL 内核]]：对比说明。前者（包含 [[x 内核]] 与 [[w 内核]]）因不支持大量存在的 [[ssr]] 节点被舍弃，后者（即 [[clash]] / [[clashn]]）因支持更全而被选作软件测速内核。
- [[v2n]] → [[测速软件]]：依赖说明。本测速软件是基于 [[v2n]]（视频中也因口误或别称提到 [[維特N]] 和 [[v2v]]）二次开发，移除了翻墙功能，仅保留并强化测速功能。
- [[yaml]] ↔ [[bas64]]：对比说明。同为软件支持导入的格式类型。前者多用于 [[CL]] 节点池的原始文本；后者常为传统机场的普通订阅链接，多用于 [[V2]]、[[vq2]] 等客端。

## 详细笔记
### 软件背景与协议兼容性
- **开发背景**：讲者 [[不良林]] 分享了一款专门用于节点筛选的软件，界面长相酷似 [[維特N]]，因为它就是基于 [[v2n]] 强化测速功能并剔除翻墙功能后的改版软件。
- **协议兼容**：软件支持 [[ss]]、[[ssr]]、[[Vmess]]、[[Trojan]]、[[socks5]]、[[http]]、[[https]] 等协议测速（注：视频语音识别字幕将部分协议错误识别为 [[Women]]、[[Children]]、[[set]]）。
- **内核变更史**：原版 [[wn]] 的 [[x 内核]] / [[w 内核]] 不兼容 [[ssr]] 协议，因此讲者将其替换为 [[clash]]；但原版 [[clash]] 不支持 [[vens]]（推断指 VLESS 协议），所以最终使用的是 [[clashn]] 内核。

### 节点导入方式与规则
- **基础操作**：`Ctrl+A` 全选，按 `Delete` 删除列表节点。可以直接通过复制 URL 或按 `Ctrl+V` 导入节点。在链接后加 `#` 可自定义别名。
- **订阅链接导入**：支持直接粘贴经过 [[bas64]] 编码的普通订阅链接（兼容 [[V2]]、[[vq2]] 的格式）。
- **节点池代码导入**：针对使用 [[CL]] 格式的 [[yaml]]（或 [[yml]]），不可直接导入完整配置文件（会提示无法解析），必须只复制从带有 `-` 符号开头的节点列表区域进行粘贴。
- **直连添加节点池**：将节点池地址填入并勾选启用。

### 测速参数设置与自动执行逻辑
- **延迟与下载参数**：包含“测延迟”与“测下载速度”选项。针对延迟测试设有“超时间隔”（默认设为「10 秒」下行动作周期，讲者原话）。
- **防拥塞控制**：针对如「2300 多个」（讲者原话）的大基数，软件会循环测延迟并剔除无效节点，只有当可用数在「100 以内」（讲者原话）时，才会开始测下载速度。
- **一键清理**：自动清理下载速度低于「0.6」（讲者原话，推断指 MB/s）的节点；最终在讲者演示中，延迟「600 以内」（讲者原话）的筛出了「9 个」（讲者原话）有效节点。

### 导出功能与右键操作限制
- **节点导出**：选中目标节点后右键 `Ctrl+C` 复制，导出的格式为常规分享链接或经 [[bas64]] 编码的订阅链接。无法直接导出 [[CL]] 订阅格式，需要用户自己寻找订阅地址转换工具（讲者明确说不演示）。
- **停止测速的正确方法**：若点“一键开始”可直接取消；若使用右键测速（多线程），必须点击专属的“停止测速”按钮并等待当前线程走完，否则强行中断会报错。
- **配置保存差异**：与 [[v2v]] 自动保存不同，该软件退出前必须手动点击“保存配置”，否则状态会丢失。也可以手动右键移除低于「一」兆（讲者原话）的低速节点。

### 高阶搜索技巧与系统环境异常处理
- **找节点池技巧**：利用 [[Google]] 搜索引擎，输入 `inurl:` 加上节点池特征 URL 字段，可以搜出大量现成的公开节点池。
- **环境兼容**：软件内置的 [[CL 内核]] 是 64 位的。如果用户的电脑系统是 32 位的 [[windows]]，必须手动去下载对应的 32 位内核并替换掉 `CL` 文件。
- **遗留 Bug**：不支持含 `<` 尖括号等特殊字符的 [[yml]] 文本导入，讲者认为遇到概率很小且修复麻烦，所以暂未处理。

## 技术栈与环境要求
- **核心技术**：基于 `[[v2n]]`（v2rayN）二次开发，测速功能基于 `[[Clash 内核]]`（因其支持 `[[ssr]]` 和 `[[Vmess]]` 等协议） [1, 2]。支持的协议包括 `[[ss]]`、`[[ssr]]`、`[[Vmess]]`、`[[Trojan]]`、`[[socks5]]`、`[[http]]`、`[[https]]` [1, 2]。支持处理 `[[base64]]` 编码和 `[[yaml]]` 格式配置 [3, 4]。
- **环境要求**：默认需 64 位 Windows 操作系统（内置的 `[[Clash 内核]]` 为 64 位）。如果使用 32 位系统，需自行下载 32 位的 Windows 内核文件替换 [5]。

## 步骤拆解
1. **导入节点/节点池**
   - 直接在软件内粘贴单链接、`[[base64]]` 订阅链接，或 `[[yaml]]` 节点池配置片段；也可在“节点池”设置中直接添加订阅 URL 并勾选启用 [3, 4, 6]。
   - 关键代码/命令：`Ctrl+A`（全选）、`Delete`（删除现有节点）、`Ctrl+V`（粘贴导入节点） [3]。
   - 注意事项：若复制 `[[yaml]]` 格式的代码段，第一行必须是特定的节点列表起始行（每个节点以杠开头），不能直接导入包含配置信息的完整 Clash 配置文件 [4, 6, 7]。

2. **配置测速参数**
   - 勾选“测延迟”或“测下载速度”，设定超时时间间隔、期望过滤的延迟测试结果数量阈值，以及清空低速节点的阈值（如设定低于 0.6 或 1 兆的节点自动删除） [7-9]。
   - 关键代码/命令：未明确提及。
   - 注意事项：测下载速度默认单次周期为 10 秒；超时时间间隔仅针对测延迟生效 [8]。

3. **执行一键测速与终止测速**
   - 点击“一键开始”，软件会自动移除重复节点，先进行延迟测试，再进行下载速度测试 [10]。若需中途停止，执行对应停止操作 [11]。
   - 关键代码/命令：未明确提及。
   - 注意事项：如果是通过“右键”菜单触发的测速，中途想停止时**不能**点击主界面的“取消”（会导致多线程报错），必须点击专用的“停止测速”按钮，且需要等待当前正在测试的线程跑完一个节点后才会完全停止 [11]。

4. **导出有效节点与保存状态**
   - 测速完成后，软件会自动删减无效节点。用户可在列表中右键手动“移除低速节点”，随后通过右键复制，将结果以分享链接或 `[[base64]]` 订阅链接形式导出 [9, 10]。退出软件前需手动保存状态 [12]。
   - 关键代码/命令：`Ctrl+C`（复制导出） [9]。
   - 注意事项：软件不支持直接导出 Clash 格式的订阅地址，如需 Clash 格式必须自行找第三方转换工具处理 [9]。软件每次退出前必须点击“保存配置”，否则所有节点和状态记录会丢失 [12]。

5. **使用特定语法检索节点池**
   - 利用 Google 搜索寻找免费的节点池 URL [12]。
   - 关键代码/命令：`inurl: [节点池URL特征参数]` [12]。
   - 注意事项：未明确提及。

## 常见坑与解决方案
- **问题描述**：直接添加或导入完整的 Clash `[[yaml]]` 配置文件会提示“无法解析” [6, 7]。
  - 解决方案：只复制文件中纯节点列表部分（杠开头的段落），或者直接把节点池的 URL 地址添加到软件自带的“节点池”订阅设置中 [4, 6, 7]。
- **问题描述**：右键手动触发测速后，点击常规的“取消”按钮会触发多线程报错 [11]。
  - 解决方案：必须点击界面上专门的“停止测速”按钮，并等待当前运行的线程结束 [11]。
- **问题描述**：退出软件后，之前导入和测速的节点数据及状态全部消失 [12]。
  - 解决方案：每次退出软件前，必须手动点击一次“保存配置” [12]。
- **问题描述**：在 32 位系统电脑上运行软件可能由于内核不兼容导致失败 [5]。
  - 解决方案：自行去相关开源库下载 32 位的 Windows `[[Clash 内核]]`，并将原软件目录内的 `CL` 文件替换掉 [5]。
- **问题描述**：节点配置信息（如密码）中包含尖括号 `<` 等特殊字符时无法解析导入 [5, 13]。
  - 解决方案：未明确提及（视频作者表示遇到此情况概率极小，故软件未做处理） [13]。

## 最佳实践
- 建议在测速前设置“延迟测试的结果的数量”参数（例如限制为 100 以内），先用极快的延迟测试筛除大量无效/不稳定节点，把基数降下来后再测下载速度，以节省大量时间 [8, 10]。
- 使用 Google 搜索节点池时，推荐搭配 `inurl:` 搜索指令精准匹配节点池的 URL 格式特征 [12]。

## 关键概念

### [[节点池]]
- **定义**：集中提供大量代理节点链接或配置地址的来源[1]。
- **视频上下文**：作者指出软件可以直接添加节点池的地址，以从中快速筛选可用节点，并分享了使用Google搜索特定URL语法（inurl）寻找节点池的技巧[1, 2]。
- **为什么重要**：它是批量获取免费节点的核心来源，也是该软件自动化测速和抓取的主要对象[1, 2]。
- **关联概念**：依赖 → [[yaml 格式]]

### [[订阅链接]]
- **定义**：用于导入和导出代理节点配置信息的专用链接形式[1]。
- **视频上下文**：作者演示了复制普通订阅链接进行导入，并提到测速后可以将筛选出的有效节点再次导出为该形式，以便分享到其他客户端去使用[3, 4]。
- **为什么重要**：是代理节点在不同平台和客户端之间流转最普遍的格式[4]。
- **关联概念**：依赖 → [[Base64 编码]]

### [[Base64 编码]]
- **定义**：一种对节点配置内容进行转换的编码格式[3]。
- **视频上下文**：作者提到普通的订阅链接以及机场提供的订阅地址通常就是经过Base64编码的，并在视频中演示了解码（decode）以查看里面原始节点内容的过程[3, 5]。
- **为什么重要**：是理解订阅链接底层结构的关键，关系到节点信息的正确解析与跨平台转换[3, 4]。
- **关联概念**：属于 → [[订阅链接]]

### [[yaml 格式]]
- **定义**：Clash和部分节点池常使用的一种特定配置文本格式[5]。
- **视频上下文**：作者强调从节点池复制yaml格式的文本时，第一行必须满足特定要求（以破折号开头）才能成功导入软件解析，并指出包含特殊字符的yaml可能导致无法导入[5, 6]。
- **为什么重要**：直接决定了Clash相关的节点和节点池配置能否被该测速软件正确识别与读取[5, 7]。
- **关联概念**：未明确提及

### [[Clash 内核]]
- **定义**：该软件底层用于实际执行节点测速功能的核心程序组件[1]。
- **视频上下文**：作者说明为了支持网上分享数量较多的SSR及Vmess协议节点，将原本自带的内核替换为了Clash内核，并提示32位系统用户需要手动下载并替换对应位数的内核文件[6, 8]。
- **为什么重要**：决定了软件能够支持检测哪些代理协议（如SSR、Trojan等），且涉及到软件在不同操作系统环境下的兼容性配置[1, 6]。
- **关联概念**：未明确提及

### [[v2rayN]]
- **定义**：一款广为人知的代理客户端软件，视频中的一键测速软件即是基于它二次开发而成的[1]。
- **视频上下文**：作者提到测速软件的界面与v2rayN（视频中称v2n或维特N）十分相似，但去除了原版的翻墙代理功能，仅保留并强化了测速功能，且修改了自动保存机制，要求用户必须手动保存配置[2, 8]。
- **为什么重要**：是理解这款测速软件界面架构、运行逻辑以及与常规代理软件功能差异的背景基础[2, 8]。
