## 一句话总结
不良林讲解如何利用 [[潘多拉]] 绕过网络与官方限制，零门槛、高速地使用并部署 [[ChatGPT]] 及 [[API]]。

## 目标受众与前置知识
- **适合谁看**：希望在国内免翻墙使用 [[ChatGPT]]，或寻求突破官方 [[API]] 速率与额度限制的开发者及普通用户（推断）。
- **前置知识**：[[ChatGPT]] 基础概念、[[API]] 调用基础、[[vps]] 与 [[Linux]] 服务器基础操作、[[GitHub]] 账号机制（推断）。

## 核心观点
1. **[[潘多拉]] 是目前最方便强大的免费使用 [[ChatGPT]] 的方式**
   - 为什么重要：解决了官方免费 API 接口被限速、额度有限且容易封号的痛点。
   - 依据：“全程无需翻墙体验更急速更丝华的chatgpt”、“把下（推断：原网页）的gpd官网所有的功能都代底了”（讲者原话）。
2. **使用 [[Token]] 登录比账号密码更安全灵活**
   - 为什么重要：避免在第三方共享平台暴露真实的账号密码，保护聊天隐私。
   - 依据：“通过账号密码可以获取token但通过token无法知道你的密码”（讲者原话）。
3. **将网页版模拟为 API（chat2api）能突破官方速度与调用限制**
   - 为什么重要：极大提升了第三方应用（如 [[webui]]）集成 [[ChatGPT]] 的响应速度，适合高频调用。
   - 依据：“网页版的免费用户是没有线速的”、“一分钟只能对话三条 这个的话可以随便发”（讲者原话）。

## 概念关系
- [[账号密码]] → [[Access Token]] / [[Session Token]]：讲者明确演示可通过账号凭证直接获取这两类 Token。
- [[Session Token]] → [[Access Token]]：讲者指出通过旧的 Session Token 可以自动获取新的 Access Token 并派生新的 Session Token。
- [[Access Token]] → [[Share Token]]：讲者演示可通过脚本使用 Access Token 生成专属的 Share Token (sh token) 用于会话隔离和 API 调用。
- [[官方 API]] ↔ [[潘多拉]] [[API]] (chat2api)：官方 API 有严格限速和计费限制，而潘多拉通过代理网页版实现极速调用，但消耗其自建系统的授权额度。

## 详细笔记
### 痛点分析与潘多拉简介
讲者 不良林 提出官方 [[API]] 存在速度慢、注册门槛高、免费5美元额度易耗尽等问题。引出核心工具 [[潘多拉]] (Pandora)，该工具不仅免翻墙，还能代理 [[ChatGPT]] 官网的全部功能。

### 基础用法：共享池轻度体验
- 对于偶尔使用的轻度用户，可直接使用作者提供的免费共享池。
- 资源量：提供 2000 多个免费共享账号（讲者原话）。
- 账号状态：绿色为空闲，红色为忙碌，金色为 plus 账号（容易被限制）。
- 使用限制：为防止滥用，限制 10 秒钟只能发送一条信息（讲者原话）。

### 进阶用法：Token 登录与获取机制
- 为保护隐私，推荐使用 [[Token]] 登录。[[Refresh Token]] 现已无法获取（讲者原话）。
- 获取方式一：通过 [[潘多拉]] 平台输入账号密码直接提取 [[Access Token]]（有效期 10 天，讲者原话）与 [[Session Token]]（有效期 3 个月，讲者原话）。
- 获取方式二：登录 [[OpenAI]] 官网，利用浏览器开发者工具（检查-应用-cookie）直接提取 [[Session Token]]，实现全程免密暴露。

### 部署防屏蔽反代服务
- 官方演示站可能被墙，建议使用 [[Cloudflare]] 的 [[Cloudflare Pages]] 反代潘多拉网站。
- 也可以使用网友搭建的中国区节点，数量达 600 多个（讲者原话），国内直连速度更快。
- 讲者提醒：使用第三方平台数据是明文的，存在隐私泄露风险。

### 潘多拉 API 调用与 VPS 部署
- 若需集成到第三方工具（如 [[webui]]）或自建应用，需开启 [[proxy模式]] (poxy模式) 部署专属服务。
- 前置条件：一台 [[vps]] 服务器和一个注册满半年的 [[GitHub]] 账号以获取 [[license ID]]。讲者账号注册2年，每24小时拥有 2000 条额度（讲者原话）。
- 部署步骤：使用 [[Linux]] 系统，通过 [[wget]] 下载并用 [[tar]] 解压。修改 config 配置文件，将监听 IP 改为 0.0.0.0，设置 Site Password 及 API 前缀。

### 自动化 Token 管理与 Python 脚本运行
- 讲解多账号 Token 提权机制：通过创建 users.txt 文件，运行 [[Python]] 脚本自动提取并整合为 [[ptoken]]。
- 额度消耗说明：调用账号密码获取接口每次消耗 100 条额度（讲者原话），讲者演示失败一次依然扣除额度（共耗 300 条额度，屏幕展示/讲者原话）。
- 脚本后台自动执行：脚本将在 Access Token 到期前自动续期，实现“三个月不用管”（推断）。

### API 限制解除与 TLS 加密配置
- 将生成的 [[ptoken]] 接入 [[webui]]。
- 注意事项：调用 API 方式发送一条信息实际会消耗 4 条额度（讲者原话/屏幕展示从368变372）。
- 解决 HTTP 协议被浏览器拦截问题：需通过 [[Cloudflare]] 的 [[DNS]] 添加记录并开启代理，设置页面规则将目标端口指向潘多拉的 8181 端口，从而套上 HTTPS 加密。

2. **获取与配置登录 Token**
   - 若不愿暴露密码，可前往 OpenAI 官网控制台提取 `cookie` 中的 `session token`，或通过 [[Pandora]] 提供的接口获取 [4-6]。

3. **部署 Pandora 开启 Proxy 模式**
   - 通过 [[GitHub]] 账号授权获取 license ID [1, 2]。将对应系统的 [[Pandora]] 程序下载至 [[VPS]] 并解压，修改配置文件 `config`，最后运行程序 [1]。

   - 注意事项：需将 IP 改为 `0.0.0.0` 监听所有网卡，保留端口号前的冒号 [1]。必须填写 `site_password` 和 `proxy_api_prefix`，且密码组合要求 8 位以上数字加字母 [1]。

4. **使用 Python 脚本自动化管理 Token**
   - 下载作者提供的 [[Python]] 脚本，编辑替换为自己的 [[API]] 接口地址、端口及前缀 [7, 8]。创建对应文本文档并运行脚本，实现 Token 的自动提取、派生与续期 [8-10]。
   - 关键代码/命令：带 `/a` 参数执行脚本（用于首次从 `users.txt` 读取账号提取 Token），后台运行指令（具体指令未明确提及，视频中演示了操作） [8, 11]。
   - 注意事项：通过脚本调用账号密码获取 `session token` 的接口每次会消耗 100 条额度 [9]。修改接口地址时需将默认的 `127.0.0.1` 替换为 [[VPS]] 的真实 IP [12]。

5. **配置第三方 WebUI 与 TLS 加密**
   - 在第三方 [[WebUI]] 设置中填入生成的 pool token (`ptoken`) 或 share token (`shtoken`) 及 API 地址 [10, 12]。通过 Cloudflare 增加 DNS 记录并开启代理，设置页面规则将目标端口转至 Pandora 的监听端口（默认 8181） [11, 12]。

## 常见坑与解决方案
- **问题描述**：官方接口有速率限制，免翻墙第三方站点演示站随时可能被墙 [6, 13]。
  - 解决方案：使用 [[Cloudflare Pages]] 部署反代站点，或寻找老友分享的国内直连站点 [6]。
- **问题描述**：GitHub 账号授权无法使用 [2]。
  - 解决方案：必须使用注册时长满半年的账号，未满半年无使用额度 [2]。
- **问题描述**：运行 [[Python]] 脚本报错无法执行 [8]。
  - 解决方案：根据返回的错误信息自行查询相关资料排查；若遇额度不足或网络提示“有点忙”导致提取 Token 失败，需通过抓包等方式手动获取 Token 并填入文本文档 [8, 9]。
- **问题描述**：在第三方 [[WebUI]] 聊天时提示错误，因浏览器禁止 HTTPS 网页调用 HTTP 的接口 [12]。
  - 解决方案：修改浏览器限制（将屏蔽不安全内容改为允许），或通过 Cloudflare 给接口套上 [[TLS]] 开启 [[HTTPS]] 访问 [11, 12]。

## 最佳实践
- 推荐使用 `session token` 进行登录，以获得长达三个月的免续期体验 [6]。
- 配置 [[Pandora]] 服务时，强烈建议设置高强度的 `site_password`（8 位以上数字+字母），防止服务被 FOFA 等空间搜索引擎收录并沦为公共库 [1]。
- 必须设置复杂的 `proxy_api_prefix`（8 位以上数字+字母），避免自己的 API 接口被他人暴力猜测并白嫖额度 [1]。
- 若只是轻度使用网页版，建议直接使用官方演示站或免翻墙共享站，不要自行搭建以避免消耗私人额度 [14]。
- 获取到固定的 pool token（`ptoken`）后，建议将其写死在 [[Python]] 脚本中，避免后续每次运行脚本时轮询池的 key 发生非预期变更 [11]。

## 关键概念

### [[潘多拉 (Pandora)]]
- **定义**：一个让用户能免翻墙、免官方限制使用 ChatGPT，并将网页版功能替代并扩展的开源项目 [1, 2]。
- **视频上下文**：作者介绍它能完美解决官方 API 限速和需要翻墙的痛点，提供共享账号池、自建服务等多种使用方式 [1, 3]。
- **为什么重要**：它是整个视频的核心工具，是实现国内直连体验和低成本调用 API 的基础平台 [1]。
- **关联概念**：扩展 → [[chat2api]]

### [[chat2api]]
- **定义**：潘多拉项目提供的一种将 ChatGPT 网页版模拟成 API 接口的机制 [4]。
- **视频上下文**：作者在讲解如何使用第三方工具调用 API 时提到，因为网页版免费用户不限速，通过此功能转化后的 API 速度比官方 API 快得多 [4]。
- **为什么重要**：打破了官方免费 API 的发信频率限制，实现了更高效的第三方应用集成 [4, 5]。
- **关联概念**：属于 → [[潘多拉 (Pandora)]]

### [[access token]]
- **定义**：一种用于登录 ChatGPT 账号的短期凭据，有效期为 10 天 [6]。
- **视频上下文**：作者演示了可通过账号密码或官方接口获取它进行免密登录，并且需要使用它来生成潘多拉专属的 share token [7-9]。
- **为什么重要**：它是生成和更新其他高级专属 token（如 share token）的基础授权凭证 [9, 10]。
- **关联概念**：依赖 → [[session token]]

### [[session token]]
- **定义**：一种用于登录 ChatGPT 账号的长期凭据，有效期为三个月 [6]。
- **视频上下文**：作者推荐使用它代替密码在第三方登录，并指出可以通过它自动获取到 access token 且派生出新的 session token [6, 9]。
- **为什么重要**：由于有效期长，它是利用 Python 脚本实现全自动维持长期有效登录和更新 API 凭证的核心 [9, 11]。
- **关联概念**：对比 ↔ [[access token]]

### [[share token (s token)]]
- **定义**：潘多拉专属的 token，用于分享给他人时进行会话隔离，也可直接作为 API 接口的 key [9]。
- **视频上下文**：作者解释通过 access token 可以生成名称不变的 share token，即使原凭证过期，只要更新底层的 access token，该 share token 依然有效 [9, 10]。
- **为什么重要**：提供了安全的账号分享机制，保护个人聊天记录不被他人看到 [3, 9]。
- **关联概念**：依赖 → [[access token]]

### [[pool token (p token)]]
- **定义**：由多个 share token 组合而成的集合 token，主要用于作为 API 调用的 key [10]。
- **视频上下文**：作者说明用它进行 API 调用时，系统会轮询使用其包含的各个底层账号的 share token，且可随时增删所包含的账号 [10, 11]。
- **为什么重要**：允许整合多个账号的调用额度，实现负载均衡并防止单账号额度耗尽 [10, 11]。
- **关联概念**：依赖 → [[share token (s token)]]

### [[Proxy 模式]]
- **定义**：潘多拉服务端的一种运行模式，开启后才能提供并调用 API 接口 [4, 12]。
- **视频上下文**：作者在讲解搭建潘多拉自建服务时，强调必须在配置文件中设置 `proxy_api_prefix` 参数来启用此模式 [4, 12]。
- **为什么重要**：只有启用此模式，第三方翻译软件或自建应用才能通过它与 ChatGPT 进行后端数据交互 [4, 12]。
- **关联概念**：依赖 → [[License ID]]

### [[License ID]]
- **定义**：用于限制和授权潘多拉 Proxy 模式每日使用额度的标识凭证 [4, 12]。
- **视频上下文**：作者演示在部署潘多拉前，需要先使用注册满半年的 GitHub 账号授权获取该 ID，并将其填入 config 配置文件中 [4, 12]。
- **为什么重要**：作为防滥用机制，它直接决定了用户自建的 API 服务每天能请求的对话额度数量 [4, 13]。
- **关联概念**：未明确提及
