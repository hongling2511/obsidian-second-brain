## 一句话总结
教你使用 [[Cloudflare]] 的 [[worker]] 搭建永久免费的私人反代 [[订阅转换]] 服务以防节点泄露。

## 目标受众与前置知识
- **适合谁看**：担心节点泄露的机场用户或自建节点用户，尤其是技术小白。
- **前置知识**：[[订阅转换]]、[[节点]]、[[vps]]（推断）。

## 核心观点
1. **使用他人的订阅转换服务存在极大的隐私泄露风险**
   - 为什么重要：节点链接是包含服务器 IP 和账号密码等信息的敏感个人数据。
   - 依据：使用别人的服务相当于共享信息，他人可以轻易获取和使用你的节点（讲者原话）。

2. **自建或本地订阅转换并非完美解决方案**
   - 为什么重要：自建存在门槛和安全隐患，本地转换对移动端不友好。
   - 依据：自建可能由于服务存在安全漏洞导致 [[vps]] 被黑客入侵；本地转换需每台设备运行工具，手机用户不方便（讲者原话）。

3. **利用 Cloudflare Worker 搭建反向代理可完美解决隐私与便利性问题**
   - 为什么重要：既能享受第三方服务的便利，又绝对不会泄露节点信息，且无需承担后端安全风险。
   - 依据：[[worker]] 扮演中间人，截获并篡改真实节点为虚假信息交给后端，待转换完成后再还原为真实信息返给用户（讲者原话）。

## 概念关系
- [[节点信息]] → [[第三方订阅转换]]：直接转换会导致隐私泄露。
- [[真实节点]] → [[worker]] → [[随机假节点]]：通过反向代理隐藏真实 IP 和密码。
- [[KV]] ↔ [[R2]]：两种可选的 Cloudflare 数据存储方式，前者免费配额少但绑定方便，后者免费配额大但需绑定支付方式。

## 详细笔记

### 订阅转换的痛点与局限
- [[订阅转换]] 工具可将节点链接转换为 [[clash]]、[[V2ray]] (v2ra)、[[Surge]] (search)、[[Quantumult X]] (圈插) 等配置格式。
- 痛点：直接使用别人的服务会暴露个人服务器 IP 及密码；由于部分在线转换是公益服务，节点被盗容易让提供者背锅。
- 现有方案局限：自建 [[vps]] 门槛高且有被黑客入侵风险；本地订阅转换需每台设备运行，不方便。

### 解决方案原理：Worker 自动化反代
- 灵感来源：评论区一位小伙伴提出“手动修改节点信息再转换再改回”的思路。
- 为实现自动化，[[不良林]] 开发了 [[psub]] 工具，利用 [[Cloudflare]] 提供的 [[worker]] 服务搭建私人订阅转换。
- 原理流程：真实节点信息 → 交给自建的 [[worker]] → 篡改 IP 和密码为随机值 → 交给别人的订阅转换服务 → 后端返回虚假节点的转换结果 → [[worker]] 将结果还原为正确的 IP 和密码 → 返回给用户。

### 搭建步骤：配置 Cloudflare Worker
- 打开 [[GitHub]] 获取 `wk.js` 代码文件。
- 注册/登录 [[Cloudflare]] 账号，进入 [[worker]] 创建服务。免费计划每天有 10 万个访问请求限制（讲者原话）。
- 清空演示代码，粘贴复制的代码并保存部署。
- 配置环境变量：添加变量名为 `backend`，值为别人的在线订阅转换后端地址（注意：[[worker]] 不支持直接访问 IP 及非标端口）。

### 搭建步骤：配置存储桶（KV 或 R2 二选一）
- **方案一：使用 [[KV]]**（每天 1000 次写入操作，每次转换消耗一个左右，讲者原话）。
  - 创建 [[KV]] 命名空间，在 [[worker]] 中添加绑定，变量名设为 `sabgbcket`，绑定刚创建的空间。
- **方案二：使用 [[R2]]**（免费计划有 1GB 存储空间，A 类写入每月 100 万次，B 类读取每月 1000 万次，讲者原话）。
  - 删除 [[KV]] 变量。开通 [[R2]] 需要绑定银行卡或 [[PayPal]]（超量才扣费）。
  - 创建存储桶，回到 [[worker]] 添加绑定，变量名为 `sgbcket`（视频展示有大写要求，讲者原话）。

### 转换测试与进阶用法
- 将 [[worker]] 网址填入前端界面（前端仅用于拼接 URL）进行测试。
- 成功将 [[SS]] 节点转为 [[clash]] 格式。后台收到的 IP（如 127 地址，推断）和密码被替换为随机字符（真实密码 41b 开头变 c99 开头，讲者原话）。同时测试了 [[SSR]] (sssr)、[[Vmess]] (VM / wm)、[[Trojan]] (秋卷) 均成功。
- **特殊协议转换**：官方版默认不支持转换 [[Vless]] (vens / vlens) + [[Reality]] 或 [[Hysteria]] (his / historia)。需使用 [[meta版本]] (0.8.2m版，讲者原话) 后端（如 [[品原]] 实验后端）。可以在链接后加 `&bd=` 拼接新网址临时覆盖环境变量。
- **机场订阅转换**：支持直接转换机场链接（以 [[专线机场白月光]] 为例），代码会自动删除数据防泄露，安全可靠。
- **格式直转**：支持将本地 [[clash]] 文件直接转为其他格式（需以 `proxies` 行开头且不能包含英文竖线符号）；支持将 [[base64]] 编码的 [[V2ray]] 数据直转。
- **自定义域名**：可在 [[worker]] 添加路由绑定自有域名，需在 [[DNS]] 添加 A 记录，指向任意 IP（如 8888，讲者原话），必须开启 [[Cloudflare]] 代理（推断为小黄云），也可通过 [[http]] 订阅地址使用。

2. **创建并部署 Worker**
   - 具体操作描述：登录 [[Cloudflare]]，进入 Workers 面板，选择免费计划并创建 Worker。随意命名后点击部署。接着编辑代码，清空原有演示代码，粘贴从 GitHub 复制的代码，点击保存并部署。
   - 关键代码/命令：未明确提及
   - 注意事项：免费计划每天有 10 万个访问请求限制；部署后直接访问会抛出异常，需先配置环境变量。

3. **配置环境变量 (后端设置)**
   - 具体操作描述：在 Worker 的设置页面添加环境变量，用于指定真实的订阅转换后端地址。
   - 关键代码/命令：变量名设为大写的 `BACKEND`，值为测试可用的在线订阅转换后端网址。
   - 注意事项：Worker 不支持直接访问 IP，也不支持非标端口；建议填入前先尝试该后端是否能正常使用。

4. **绑定存储空间 (KV 或 R2)**
   - 具体操作描述：创建 [[Cloudflare KV]] 命名空间或 [[Cloudflare R2]] 存储桶，并在 Worker 设置中添加绑定。两者只能二选一，若需切换必须先删除原有的绑定。
   - 关键代码/命令：变量名称填写大写的 `SUB_BUCKET`（字幕识别为 sabgbcket / sgbcket）。
   - 注意事项：
     - KV 免费计划每天约有 1000 次写入操作。
     - R2 免费计划每月有 100 万次写入和 1000 万次读取，但绑定 R2 需添加银行卡或 PayPal 账单信息（不超额不扣费）。

5. **配置前端与测试**
   - 具体操作描述：在订阅转换前端界面的“后端地址”栏中填入 Worker 的域名，随后进行节点或订阅链接转换测试。
   - 关键代码/命令：未明确提及
   - 注意事项：填入后端地址时，域名后面的 `s`（指 https）不要删掉；前端的作用仅用来拼接 URL。

6. **自定义域名绑定（可选）**
   - 具体操作描述：在 Worker 中添加路由绑定自有域名，随后在 DNS 处添加对应的 A 记录。
   - 关键代码/命令：Worker 路由设置为 `域名/前缀/*`（字幕识别为斜杠心）；DNS 添加 A 记录时，地址可随便设置（例如 `8.8.8.8`）。
   - 注意事项：DNS 记录中的代理状态（Proxy）**必须开启**，否则无法路由到 Worker。

## 常见坑与解决方案
- **问题描述**：直接访问刚部署的 Worker 会抛出异常。
  - 解决方案：需要按照步骤配置环境变量 `BACKEND` 以指定后端地址。
- **问题描述**：转换 Vless 和 Hysteria 节点失败。
  - 解决方案：官方版本的订阅转换工具（如 0.8.1）默认不支持这些协议，需使用 meta 版本（如 0.8.2m）的后端。可在订阅链接最后加上 `&bd=` （字幕识别为 endbd 等于）并附上 meta 版后端网址，以临时覆盖 Worker 中设置的默认环境变量。
- **问题描述**：直接转换 [[Clash]] 的节点时无法解析。
  - 解决方案：直接转换的内容中不能包含英文的竖线符号（`|`），需使用文本替换工具将所有的竖线符号删掉。

## 最佳实践
- 强烈建议用户自己动手搭建专属的私人转换服务（“信别人不如信自己”），以彻底规避隐私泄露及他人服务端被黑客入侵的连带风险。

- 当 KV 的每日 1000 次写入限额无法满足需求时，推荐切换使用配额极大的 R2 存储桶。
