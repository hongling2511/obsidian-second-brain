## 一句话总结
通过解析不同 [[NAT]] 类型及防火墙策略，实战演示使用 [[P2P]] [[VPN]] 工具在无公网 IP 环境下实现内网穿透与直连。

## 目标受众与前置知识
- **适合谁看**：有异地访问家庭内网、游戏联机或 BT 下载需求，且面临运营商未提供公网 IP 环境的用户（推断）。
- **前置知识**：[[IP]] 地址、端口、局域网与公网概念、[[路由器]]基础工作原理、上期视频提及的 [[ST]] 打洞穿透方案。

## 核心观点
1. **防火墙过滤和 NAT 映射行为共同决定网络类型**
   - 为什么重要：这是理解各网络环境（[[Net 1]] 至 [[Net 4]]）能否成功进行打洞的底层基础。
   - 依据：讲者详细拆解了三种防火墙过滤行为（端点无关、地址相关、地址和端口相关）和三种 [[NAT]] 映射行为，并以浏览器通过 [[htp]] 访问 [[百度]]（IP 66，讲者原话）和 [[支呼]]（IP 77，讲者原话）为例进行了详细的原理推演。
2. **借助 P2P VPN 工具，无公网 IP 也能实现内网直连**
   - 为什么重要：解决了传统方案仅适用极宽松网络（[[Focal]] 的 Net 一），无法在复杂运营商大内网中穿透的痛点。
   - 依据：讲者实测使用 [[TS]] 和 [[ET]] 在 [[MAC]] 电脑与 [[安卓]] 手机间，成功实现了跨越 [[Net 3]] 甚至部分 [[Net 4]] 的 [[P2P]] 直连打洞，将节点纳入 100 开头的虚拟局域网（屏幕展示/讲者原话）。
3. **两个对称网络（Net 4）之间的强制打洞代价极大且极易失败**
   - 为什么重要：指导用户规避无效的穿透尝试，防止对家庭路由器造成毁灭性负担。
   - 依据：讲者指出由于双边端点均无规律，需盲目碰撞，使用 [[Y]] 抓包会发现工具正疯狂发送 [[UDP]] 打洞数据。这种行为类似网络攻击，会瞬间占满 NAT 表导致家庭网络崩溃（讲者原话）。

## 概念关系
- [[防火墙过滤策略]] ↔ [[NAT映射行为]]：前者针对外部发往内部的数据，后者针对内部发往外部的数据，两者结合划分了 [[RFC3489]] 定义的四种 [[NAT]] 类型。
- [[TS]] ↔ [[ETEL]]（[[ET]]）：均为 [[P2P]] 组网工具，但 [[ET]] 支持了更多的穿透方案，利用端口预测扫描技术支持 [[TS]] 无法直连的 [[Net 3]] 对 [[Net 4]] 场景。
- [[Net 1]]（[[net一佛全追]] / [[Easynet]]） → 打洞成功率：因为拥有极宽松的端点无关机制，可轻易接受外部任意端点连接，是打洞直连最理想的一端。

## 详细笔记

### 网络基础与防火墙“打洞”机制
讲者（[[不良林]]）指出，即使在运营商未提供公网 IP 的环境下，通过 [[T Zerotil]]、[[Netb Easytail]] 等 [[P2P]] [[VPN]] 方案，依然可以实现 [[Net 2]] 甚至 [[Net 4]] 的直连，打破了前作 [[ST]] 穿透仅支持 [[Focal]] 的 Net 一环境的局限。
讲者以一台 [[MAC]] 电脑和一台 [[安卓]] 手机作为测试环境，讲解路由器通过 [[pppe]] 拨号获取公网 IP（假设为 5.5.5.5，讲者原话）。当内网通过 [[htp]] 访问外部时，防火墙会临时打开一个“洞”并记录状态。防⽕墙对外部数据进⼊内部有三种过滤策略：端点无关过滤、地址相关过滤、地址和端口相关过滤。

### NAT 出站映射与四种网络类型
介绍了路由器的网络地址转换（[[NAT]]）功能。通过访问 [[支呼]] 的过程，推演出 [[NAT]] 的三种出站映射行为（端点无关映射、地址相关映射、地址和端口相关映射）。讲者强调，出站映射与入站过滤行为共同决定了 [[RFC3489]] 定义的四种常见 [[NAT]] 类型（内容提及 [[RFC5]] 处视频截断未深入）。

### TS 组网实战与宽松网络直连
演示两端均为 [[net一佛全追]]（Net 1）环境的打洞。使用 [[微软]] 邮箱随意注册后，在 [[CMD]] 终端输入指令连接。[[TS]] 客户端先与美国的 DP 中转服务器连接，获取各自公网映射后，双方向对方端点发送数据。因双方都是端点无关，立刻实现 [[P2P]] 直连。直连后，[[WG]] 加密数据直接互传，大幅降低延迟。

### 混合网络打洞与网络封锁干扰
演示 [[Net 1]] 对端口限制型 [[Net 3]]，以及双端均为 [[Net 3]] 的情况。虽存在 IP 与端口绑定的过滤行为，但在出站端点无关的映射下，双方不断重试发包即可错开时差“建洞”成功。期间讲者提到，[[TS]] 节点在国外，在 [[GFW]] 的加持下可能出现连接不稳定，但最终均成功直连。随后开启了 [[Easynet]] 对 [[HnetNet]] 的高难度演示。

### Net 4 (对称网络) 与 ET 端口预测扫描
当处于 [[Net 3]] 对 [[Net 4]] 时，[[TS]] 无法直连只能使用中转。讲者换用支持更多方案的 [[ETEL]]（[[ET]]，目前不支持 [[苹果]] 的 [[iOS]] 系统）进行演示。针对 [[Net 4]] 的严格端口变化，[[ET]] 通过“端口预测进行端口扫描”强行打洞。若逐个扫描 65535 个端口，极端情况需 18 个小时，一秒尝试 100 次也要 10 分钟（讲者原话）；而该工具通过并发上百条链接，通常在 30 秒内即可出结果打通（讲者原话）。

### 极端双边 Net 4 与环境优化建议
演示 [[Net 4]] 对 [[Net 4]] 的打洞。双方完全没有固定端口，需通过不断碰撞碰运气。讲者建议放弃此类尝试，因为疯狂的高频 [[UDP]] 发包会耗尽家庭路由器的 NAT 资源。
应对策略：
- 有公网 IP 的用户：尝试在路由器内开启 [[UPNP]] / [[NETPNP]]、设置 [[dmz]] 区域，或使用软路由开启 [[Focalet]] 改变本地 NAT 类型。
- 处在运营商大内网（[[CGNET]]）的用户：由于经历两次 NAT 转换，即使电脑直接用 [[pppoPOE]] 拨号也无法改变运营商的限制，建议联系运营商更改类型或自建中转。
- 终极方案：若两端均支持 [[IPV6]]，则天然可直连，不适用于本 [[IPV4]] 教程。

## 关键概念

### [[NAT (Network Address Translation)]]
- **定义**：将局域网IP和端口替换为运营商分配的公网IP和随机空闲端口（或反之）的网络地址转换技术 [1]。
- **视频上下文**：作者解释路由器如何处理内网设备发往互联网的数据包，指出NAT的出站映射行为和入站过滤行为共同决定了你家网络的NAT类型 [1, 2]。
- **为什么重要**：它是理解后续所有NAT类型和P2P打洞原理的最底层基础机制 [1, 3]。
- **关联概念**：扩展 → [[Full Cone NAT (NAT1)]]

### [[Endpoint Independent Filtering (端点无关过滤)]]
- **定义**：只要防火墙的“洞”还开着，就允许任何外部IP和任何端口发送的数据通过该洞进入内网的过滤行为 [4]。
- **视频上下文**：作者在解释防火墙的三种过滤策略时，将其作为最宽松的一种进行介绍，并指出它是构成NAT1的关键条件 [3, 4]。
- **为什么重要**：它是理解NAT入站规则的关键维度，直接决定了外部设备主动连接内网的难易程度 [4, 5]。
- **关联概念**：对比 → [[Address and Port Dependent Filtering (地址和端口相关过滤)]]

### [[Address and Port Dependent Filtering (地址和端口相关过滤)]]
- **定义**：外部发送方的数据包要进入内网，其源IP地址和源端口必须与防火墙状态中记录的打洞连接完全关联的严格过滤行为 [4]。
- **视频上下文**：作者指出这是最严格的过滤情况，用于解释NAT3和NAT4在接收外部数据时为何会拒绝并丢弃未主动连接过的数据包 [4, 6, 7]。
- **为什么重要**：解释了为何不同NAT类型间的P2P直连经常失败，是理解复杂网络环境下打洞难点的核心 [4, 6]。
- **关联概念**：未明确提及

### [[Endpoint Independent Mapping (端点无关映射)]]
- **定义**：不管内网设备要访问的目标端点是什么，只要是从同一个内网源IP和端口发出的数据，路由器就复用同一条公网端点映射规则 [1, 2]。
- **视频上下文**：作者用它来定义“EasyNAT”，说明NAT1、NAT2、NAT3在出站映射上都属于此行为，只要内网源端口不变，公网端口就固定不变 [1-3]。
- **为什么重要**：它是P2P打洞中最被依赖的特性，固定不变的公网端口使得对方能够精准预测并建立直接连接 [5, 6]。
- **关联概念**：对比 → [[Address and Port Dependent Mapping (地址和端口相关映射)]]

### [[Address and Port Dependent Mapping (地址和端口相关映射)]]
- **定义**：只要内网设备访问的外部目标IP或目标端口不同，路由器就会使用新的空闲公网端口创建一条全新的映射规则 [2]。
- **视频上下文**：作者指出这种行为是NAT4独有的特征，导致设备每次连接不同的外部目标都会被分配不可预知的、全新的公网端口 [2, 3, 7]。
- **为什么重要**：这种映射行为直接导致常规的P2P打洞方法失效，因为双方无法提前知晓分配给对方的公网端口 [7, 8]。
- **关联概念**：属于 → [[Symmetric NAT (NAT4)]]

### [[Full Cone NAT (NAT1)]]
- **定义**：映射行为是“端点无关映射”，且过滤行为是“端点无关过滤”的最宽松NAT类型 [3]。
- **视频上下文**：视频开篇演示中，当两端均为NAT1时，可以直接通过P2P VPN工具极速实现互相发送数据的直连 [3, 5, 9]。
- **为什么重要**：它是最理想的家庭网络P2P环境，无任何打洞门槛，成功率最高 [3, 5]。
- **关联概念**：属于 → [[EasyNAT]]

### [[Port Restricted Cone NAT (NAT3)]]
- **定义**：映射行为是“端点无关映射”，且过滤行为是严格的“IP地址和端口相关过滤”的NAT类型 [3]。
- **视频上下文**：作者将其作为现实中最典型的网络环境，演示了它与NAT1和NAT3之间需依靠双方互相发送数据包以在各自的防火墙上“开洞”的完整过程 [3, 6, 10]。
- **为什么重要**：它是现实生活中最常见的家庭网络环境之一，理解其打洞机制对日常网络实战最有帮助 [6]。
