## 一句话总结
本教程讲解如何利用亚马逊云的免费 [[CloudFront]] 替代被严重封锁的 [[Cloudflare]]，为 [[VPS]] 节点提速及优选 IP。

## 目标受众与前置知识
- **适合谁看**：搭建了代理节点，且面临 [[VPS]] 线路较差或 [[Cloudflare]] 的 [[CDN]] IP 频繁被墙问题的用户（推断）。
- **前置知识**：[[VPS]]、[[CDN]] 原理、[[Cloudflare]]、[[xui]]、[[DNS解析]]、[[域名绑定]]

## 核心观点
1. **[[CloudFront]] 可作为 [[Cloudflare]] 的优质替代方案**
   - 为什么重要：解决 [[Cloudflare]] 因被大量滥用而被 [[gfw]] 重点关照，导致 IP 被秒封的“半残”问题。
   - 依据：讲者提到亚马逊云注册需信用卡，有一定门槛，能杜绝大部分滥用情况，且目前网络速度表现不错（讲者原话）。
2. **必须自建测速地址以避免耗尽免费流量额度**
   - 为什么重要：防止公用测速地址被他人滥用，导致个人账户的免费流量快速超标而产生扣费。
   - 依据：讲者指出测一次大约消耗 1GB（原话为“1记”）流量，而每账户每月只有 1TB（1T）的免费额度，共享链接很快会耗尽额度（讲者原话）。
3. **[[CloudFront]] 免费额度对绝大部分个人用户足够**
   - 为什么重要：消除用户对亚马逊云易产生意外扣费的普遍担忧。
   - 依据：讲者展示其为 [[甲骨文]] vps 套用测试的账单，从6月份开始测试消耗 30g 流量，本月消耗 19g，均未产生扣费（屏幕展示/讲者原话）。

## 概念关系
- [[Cloudflare]] ↔ [[CloudFront]]：对比关系。两者均为 CDN 服务，前者完全免费但因滥用严重导致 IP 质量差；后者需信用卡注册且每月限 1TB 流量，但当前网络节点质量更好、不易被墙。
- [[xui]] → [[VPS]]：依赖关系。[[xui]] 是安装在 [[VPS]] 上的面板工具，用于搭建支持特定传输协议的网络节点。
- [[优选IP]] → [[CDN]]：因果关系。通过本地终端进行测速，筛选出延迟最低、速度最快的 [[CDN]] 节点 IP，从而提升代理节点的实际连接速度。

## 详细笔记

### 替代 CF 的背景与优势
- 讲者 [[布良玲]] (不良林) 介绍，因 [[Cloudflare]] (简称 CF) 提供的 [[DNS]] 和 [[CDN]] 流量完全免费，导致过度滥用，被 [[gfw]] 重点关照，节点 IP 处于半残状态。
- 提出使用 [[亚马逊云]] (AWS) 的永久免费 [[CDN]] 服务 [[CloudFront]] (简称 cft) 进行替代。
- 注册需信用卡，提高了门槛，减少了滥用（推断：因此连通性更好）。

### 安装 xui 面板与节点准备
- 在一台 [[VPS]] 上通过官方脚本安装 [[xui]]，默认访问端口为 5321（讲者原话）。
- 将域名绑定至 [[VPS]] IP，通过 [[Windows]] 的 [[cmd]] 面板使用 ping 命令验证 [[DNS解析]] 已生效（耗时约几十秒，讲者原话）。
- 讲者演示用的机器为带有丢包情况的 [[NTT]] 线路（屏幕展示）。
- 在 [[xui]] 中创建节点，将传输协议必须改为 [[ws]]，因 cft 目前仅支持该协议（讲者原话）。
- 提及 wm、wens、秋卷（推断：指代 vmess、vless、trojan 等协议/工具）都可修改传输协议为 [[ws]]。端口建议设为 80（讲者原话）。

### 创建 cft 测速专用分配 (Distribution)
- 进入 [[亚马逊云]] 控制台，搜索并收藏 `cloudfont`（即 [[CloudFront]]）。
- 因每月仅有 1TB 免费流量（讲者原话），必须自行创建测速分配，以防公开链接导致流量被耗尽。
- 填写源域为一个专门测试 100兆文件（讲者原话）下载速度的地址。
- 协议设置：选择“匹配查看器”，允许 [[CDN]] 根据客户端请求自动匹配 [[http]] 或 [[HTTPS]] 访问源服务器（讲者原话）。默认端口 80 和 443（讲者原话）。
- 关闭“自动压缩对象”，将“缓存设置”切换为第二项，点击部署并等待生效。

### 优选 IP 流程
- 下载优选 IP 主程序（.exe 格式，同时支持 CF 和 cft）。
- 获取 cft 的 IP 段，剔除掉中国区 IP 段（因为套中国 [[CDN]] 需要备案，讲者原话）。
- 在终端运行主程序：带参数 `/t 1` 表示只测一次延迟以节省时间（讲者原话）。
- 输入 cft 给测速服务器分配的随机域名，程序自动测速并返回可用的优质 IP。

### 为节点套用 cft CDN
- 再次回到 cft 控制台，新建一个分配，源域填写刚才搭建的节点域名。
- 因节点未配证书，协议仅能选 [[http]]，端口设为 80（讲者原话）。
- 部署完成后，在浏览器输入 `cft域名/ws路径` 验证，若返回 `bad request` 则说明生效（讲者原话）。
- 回到 [[xui]] 面板，将节点伪装域名改为 cft 的随机域名，并将 IP 替换为优选出的 IP。测速对比显示套用后速度有明显提升。

### 启用 TLS 与清理操作
- 演示开启 [[TLS]]：不需要自己申请证书，只需在节点编辑中把端口改为 443，传输安全选为 tls 即可（讲者原话）。
- 演示删除分配：需先勾选点击“禁用”，确认后再点击“删除”。

### cft 免费额度与防扣费注意事项
- 每月免费额度包含：1TB 流量、1千万次 [[http]] 和 [[HTTPS]] 请求、200万次函数调用（类似 CF 的 [[worker]]）（讲者原话）。
- 流量不可结转至下月。
- 讲者展示其 [[甲骨文]] 云节点的 cft 消耗情况：6月份消耗 30g，本月消耗 19g，均未超出额度且未扣费（讲者原话/屏幕展示）。

- **核心技术**：`[[CloudFront]]` (简称 cft)、`[[CDN]]`、`[[VPS]]`、`[[x-ui]]`、`[[ws]]` (WebSocket)、`[[HTTP]]`、`[[HTTPS]]`、`[[TLS]]`、`[[vmess]]`、`[[vless]]`、`[[trojan]]` [1-5]。

- **前置技能**：具备 AWS 账号（需信用卡注册）、域名解析与绑定 [1, 2]。

## 步骤拆解
1. **搭建节点与域名解析**
   - 具体操作描述：在 `[[VPS]]` 上安装 `[[x-ui]]`，将域名解析到 VPS IP，关闭代理状态（仅 DNS 解析）。在面板创建节点，传输协议必须改为 `[[ws]]`。
   - 关键代码/命令：在 Windows 中使用 `cmd` 运行 `ping [域名]` 验证解析 [2]。`[[x-ui]]` 的安装命令未明确提及。
   - 注意事项：`[[CloudFront]]` 目前只支持传输协议为 `[[ws]]` 的节点；建议端口设置为 80 [3]。

2. **创建测速专用的 cft 分配**
   - 具体操作描述：在 AWS 控制台创建 `[[CloudFront]]` 分配，源域填写支持下载测试文件的地址（如 100M 测速文件地址）。
   - 关键代码/命令：未明确提及。
   - 注意事项：协议设置选择“匹配查看器”；端口保持默认 80 和 443；必须关闭“自动压缩对象”；缓存设置切换为第二项。部署需等待状态变为具体时间 [4, 6]。

3. **执行优选 IP**
   - 具体操作描述：下载优选 IP 主程序，将默认的 CF IP 段文本文件替换为剔除国内 IP 后的 cft IP 段文件，在终端中执行测速命令。
   - 关键代码/命令：在终端拖入主程序后输入 ` /t 1 URL [套了cdn之后的测速地址]` [6, 7]。
   - 注意事项：切勿将反代后的测速地址分享给他人，以免耗尽自己账户的 1TB 免费流量；必须剔除中国 IP 段以规避域名备案限制 [3, 6, 7]。

4. **为节点套用 CDN**
   - 具体操作描述：再次创建 `[[CloudFront]]` 分配，原域输入节点域名。部署完成后，在浏览器验证 `[[ws]]` 路径。最后在客户端中修改节点配置，将分配的随机域名填入“伪装域名”，并将节点 IP 替换为优选出的 IP。
   - 关键代码/命令：在浏览器访问 `http://[分配的域名]/[ws路径]`，预期返回 `bad request` [5, 7]。
   - 注意事项：未配置证书时，CDN 回源协议必须选 `[[HTTP]]`；若节点需套用 `[[TLS]]`，无需自行申请证书，直接在客户端将端口改为 443、传输安全改为 `[[TLS]]` 即可 [5, 7]。

## 常见坑与解决方案
- **问题描述**：域名解析后 `ping` 测试出现丢包。
  - 解决方案：说明当前网络与 VPS 连接状态不佳（如 NTT 线路），习惯就好 [2]。
- **问题描述**：优选 IP 测速时意外消耗大量 CDN 免费流量。
  - 解决方案：必须自己创建测速地址，且反代后的测速地址不可分享给别人测速 [3, 7]。
- **问题描述**：使用官方获取的 cft IP 段可能包含中国区 CDN 导致无法使用。
  - 解决方案：中国区 IP 套 CDN 需域名备案，需将其全从 IP 段文本中剔除 [6]。
- **问题描述**：浏览器访问 `[[ws]]` 路径时没有返回 `bad request`。
  - 解决方案：检查 `[[ws]]` 路径是否填写错误，或检查设置项是否与教程不一致 [5]。

## 最佳实践
- 节点端口建议设置为 80，以降低出错概率 [3]。
- 运行优选 IP 程序时，添加 `/t 1` 参数（表示只进行一次延迟测试），可大大节省测延迟时间 [6]。
- 时刻关注账单控制面板中的使用量（每月 1TB 流量及 1000 万次请求额度），避免超额产生扣费 [8]。
- 若需彻底删除已创建的 cft 分配，必须先勾选并点击“禁用”，等待其被禁用后再勾选点击“删除” [5]。

## 关键概念

### [[CDN]]
- **定义**：一种可以拯救被墙的虚拟专用服务器并且给垃圾线路提速的网络服务 [1]。
- **视频上下文**：作者指出套了 CDN 之后可以拯救被墙的 vps，视频重点探讨了通过优选 IP 来发挥 CDN 的提速作用 [1]。
- **为什么重要**：它是整个教程的技术基础，用于隐藏真实节点 IP 并优化网络连接速度。
- **关联概念**：扩展 → [[CloudFront]]

### [[CloudFront]]
- **定义**：亚马逊云（AWS）提供的永久免费 CDN 服务，在视频中简称为 cft [1]。
- **视频上下文**：作者将其作为 Cloudflare 的替代方案进行详细教学，指出其注册有信用卡门槛、每月有 1TB 免费流量额度，且目前速度不错 [1-3]。
- **为什么重要**：本教程的主题核心，用于代替目前表现不佳的 Cloudflare，以提供更稳定、快速的代理节点连接 [1]。
- **关联概念**：对比 → [[Cloudflare]]

### [[Cloudflare]]
- **定义**：一家提供免费 DNS 和 CDN 服务的服务商，在视频中简称为 CF [1]。
- **视频上下文**：作者提到因为其完全免费导致被大量使用，从而被防火墙（gfw）重点关照，其优选 IP 经常刚选出来就被墙，处于“半残”状态 [1]。
- **为什么重要**：作为背景引入，说明了为什么当前的节点环境需要寻找 CloudFront 作为新的替代方案 [1]。
- **关联概念**：对比 → [[CloudFront]]

### [[vps]]
- **定义**：被用来搭建网络节点的虚拟专用服务器 [1, 4]。
- **视频上下文**：作者演示了如何在一台 vps 上使用 xui 面板搭建节点，并通过绑定域名和套用 CDN 来为其改善网络线路 [4]。
- **为什么重要**：它是承载代理节点的实体服务器，也是 CDN 服务的加速和保护对象 [1]。
- **关联概念**：依赖 → [[xui]]

### [[优选IP]]
- **定义**：通过本地网络去寻找并测试 CDN 下载文件速度，将下载速度快的 IP 筛选出来的过程 [2]。
- **视频上下文**：作者提供了专门的测速地址和工具程序，指导观众通过消耗自己 CDN 流量的方式筛选出速度最快的 CloudFront IP，并将其替换到节点配置中 [2, 5, 6]。
- **为什么重要**：不经过优选的节点在高峰期会极不稳定甚至无法使用，优选 IP 是提升节点速度和体验的关键核心步骤 [2, 6]。
- **关联概念**：依赖 → [[CloudFront]]

### [[xui]]
- **定义**：一个用于搭建和管理代理节点的面板工具 [4]。
- **视频上下文**：作者演示了通过官方项目地址在 vps 上安装 xui 面板，并在其中创建传输协议为 ws 的节点以配合 CloudFront 使用 [2, 4]。
- **为什么重要**：它是视频中实现节点创建、参数修改（如设置伪装域名和端口）的具体操作平台 [4, 6]。
- **关联概念**：依赖 → [[vps]]

### [[ws]]
- **定义**：WebSocket 协议的简称，是目前 CloudFront 唯一支持的节点传输协议 [2]。
- **视频上下文**：在 xui 面板中创建节点时，作者强调必须将传输协议选项修改为 ws，并配置相应的路径 [2]。
- **为什么重要**：如果不将节点传输协议设置为 ws，将无法成功套用并使用 CloudFront 的 CDN 加速服务 [2]。
- **关联概念**：未明确提及

### [[tls]]
- **定义**：一种传输安全协议，用于配合 HTTPS 节点使用 [6, 7]。
- **视频上下文**：作者提到如果要给节点套 tls，不需要自己去申请证书，只需在节点的编辑界面中将端口改为 443，并将传输安全改为 tls 即可直接连接测试 [6]。
- **为什么重要**：用于为节点建立安全的加密连接，是配置高级节点协议的技术手段。
- **关联概念**：未明确提及
