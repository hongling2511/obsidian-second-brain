## 一句话总结
[[不良林]] 面向纯小白提供的三种基于 [[VPS]] 的高速安全节点搭建保姆级教程。

## 目标受众与前置知识
- **适合谁看**：想搭建专属稳定节点、不知如何选择协议的纯小白用户。
- **前置知识**：未明确提及（讲者原话强调“零基础”、“不需要敲代码”）。

## 核心观点
1. **标题**：[[自建节点]]与[[机场节点]]是互补关系
   - 为什么重要：帮助用户理解两种方式的优劣，明确自建节点的必要性。
   - 依据：讲者原话将[[自建节点]]比喻为私家车（私人专享、稳定、IP固定、无[[内容审计]]），[[机场节点]]比喻为公交车（易受攻击、IP变动、有内容审计）。
2. **标题**：[[VPS]] 线路质量决定[[科学上网]]体验
   - 为什么重要：解释了晚高峰卡顿的原因，指导用户购买合适的服务器。
   - 依据：讲者原话将普通线路比喻为城市道路（线速60码），将 [[cn2gia]] 线路比喻为网络中的高速公路（线速120码），指出同配置价差在于线路不同。
3. **标题**：目前没有绝对不被封的节点，需掌握排查方法
   - 为什么重要：建立合理的预期，并提供应对端口封锁的解决思路。
   - 依据：讲者明确指出“目前并没有哪一种节点可以保证不被封”（讲者原话），并演示了通过访问 `IP:端口` 测试端口连通性的具体方法。

## 概念关系
- [[机场节点]] ↔ [[自建节点]]：前者共享、易变动、有[[内容审计]]；后者独享、稳定、无审查。
- 普通线路 ↔ [[cn2gia]]：前者拥堵如城市道路；后者高速稳定如高速公路。
- [[tls1.3]] → 目标网站：目标网站必须支持该协议，才能作为 [[vlens]]+[[vision]]+[[reality]] 进阶伪装的合适网站，不支持的（如 [[tls1.2]]）需排除。

## 详细笔记
### 节点类型对比与教程简介
概念：[[机场节点]]（地区丰富上手简单，但不稳定、IP地址易变、有[[内容审计]]）；[[自建节点]]（私人专属、稳定、IP固定、无内容审计）。
因果：之前教程存在 [[tls]] 特征被针对导致端口被封，故本期提供三种未被[[防火墙]]针对的最稳定、最安全的搭建方式。

### 准备工作：选购 [[VPS]]
概念与因果：[[VPS]] 线路质量决定速度。晚高峰卡顿是因为线路拥堵。[[cn2gia]] 属于网络中的高速公路，最适合电信用户。
步骤与公司：推荐在 [[班瓦公]] 购买，提供美国、香港、日本线路（普通用户选美国）。最低配置速率 2.5g，月流量 1000g（讲者原话/屏幕展示）。价格为季付50刀、半年付90刀、年付170刀（讲者原话）。注册需填入真实手机号，通过邮箱验证并支付，商家支持30天全额退款。

### 连接 [[VPS]] 与安装面板
步骤与工具：获取 [[VPS]] 的 IP地址、端口、用户名（ROOT）和密码。使用 [[Windows]] 下的 [[cmd]]。
步骤：输入 `ssh 用户名@IP地址 -p 端口号` 连接，确认 yes 并输入密码。执行代码安装 [[XY面板]]，设置账号、密码和 60000 以下（如10000）的端口（讲者原话）。浏览器输入 `IP:端口` 登录，切换版本至 1.8.3（讲者原话/屏幕展示）。

### 第一种方式：[[wmens]] + [[ws]]
步骤与协议：在面板添加入站，协议选 [[wmens]]，网络选 [[ws]]。复制ID前段粘贴到路径。
特点：最简单，对 [[tls]] 封端口地区有奇效，基本所有客户端支持。
验证：导入代理工具后测试真连接延迟。讲者原话称自家 100兆 的宽带可轻松跑满，成功无卡顿访问 [[谷歌]] 与 [[YouTube]]。

### 第二种方式：[[vlens]] + [[vision]]
步骤与工具：面板添加入站，选 [[vlens]]，开启 [[vision]]。在 [[TS flow]] 选项中选 vision。修改IP格式并在结尾加 `.niip.io`。在 [[cmd]] 命令行执行三条指令，获取公钥和密钥路径填入面板。
特点：稍麻烦，需使用推荐客户端，建议将节点网址改为IP地址。

### 第三种方式：[[vlens]] + [[vision]] + [[reality]]
步骤：添加入站，端囗改为 443（讲者原话），开启 [[reality]]，目标网站填 `1.1.1.1:443`。复制ID前一小部分到可选域名。
进阶伪装：通过查询 [[asn]]（如洛杉矶US查询到25820，讲者原话），寻找同地区的合适目标网站。需排除无域名、中文、xyz小众域名、非正常网页以及使用 [[sped text]] 相关的域名，且必须利用开发者工具确认网站支持 [[tls1.3]] 协议。
因果：进阶伪装最好且延迟极低，但可能出现节点没被封但不能上网的不稳定情况。

### 端口封锁排查与维护
步骤：浏览器访问网址加IP再加端口号排查。全绿代表未被封；全红代表均无法访问（需检查VPS相关服务）；仅中国红明确代表端口被封。
解决方案：端口被封后可使用其他搭建方式，或在面板将端口改为 65535 以下的新端口重新导入（讲者原话）。由于 [[XY面板]] 设置会自动更改网址路径，建议保存新网址为书签。
互动：讲者 [[不良林]] 呼吁点赞关注，遇问题可通过邮件反馈。

## 步骤拆解
1. **连接服务器**
   - 准备好 [[VPS]] 的 IP、ROOT 用户名、密码和端口，打开 [[cmd]] 进入命令行终端进行连接 [1]。
   - 关键代码/命令：`ssh root@IP地址 -p 端口号` [1]。
   - 注意事项：在命令行中输入密码时，点击鼠标右键即已粘贴，屏幕不会有任何字符显示，直接回车即可 [2]。

2. **安装面板与基础配置**
   - 执行面板安装命令，根据提示设置账号、密码以及 60000 以下的端口号 [2, 3]。通过浏览器访问 `IP:端口号` 登录面板，并将面板版本切换 [3]。
   - 关键代码/命令：面板安装命令未明确提及（视频提示在说明栏获取）[2]。
   - 注意事项：安装过程中若进度条卡住，需耐心等待，不要中断操作 [2]；切换版本时需选择 1.8.3 版本 [3]；首次在面板设置中更改网址路径后会自动跳转新网址，必须将其保存为书签以防遗忘 [10]。

3. **搭建第一种节点（[[vmess]] + [[ws]]）**
   - 在面板填写入站备注，协议选择 [[vmess]]，添加用户后将网络改为 [[ws]]，将 ID 的前一小段复制并粘贴到路经中 [3, 4]。
   - 关键代码/命令：未明确提及。
   - 注意事项：路径前面的斜杠（/）不能删掉；此节点配置不挑客户端，且对 [[tls]] 封端口的地区有奇效 [4]。

4. **搭建第二种节点（[[vless]] + [[vision]]）**
   - 添加入站并备注，打开 flow 选择 [[vision]]，将 IP 地址修改并加上 `.nip.io` 后缀 [5]。在服务器命令行执行三条指令生成密钥，将生成的路径分别填入公钥和密钥文件路径中 [5, 6]。
   - 关键代码/命令：生成密钥的三条指令未明确提及（视频提示复制执行）[5]。
   - 注意事项：此节点并非所有代理软件都支持，建议使用视频推荐的客户端 [6]。

5. **搭建第三种节点（[[vless]] + [[vision]] + [[reality]]）**
   - 添加入站，开启 [[reality]] 功能，将端口改为 443 [6]。将目标网站改为 `1.1.1.1:443`，将 ID 前缀作为可选域名并加上 `.com` [6]。
   - 关键代码/命令：未明确提及。
   - 注意事项：如果想提升伪装效果，需查询 [[VPS]] 的 ASN 与地区缩写，筛选出拥有正常非中文网页、无证书警告且支持 [[tls]] 1.3 的网站替换目标网站 [7, 8]。

## 常见坑与解决方案
- **问题描述**：晚高峰节点卡顿严重 [11]。
  - 解决方案：更换线路质量更好的 [[VPS]]（如电信用户建议选择 CN2 GIA 线路）[11]。
- **问题描述**：配置完成后尝试科学上网提示错误 [4]。
  - 解决方案：退出代理工具，重新打开并刷新网页即可正常访问 [4, 5]。
- **问题描述**：节点突然无法使用（疑似端口被封）[10]。
  - 解决方案：在特定检测网址后加上 `VPS的IP:节点端口号` 进行排查。如果仅中国地区显示红色，说明端口被封。需更换其他搭建方式，或将节点端口修改为其他端口（65535 以下，如 65000），重导节点即可 [10]。
- **问题描述**：高级伪装节点没有被封但无法上网 [12]。
  - 解决方案：此为追求极致伪装带来的不稳定现象，需自行权衡利弊，解决方案未明确提及 [7, 12]。

## 最佳实践
- 面板版本控制：即便后续发布了新版本，建议依然选择 1.8.3 版本的面板 [3]。
- 客户端配置优化：使用第二种节点（[[vision]]）时，建议在客户端中将节点的网址改为 IP 地址，连接效果会更好 [6]。
- [[reality]] 伪装目标选择反模式：不要使用 Speedtest 作为目标网站 [12]。
- 伪装网站筛选原则：寻找目标网站时，必须排除只有 IP 没有域名的网站、有黄色证书警告的网站、内容为中文的网站以及 `.xyz` 等小众域名的网站 [7]。并在不翻墙的情况下确认该网站是一个支持 [[tls]] 1.3 的正常网页 [7]。

## 关键概念

### [[机场节点]]
- **定义**：一群人共同使用的科学上网节点 [1]。
- **视频上下文**：作者将其比喻为“公交车”，指出其可能因拥挤出现不稳定、IP随时变化、有内容审计等问题，但优势是地区丰富、上手简单 [1]。
- **为什么重要**：理解当前科学上网的主要方式之一，是说明为什么要自己搭建节点的重要背景。
- **关联概念**：对比 → [[自建节点]]

### [[自建节点]]
- **定义**：个人专属搭建的科学上网节点 [1]。
- **视频上下文**：作者将其比喻为“私家车”，强调其具有私人专享、比较稳定、IP固定、无内容审计的优势，是本教程的核心目标 [1]。
- **为什么重要**：解决机场节点不稳定、IP变动和内容审计等痛点的核心手段。
- **关联概念**：对比 → [[机场节点]]

### [[vps]]
- **定义**：搭建自建节点所必须准备的服务器 [2]。
- **视频上下文**：作者指出搭建节点前需要购买一台VPS，且VPS的线路质量直接决定了晚高峰时段科学上网是否会卡顿 [2, 3]。
- **为什么重要**：自建节点的基础物理/虚拟硬件前提。
- **关联概念**：扩展 → [[cn2gia]]

### [[cn2gia]]
- **定义**：个人用户能买到的最高端、类似于“高速公路”的网络线路 [3]。
- **视频上下文**：作者解释相较于拥堵的城市道路，走cn2gia线路尤其能保证电信用户在晚高峰拥有高速稳定的科学上网体验，但价格较贵 [3]。
- **为什么重要**：决定了节点在网络拥堵时段的稳定性和连接速度。
- **关联概念**：属于 → [[vps]]

### [[xyi 面板]]
- **定义**：安装在VPS上用于管理和搭建节点的可视化控制面板 [4, 5]。
- **视频上下文**：作者通过命令行一键安装该面板，设置账号密码和端口后，在浏览器中登录并在其中完成所有节点协议的添加与配置 [4, 5]。
- **为什么重要**：让纯小白免除编写代码，通过图形化界面即可完成节点搭建的核心工具。
- **关联概念**：依赖 → [[vps]]

### [[ws 协议]]
- **定义**：一种最简单且兼容性最强的节点搭建网络协议 [5, 6]。
- **视频上下文**：作者演示的第一种搭建方式（配合 wmens 使用），指出其对 TLS 封端口的地区有奇效，且基本任何代理客户端都支持 [6]。
- **为什么重要**：最基础的搭建方式，适合作为保底的节点类型。
- **关联概念**：未明确提及

### [[vision]]
- **定义**：第二种稍复杂的节点搭建协议组件 [7]。
- **视频上下文**：需要通过命令行生成并配置公钥和密钥路径进行搭建，并非所有代理软件都支持，建议使用特定客户端 [7, 8]。
- **为什么重要**：提供不同于ws协议的进阶选择。
- **关联概念**：未明确提及

### [[reality]]
- **定义**：第三种节点搭建协议，通过伪装目标网站来隐藏节点流量 [8, 9]。
- **视频上下文**：作者演示将其目标网站设为 1.1.1.1 或自定义网址，搭建简单且相对稳定，但自定义高级伪装可能会带来不稳定 [8-10]。
- **为什么重要**：当前较为主流且具伪装特性的节点搭建方式。
- **关联概念**：依赖 → [[tls1.3]]

### [[tls1.3]]
- **定义**：作为 reality 节点高级伪装目标网站时必需支持的安全协议版本 [10, 11]。
- **视频上下文**：作者强调在寻找优质伪装目标网站时，必须在浏览器开发者工具中确认该网站支持 tls1.3，否则无法用作目标网站 [10, 11]。
- **为什么重要**：决定目标网站是否能用于 reality 协议伪装的强制性技术门槛。
- **关联概念**：属于 → [[reality]]

### [[端口]]
- **定义**：节点与外界通信的网络通道标识（数值需在65535以下） [12]。
- **视频上下文**：作者提到之前的搭建方式存在被防火墙针对导致“端口被封”的情况；如果节点无法使用且检测工具显示中国地区飘红，则需更换新的端口 [2, 12]。
- **为什么重要**：节点连接成功的关键参数，也是防封锁排查及恢复节点的主要操作对象。
- **关联概念**：未明确提及
