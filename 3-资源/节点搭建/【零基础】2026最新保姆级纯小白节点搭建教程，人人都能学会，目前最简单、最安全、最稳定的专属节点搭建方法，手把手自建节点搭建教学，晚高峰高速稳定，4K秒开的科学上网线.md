## 一句话总结
[[不良林]]为纯小白演示如何购买 [[VPS]]、通过脚本搭建专属节点及配置多平台客户端的完整流程。

## 目标受众与前置知识
- **适合谁看**：完全不懂节点搭建、想要有自己专属独享节点的新手小白。
- **前置知识**：只需熟练掌握电脑的开关机。

## 核心观点
1. **[[VPS]] 线路质量直接决定节点稳定性**
   - 为什么重要：很多朋友搭建的节点晚高峰卡顿，大部分都是线路太差导致。
   - 依据：讲者指出若对速度稳定要求高，需选择有线路优化的 [[VPS]]（如 [[CN2GIA]]），而最便宜的 [[KVM]] 晚高峰大概率会卡（讲者原话）[1, 2]。
2. **延迟高低不是衡量节点速度的绝对标准**
   - 为什么重要：避免用户陷入唯延迟论的误区。
   - 依据：讲者明确表示延迟不是衡量节点速度的标准，主要看线路的好坏，美国节点的正常握手延迟并不会影响网速（讲者原话）[3, 4]。

## 概念关系
- [[VPS]] 线路 → 节点速度与稳定性：晚高峰是否卡顿由机房和线路质量（如是否为 [[CN2GIA]]）决定。
- [[美国]] [[DC6]] ↔ [[日本]] [[软银]]：前者为默认 GIA 线路，后者虽非 GIA 但速度不错且地理位置近、延迟更低（对比）。
- [[TLS]] → [[reality]]：在面板配置入站节点时，需先创建基于 reality 协议的 TLS 模板（依赖）。

## 详细笔记
### 购买与配置 [[VPS]]
- **选择线路**：推荐使用 [[美国]] [[CN2GIA]]（讲者口误提及 [[CR2GIA]] / [[CNGIA]]）线路，或 [[日本]]、[[香港]] 的 GIA 线路。预算有限可买最便宜的 [[KVM]]，但晚高峰易卡顿（讲者原话）[1, 2]。
- **下单流程**：通过专属链接购买（支持季付、半年付、年付），默认选择 [[美国]] [[DC6]] 机房或 [[日本]] [[软银]] 机房。输入优惠码结账，注册填入邮箱和密码即可。购买后一个月内可申请退款（讲者原话）[2, 5]。
- **获取信息**：付款后会收到包含 IP、端口及 root 密码的邮件 [5]。

### 连接服务器与执行搭建脚本
- **系统连接**：[[Windows]] 用户运行 [[CMD]]，[[Mac]]（[[MAC]]）用户运行自带的 [[终端]] 程序。输入 `SSH root@IP -P 22`（定量：讲者原话），随后粘贴密码并回车 [5, 6]。
- **运行脚本**：粘贴专属指令并执行，等待过程需三五分钟（讲者原话）。提示 `y/-n` 时按回车。保存生成的用户名和密码 [6]。
- **测试延迟**：执行选域名指令，推荐选择延迟低的域名（如 [[icloud.com]]，延迟 30 多毫秒，讲者原话），保存域名后输入 `exit` 退出 [6, 7]。

### 配置 [[SUI面板]] 与入站节点
- **登录面板**：执行端口转发指令后，将连接中的 IP 替换为 127.0.0.1（讲者原话），在浏览器打开并使用刚才保存的账号密码登录 [7]。
- **配置 [[TLS]]**：
  1. 添加 [[reality]] 协议：输入保存的低延迟域名作为 [[SNI]] 和握手服务器，生成公钥/私钥，启用 [[UTLS]]。
  2. 添加备用 TLS：启用“允许不安全”、[[ALPN]] 和 [[SNI]]，填入相同域名 [7]。
- **添加入站**：
  1. 添加 [[WS]] 类型（推断：字幕显示 WS，通常指 WebSocket）：端口改为 443（讲者原话），绑定 reality 模板。
  2. 添加 [[TK]] 类型（推断：字幕显示 TK，指代 TCP 或 Trojan 等协议）：端口改为 443，绑定备用 TLS 模板。
  3. 在用户管理中勾选这两人个节点并保存 [7]。

### 多平台客户端配置
- **电脑端 ([[Windows]] / [[Mac]])**：
  - 下载 [[VN]]（或称 [[WN]]，推断：v2rayN）或 [[微头]]（推断：v2rayU/v2rayA）。
  - [[Mac]] 需区分处理器：[[CPU]]（推断：指代旧版 Intel）下载对应版本，[[M芯片]] 下载 [[M64]] 版本 [3]。
  - 将面板生成的二维码链接导入。将节点地址修改为 [[VPS]] 的真实 IP。[[TK]] 节点需将“跳过证书验证”改为 true（讲者原话）[3]。配置完成后可访问 [[谷歌]] 等被强网站，[[Macem]]（推断：指代 Mac 端）同理 [8]。
- **移动端 ([[iPhone]] / [[安卓]])**：
  - [[iPhone]]：使用非国区 ID 在 [[苹果商店]] 下载 [[shadow Rcket]]（[[小火箭]]），扫码或复制导入节点，修改节点 IP 并开启“允许不安全”[8]。
  - [[安卓]]（Android）：下载 [[NO Box]]（推断：NekoBox）的 [[V8A]] 安装包，扫码导入，修改服务器 IP，并在最下方启用“允许不安全的链接”[8, 9]。

### 后续维护与机房迁移
- **续费管理**：VPS 需定期续费（例如季付会在 3 个月后的 10 月 21 号收到通知，讲者原话）。收到邮件后有 7 天时间（讲者原话）操作续费，逾期会被回收 [9]。面板支持将套餐升级为 40G、80G、160G 月流量（屏幕展示）[4]。
- **免费迁移**：进入控制面板选择“迁移到其他 DC”，可免费从美国 GIA 切换至日本 [[软银]] 等机房。迁移无次数限制且会更换 IP，但会消耗流量。迁移后需在客户端中修改为新 IP 才能恢复科学上网 [4]。

## 步骤拆解
1. **购买与准备 `[[VPS]]`**
   - 具体操作描述：选择并购买带有优化线路（如 CN2GIA 或软银）的 `[[VPS]]`，完成注册和付款，随后通过邮件获取 IP 地址、端口和 root 密码 [1, 2, 7]。
   - 关键代码/命令：未明确提及。
   - 注意事项：如果在视频推荐的特定页面购买，可在一个月内申请退款试用 [7]。

2. **连接 `[[VPS]]` 服务器**
   - 具体操作描述：Windows 用户运行 `[[CMD]]`，Mac 用户打开终端，通过 `[[SSH]]` 建立连接并输入 root 密码 [2, 6]。
   - 关键代码/命令：`ssh root@IP地址 -p 22` [2]。
   - 注意事项：粘贴密码时屏幕不会有任何显示，且需注意区分大小写 [2, 6]。

3. **运行一键搭建脚本**
   - 具体操作描述：在终端粘贴特定长指令执行，等待代码跑完，遇到提示时按回车，并妥善保存生成的用户名和密码 [6]。
   - 关键代码/命令：未明确提及。
   - 注意事项：若中途出现卡死或意料之外的情况，可直接关闭窗口重新操作 [6]。

4. **测试并寻找低延迟域名**
   - 具体操作描述：复制并执行测速指令，选择一个延迟较低的域名并保存，随后退出 `[[VPS]]` 的连接 [3, 6]。
   - 关键代码/命令：`exit` [3, 6]。
   - 注意事项：域名延迟越低越好，若不满意可换一批重新测试 [6]。

5. **配置 `[[SUI]]` 面板**
   - 具体操作描述：在浏览器通过 IP 访问面板并登录 [3]。添加 `[[TLS]]` 设置，配置 `[[reality]]` 模板（填写测速域名、生成公私钥、启用 `[[UTLS]]` 和浏览器指纹） [3]。添加备用的 `[[TLS]]` 模板（生成证书和私钥、启用允许不安全、开启 `[[ALPN]]` 和 `[[SNI]]`） [3]。在入站管理中添加端口为 443 的节点并绑定对应的模板，最后在用户管理中分配节点 [3]。
   - 关键代码/命令：未明确提及。
   - 注意事项：在节点搭建完成之前，绝对不要关闭跑脚本的黑窗口 [3]。

6. **客户端配置与连接**
   - 具体操作描述：根据不同设备下载代理客户端（如 `[[Shadowrocket]]` 或 `[[NekoBox]]`），通过二维码或链接导入刚创建的两个节点，随后将节点的地址手动修改为 `[[VPS]]` 的 IP 地址并测试延迟 [4, 5, 8]。
   - 关键代码/命令：未明确提及。
   - 注意事项：针对备用的 T 节点，必须手动将“跳过证书验证”或“允许不安全的链接”设为开启 [4, 5, 8]。确认节点无误后，将其设为自动配置系统代理即可使用 [4]。

## 常见坑与解决方案
- **问题描述**：`[[SSH]]` 连接时密码粘贴失败或提示密码错误 [2]。
  - 解决方案：手动输入密码，并注意区分大小写 [2, 6]。
- **问题描述**：搭建脚本运行中途出现卡死或意料之外的情况 [6]。
  - 解决方案：直接关闭黑窗口，重新跟着教程从头操作 [6]。
- **问题描述**：由于客户端版本不同导致无法使用 [8]。
  - 解决方案：重新下载视频中提供的特定版本客户端覆盖安装 [8]。
- **问题描述**：在进行机房迁移中途出现错误 [9]。
  - 解决方案：翻译错误提示，通常需要先关闭 `[[VPS]]` 再进行迁移操作 [9]。
- **问题描述**：忘记续费导致 `[[VPS]]` 节点被回收 [5]。
  - 解决方案：在收到续费通知后的 7 天内完成续费操作；若已被彻底回收，则必须重新购买并重新进行搭建操作 [5]。

## 最佳实践
- 日常推荐使用 `[[reality]]` 节点，将另一个配置的 T 节点仅当作备用线路使用 [4]。
- 购买带有线路优化（如 CN2GIA 线路）的 `[[VPS]]` 以保障速度和稳定性，避免使用无优化线路导致晚高峰期卡顿 [1, 7]。
- 反模式：非必要情况不要频繁进行机房迁移，因为迁移过程会额外消耗服务器流量 [9]。

## 关键概念

### [[VPS]]
- **定义**：搭建专属节点的服务器基础，其线路质量直接决定节点的速度和稳定性 [1]。
- **视频上下文**：作者强调搭建节点必须先购买一台 VPS，并指导了购买、连接和后台切换机房线路的完整操作 [1-3]。
- **为什么重要**：它是自建节点的物理或虚拟承载基础，线路优劣大部分时候是导致晚高峰网络卡顿的核心原因 [1]。
- **关联概念**：扩展 → [[GIA 线路]], [[软银线路]]

### [[GIA 线路]]
- **定义**：目前个人用户能买到的最高端 VPS 线路，拥有极佳的速度和稳定性，且价格较贵 [1, 2]。
- **视频上下文**：作者推荐对速度要求高的用户选择具备此线路的美国、日本或香港机房，以保证晚高峰不卡顿 [1, 2]。
- **为什么重要**：决定了节点在极端网络环境下的高质量访问体验 [1]。
- **关联概念**：对比 → [[软银线路]]

### [[软银线路]]
- **定义**：一种日本机房的 VPS 线路，虽不是 GIA 线路但也具备不错的速度和低延迟表现 [2, 3]。
- **视频上下文**：作者演示了在控制面板中，如何将购买的 VPS 从美国机房免费迁移到日本软银线路，从而大幅降低物理延迟 [3]。
- **为什么重要**：提供了一个高性价比且地理位置较近的低延迟备选线路方案 [2, 3]。
- **关联概念**：对比 → [[GIA 线路]]

### [[SSH]]
- **定义**：一种用于在电脑命令行终端远程连接并控制 VPS 的指令协议 [4, 5]。
- **视频上下文**：教程要求在 Windows (CMD) 或 Mac (终端) 输入含有 IP 和端⼝号的 SSH 指令，并输入 root 密码来登录服务器执行安装代码 [4, 5]。
- **为什么重要**：是普通个人电脑与云端 VPS 建立安全管理连接的必要前置步骤 [4, 5]。
- **关联概念**：依赖 → [[VPS]]

### [[SUI 面板]]
- **定义**：部署在 VPS 上、通过浏览器访问的图形化节点管理后台 [6]。
- **视频上下文**：通过终端跑完代码后，作者指导使用浏览器访问特定 IP 地址进入该页面，并在其中添加和配置 TLS、入站规则等信息 [6]。
- **为什么重要**：将复杂的节点代码配置转化为直观的网页界面操作，大幅降低了小白用户的学习门槛 [6]。
- **关联概念**：依赖 → [[VPS]]

### [[Reality]]
- **定义**：SUI 面板中用于节点配置的一种特定 TLS 模板类型 [6]。
- **视频上下文**：在添加入站管理规则时，作者指导用户将 TLS 设置切换为 reality，并建议将其作为日常科学上网的首选节点 [6, 7]。
- **为什么重要**：是视频中推崇的最简单、安全、稳定的专属节点搭建方法的核心配置参数 [1, 6]。
- **关联概念**：依赖 → [[SUI 面板]]

### [[SNI]]
- **定义**：节点 TLS 设置中的一个需要填入低延迟域名的网络参数 [6]。
- **视频上下文**：作者在 SUI 面板配置 reality 节点时，要求在 SNI 以及握手服务器处填入之前通过终端测试过延迟较低的域名（如 icloud.com） [5, 6]。
- **为什么重要**：与节点的伪装及连接建立直接相关，其测试延迟越低代表网络表现越好 [5, 6]。
- **关联概念**：未明确提及

### [[代理工具]]
- **定义**：安装在用户本地设备上，用于导入已创建的节点并实现网络代理的客户端软件 [8]。
- **视频上下文**：视频列举了 Windows 使用 v2rayN、Mac 使用 v2rayU/Macem、iOS 使用 Shadowrocket 等，并演示了如何通过扫描二维码或粘贴链接导入节点并修改 IP [7-9]。
- **为什么重要**：是打通 VPS 节点与用户本地设备，最终实现科学上网的落地执行工具 [7, 9]。
- **关联概念**：依赖 → [[VPS]]
