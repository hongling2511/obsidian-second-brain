## 一句话总结
[[不良林]]演示零基础使用[[搬瓦工]] [[CN2 GIA]]线路及[[x-ui]]面板搭建安全防封锁专属节点的完整步骤。

## 目标受众与前置知识
- **适合谁看**：想要拥有自己专属稳定节点、不太愿意折腾且希望一次操作安心使用的零基础小白用户。
- **前置知识**：未明确提及（讲者明确定位为“零基础保姆级搭建教程”）。

## 核心观点
1. **安全是节点搭建的首要前提**
   - 为什么重要：只有安全了才能实现省时省心，跳出“翻墙出来学翻墙”的怪圈。
   - 依据：讲者原话：“虽然会牺牲一些效率但这是目前最安全的搭建方式……只有安全了才能实现省时省心”。
2. **高速稳定高度依赖优质的物理线路**
   - 为什么重要：高速稳定只有靠“钞能力”解决，普通线路体验不佳。
   - 依据：讲者推荐[[搬瓦工]]的[[CN2 GIA]]线路，称其经过三网优化，历史稳定性达 99.9%（屏幕展示），传输速率达 2.5Gbps（屏幕展示）。
3. **流量伪装必须彻底模仿正常网站行为**
   - 为什么重要：[[GFW]]处于黑盒状态，不能心存侥幸猜测其探测机制。
   - 依据：讲者原话：“既然翻墙流量伪装成了[[HTTPS]]……就要尽可能表现的和正常网站的行为是一样的，而不是去猜测[[GFW]]会不会来探测你的伪装”。

## 概念关系
- [[CN2 GIA]] → [[VPS]]：提供底层高速、高稳定性的物理网络基础（推断）。
- [[x-ui]] → [[VMess]] / [[VLESS]]：作为可视化面板，用于配置和管理底层代理协议节点。
- [[acme]] → [[TLS]]：通过工具自动申请并部署证书，为节点传输提供加密安全层。
- [[Nginx]] ↔ [[x-ui]]：配合使用，[[Nginx]]负责处理前端[[Web]]流量伪装和反向代理，隐藏后端的[[x-ui]]面板和真实节点端口。

## 详细笔记
### 课程介绍与线路选择
- 讲者[[不良林]]针对基础小白推出零基础教程，目标是搭建高速、稳定、安全、防封、省时、省心的节点。
- 采用[[x-ui]]面板搭建[[VMess]]或[[VLESS]] + [[ws]] + [[TLS]] + [[Web]]的节点，讲者称这是目前最安全的搭建方式。
- 推荐使用[[搬瓦工]]的洛杉矶[[CN2 GIA]]线路，对电信、移动、联通三网优化。套餐月流量 1000G（双向计费，实际每月可用约 500G，讲者原话），速率 2.5Gbps（屏幕展示）。
- 新用户支持购买后一个月内不满意申请退款。

### VPS 购买与环境初始化
- 需在[[搬瓦工]]特定页面搜寻并购买[[CN2 GIA]]线路的美国[[VPS]]，支持使用[[支付宝]]付款结算。
- 进入服务管理面板，停止默认安装的[[CentOS]]系统，重新安装为最新版[[Ubuntu 22]]操作系统，系统会自动分配新的端口和root密码。
- 下载并安装[[FinalShell]]客户端（支持[[Windows]]、[[macOS]]、[[Linux]]），输入IP、分配的端口、用用户名root及密码，通过[[SSH]]连接至[[VPS]]。

### 域名注册与 DNS 解析设置
- 在域名网站注册账号并购买廉价域名（演示选中 1.88 美金域名，使用优惠码后实付 1 美金/年，讲者原话），支持[[支付宝]]支付。
- 进入域名[[DNS]]解析后台，删除默认记录，添加一条泛解析（主机记录填入*号）和一条 A 记录，均指向[[VPS]]的IP地址。
- 解析生效大约需要15到30分钟。使用本地[[cmd]]运行ping命令测试域名，若返回的IP与[[VPS]]一致即代表解析生效。

### 系统更新与面板安装
- 使用命令行更新[[Ubuntu 22]]操作系统组件（耗时约10分钟，讲者原话），完成后使用reboot命令重启服务器。
- 演示执行开启[[BBR]]拥塞控制算法的命令（针对旧系统，[[Ubuntu 22]]已默认开启）。
- 执行脚本安装[[x-ui]]面板，设置自定义的账号、密码及访问端口（演示为9999）。
- 安装[[acme]]工具，并执行命令自动为域名申请[[TLS]]证书及完成安装部署。

### 伪装网站与 Nginx 部署
- 在[[Google]]搜索他人搭建的私人网盘，确认可正常访问后将其选为伪装站点，因其符合单节点大流量的特征。
- 找到并修改服务器上的[[Nginx]]配置文件，将申请证书的域名以及伪装网盘网站的URL替换至配置模板中。

### 节点配置与防封锁隐藏
- 通过 `域名:端口`（如9999）登录[[x-ui]]面板，更新到最新版。创建监听IP为 127.0.0.1、传输协议为[[ws]]的[[VMess]]或[[VLESS]]节点，记录生成的端口（如10000）与ID。
- 修改[[x-ui]]面板设置，将面板本身监听IP设为 127.0.0.1，并配置面板的URL根路径（填入节点ID + /xui）。
- 将面板路径和端口信息同步填入[[Nginx]]配置文件，覆盖清空后的[[VPS]]默认[[Nginx]]配置，重载[[Nginx]]服务。
- 此时公网443端口仅展示私人网盘，真实节点和面板隐藏在其后。
- 客户端导入节点后需修改参数：地址改域名、端口改443、加密方式改[[Zero]]、传输安全改[[TLS]]。

### 单端口多用户分流合租（进阶）
- 预算有限的用户可通过分流[[ws]]路径实现合租分摊费用。
- 在[[x-ui]]中新建支持[[ws]]传输的节点（必须是[[VMess]]或[[VLESS]]，不能是[[Trojan]]因其强制要求设置TLS证书）。
- 设定监听IP为 127.0.0.1，自定义新端口（如101），可设置总流量限制（如10G，讲者原话）和到期时间。
- 将新节点的ID和端口追加写入[[Nginx]]配置文件中，重载[[Nginx]]后即可生效。超出流量或到期后对应节点自动失效。

## 步骤拆解
1. **购买VPS与重装系统**
   - 购买搬瓦工 CN2 GIA 线路 VPS，在管理面板中将默认的 CentOS 系统重装为最新版 Ubuntu 22.04 [2, 7, 8]。
   - 关键代码/命令：未明确提及。
   - 注意事项：留意重装系统后自动生成的 ROOT 密码和 SSH 连接端口 [2]。

2. **通过SSH连接VPS**
   - 下载安装 [[FinalShell]]，添加 SSH 连接，填入 VPS 的 IP、端口、用户名（ROOT）及密码进行连接 [2]。
   - 关键代码/命令：未明确提及。
   - 注意事项：当 VPS 后台显示系统重装完成界面时再进行连接 [2]。

3. **购买域名与设置DNS解析**
   - 注册购买低价域名，进入 DNS 解析设置，删除默认记录，添加一条泛解析（输入 `*`）A记录和一条直接解析（输入 `@`）A记录，指向 VPS 的 IP 地址 [9, 10]。
   - 关键代码/命令：未明确提及。
   - 注意事项：DNS 解析生效大约需要 15 到 30 分钟 [10]。

4. **更新VPS操作系统**
   - 复制并执行系统更新命令，期间出现提示直接回车，完成后重启系统并重新连接 [10]。
   - 关键代码/命令：`reboot` [10]。
   - 注意事项：搬瓦工的 VPS 默认为最小化安装，缺失很多功能，必须先进行更新（其他商家 VPS 可忽略） [10]。

5. **验证DNS解析是否生效**
   - 在本地系统运行 cmd 工具，ping 购买的域名，检查返回的 IP 是否与 VPS 的 IP 一致 [3]。
   - 关键代码/命令：`ping [域名]` [3]。
   - 注意事项：如果 IP 相同则解析生效；如果不同说明还未生效，需耐心等待 [3]。

6. **开启BBR与安装x-ui面板**
   - 更新软件源，执行命令启用 [[BBR]] 拥塞控制算法，然后执行命令安装 [[x-ui]]，并按提示设置账号、密码以及面板的访问端口（如 9999） [3]。
   - 关键代码/命令：未明确提及。
   - 注意事项：必须记住设置的账号、密码和端口，后续步骤会用到 [3]。

7. **申请与安装SSL证书**
   - 安装 [[acme.sh]] 自动申请证书，添加软链接并切换 CA 机构，随后使用域名申请证书并安装 [3, 4]。
   - 关键代码/命令：未明确提及。
   - 注意事项：操作这一步时一定要确保域名的 DNS 解析已经生效 [4]。

8. **配置Nginx伪装网站**
   - 在搜索引擎寻找一个私人网盘网站作为伪装站点，进入 [[nginx]] 的配置文件目录修改配置文件，将域名、伪装网址等信息替换进去 [4]。
   - 关键代码/命令：未明确提及。
   - 注意事项：所选的伪装网站必须确认能正常访问 [4]。

9. **配置x-ui节点与面板路径**
   - 登录 [[x-ui]] 面板，切换至最新版 x-ray，添加一个 [[vmess]]/[[vless]] 节点，监听 IP 设为 127.0.0.1，设置一个端口（如10000），传输设为 [[ws]]，将节点 ID 复制并替换到 [[nginx]] 配置文件的路径中。接着在面板设置中将面板监听 IP 改为 127.0.0.1，根路径设为包含刚才复制的 ID 的路径 [5]。
   - 关键代码/命令：未明确提及。
   - 注意事项：确认 Nginx 配置中的端口和节点监听的端口保持一致；面板监听 IP 和路径修改后，原面板地址将无法直接访问 [5]。

10. **应用Nginx配置与配置客户端**
    - 清空 VPS 默认的 [[nginx]] 配置文件，粘贴修改好的配置并保存，重载 [[nginx]]。在 [[x-ui]] 面板复制节点导入客户端，修改相关连接信息 [5, 11]。
    - 关键代码/命令：`systemctl reload nginx` （视频原文读作 System control reload engin） [5, 11]。
    - 注意事项：导入客户端后，需将地址改为域名，端口改为 443，传输安全改为 [[tls]]，加密方式建议改为 zero [11]。

11. **配置单端口多用户合租（可选）**
    - 在 [[x-ui]] 中创建新节点，协议选择支持 [[ws]] 的 [[vmess]] 或 [[vless]]，监听 IP 为 127.0.0.1，分配新端口并复制 ID，设置流量或时间限制；然后在 [[nginx]] 配置文件中新增对应的 ws 分流路径，最后重载 [[nginx]] [6, 12]。
    - 关键代码/命令：`systemctl reload nginx` [12]。
    - 注意事项：不要在 [[x-ui]] 中创建 Trojan 节点用于 ws 分流，因为它会强制要求设置 TLS 证书 [12]。

## 常见坑与解决方案
- **问题描述**：DNS解析未生效导致证书申请失败 [3, 4]。
  - 解决方案：通过 cmd 使用 ping 命令确认域名返回的 IP 与 VPS 的 IP 相同后，再进行证书申请操作 [3, 4]。
- **问题描述**：更新系统后输入 reboot 重启，重新连接失败 [3, 10]。
  - 解决方案：说明系统还没有重新启动完成，稍等片刻后重试连接即可 [3]。
- **问题描述**：通过 ws 分流配置多用户节点时，使用 Trojan 协议导致无法实现 [12]。
  - 解决方案：仅创建 [[vmess]] 或 [[vless]] 节点，不要创建 Trojan 节点 [12]。

## 最佳实践
- **推荐做法**：
  - 套餐购买推荐：如果预算充足选择搬瓦工 CN2 GIA 线路，付款周期选择年付更划算 [7, 8]。
  - 伪装网站选择：单节点大流量代理推荐使用“私人网盘”类网站作为伪装站点，更符合大流量特征 [4]。
  - 节点客户端加密设置：[[vmess]] 的加密方式建议改成 zero，效率更高 [11]。
  - 端口安全：将所有翻墙服务隐藏在 443 端口中，从公网看 VPS 仅开放了伪装的私人网盘服务 [11]。
- **反模式**：
  - 存在侥幸心理，在没有伪装或没有套 [[tls]] 的情况下裸奔翻墙，去猜测 GFW 不会进行探测 [6, 11]。

## 关键概念

### [[x-ui面板]]
- **定义**：一款用于搭建节点和实现分流的可视化控制面板 [1-3]。
- **视频上下文**：作者通过它来搭建安全协议节点，并通过它管理单端口多用户的合租设置、限制流量和设置到期时间 [1, 3, 4]。
- **为什么重要**：它是实现省时省心管理，以及实现多用户合租分摊费用的核心工具 [1, 2, 4]。
- **关联概念**：依赖 → [[VPS]]

### [[CN2 GIA线路]]
- **定义**：一种经过电信、移动、联通三网优化的优质网络线路 [1]。
- **视频上下文**：作者推荐使用搬瓦工的这种线路部署服务，并在购买环节专门演示了如何寻找该线路套餐 [1, 2, 5]。
- **为什么重要**：它是保证节点高速、稳定体验的关键，能避免晚高峰拥堵的情况 [1, 6]。
- **关联概念**：未明确提及

### [[VPS]]
- **定义**：用于承载节点服务和伪装网站的云端虚拟服务器主机 [2, 6, 7]。
- **视频上下文**：作者演示了购买VPS、重装Ubuntu系统，以及使用SSH工具连接服务器的完整过程 [2, 7]。
- **为什么重要**：它是搭建所有代理服务、部署控制面板和配置网络环境的基础设施 [3, 7]。
- **关联概念**：扩展 → [[CN2 GIA线路]]

### [[DNS解析]]
- **定义**：将注册的域名映射绑定到VPS的IP地址的过程 [8-10]。
- **视频上下文**：作者在域名管理后台添加了泛解析记录和A记录，并使用ping命令测试解析是否生效 [9, 10]。
- **为什么重要**：只有解析生效后，才能进行后续的节点正式搭建和自动申请TLS安全证书 [10, 11]。
- **关联概念**：依赖 → [[VPS]]

### [[TLS]]
- **定义**：一种需要配合域名和安全证书使用的网络传输安全协议 [1, 6]。
- **视频上下文**：作者通过acme脚本自动申请证书，并在客户端配置时将传输安全选项强制改为TLS [6, 10, 11]。
- **为什么重要**：结合伪装网站，它能将代理流量伪装成正常的HTTPS流量，是实现最安全搭建方式的必要环节 [4, 6]。
- **关联概念**：依赖 → [[DNS解析]]
