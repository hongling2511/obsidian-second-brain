## 一句话总结
通过 [[BGP]] 广播自定义 [[ASN]] 与 IP 段，搭建纯净 IP 站群服务器并实现每次连接随机落地 IP。

## 目标受众与前置知识
- **适合谁看**：有跨境电商防封控需求、爬虫业务需求或希望搭建纯净网络代理节点的用户。
- **前置知识**：[[VPS]] 基础、[[IPv4]] 与 [[IPv6]] 基础、[[Linux]] 命令行操作（推断）。

## 核心观点
1. **自带干净的 IP 与 ASN 可以解决被平台封控的痛点**
   - 为什么重要：很多廉价 [[VPS]] 商家分配的 IP 因被滥用，导致被互联公司（如 [[奈飞]]）整段封禁。
   - 依据：讲者指出自己申请 [[ASN]] 并租用 IP 地址后，由于是“自己的地盘”，可以保证 [[ASN]] 和 IP 的干净程度，不被封禁（讲者原话）。
2. **原生 IP 和广播 IP 的本质区别在于机房位置与 IP 归属地库是否一致**
   - 为什么重要：打破了行业内对“原生IP”必定能解锁流媒体的误解。
   - 依据：[[ipino]] 等平台是通过 `trrot`（路由追踪）来判断 IP 归属地并更新数据库的。讲者发邮件催促更新 IP 归属地花了 3 天时间，在此之前美国机房的 IP 曾显示为荷兰（讲者原话）。对流媒体平台而言，实际上只看 IP 是否在黑名单里（讲者原话）。
3. **利用最新的代理工具更新可实现每次 TCP 连接使用完全随机的落地 IP**
   - 为什么重要：极大方便了爬虫用户，避免单一 IP 请求过频被目标网站阻断。
   - 依据：将 [[插瑞]]（推断：Xray）更新到最版的 1.8.10，配合 [[xu面板]] 修改配置模板，访问支持 IPv6 的网站时，每次新的 [[TCP]] 连接都会在 [[IPv6]] 前缀后生成完全随机的落地 IP（讲者原话）。

## 概念关系
- [[AS]] → [[ASN]]：每个自治域（[[AS]]）作为一个独立的网络，都拥有唯一的编号，即 [[ASN]]。
- [[BGP]] → 路由宣告：通过 [[BGP]] 协议将两个不同的自治域连接起来并传递路由信息，将 IP 段发送给相邻网关的行为称为路由宣告。
- [[Transit]] → [[BGP]] 连接：需要通过寻找提供 Transit 服务（如 [[w]] 家）的地方，才能建立 BGP 连接并将 IP 广播出去。

## 详细笔记

### 背景介绍与核心概念
- 讲者 [[不良林]] 介绍常规 [[VPS]] 只有单个公网 IP，若需成百上千个 IP 需使用 [[BGP]]。
- 介绍 [[BGP]]（边界网关协议）用于在不同自治域（[[AS]]）之间传递路由信息。
- 举例：[[中国电信]] 的 [[ASN]] 是 4134，[[中国联通]] 的 [[ASN]] 是 4837。不同 [[AS]] 像不同局域网，需通过 [[BGP]] 连接。[[BGP]] 出问题可能导致国家断网。

### 跨境代理的 IP 痛点与解决方案
- 廉价 [[VPS]]（如 [[w]] 家，推断指 Vultr）的 IP 因门槛低被滥用，被 [[奈飞]] 等公司将整个 [[AS]] 封禁。
- 解决方案：自行申请 [[ASN]]（通过 LIR 服务商，一年几百块钱）和租用 IP 地址（一段 /48 的 [[IPv6]] 每月 20 刀，一段 /24 的 [[IPv4]] 每月 200 刀）（屏幕展示与讲者原话）。
- 需寻找提供 [[Transit]] 的商家（如走 [[cn2gia]] 线路需花钱，[[w]] 提​​供免费 Transit）。使用 [[KVM]] 或 [[OpenVZ]] 技术可创建 VPS 做超售。

### 在 W 家平台配置 BGP 广播
- 使用 [[w]] 平台（新注册绑定支付方式送 100 美金试用 14 天）。
- 在 Network 的 BGP 页面设置：填入自有的 [[ASN]]（不填则系统分配私有 ASN），输入租用的 IP 段，上传 LOA 授权文件。
- 发送邮件向 IP 所有者验证，联系商家通过验证后即可添加 IP 段。

### 服务器端环境部署与路由收敛
- 部署 [[半图]] 22.04（推断：Ubuntu）系统的 VPS。
- 安装与配置 [[BD]]（推断：BIRD）：
  1. 使用 `ufw disable` 关闭防火墙。
  2. 使用 `apt install bird` 安装。
  3. 用 [[VM编辑器]]（推断：Vim）修改配置文件，填入 BGP 密码和租用的 IP 段。
  4. 重载配置后，检查状态显示 `H` 即表示与 [[w]] 成功建立 BGP 连接。
- **路由宣告与收敛**：向外界发送路由信息，直到全球 BGP 路由都知道如何访问该 IP 段的过程称为路由收敛。
- 将 IP 添加到本地路由表并使套接字支持绑定非本地地址，实现站群服务器效果（内外部互相 Ping 通）。

### IP 归属地与流媒体解锁的迷思
- 在 [[ipino]] 查询刚广播的 IP 会显示 [[ASN]] 信息及类型（如 hosting 或 business），类型由商业公司自行推断，非注册时选择。
- 演示机房在美国，但 IP 归属地显示荷兰（nl），引出**原生 IP**和**广播 IP**的常见分类。
- 辟谣：能否解锁流媒体只看是否在平台黑名单里。归属地显示是由检测平台的路由追踪（Traceroute）更新速度决定的（更新可能需 1 天到半个月）。
- 本演示的 IP 挂在 [[w]] 的 [[ASN]] 下，因该 ASN 被拉黑导致原可解锁荷兰 [[奈飞]] 的 IP 变为不能解锁。

### 多地区宣告与 Xray 随机落地 IP 实操
- **任播 IP**：支持在多数据中心宣告同一 IP 段，实现类似 [[cloudfare]] 1.1.1.1 的 Anycast 任播效果（基于物理距离路由）。
- **随机落地 IP 配置**：
  1. 将 [[插瑞]]（推断：Xray）更新至 1.8.10 版本。
  2. 在 [[xu面板]]（推断：x-ui）建两个节点（演示端口为 33333 和 44444），导入 [[v2n]]（推断：v2rayN）。
  3. 修改端口 33333 的配置文件模板，实现每次新建 TCP 连接（如浏览器无痕模式）时，IPv6 后半段完全随机变化。
- **本地自带 IPv6 随机化补充**：
  - 若无租用 IP，可利用 VPS 默认前缀为 64 位的 IPv6 地址。
  - 安装并配置 [[ndp代理工具]] 后，修改配置模板中的前缀，同样可实现特定端口代理时的随机落地 IP。

## 案例分析
### 案例：通过自行租用 IP 与 [[BGP]] 广播搭建纯净站群节点
- **背景**：常规廉价 [[VPS]] 商家分配的 IP 上手门槛低，导致 IP 经常被滥用，进而被 [[Netflix]]（奈飞）等流媒体或互联网公司将整个 [[ASN]] 拉黑封禁。跨境电商及相关用户难以获得高质量、防封控的纯净 IP [4]。
- **策略**：用户以个人身份申请或使用服务商的私有 [[ASN]]，向提供 IP 租赁的商家租用干净的 IP 段（如一段 /32 的 [[IPv6]] 地址）[1][2]。随后寻找提供 Transit 服务的平台（如 [[w]] 提供的免费 Transit），建立 [[BGP]] 连接，将属于自己的 IP 段广播（路由宣告）至全球网络，并绑定到具体的 [[VPS]] 上 [2][5][6]。
- **结果**：实现了单台 [[VPS]] 拥有海量纯净 IP 的“站群服务器”效果，解决了 IP 不干净被封控的问题 [1][6]。此外，通过在不同地区（如美国和日本）的数据中心同时宣告同一个 IP 段，成功实现了类似 [[Cloudflare]] 1.1.1.1 的 [[Anycast]]（任播）效果，根据访问者地理位置实现就近路由 [7]。
- **启示**：跨境业务从业者或爬虫开发者可以通过掌握底层的 [[BGP]] 广播逻辑并独立持有网络资源（IP和ASN），从根本上摆脱对单一 [[VPS]] 厂商分配劣质 IP 的依赖，提升业务的网络稳定性和抗封锁能力。

## 竞争格局
- **流媒体平台风控与 [[VPS]] 厂商的对抗**：以 [[Netflix]] 为代表的流媒体平台与低门槛 [[VPS]] 提供商（如 [[w]]）之间存在强烈的对抗关系。当平台察觉某厂商 IP 被大量滥用时，会直接采取“一刀切”策略，将该厂商整个 [[ASN]] 封禁入黑名单 [4][8]。
- **国际路由 Transit 服务的层级差异**：提供 [[BGP]] 广播介入的 Transit 服务存在明显的品质与成本分层。例如高品质的付费 Transit 需依附于 [[CN2 GIA]] 等优质线路商家，而基础或免费 Transit（如 [[w]] 提供的服务）虽降低了个人玩家的门槛，但在网络连接速度上存在劣势，晚高峰时期往往需要借助中转才能达到专线水平 [2][9]。
- **IP 归属地数据提供商的更新效率博弈**：提供 IP 数据查询的商业公司（如 [[ipinfo]]）与传统静态检测平台之间存在准确度的竞争。[[ipinfo]] 采用动态的 [[traceroute]]（路由追踪）技术，当监测到广播 IP 的真实数据中心发生物理位移（如从荷兰变更为美国）时，会更新其数据库，而部分技术滞后的检测网站仍会显示错误的静态归属地信息 [8]。

## 关键概念

### [[BGP]]
- **定义**：全称边界网关协议，主要作用是在不同的自治域之间传递路由信息，将全球所有自治域连接在一起组成互联网世界 [1, 2]。
- **视频上下文**：作者指出无论是机场节点还是自建VPS都经常宣传此概念，并在视频中实操通过该协议广播自己租用的IP段，实现一台VPS拥有无数IP地址 [1, 3, 4]。
- **为什么重要**：它是整个互联网世界互联的基础核心协议，出现问题可能会导致整个国家断网 [2]。
- **关联概念**：依赖 → [[自治域]]

### [[自治域]]
- **定义**：可以被视为一个独立网络的区域（简称 AS），每个自治域内部网络是相对独立的局域网 [1, 2]。
- **视频上下文**：作者以中国电信和中国联通为例，说明它们属于不同的自治域，两者由于网络独立，必须通过 BGP 协议连接后才能互相发送数据 [1, 2]。
- **为什么重要**：理解互联网如何划分边界、以及跨网络数据如何互通的基础单位 [1, 2]。
- **关联概念**：扩展 → [[ASN]]

### [[ASN]]
- **定义**：每个自治域拥有的唯一编号 [1]。
- **视频上下文**：作者介绍电信的编号是 4134，联通是 4837，并演示利用 ipinfo 等网站检测 IP 地址所属的编号以查看宽带运营商、IP所有者或其提供的服务类型 [1, 5]。
- **为什么重要**：它是追踪 IP 地址网络归属和身份鉴定的重要标识 [1, 5]。
- **关联概念**：属于 → [[自治域]]

### [[路由宣告]]
- **定义**：将自己的IP段通过 BGP 协议发送给邻居节点，告诉全网如果有人要访问该IP段应该往自己的方向路由的行为（即“IP广播”） [6]。
- **视频上下文**：作者在 VPS 上建立 BGP 连接后，将自己拥有的 IP 段宣告出去，解决外部网络因不知道该 IP 段在哪里而访问失败的问题 [6]。
- **为什么重要**：这是让自有或私有 IP 段在全球互联网中变得“可达”的必经操作 [6]。
- **关联概念**：依赖 → [[BGP]]

### [[路由收敛]]
- **定义**：路由信息通过邻居节点层层传递，直到全球的 BGP 路由都知道怎么到达你宣告的 IP 段的状态 [6]。
- **视频上下文**：作者提到，当发出路由宣告后，周边节点会继续传递信息，直到全球网络都获悉路径，此时收敛完成，目标IP便能根据规则接收到数据 [6]。
- **为什么重要**：标志着 IP 广播过程最终生效，全网通信线路建立完毕 [6]。
- **关联概念**：依赖 → [[路由宣告]]

### [[广播IP]]
- **定义**：VPS 机房的实际物理位置和 IP 的数据归属地不在同一地区的 IP 地址 [5]。
- **视频上下文**：作者举例自己在由于路由宣告操作，使得机房在美国的 VPS 使用了归属地显示为荷兰的 IP，从而解释了这一术语 [5]。
- **为什么重要**：在跨境电商和流媒体圈常用于判断 IP 属性，但作者强调通过归属地和机房位置判断是否能解锁流媒体其实是一个误区 [5]。
- **关联概念**：对比 → [[原生IP]]

### [[原生IP]]
- **定义**：VPS 机房的物理位置和 IP 的数据归属地在同一地区的 IP 地址 [5]。
- **视频上下文**：作者在解释完广播IP后，相对应地提出了这个概念，并指出两者主要被大众用来区分能否解锁奈飞等流媒体平台 [5]。
- **为什么重要**：常作为评估节点“纯净度”的商业卖点，但其真实机制受制于 IP 平台的数据库更新情况 [5, 7]。
- **关联概念**：对比 → [[广播IP]]

### [[任播IP]]
- **定义**：支持在多个不同地区的数​​据中心同时进行路由宣告，让用户访问时自动连入距离自己最近的服务器的 IP [8]。
- **视频上下文**：作者举例 Cloudflare 的 1.1.1.1 服务，并实操在美国和日本的 VPS 绑定同一 IP 段，实现了新加坡用户连日本、加拿大用户连美国的就近访问效果 [8]。
- **为什么重要**：是实现全球就近接入、降低网络延迟的高级网络架构技术 [8]。
- **关联概念**：依赖 → [[路由宣告]]

### [[路由追踪]]
- **定义**：一种用于探测数据包到达目标过程中所经过的实际路由路径和最终物理位置的技术手段 [7, 8]。
- **视频上下文**：作者解释说 ipinfo 这类数据库平台就是通过此手段监测到 IP 实际上被路由到了美国的某个数据中心，进而去更新并显示正确的 IP 归属地 [7]。
- **为什么重要**：揭示了 IP 归属地判定和更新的底层技术逻辑，能识破表面上的归属地伪装 [7]。
- **关联概念**：未明确提及
