## 一句话总结
教大家脱离 [[洋葱浏览器]]，使用普通浏览器通过 [[Tor]]（推断，原文作“拓/唾”）网络访问暗网、搭建暗网服务及配置链式代理。

## 目标受众与前置知识
- **适合谁看**：需要极高隐私保护、想要访问暗网或搭建暗网服务的网络技术学习者。
- **前置知识**：[[VPS]] 操作基础、[[TCP]] 与 [[DNS]] 网络概念、[[代理]] 原理、[[XUI]]（推断，原文作“擦面板”）节点配置基础。

## 核心观点
1. **[[Tor]] 网络的初衷是保护隐私而非用于犯罪**
   - 为什么重要：纠正了大众将 [[Tor]] 仅等同于犯罪暗网的误解。
   - 依据：讲者原话提及「曝光美国监控计划的 [[爱德华斯诺登]] 推荐使用来保护隐私和匿名通信」。
2. **[[Tor]] 的三重加密与随机节点机制极难追踪**
   - 为什么重要：阐明了其提供高度隐匿性的核心技术原理。
   - 依据：讲者原话说明追踪必须同时控制三个中继节点，且「拖每隔 10 分钟就会更换节点」（讲者原话）。
3. **后置代理可解决 [[Tor]] 出口节点被表网封锁的问题**
   - 为什么重要：[[Tor]] 节点常被表网阻挡，此举显著提升了访问体验和连通性。
   - 依据：讲者原话指出出口节点「IP 风险程度爆表百分百」，并演示通过 [[VPS]] 或住宅 IP 的 [[Socks]]（推断，原文作“sox”）作为后置代理，成功向 [[谷歌]] 隐藏了高风险的出口 IP。

## 概念关系
- [[Tor]] ↔ [[洋葱浏览器]]：[[洋葱浏览器]] 默认内置了 [[Tor]]；通过在 [[VPS]] 配置 [[Tor]] 节点，也可使用普通浏览器（如 [[谷歌浏览器]]）访问暗网，但普通浏览器隐匿性不如 [[洋葱浏览器]]（讲者明确指出普通浏览器会收集访问数据）。
- [[Xray]]（推断，原文作“插瑞”） → [[Tor]]：利用 [[XUI]] 面板中的 [[Xray]] 路由功能，将特定流量（如暗网流量）转发至本地 9050 端口的 [[Tor]] 入口。
- [[fakeip]] → 暗网域名：在 [[安卓]] 设备（如 [[nic Box]]）上，如果使用需本地解析的 Tun 模式，必须开启 [[fakeip]] 才能使节点直接处理并路由没有真实 IP 的暗网域名，避免本地 [[DNS]] 报错。

## 详细笔记
### 澄清误解与 [[Tor]] 简介
- 讲者 [[不良林]] 指出，大众常将 [[Tor]] 等同于犯罪法外之地，但其初衷是保护隐私，并曾获 [[爱德华斯诺登]] 推荐。
- [[Tor]] 不仅可访问暗网和表网，还支持代理 [[SSH]]、[[FTP]] 等其他 [[TCP]] 流量。[[比特币钱包]] 和 [[Sbox]] 也支持针对 [[Tor]] 节点进行单独配置出站。

### [[Tor]] 网络工作原理
- 流量经过三个随机中继节点。通过握手协商共享密钥，并使用节点 3、节点 2、节点 1 的密钥依次对数据进行加密（讲者明确说明此顺序）。
- 解密时层层剥开：节点 1 只知道你的真实 IP，节点 3（出口节点）只知道你的意图是访问 [[谷歌]]。由于不知道彼此的完整链路，极难追踪。

### 在 [[VPS]] 上配置 [[Tor]] 前置代理
- 在 [[VPS]] 安装 [[XUI]] 面板，搭建普通 [[SS]] 节点。
- 安装 `tor`，默认开启 `9050` 端口（讲者原话）作为入口。
- 在 [[Xray]] 设置中添加指向 `127.0.0.1:9050`（屏幕展示）的出站，并在路由规则中设置将暗网域名（推断，原文作“阿娘/安/奥”域名）流量转入该出站。此时普通浏览器即可访问 [[BBC]] 等暗网版。
- 也可为 [[SS]] 添加新客户端，通过路由规则实现所有代理由 [[Tor]] 接管（讲者提示全走 [[Tor]] 速度较慢）。

### 移动端配置与 [[DNS]] 限制
- **[[安卓]]端**：以 [[nic Box]] 为例。若使用 [[Socks]] 模式可直接将暗网域名交给节点处理；若使用需本地解析的 Tun 模式，必须开启 [[fakeip]]（Fake DNS），否则会因找不到暗网 IP 报错。
- **[[iOS]]端**：讲者测试 [[Safari]] 和 [[谷歌浏览器]]（Chrome），发现因系统限制压根不处理暗网域名，无法访问暗网。

### 配置后置代理解决 IP 风险
- [[Tor]] 出口节点访问表网被视为高风险（讲者原话：信任分为零的高风险 IP）。
- 通过在面板中再添加一个节点（例如讲者演示的 [[wimense]] 节点，推断为 [[VMess]]），修改原节点路由的出口为该节点，使数据经 [[Tor]] 加密后再由 [[VPS]] 直连访问表网。讲者提及此步骤也可以套个 [[CDN]]（讲者原话）。
- 也可添加另一个具备干净美国住宅 IP 的 [[Socks]] 节点作为落地后置代理，从而获取干净的落地 IP。

### 搭建暗网网站服务
- 修改 [[Tor]] 的 `torrc` 配置文件，取消隐藏数据保存目录和端口映射的注释（例如将暗网 80 端口映射到本地的 80 端口）。
- 可在 80 端口使用 [[inx]]（推断为 [[nginx]]）搭建网站，或直接映射为 [[XUI]] 面板端口进行演示。
- 重启 [[Tor]]，查看 `hostname` 文件获取分配的暗网域名。如果访问空白，需检查浏览器是否自动加上了 [[HTTPS]]，讲者明确提示应改为 [[HTTP]] 访问即可成功进入。

3. **配置 Xray 路由规则导入 Tor**

5. **手机端配置访问暗网**
   - 具体操作描述：以安卓端 [[Nekobox]] 为例，将节点导入后，开启 [[FakeIP]] 功能，让暗网域名无需在本地强行请求真实 IP，而是交由远端节点处理并在 [[Tor]] 网络中完成路由。

## 常见坑与解决方案
- **问题描述**：在面板中直接点击重启 [[Xray]] 时卡住无响应。
  - 解决方案：先在客户端切换连接到其他可用节点再点重启，或者直接在 [[VPS]] 命令行界面执行 `XUI Restart` 命令。
- **问题描述**：普通浏览器访问暗网域名的隐私泄露风险。
  - 解决方案：如果需要更强的隐私保护，推荐使用专用的洋葱浏览器，避免普通浏览器收集访问历史。
- **问题描述**：安卓手机开启 [[TUN]] 模式访问 `.onion` 域名时爆出“域名无法解析IP”错误。
  - 解决方案：在 [[Nekobox]] 的设置中开启 `Fake DNS`（即 [[FakeIP]]）选项，避免本地发起的真实 DNS 解析失败。
- **问题描述**：[[Tor]] 网络出口节点 IP 高度危险（通常信任分为 0），用来访问表网时会遭遇各类限制、滥用阻挡和频繁人机验证。
  - 解决方案：配置后置代理，将流量从 [[Tor]] 出口再导入到一个纯净的 [[VPS]] 或住宅 [[Socks]] 节点，以干净的 IP 落地完成访问。
- **问题描述**：访问搭建好的暗网服务域名时返回一片空白/无反应。
  - 解决方案：检查浏览器是否错误地自作主张加上了 HTTPS，改为 HTTP 并在 URL 尾部加上正确的路由路径（如面板特定路径）即可访问。

## 最佳实践
- 如果隐私要求不是特别高，仅推荐将访问暗网的流量路由进 [[Tor]] 网络。全量代理走 [[Tor]] 网络的方案会导致网速严重降低，是反模式做法。
- 当需要利用 [[Tor]] 网络极强的隐匿性同时又需要流畅访问表网时，推荐采取“前置代理+Tor网络+后置干净IP节点”的链式代理配置方案，完美解决出口节点 IP “太脏”导致的使用受限。

## 关键概念

### [[tor网络]]
- **定义**：一个由全球志愿者搭建的去中心化的免费三层代理网络，旨在保护用户隐私和匿名通信 [1, 2]。
- **视频上下文**：作者澄清它不等于暗网，还可以用来访问表网以及代理其他TCP流量，并演示了如何让普通代理节点接入该网络 [1, 3]。
- **为什么重要**：它是实现极致隐私、构建三重“链式代理”以及访问暗网的核心基础设施 [1]。
- **关联概念**：包含 → [[中继节点]]，扩展 → [[暗网]]

### [[暗网]]
- **定义**：tor网络的众多应用之一，是一个不受常规监管、包含特定隐藏域名的网络空间 [1, 4]。
- **视频上下文**：作者指出很多人错误地将它与tor网络画等号，并演示了如何通过配置让普通谷歌浏览器成功访问暗网版BBC，以及如何在VPS上搭建自己的暗网服务 [1, 4, 5]。
- **为什么重要**：常被大众误解为纯粹的犯罪之地，但其本质是利用tor网络提供高度匿名的访问和服务托管 [1]。
- **关联概念**：依赖 → [[tor网络]]，对比 ↔ [[表网]]

### [[表网]]
- **定义**：普通的公共互联网或常规网站（如谷歌），与需要特定路由的暗网相对应 [1, 4]。
- **视频上下文**：作者提到使用tor网络不仅可以访问暗网也能访问表网，但由于被滥用，使用tor出口节点访问表网会面临极高的IP风险和限制 [1, 6]。
- **为什么重要**：用作参照物说明tor网络出口节点的弊端，引出了使用后置代理解决访问限制的需求 [6, 7]。
- **关联概念**：对比 ↔ [[暗网]]

### [[中继节点]]
- **定义**：tor网络中用于建立连接和转发数据的服务器，每次请求会随机选择三个此类节点 [2]。
- **视频上下文**：作者在解释tor工作原理时，详细说明了数据如何通过三个中继节点进行逐层加密握手和解密转发，且每隔十分钟会自动更换 [2]。
- **为什么重要**：这是实现tor网络匿名追踪几乎不可能的技术基础 [2]。
- **关联概念**：属于 → [[tor网络]]，包含 → [[首节点]]，包含 → [[出口节点]]

### [[首节点]]
- **定义**：tor网络随机选择的三个中继节点中的第一个，也叫节点一 [2]。
- **视频上下文**：作者说明用户直接将加密数据发给首节点，因此它是整个网络中唯一知道用户真实IP地址的节点，但它无法解密查看数据内容 [2]。
- **为什么重要**：定义了用户真实身份在tor网络中暴露的边界 [2]。
- **关联概念**：属于 → [[中继节点]]

### [[出口节点]]
- **定义**：tor网络三层代理中的第三个节点，负责最终解密用户意图并代为访问目标网站 [2]。
- **视频上下文**：作者指出它知道你要访问哪里但不知道你的真实IP，同时强调对于表网来说，tor的出口节点无一例外全都是信誉分为零的高风险IP [2, 6]。
- **为什么重要**：直接解释了为何通过tor网络访问普通网站会频繁遇到人机验证和阻挡 [6, 8]。
- **关联概念**：属于 → [[中继节点]]

### [[后置代理]]
- **定义**：在tor网络出口之后再连接的代理节点（如住宅IP或VPS） [6, 7]。
- **视频上下文**：作者演示了如何在路由规则中将tor节点的出口改为另一个节点，使数据从tor网络出来后通过后置代理访问目标网站 [6, 7]。
- **为什么重要**：用于掩盖tor出口节点的高风险IP，从而顺利访问受限的表网资源 [5-7]。
- **关联概念**：依赖 → [[tor网络]]

### [[前置代理]]
- **定义**：用户发出的流量在进入tor网络之前所经过的第一层代理节点（如用户自己搭建的普通VPS代理） [7]。
