## 一句话总结
通过免域名和证书的 [[CDN]] 搭配 [[vmess]] 与 [[WebSocket]] 协议实现 [[域前置]]，搭建防封锁节点的完整教程。

## 目标受众与前置知识
- **适合谁看**：需要搭建高传输效率、防封锁（永不被墙）科学上网节点的用户（推断）。
- **前置知识**：[[CDN]]、[[HTTP]]、[[TCP]]、[[WebSocket]]、[[TLS]]、[[节点搭建]]（推断）。

## 核心观点
1. **[[WebSocket]] 协议传输效率高于 [[HTTP]]**
   - 为什么重要：解决半双工传输效率低的问题，能够满足日常科学上网需求。
   - 依据：讲者原话表明 [[WebSocket]] 基于 [[TCP]] 可实现全双工双向通信，对比基于 [[HTTP]] 的半双工 [[Mic协议]]，速度和效率有很大提升 [1]。
2. **[[0-RTT]] 可解决 [[WebSocket]] 连接延迟问题**
   - 为什么重要：弥补建立 [[WebSocket]] 需等待 [[HTTP]] 升级响应带来的一次 [[RTT]] 延迟。
   - 依据：讲者原话提及在 [[ws]] 路径后加 `?ED=2048`，可在请求升级阶段携带 [[vmess]] 数据，从而实现 [[0-RTT]] [2]。
3. **[[HTTPUpgrade]] 协议可进一步提升性能**
   - 为什么重要：减轻客户端和服务器端在数据封装上的压力。
   - 依据：讲者原话说明该协议升级成功后，连接中直接传输 [[vmess]] 数据，不再封装成 [[ws]] 帧，但部分 [[CDN]] 可能不支持 [3]。
4. **利用 [[CDN]] 实现 [[域前置]]（视频称预前置）可隐蔽真实节点**
   - 为什么重要：使节点流量伪装成访问正规大厂网站，防止 [[VPS]] 的 IP 被墙。
   - 依据：视频演示在 [[Fastly]] 和 [[Gcore]] 设置伪装域名（如 `gov.uk`），且讲者明确表示这些平台不验证域名所有者 [4-6]。

## 概念关系
- [[HTTP]] ↔ [[WebSocket]]：[[HTTP]] 为半双工通信，而升级后的 [[WebSocket]] 接管连接并基于 [[TCP]] 实现全双工通信 [1]。
- [[WebSocket]] ↔ [[HTTPUpgrade]]：建立连接步骤相同，但 [[HTTPUpgrade]] 后续不传输标准 [[ws]] 帧，直接传底层数据，对 [[CDN]] 兼容性要求更高 [3]。
- [[vmess]] ↔ [[vless]]：搭建节点推荐使用 [[vmess]]，因为 [[vless]] 在未套用 [[TLS]] 时发送的是明文数据 [3]。
- [[CDN]] → [[域前置]]：通过给节点套用特定的 [[CDN]]（如 [[Fastly]] 或 [[Gcore]]）并配置 [[SNI]] 伪装域名，达成隐藏真实 [[VPS]] IP 的 [[域前置]] 效果 [5, 6]。

## 详细笔记

### 协议原理：WebSocket与0-RTT
- 视频由 [[不良林]] 主讲。指出基于 [[HTTP]] 的 [[Mic协议]] 效率低，引出 [[WebSocket]]（视频称 [[wapsocket]]/[[ws]]）[1]。
- [[ws]] 通信过程：客户端（如 [[浏览器]] 访问 [[Google]]）请求发给 [[v2a]]（推断：[[v2ray]]），数据封装成 [[VM]]（[[vmess]]）格式并进行 [[as加密]]（推断：AES加密）。客户端先发 [[HTTP]] 升级请求，[[CDN]] 转发至 [[VPS]]，[[VPS]] 回复 101 响应成功后，开始用 [[ws]] 帧传输数据 [1, 2]。
- 延迟问题：对比裸 [[TCP]]，[[ws]] 增加了一个 [[RTT]] 延迟 [2]。
- 解决方案：[[x 瑞]]（推断：[[Xray]]）客户端通过在路径加 `?ED=2048` 字段携带数据实现 [[0-RTT]]（视频称 0rdt）[2]。

### HTTPUpgrade 协议解析
- 升级步骤与 [[ws]] 相同（回复101响应），但建立连接后直接传输 [[vmess]] 数据，不封装成 [[ws]] 帧 [3]。
- 优势：减轻客户端和服务器压力，提升性能 [3]。
- 劣势：并非所有 [[CDN]] 接受非标准 [[ws]] 帧，需要平台支持 [3]。

### Fastly CDN 配置与域前置演示
- 前置准备：推荐搭建 [[vmess]] 加 [[ws]] 节点（讲者端囗：18880），导入 [[VN]] 客户端测试真连接延迟 [3]。
- [[Fastly]] 配置：邮箱注册登录，添加随意域名（如“我是我是颜色不一样的烟火”），回源设为节点 IP 及端口 18880。开启 [[ws]] 服务并部署一段名为 ws 的 [[vcl脚本]] [4]。
- 伪装设置：节点套用 [[Fastly]] 伪装域名，利用挂载在 [[Fastly]] 上的高权重域名（如 `gov.uk`），此时端口为 80，测试可通但未加密 [4, 5]。
- 完全体（[[TLS]]）：复制节点将端口改 443，开启 [[TLS]] 与 [[SNI]] 并填入伪装域名。此时中间人视角流量显示为访问 `gov.uk`，实现完美的 [[域前置]] [5]。

### Gcore CDN 配置与域前置演示
- [[Gcore]]（视频称 [[GC]]/[[gco]]）配置：邮箱注册，选免费计划（讲者原话：每个月有1TB的流量）。创建 [[CDN]] 资源，填随意域名，[[ipv4]] 填节点 IP [5]。
- [[DNS]] 限制验证：讲者实测发现无需修改域名的 [[NS记录]] 即可使用（即不验证域名所有权）[5]。
- 伪装设置：取消默认端口，改设为 18880 [6]。
- 完全体（[[TLS]]）：复制节点，端口改 443 开启 [[TLS]] 和 [[SNI]]。讲者提及 [[Gcore]] 有限局，只能用其特定子域或主域（如 `gc.com`）作为前置域名 [6]。测试连接有较长生效时间（讲者原话：可能需要等待10到20分钟）[6]。

### 节点优选与自动测速
- 获取 IP：使用优选工具输入节点，[[CDN]] 提供商选择 [[Fastly]]，可提取出 1000 个（屏幕展示/讲者原话）他家 IP 的节点 [6]。
- 自动测速：将生成的节点与 [[Gcore]] 节点复制到全自动测速软件列表中，建议改用 [[Xray]]（视频称插瑞）内核。软件自动进行真连接延迟测试与下载速度测试，以获取低延迟优选 IP 节点 [7]。

## 步骤拆解
1. **搭建并导入基础节点**
   - 具体操作描述：搭建基于 vmess 或 vless 加 ws 的节点并导入客户端，随后测试真连接延迟验证可用性 [2]。
   - 关键代码/命令：节点端口为 `18880` [2]；ws 路径后可添加参数 `?ed=2048` [7]。
   - 注意事项：强烈建议使用 vmess，因为 vless 在没有套 TLS 的情况下会发送明文数据 [2]。在 ws 路径后加 `?ed=2048` 可以在建立连接时携带数据以实现 0-RTT，且即使客户端不支持也不会报错 [2, 7]。

2. **配置 Fastly CDN 服务**
   - 具体操作描述：注册账号并创建 CDN 资源，填写任意伪装域名并将其 IP 指向节点 IP 地址 [3]。在设置中将回源端口修改为节点端口，开启 WebSocket 试用服务，最后创建名为 ws 的 VCL 脚本片段并部署 [3]。
   - 关键代码/命令：回源端口为 `18880` [3]；VCL 片段名称写 `ws`（具体脚本内容未明确提及）[3]。
   - 注意事项：Fastly 的 ws 回源功能并非免费，仅能试用一个月 [3]。注册过程很宽松，可以使用临时邮箱 [3]。部署后需要等待一两分钟才会生效 [3]。

3. **配置客户端 Fastly 节点**
   - 具体操作描述：配置免 TLS 节点时，将伪装域名设为 Fastly 中的自定义伪装域名，地址填入 Fastly 优选 IP 或挂载其上的已有域名，并将端口改为 80 [3]。配置 TLS 预前置完全体时，复制上述节点，将端口改为 443，开启 TLS 并在 SNI 处填入所选的前置域名 [4]。
   - 关键代码/命令：免 TLS 端口 `80` [3]；TLS 端口 `443` [4]。
   - 注意事项：用于地址伪装的 Fastly 挂载域名并非 100% 可用，搜索结果可能过期，需多换几个尝试 [3]。

4. **配置 Gcore CDN 服务**
   - 具体操作描述：注册后选择免费计划激活服务，创建 CDN 资源并输入自定义伪装域名 [4]。添加 DNS 的 IPv4 记录指向节点 IP 地址，并在资源设置中取消默认端口，修改为节点实际端口 [4, 5]。
   - 关键代码/命令：IPv4 填节点 IP [4]；源站端口 `18880` [5]。
   - 注意事项：免费计划每月包含 1TB 流量 [4]。Gcore 不会验证域名所有者，即使不修改域名的 NS 记录也能正常使用 [4]。

5. **配置客户端 Gcore 节点**
   - 具体操作描述：免 TLS 节点配置与 Fastly 类似，端口改 80，伪装域名设为自定义域名 [5]。TLS 预前置完全体节点则复制一个节点，端口改 443，开启 TLS，并将地址和 SNI 替换为收集到的 Gcore 特定子域名 [5]。
   - 关键代码/命令：免 TLS 端口 `80` [5]；TLS 端口 `443` [5]。
   - 注意事项：Gcore 生效时间较长，配置完需等待 10 到 20 分钟 [5]。在使用预前置完全体时，有前置域限制，不建议直接使用 `gcore.com` 这个主域 [5]。

6. **提取并优选 IP**
   - 具体操作描述：通过优选工具提取 Fastly 或 Gcore 提供的 1000 个 IP 节点，将结果导入全自动测速软件中，进行真连接延迟与下载速度自动化测试 [5, 6]。
   - 关键代码/命令：未明确提及 [6]。
   - 注意事项：建议将全自动测速软件的内核改为 Xray 内核进行测试 [6]。

## 最佳实践
- **推荐做法**：建立节点时首选 [[vmess]] 协议而非 [[vless]]，避免在无 [[TLS]] 时发送明文数据暴露流量特征 [2]。建议在 ws 路径后添加 `?ed=2048` 以实现 0-RTT 的提速，且不支持该功能的客户端也不会因此报错 [2, 7]。在进行 IP 优选时，推荐将测速软件内核更改为 [[Xray]] [6]。
- **反模式**：在配置 Gcore 预前置完全体时，直接使用 `gcore.com` 主域作为前置域和 [[SNI]] 属于不建议的做法 [5]。

## 关键概念

### [[WebSocket (ws)]]
- **定义**：一种基于 TCP 的全双工双向通信协议，由 HTTP 协议头部插入 upgrade 字段升级而来 [1]。
- **视频上下文**：作者将其与半双工的 HTTP 协议对比，说明 ws 传输效率更高、速度更快，并详细讲解了数据如何封装在 ws 帧中经过 CDN 转发 [1, 2]。
- **为什么重要**：它是实现高效代理和节点防封锁的核心底层传输机制 [1]。
- **关联概念**：对比 → [[HTTP]]

### [[HTTP]]
- **定义**：一种半双工的传输协议，必须由客户端先发起请求服务端才能推送数据 [1]。
- **视频上下文**：作者指出其效率低、速度有限，仅能作为应急备用，并解释它是升级为 WebSocket 协议的必经前置阶段 [1, 2]。
- **为什么重要**：理解 WebSocket 升级机制和传统网络代理效率瓶颈的基础 [1]。
- **关联概念**：对比 → [[WebSocket (ws)]]

### [[HTTPUpgrade]]
- **定义**：一种新的传输协议，同样通过 HTTP 升级为 ws 协议，但在已建立的连接中直接传输代理数据，不封装成标准的 ws 帧格式 [3]。
- **视频上下文**：作者提到它可以减轻客户端和服务端压力、提升性能，但部分 CDN 可能不接受这种非标准的 ws 帧数据 [3]。
- **为什么重要**：代表了代理协议在性能优化上的新方向，但部署时需权衡与现有 CDN 的兼容性 [3]。
- **关联概念**：属于 → [[WebSocket (ws)]]

### [[0-RTT]]
- **定义**：在请求升级为 ws 的阶段就携带代理数据，从而减少一个往返延迟时间（RTT）的技术 [2]。
- **视频上下文**：作者说明在 ws 路径后加上 `?ed=2048`，可以在 HTTP 升级 ws 的请求头（协议字段）中直接放入数据，从而实现零延迟 [2]。
- **为什么重要**：解决了 WebSocket 相比裸 TCP 多出一个 RTT 延迟的劣势，大幅提升了连接建立的速度 [2]。
- **关联概念**：依赖 → [[WebSocket (ws)]]

### [[预前制 (域前置)]]
- **定义**：一种利用中间节点隐藏真实 VPS IP 和自定义域名的防封锁技术 [1, 4]。
- **视频上下文**：作者演示了如何通过配置 Fastly 或 Gcore 等 CDN 服务，并伪装成高信誉域名（如 gov.uk），实现节点永不被墙的“完全体” [4-6]。
- **为什么重要**：这是本视频介绍的核心应用技巧，让网络审查者在中间人角度只能看到访问正规网站的正常流量 [4, 7]。
- **关联概念**：依赖 → [[CDN]]

### [[CDN]]
- **定义**：支持转发 WebSocket 请求的内容分发网络服务（如视频中的 Fastly 和 Gcore） [2, 4, 5]。
- **视频上下文**：作者演示了如何利用 CDN 接收数据并转交给节点，从而实现真实 IP 的隐藏和快速优选 IP [2, 5-7]。
- **为什么重要**：提供网络请求的中间转发层，是实现预前制和防止 VPS 节点被墙的必备基础设施 [1, 4, 7]。
- **关联概念**：扩展 → [[预前制 (域前置)]]

### [[TLS / SNI]]
- **定义**：用于建立加密连接并指定目标服务器名称的网络协议配置 [4, 6]。
- **视频上下文**：作者指出要实现预前制的“完全体”，需要开启 TLS SNI 并填入伪装的高信誉域名，这样连自定义域名都能彻底隐藏 [4, 6]。
- **为什么重要**：实现绝对流量伪装的关键步骤，使得抓包时只能看到对他人合法证书的使用，看起来更加真实 [4, 7]。
- **关联概念**：依赖 → [[预前制 (域前置)]]
