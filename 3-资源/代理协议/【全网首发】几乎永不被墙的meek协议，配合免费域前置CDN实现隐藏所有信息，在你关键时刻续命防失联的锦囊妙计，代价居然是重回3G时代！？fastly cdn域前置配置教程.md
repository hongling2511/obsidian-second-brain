## 一句话总结
本视频教程讲解如何利用极慢但隐蔽的 [[meek]] 协议配合 [[Fastly]] [[CDN]] 实现 [[域前置]]，搭建防失联应急代理节点。

## 目标受众与前置知识
- **适合谁看**：身处白名单地区、担心失联，需要搭建备用/应急翻墙手段的用户。
- **前置知识**：[[v2ray]]、[[CDN]]、[[VPS]]、[[http]]、[[tls]]

## 核心观点
1. **[[meek]] 协议是保命防失联的锦囊，但速度极慢**
   - 为什么重要：作为所有翻墙手段全失效时的终极应急方案。
   - 依据：讲者提到其速度不如 3G 时代，每次最多只能传输 64k 的数据且必须按顺序接收（讲者原话）[1, 2]。
2. **配合支持 [[域前置]] 的 [[CDN]] 才能发挥 [[meek]] 最大优势**
   - 为什么重要：可以完全隐藏源服务器 IP 和真实域名，防范防火墙的污染与阻断。
   - 依据：外层 [[SNI]] 目标和内层 [[host]] 目标不同，防火墙只能看到伪装的白名单域名（讲者原话）[3, 4]。
3. **支持 [[域前置]] 的 [[CDN]] 商家目前极少**
   - 为什么重要：导致该技术使用门槛变高。
   - 依据：由于合规问题，[[Cloudflare]] (cft)、[[Google]] (谷歌) 等已不再支持，目前仅剩 [[Microsoft]] (微然) 的 [[AZ]] 和 [[Fastly]] (fast特力) 等少数商家支持（讲者原话）[4]。

## 详细笔记
### [[meek]] 协议背景与工作原理
- [[meek]] 在 2014年 就存在，曾作为 [[Tor Browser]] (洋葱浏览器) 的可插拔传输组件（讲者原话）[1]。
- 5月31号 [[v2ray]] 更新的 5.7版本 中加入了对 [[meek]] 的支持（讲者原话）[1]。
- **传输痛点**：普通 [[http]] 无法由服务器主动推送数据，客户端需不断主动发送 [[http]] 请求要数据，每次最多传输 64k（讲者原话），导致其速度极慢，无法作为日常使用 [2, 5]。
- **核心优势**：伪装成标准 [[http]] 请求，可套用任何 [[CDN]]（不像 [[websocket]] 必须要求商家支持 ws）[2]。

### [[域前置]] 技术解析
- 即使套用普通 [[CDN]]，防火墙仍可通过 [[DNS]] 或 [[SNI]] 看到域名并进行阻断 [3]。
- **原理演示**：假设 [[Baidu]] (百度) 使用的 [[CDN]] 支持 [[域前置]]。[[v2ray]] 将数据通过 [[meek]] 封装，目标设为自有域名，但该请求外层使用 [[tls]] 加密，外层访问目标（[[SNI]]）设为 [[Baidu]]。防火墙只看到访问 [[Baidu]]，而 [[CDN]] 解密后根据内层 [[host]] 将数据发给自有域名绑定的 [[VPS]] [3, 4]。
- 讲者提到 [[Cloudflare]]、[[Google]] 等大厂由于合规问题已不支持 [[域前置]]，目前演示使用的是 [[Fastly]]，另提及 [[Microsoft]] 的 [[AZ]] 也支持 [4]。
- （推断）讲者顺带提及 [[v2ray]] 官方近期添加了 T模式 和新传输协议 [[http upgrade]] [6]。

### [[Fastly]] 注册与 [[CDN]] 节点配置
- [[Fastly]] 目标用户是企业，每月最低消费 50刀，但提供每月 50刀 流量的免费试用，试用期可能为 1个月（讲者原话提及网传与官网信息）[7]。
- 可使用临时邮箱注册。创建 [[CDN]] 服务时，随便填写一个未被占用的域名（无需验证所有权）[7]。
- 填写服务器 IP，讲者建议将回源 80端口 改为不常用端口（如 1888 或 18880）（讲者原话/推断）[7, 8]。
- [[Fastly]] 部署后配置即锁定，如需修改端口等配置，必须克隆新版本再重新部署 [8]。

### 服务端与客户端部署测试
- **服务端**：下载相应操作系统版本的 [[v2ray]]，解压并一键安装。修改配置文件监听端口与 [[CDN]] 端口一致，并在 [[VPS]] 防火墙放行该端口。启动服务并设置开机自启 [8]。
- **客户端**：将配置中的域名替换为在 [[CDN]] 设置的随机域名，或寻找一个真实挂在 [[Fastly]] 上的正经网站域名作为伪装 [8, 9]。
- 启动 [[v2ray]] 后，打开系统代理（http 代理端口为 1081）（讲者原话），查询 IP 确认经过代理 [9]。
- **测速与抓包**：访问 [[Google]] 网页打开速度尚可，缓冲视频需约 10秒（讲者原话）。抓包显示目标 IP 为 [[Fastly]] [[CDN]]，目标域名为伪装网站，无自有信息暴露 [9, 10]。
- 讲者预告：若支持 [[域前置]] 的 [[CDN]] 同时也支持 [[websocket]] (ws/wopsocket)，则 [[websocket]] 节点也能实现 [[域前置]] [10]。

## 关键概念

### [[meek]]
- **定义**：一个速度极慢的底层传输协议，通过标准的 HTTP 格式将数据封装并传输到代理服务器[1, 2]。
- **视频上下文**：早期作为洋葱浏览器的可插拔传输层使用，V2Ray 5.7 版本加入了对它的支持，视频主要讲解如何用它搭建备用节点[1, 3]。

### [[域前置]]
- **定义**：一种使外层 SNI 目标和内层 Host 目标不一致，从而隐藏真实访问域名的技术[4]。
- **视频上下文**：视频指出 meek 必须配合支持域前置的 CDN（如 Fastly）才能发挥最大作用，实现白名单效果[3, 5, 6]。
- **为什么重要**：它能让防火墙看不到你的真实域名，只看到伪装的白名单域名（如百度等），从而防止域名被污染和阻断[4, 7]。
- **关联概念**：依赖 → [[SNI]]

### [[CDN]]

### [[轮询]]
- **定义**：客户端主动且不断地发送 HTTP 请求找服务器索要数据的传输方式[5]。

### [[SNI]]
- **定义**：在域前置请求中，外层的目标域名（Server Name Indication），也是防火墙唯一能看到的域名[4, 7]。

### [[Host]]

- **视频上下文**：当 CDN 收到了伪装在 SNI 下的数据并解密后，会根据内层的 Host 目标去寻找真实的服务器获取数据[4, 7]。
- **为什么重要**：它是确保 CDN 能把伪装流量准确转发回用户真实 VPS 节点的关键指向标[4]。
